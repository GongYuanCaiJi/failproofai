## Dﺣ۸marrage rapide

```python
import agenteye

agenteye.configure(environment="production")

agenteye.event.agent_start(session_id="run-001", agent_id="planner", goal="answer user query")

agenteye.event.tool_use(
    session_id="run-001",
    agent_id="planner",
    tool_name="web_search",
    tool_call_id="toolu_01",
    input={"query": "latest AI research"},
)

agenteye.event.tool_result(
    session_id="run-001",
    agent_id="planner",
    tool_name="web_search",
    tool_call_id="toolu_01",
    output={"results": ["..."]},
)

agenteye.event.agent_end(session_id="run-001", agent_id="planner", outcome="success")
```

### Instrumenter un vrai appel

En pratique, vous enveloppez votre code d'agent existant. Encadrez un appel de modﺣ۷le avec `model_request` avant et `model_response` aprﺣ۷s, afin que les deux ﺣ۸vﺣ۸nements couvrent la vraie requﺣ۹te et que Failproof AI Observability puisse les associer :

```python
import anthropic
import agenteye

agenteye.configure(environment="production")
client = anthropic.Anthropic()

messages = [{"role": "user", "content": "Summarise today's incidents."}]

agenteye.event.model_request(
    session_id="run-001",
    agent_id="planner",
    model="claude-sonnet-4-6",
    messages=messages,
)

reply = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=512,
    messages=messages,
)

agenteye.event.model_response(
    session_id="run-001",
    agent_id="planner",
    model=reply.model,
    stop_reason=reply.stop_reason,
    input_tokens=reply.usage.input_tokens,
    output_tokens=reply.usage.output_tokens,
    content=[block.model_dump() for block in reply.content],
)
```

Enveloppez les appels d'outils de la mﺣ۹me faﺣ۶on avec `tool_use` et `tool_result`, en rﺣ۸utilisant le mﺣ۹me `tool_call_id` pour la paire.

## configure()

```python
agenteye.configure(
    base_dir=None,           # Path | str | None. Default: $AGENTEYE_HOME or ~/.agenteye
    flush_interval=0.5,      # float, seconds between flush cycles
    environment=None,        # str | None. Deployment environment label
)
```

ﺣ appeler une seule fois avant tout appel ﺣ  `event.*`. Peut ﺣ۹tre omis sans risque ; les valeurs par dﺣ۸faut fonctionnent telles quelles. Tous les arguments sont uniquement par mot-clﺣ۸ ; passez-les par nom comme indiquﺣ۸ ci-dessus.

Lorsque `base_dir` vaut `None` (la valeur par dﺣ۸faut), le SDK lit `$AGENTEYE_HOME` s'il est dﺣ۸fini,
sinon il revient ﺣ  `~/.agenteye`. Cela correspond ﺣ  la propre rﺣ۸solution du collecteur,
de sorte qu'une seule variable d'environnement `AGENTEYE_HOME` configure le spool d'ﺣ۸vﺣ۸nements partagﺣ۸
pour le SDK et le collecteur.

## Environnement

ﺣtiquetez chaque ﺣ۸vﺣ۸nement avec un environnement de dﺣ۸ploiement (`production`, `staging`, `qa`, `canary`, etc.). Dﺣ۸finissez-le une fois ; le SDK l'associe automatiquement ﺣ  chaque ﺣ۸vﺣ۸nement.

**Option 1 : via `configure()` :**

```python
agenteye.configure(environment="production")
```

**Option 2 : via une variable d'environnement :**

```bash
export AGENTEYE_ENVIRONMENT=production
```

**Prioritﺣ۸ :** `configure(environment=...)` prend le dessus sur la variable d'environnement. Si aucun des deux n'est dﺣ۸fini, la valeur par dﺣ۸faut est `"dev"`.

La valeur d'environnement apparaﺣ؟t comme un filtre de premier ordre dans le tableau de bord et est stockﺣ۸e sur le serveur pour des requﺣ۹tes rapides.

> **Avertissement :** Les valeurs d'environnement ne doivent pas contenir une virgule `,` littﺣ۸rale. Les filtres du tableau de bord utilisent une multi-sﺣ۸lection sﺣ۸parﺣ۸e par des virgules sur le fil (`?environment=prod,staging`), donc un environnement nommﺣ۸ `prod,blue` serait divisﺣ۸ en deux valeurs. Les ﺣ۸vﺣ۸nements dont les environnements contiennent des virgules sont rejetﺣ۸s ﺣ  l'ingestion.

## Donnﺣ۸es et confidentialitﺣ۸

Le SDK n'enregistre que les champs que vous passez explicitement. Les prompts, messages, entrﺣ۸es et sorties d'outils, et le contenu des modﺣ۷les sont capturﺣ۸s uniquement parce que vous les transmettez ﺣ  un appel `event.*`. Rien n'est lu depuis votre processus ni capturﺣ۸ implicitement. Tout champ que vous laissez non dﺣ۸fini est omis de l'ﺣ۸vﺣ۸nement ; il n'est pas ﺣ۸crit sur le disque.

La suppression des donnﺣ۸es est donc votre choix et votre responsabilitﺣ۸. Si un prompt ou une charge utile d'outil contient des donnﺣ۸es personnelles ou des secrets que vous prﺣ۸fﺣ۸rez ne pas stocker, nettoyez-les ou masquez-les avant de les passer ﺣ  la mﺣ۸thode d'ﺣ۸vﺣ۸nement.

## Rﺣ۸fﺣ۸rence des ﺣ۸vﺣ۸nements

La plupart des ﺣ۸vﺣ۸nements viennent par paires dﺣ۸but/fin qui partagent un identifiant de corrﺣ۸lation : `tool_use` et `tool_result` partagent un `tool_call_id`, `hook_triggered` et `hook_completed` partagent un `hook_id`, et `human_wait` et `human_input` partagent un `input_id`. ﺣmettez l'ﺣ۸vﺣ۸nement de dﺣ۸but, effectuez le travail, puis ﺣ۸mettez l'ﺣ۸vﺣ۸nement de fin avec le mﺣ۹me identifiant. Failproof AI Observability associe la paire et calcule `duration_ms` pour vous, vous n'avez donc jamais ﺣ  le passer vous-mﺣ۹me.

![Le graphe d'exﺣ۸cution de style git d'une session ﺣ  cﺣﺑtﺣ۸ de sa chronologie d'ﺣ۸vﺣ۸nements, reconstruit ﺣ  partir des ﺣ۸vﺣ۸nements appariﺣ۸s, avec le panneau de rﺣ۸partition outil/modﺣ۷le/hook](/agenteye/images/session-detail.png)

Toutes les mﺣ۸thodes d'ﺣ۸vﺣ۸nement requiﺣ۷rent ces deux champs :

| Champ | Type | Description |
|---|---|---|
| `session_id` | `str` | Identifie l'exﺣ۸cution de l'agent de niveau supﺣ۸rieur |
| `agent_id` | `str` | Identifie quel agent dans la session a ﺣ۸mis l'ﺣ۸vﺣ۸nement |

Toutes les mﺣ۸thodes acceptent ﺣ۸galement des `**kwargs` arbitraires pour des mﺣ۸tadonnﺣ۸es personnalisﺣ۸es (voir [Champs personnalisﺣ۸s](#custom-fields)).

### `event.agent_start()`

ﺣmis quand un agent commence ﺣ  travailler.

```python
agenteye.event.agent_start(
    session_id="run-001",
    agent_id="planner",
    goal="answer user query",   # str | None
    parent_id=None,             # str | None - parent agent_id for nested agents
)
```

---

### `event.agent_end()`

ﺣmis quand un agent termine son travail.

```python
agenteye.event.agent_end(
    session_id="run-001",
    agent_id="planner",
    outcome="success",          # str | None
    summary="Answered query",   # str | None
)
```

---

### `event.tool_use()`

ﺣmis quand un agent invoque un outil. ﺣ associer avec `tool_result` ; le SDK calcule automatiquement `duration_ms`.

```python
agenteye.event.tool_use(
    session_id="run-001",
    agent_id="planner",
    tool_name="web_search",     # str, required
    tool_call_id="toolu_01",    # str, required - correlation key for the matching tool_result
    input={"query": "..."},     # dict | None
)
```

---

### `event.tool_result()`

ﺣmis quand un outil retourne un rﺣ۸sultat. Corrﺣ۸lﺣ۸ avec `tool_use` via `tool_call_id`.

```python
agenteye.event.tool_result(
    session_id="run-001",
    agent_id="planner",
    tool_name="web_search",
    tool_call_id="toolu_01",        # must match the prior tool_use
    output={"results": ["..."]},    # Any | None
    error=None,                     # str | None - set if the tool raised
    # duration_ms is computed automatically - do not pass it
)
```

---

### `event.model_request()`

ﺣmis juste avant l'envoi d'un prompt ﺣ  un LLM.

```python
agenteye.event.model_request(
    session_id="run-001",
    agent_id="planner",
    model="claude-sonnet-4-6", # str | None - any provider/model string; not validated
    messages=[                   # list[dict] | None - conversation turns
        {"role": "user", "content": "..."},
    ],
    system="You are helpful.",   # Any | None - str or list of content blocks
    tools=[                      # list[dict] | None - tool schemas offered to the model
        {"name": "search", "input_schema": {"type": "object"}},
    ],
)
```

Les entrﺣ۸es `messages` acceptent soit un `content` sous forme de chaﺣ؟ne simple, soit un `content` sous forme de liste de blocs de style Anthropic. Les paramﺣ۷tres d'ﺣ۸chantillonnage (`temperature`, `max_tokens`, etc.) peuvent ﺣ۹tre passﺣ۸s comme kwargs supplﺣ۸mentaires.

---

### `event.model_response()`

ﺣmis quand le LLM retourne une rﺣ۸ponse.

```python
agenteye.event.model_response(
    session_id="run-001",
    agent_id="planner",
    model="claude-sonnet-4-6", # str | None - any provider/model string; not validated
    stop_reason="end_turn",     # str | None
    input_tokens=1024,          # int | None
    output_tokens=256,          # int | None
    content=[                    # Any | None - str, or list of content blocks
        {"type": "text", "text": "..."},
    ],
    role="assistant",            # str | None
)
```

`content` accepte soit une chaﺣ؟ne simple (fournisseurs gﺣ۸nﺣ۸riques) soit une liste de blocs de contenu de style Anthropic. Les appels d'outils se trouvent dans `content` sous forme de blocs `{"type": "tool_use", ...}`, sans champ `tool_calls` sﺣ۸parﺣ۸.

---

### `event.hook_triggered()`

ﺣmis quand un hook se dﺣ۸clenche. ﺣ associer avec `hook_completed` ; le SDK calcule automatiquement `duration_ms`.

```python
agenteye.event.hook_triggered(
    session_id="run-001",
    agent_id="planner",
    hook_name="pre_tool_use",   # str, required
    hook_id="hook-abc",         # str, required - correlation key
    trigger_event="tool_use",   # str | None
    input={"tool": "search"},   # Any | None
)
```

---

### `event.hook_completed()`

ﺣmis quand un hook se termine. Corrﺣ۸lﺣ۸ avec `hook_triggered` via `hook_id`.

```python
agenteye.event.hook_completed(
    session_id="run-001",
    agent_id="planner",
    hook_name="pre_tool_use",
    hook_id="hook-abc",         # must match the prior hook_triggered
    outcome="allow",            # str | None
    output=None,                # Any | None
    error=None,                 # str | None
    # duration_ms is computed automatically - do not pass it
)
```

---

### `event.error()`

ﺣmis quand une erreur non gﺣ۸rﺣ۸e se produit.

```python

    session_id="run-001",
    agent_id="planner",
    error_type="TimeoutError",  # str, required

    traceback="Traceback...",   # str | None

```

## ﺣvﺣ۸nements humains dans la boucle

Les ﺣ۸vﺣ۸nements humains dans la boucle vous donnent une visibilitﺣ۸ sur les moments oﺣﺗ une personne intervient dans l'exﺣ۸cution de l'agent (en attente d'approbation, en fournissant une entrﺣ۸e, en mettant en pause ou en arrﺣ۹tant l'agent). Ils vous permettent de mesurer le temps que les humains mettent ﺣ  rﺣ۸pondre (le SDK calcule automatiquement `duration_ms` sur les ﺣ۸vﺣ۸nements appariﺣ۸s), d'auditer qui a mis en pause ou interrompu un agent, et de construire des flux d'approbation et de supervision qui apparaissent dans le tableau de bord.

### `event.human_wait()`

ﺣmis quand l'agent suspend son exﺣ۸cution pour attendre une entrﺣ۸e humaine. ﺣ associer avec `human_input` ; le SDK calcule automatiquement `duration_ms` (le temps que l'humain a mis ﺣ  rﺣ۸pondre).

```python
agenteye.event.human_wait(
    session_id="run-001",
    agent_id="planner",
    input_id="inp-abc",                          # str, required - correlation key for the matching human_input
    prompt="Do you approve this action?",        # str | None - the question shown to the human
    options=["approve", "reject", "defer"],      # list[str] | None - choices presented to the human
    reason="approval_required",                  # str | None - why the agent is waiting
)
```

### `event.human_input()`

ﺣmis quand un humain fournit une entrﺣ۸e et que l'agent reprend. Corrﺣ۸lﺣ۸ avec `human_wait` via `input_id`. `duration_ms` est calculﺣ۸ automatiquement et ne doit pas ﺣ۹tre passﺣ۸ par l'appelant.

```python
agenteye.event.human_input(
    session_id="run-001",
    agent_id="planner",
    input_id="inp-abc",    # str, required - must match the prior human_wait
    response="approve",    # str | None - the human's answer (free text or selected option)
    # duration_ms is computed automatically - do not pass it
)
```

### `event.human_pause()`

ﺣmis quand un humain met activement l'agent en pause (par exemple via un contrﺣﺑle du tableau de bord). L'agent est suspendu mais pas terminﺣ۸.

```python
agenteye.event.human_pause(
    session_id="run-001",
    agent_id="planner",
    reason="user_requested",  # str | None
    user_id="usr_42",         # str | None - who paused the agent
)
```

### `event.human_interrupt()`

ﺣmis quand un humain arrﺣ۹te activement l'agent en cours d'exﺣ۸cution. Contrairement ﺣ  `human_pause`, le travail de l'agent est terminﺣ۸ plutﺣﺑt que suspendu.

```python
agenteye.event.human_interrupt(
    session_id="run-001",
    agent_id="planner",
    reason="output_incorrect",       # str | None
    user_id="usr_42",                # str | None - who interrupted the agent
    at_step="tool_use:web_search",   # str | None - what the agent was doing when stopped
)
```

## Champs personnalisﺣ۸s

Tout argument par mot-clﺣ۸ supplﺣ۸mentaire est ajoutﺣ۸ ﺣ  l'ﺣ۸vﺣ۸nement aprﺣ۷s les champs standard :

## Comment les ﺣ۸vﺣ۸nements sont ﺣ۸crits

Les ﺣ۸vﺣ۸nements sont mis en mﺣ۸moire tampon dans le processus et vidﺣ۸s sur le disque toutes les `flush_interval` secondes (500 ms par dﺣ۸faut). Chaque vidage ﺣ۸crit un fichier JSONL :

```text
~/.agenteye/events/event-2026-04-01T12-00-00-000Z.jsonl
```

Le collecteur surveille ce rﺣ۸pertoire et tﺣ۸lﺣ۸charge les fichiers automatiquement. Vous n'avez pas besoin de gﺣ۸rer ces fichiers directement.

Chaque fichier est ﺣ۸crit de maniﺣ۷re atomique : le SDK ﺣ۸crit dans un fichier temporaire puis le renomme ﺣ  sa place, de sorte que le collecteur ne voit jamais un fichier partiellement ﺣ۸crit. Un vidage final est ﺣ۸galement effectuﺣ۸ ﺣ  la fermeture de votre processus, afin que les ﺣ۸vﺣ۸nements mis en mﺣ۸moire tampon dans le dernier intervalle ne soient pas perdus. Si le collecteur est hors ligne, les ﺣ۸vﺣ۸nements s'accumulent simplement sous forme de fichiers sur le disque et sont envoyﺣ۸s dﺣ۷s qu'il revient en ligne.
