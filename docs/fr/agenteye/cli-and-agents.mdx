Tout votre déploiement Failproof AI Observability accessible en une seule commande. Vérifiez la production, générez une clé API ou acquittez un incident sans quitter votre terminal, puis scriptez n'importe quelle opération dans votre CI, ou laissez un agent de code le faire pour vous en langage naturel.

```bash
pipx install agenteye
agenteye login --email vous@example.com      # un code à 6 chiffres arrive dans votre boîte mail
agenteye --json sessions --since 24h         # toutes les exécutions d'agents des dernières 24h, les plus récentes en premier
```

*La CLI `agenteye` communique avec votre tableau de bord. C'est un outil différent du collecteur, qui envoie les événements vers le serveur.*

## Tout votre déploiement, en une seule commande

Fini de jongler entre les onglets pour répondre à une question rapide. La CLI `agenteye` lit vos données et administre votre organisation depuis un binaire unique : une vérification qui nécessitait auparavant de naviguer dans le tableau de bord devient une ligne de commande que vous pouvez relancer, créer en alias ou coller dans un runbook. Quatre surfaces à votre disposition :

- **Lire vos données :** `sessions`, `events`, `evals` et `errors`, filtrés par période, agent et environnement.
- **Gérer votre organisation :** `keys`, `users`, `settings`, `alerts` et `incidents`.
- **Lancer des analyses :** requêtes SQL sauvegardées et runner `query` ad hoc sur vos données d'événements.
- **Interroger l'assistant :** `agent ask` accède au même analyste en lecture seule que vous utilisez dans le tableau de bord.

Installez-le une fois avec `pipx`, connectez-vous via un code à 6 chiffres reçu par e-mail, et vous êtes prêt. La session dure environ une journée ; relancez `agenteye login` à expiration. Utilisez-le pour vérifier la production, provisionner une clé ou trier un incident en cours, le tout sans ouvrir de navigateur :

```bash
agenteye errors --since 24h --aggregate      # ce qui plante, regroupé par type d'erreur
agenteye incidents list --state firing       # ce qui est en feu en ce moment
agenteye keys create ci --add events:add     # une clé qui ne peut qu'envoyer des événements, secret affiché une seule fois
```

Un point important à retenir : les options globales comme `--json` se placent avant la commande. `agenteye --json sessions` est correct ; `agenteye sessions --json` ne l'est pas.

## Scriptez-le, intégrez-le à votre CI

Chaque commande accepte `--json`, et ça change tout. Le JSON propre est envoyé sur stdout tandis que les messages de statut et avertissements lisibles par l'humain vont sur stderr — ainsi, une capture `--json` se pipe directement dans `jq` sans aucune ligne parasite à éliminer. C'est ce qui rend la CLI aussi efficace pour vous en ligne de commande que pour un agent de code qui analyse les sorties :

```bash
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'
```

Elle est conçue pour fonctionner sans surveillance. Les invites de confirmation sont automatiquement ignorées lorsqu'aucun terminal n'est attaché, ce qui évite tout blocage dans un pipeline, et chaque commande retourne un code de sortie explicite : `0` succès, `4` non connecté, `5` permission manquante (le message l'indique précisément, par exemple `alerts:write`), `3` tableau de bord inaccessible. Un script peut brancher sur un `4` pour se ré-authentifier ou sur un `5` pour savoir exactement quelle permission demander à un administrateur, au lieu d'échouer silencieusement.

## Laissez un agent de code piloter en langage naturel

Mieux encore, vous ne devriez pas avoir à mémoriser ces options du tout. La **compétence CLI** est un petit dossier Agent Skill nommé `agenteye-cli` qui apprend à un agent de code comme Claude Code ou Codex à piloter la CLI à partir de requêtes en langage naturel. Demandez « est-ce que quelque chose ne va pas aujourd'hui ? » et l'agent choisit la commande, l'exécute en votre nom et répond en prose.

Pour Claude Code, déposez le dossier `agenteye-cli` dans `~/.claude/skills/` et il est découvert automatiquement. Failproof AI Observability fournit le dossier ; il n'y a rien d'autre à installer, car il se contente de piloter la CLI que vous avez déjà installée. Connectez-vous vous-même au préalable : la compétence ne peut pas effectuer la connexion par code e-mail à votre place.

L'agent exécutant la CLI en votre nom, il peut tout faire selon vos droits de connexion, lectures comme écritures : créer des clés, modifier des paramètres, résoudre des incidents. L'invite de confirmation de la CLI ne se déclenche pas pour un agent, c'est pourquoi la compétence est écrite pour indiquer la commande exacte et attendre votre validation avant tout changement. Vous êtes l'étape de confirmation.

```text
vous    Pourquoi la session run-001 a-t-elle échoué ?

agent   Exécution : agenteye --json events --session-id run-001 --all
        Le checkout-agent a rencontré une TimeoutError lors de son troisième appel d'outil.
```

Les lectures restent instantanées, et chaque écriture marque une pause pour vous :

```text
vous    Donne à CI une clé qui ne peut qu'envoyer des événements.

agent   Cela crée une clé API. Je vais exécuter :
          agenteye keys create ci --add events:add
        Voulez-vous continuer ?

vous    oui

agent   Fait. Clé "ci" créée avec events:add uniquement. Le secret n'est affiché qu'une seule fois, enregistrez-le maintenant.
```
