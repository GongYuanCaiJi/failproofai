Toàn bộ triển khai FailproofAI Observability của bạn, chỉ cách một lệnh. Kiểm tra môi trường production, tạo khóa API, hoặc xác nhận một sự cố mà không cần rời khỏi terminal của bạn, sau đó có thể viết kịch bản bất kỳ điều gì vào CI, hoặc để một coding agent thực hiện nó cho bạn bằng tiếng Anh thường ngôn.

```bash
pipx install agenteye
agenteye login --email you@example.com      # a 6-digit code lands in your inbox
agenteye --json sessions --since 24h         # every agent run from the last day, newest first
```

*CLI `agenteye` kết nối với bảng điều khiển của bạn. Đây là một công cụ khác với collector, công cụ này gửi các sự kiện đến máy chủ.*

## Toàn bộ triển khai của bạn, chỉ cách một lệnh

Dừng việc chuyển tab để trả lời một câu hỏi nhanh. CLI `agenteye` đọc dữ liệu của bạn và quản lý tổ chức của bạn từ một tệp nhị phân duy nhất, vì vậy một kiểm tra mà trước đây có nghĩa là nhấp qua bảng điều khiển trở thành một dòng mà bạn có thể chạy lại, tạo bí danh hoặc dán vào một runbook. Bạn có bốn giao diện:

- **Đọc dữ liệu của bạn:** `sessions`, `events`, `evals`, và `errors`, được lọc theo thời gian, agent và môi trường.
- **Quản lý tổ chức của bạn:** `keys`, `users`, `settings`, `alerts`, và `incidents`.
- **Chạy phân tích:** SQL đã lưu cộng với một trình chạy `query` ad-hoc trên dữ liệu sự kiện của bạn.
- **Hỏi trợ lý:** `agent ask` kết nối với cùng một nhà phân tích chỉ đọc mà bạn trò chuyện trong bảng điều khiển.

Cài đặt một lần với `pipx`, đăng nhập bằng mã 6 chữ số gửi qua email, và bạn đã sẵn sàng. Phiên kéo dài khoảng một ngày; chạy lại `agenteye login` khi nó hết hạn. Sử dụng nó để kiểm tra nhanh production, cấp phát một khóa, hoặc sàng lọc một sự cố đang kích hoạt, tất cả mà không cần mở trình duyệt:

```bash
agenteye errors --since 24h --aggregate      # what is breaking, grouped by error type
agenteye incidents list --state firing       # what is on fire right now
agenteye keys create ci --add events:add     # a key that can only push events, secret shown once
```

Một thói quen cần biết: các tùy chọn toàn cục như `--json` đặt trước lệnh. `agenteye --json sessions` là đúng; `agenteye sessions --json` là sai.

## Viết kịch bản, kết nối nó vào CI

Mỗi lệnh nhận `--json`, và điều đó thay đổi mọi thứ. JSON sạch được gửi đến stdout trong khi các cảnh báo và trạng thái dành cho con người được gửi đến stderr, vì vậy việc capture `--json` piped thẳng vào `jq` mà không có dòng lạc. Đó là những gì làm cho CLI tốt ngang nhau cho bạn ở dấu nhắc và cho một coding agent phân tích đầu ra:

```bash
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'
```

Nó được xây dựng để chạy không giám sát. Các lời nhắc xác nhận tự động bỏ qua khi không có terminal được đính kèm, vì vậy không có gì treo trong pipeline, và mỗi lệnh trả về mã thoát có ý nghĩa: `0` thành công, `4` chưa đăng nhập, `5` thiếu quyền (thông báo đặt tên cho nó, ví dụ `alerts:write`), `3` bảng điều khiển không thể truy cập. Một kịch bản có thể phân nhánh trên `4` để xác thực lại hoặc `5` để cho bạn biết chính xác những gì cần yêu cầu từ một quản trị viên, thay vì thất bại mù quáng.

## Để một coding agent điều khiển nó bằng tiếng Anh thường ngôn

Tốt hơn nữa, bạn không phải nhớ bất kỳ cờ nào trong số này. **Kỹ năng CLI** là một thư mục Agent Skill nhỏ có tên `agenteye-cli` dạy một coding agent chẳng hạn như Claude Code hoặc Codex cách điều khiển CLI từ các yêu cầu bằng tiếng Anh thường ngôn. Hỏi "có gì bị hỏng hôm nay không?" và agent chọn lệnh, chạy nó dưới tư cách bạn, và trả lời bằng văn bản.

Đối với Claude Code, thả thư mục `agenteye-cli` vào `~/.claude/skills/` và nó được tự động phát hiện. FailproofAI Observability cung cấp thư mục; không có gì thêm để cài đặt, vì nó chỉ điều khiển CLI bạn đã cài đặt. Đăng nhập bạn trước tiên: kỹ năng không thể hoàn thành việc đăng nhập mã gửi qua email cho bạn.

Vì agent chạy CLI dưới tư cách bạn, nó có thể làm mọi thứ mà đăng nhập của bạn cho phép, cả đọc và ghi: tạo khóa, thay đổi cài đặt, giải quyết sự cố. Lời nhắc "bạn có chắc chắn không?" của CLI không kích hoạt cho một agent, vì vậy kỹ năng được viết để nêu lệnh chính xác và chờ OK của bạn trước bất kỳ thay đổi nào. Bạn là bước xác nhận.

```text
you    Why did session run-001 fail?

agent  Running: agenteye --json events --session-id run-001 --all
       The checkout-agent hit a TimeoutError on its third tool call.
```

Các lần đọc vẫn tức thì, và mỗi lần ghi tạm dừng cho bạn:

```text
you    Give CI a key that can only push events.

agent  This creates an API key. I'll run:
         agenteye keys create ci --add events:add
       OK to proceed?

you    yes

agent  Done. Key "ci" created with events:add only. The secret is shown once, so store it now.
```
