## Yêu cầu

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (tùy chọn - chỉ cần để xây dựng từ nguồn)

---

## Cài đặt

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

## Bắt đầu nhanh

<Steps>
  <Step title="Bật các chính sách">
    Các chính sách là những quy tắc chạy trước và sau mỗi lệnh gọi công cụ của agent. Chúng bắt các lệnh phá hủy, rò rỉ bí mật và các chế độ lỗi khác trước khi chúng gây ra thiệt hại.

    ```bash
    failproofai policies --install
    ```

    Lệnh này ghi các mục hook vào CLIs agent đã cài đặt của bạn (tệp `~/.claude/settings.json` của Claude Code, tệp `~/.codex/hooks.json` của OpenAI Codex, tệp `~/.copilot/hooks/failproofai.json` của GitHub Copilot CLI, tệp `~/.cursor/hooks.json` của Cursor Agent, plugin shim được tạo tại `~/.config/opencode/plugins/failproofai.mjs` của OpenCode cùng mục đăng ký trong mảng `plugin` của `~/.config/opencode/opencode.json`, tệp `~/.pi/agent/settings.json` của Pi hoặc tệp `~/.gemini/settings.json` của Gemini CLI). Khi có nhiều hơn một, bạn sẽ được yêu cầu; chuyển `--cli claude codex copilot cursor opencode pi gemini` (bất kỳ tập hợp con nào) để bỏ qua lời nhắc.

    Hỗ trợ GitHub Copilot CLI, Cursor Agent, OpenCode, Pi và Gemini CLI đang ở trạng thái **beta** — cài đặt bằng `--cli copilot`, `--cli cursor`, `--cli opencode`, `--cli pi` hoặc `--cli gemini`.

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install --cli copilot --scope project
    failproofai policies --install --cli cursor --scope project
    failproofai policies --install --cli opencode --scope project
    failproofai policies --install --cli pi --scope project
    failproofai policies --install --cli gemini --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="Xác minh">
    ```bash
    failproofai policies
    ```

    Hiển thị mọi chính sách, xem nó có được bật hay không và bất kỳ tham số nào được cấu hình.
  </Step>
  <Step title="Khởi chạy bảng điều khiển">
    ```bash
    failproofai
    ```

    Mở bảng điều khiển cục bộ tại `http://localhost:8020` nơi bạn có thể duyệt các phiên, kiểm tra các lệnh gọi công cụ và quản lý các chính sách.
  </Step>
  <Step title="Chạy agent của bạn">
    Khởi chạy Claude Code như bình thường. Nếu agent cố gắng làm điều gì đó rủi ro, failproofai sẽ chặn nó tự động. Để nó chạy mà không cần giám sát và xem lại những gì đã xảy ra trong bảng điều khiển.
  </Step>
</Steps>

## Các chính sách hoạt động như thế nào

Mỗi khi một agent chạy một công cụ, Claude Code gọi failproofai như một tiến trình con:

```text
Claude Code  →  failproofai --hook PreToolUse  →  reads stdin JSON
                                                 evaluates policies
                                                 writes decision to stdout
```

Mỗi chính sách trả về một trong ba quyết định:

- **allow** - agent tiếp tục bình thường
- **deny** - hành động bị chặn, agent được cho biết lý do
- **instruct** - ngữ cảnh bổ sung được thêm vào lời nhắc của agent

<Note>
Các chính sách chạy trong quy trình cục bộ của bạn. Không có gì được gửi đến dịch vụ từ xa.
</Note>

## Thiết lập các chính sách nhóm với các chính sách dựa trên quy ước

Cách nhanh nhất để thiết lập các tiêu chuẩn chất lượng trên toàn bộ nhóm của bạn là quy ước `.failproofai/policies/`. Thả các tệp chính sách vào thư mục này và chúng sẽ được tải tự động — không có cờ, không có thay đổi cấu hình, không có lệnh cài đặt.

<Steps>
  <Step title="Tạo thư mục chính sách">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="Thêm các tệp chính sách">
    Sao chép các ví dụ khởi động hoặc viết của riêng bạn:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    Hoặc tạo một cái mới:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Run tests before committing.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Cam kết vào git">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```

    Mọi thành viên nhóm đã cài đặt failproofai sẽ tự động nhận những chính sách này. Không cần thiết lập cho mỗi nhà phát triển.
  </Step>
</Steps>

<Tip>
Cam kết `.failproofai/policies/` vào kho của bạn để toàn bộ nhóm chia sẻ các tiêu chuẩn tương tự. Khi nhóm của bạn khám phá các chế độ lỗi mới, hãy thêm các chính sách và đẩy — mọi người sẽ nhận được bản cập nhật khi họ thực hiện `git pull` tiếp theo. Theo thời gian, những chính sách này trở thành một tiêu chuẩn chất lượng sống động mà tiếp tục cải thiện.
</Tip>

## Lưu trữ dữ liệu

Tất cả cấu hình và nhật ký nằm trên máy của bạn:

| Đường dẫn | Nó lưu trữ cái gì |
|------|----------------|
| `~/.failproofai/policies-config.json` | Cấu hình chính sách toàn cầu |
| `~/.failproofai/hook-activity.jsonl` | Lịch sử thực thi hook |
| `~/.failproofai/hook.log` | Nhật ký gỡ lỗi cho các lỗi hook tùy chỉnh |
| `.failproofai/policies-config.json` | Cấu hình cho mỗi dự án (đã cam kết) |
| `.failproofai/policies-config.local.json` | Ghi đè cá nhân (được gitignore) |

---

## Gỡ cài đặt

```bash
failproofai policies --uninstall
```

Xóa các mục hook khỏi `~/.claude/settings.json`. Các tệp cấu hình trong `~/.failproofai/` được giữ lại.
