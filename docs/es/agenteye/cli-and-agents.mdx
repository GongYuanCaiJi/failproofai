Todo tu despliegue de Failproof AI Observability a un solo comando de distancia. Revisa producción, genera una clave de API o reconoce un incidente sin salir de tu terminal, luego automatiza cualquiera de estas acciones en CI, o deja que un agente de código lo haga por ti en lenguaje natural.

```bash
pipx install agenteye
agenteye login --email you@example.com      # un código de 6 dígitos llega a tu bandeja de entrada
agenteye --json sessions --since 24h         # todas las ejecuciones de agentes del último día, las más recientes primero
```

*La CLI `agenteye` se comunica con tu dashboard. Es una herramienta distinta al colector, que envía eventos al servidor.*

## Todo tu despliegue a un solo comando de distancia

Deja de cambiar de pestaña para responder una pregunta rápida. La CLI `agenteye` lee tus datos y administra tu organización desde un único binario, así que una comprobación que antes implicaba navegar por el dashboard se convierte en una línea que puedes volver a ejecutar, añadir como alias o pegar en un runbook. Dispones de cuatro superficies:

- **Lee tus datos:** `sessions`, `events`, `evals` y `errors`, filtrados por tiempo, agente y entorno.
- **Administra tu organización:** `keys`, `users`, `settings`, `alerts` e `incidents`.
- **Ejecuta análisis:** SQL guardado más un ejecutor ad-hoc `query` sobre tus datos de eventos.
- **Consulta al asistente:** `agent ask` accede al mismo analista de solo lectura con el que chateas en el dashboard.

Instálala una vez con `pipx`, inicia sesión con un código de 6 dígitos enviado por correo y ya estás listo. La sesión dura aproximadamente un día; vuelve a ejecutar `agenteye login` cuando expire. Úsala para revisar producción de forma puntual, provisionar una clave o clasificar un incidente activo, todo sin abrir un navegador:

```bash
agenteye errors --since 24h --aggregate      # qué está fallando, agrupado por tipo de error
agenteye incidents list --state firing       # qué está en llamas ahora mismo
agenteye keys create ci --add events:add     # una clave que solo puede enviar eventos, el secreto se muestra una vez
```

Un hábito importante: las opciones globales como `--json` van antes del comando. `agenteye --json sessions` es correcto; `agenteye sessions --json` no lo es.

## Automatízalo, intégralo en CI

Cada comando acepta `--json`, y eso lo cambia todo. El JSON limpio va a stdout mientras los mensajes de estado y advertencias van a stderr, por lo que una captura con `--json` se puede pasar directamente a `jq` sin necesidad de eliminar ninguna línea extra. Eso es lo que hace que la CLI sea igual de útil para ti en un prompt y para un agente de código que analiza la salida:

```bash
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'
```

Está diseñada para ejecutarse de forma desatendida. Las confirmaciones de prompts se omiten automáticamente cuando no hay terminal conectada, así que nada se queda bloqueado en un pipeline, y cada comando devuelve un código de salida con significado: `0` éxito, `4` no autenticado, `5` falta un permiso (el mensaje lo indica, por ejemplo `alerts:write`), `3` dashboard inaccesible. Un script puede bifurcarse en un `4` para reautenticarse o en un `5` para saber exactamente qué pedir a un administrador, en lugar de fallar sin información.

## Deja que un agente de código lo maneje en lenguaje natural

Mejor aún, no deberías tener que recordar ninguno de estos parámetros. La **CLI skill** es una pequeña carpeta de Agent Skill llamada `agenteye-cli` que enseña a un agente de código como Claude Code o Codex a manejar la CLI mediante peticiones en lenguaje natural. Pregunta «¿hay algo roto hoy?» y el agente elige el comando, lo ejecuta como tú y responde en prosa.

Para Claude Code, coloca la carpeta `agenteye-cli` en `~/.claude/skills/` y se descubre automáticamente. Failproof AI Observability proporciona la carpeta; no hay nada extra que instalar, ya que solo maneja la CLI que ya instalaste. Inicia sesión tú primero: la skill no puede completar el inicio de sesión con código por correo en tu lugar.

Como el agente ejecuta la CLI como tú, puede hacer todo lo que tu sesión permita, tanto lecturas como escrituras: crear claves, cambiar configuraciones, resolver incidentes. El prompt de confirmación de la CLI no se activa para un agente, por lo que la skill está diseñada para indicar el comando exacto y esperar tu visto bueno antes de cualquier cambio. Tú eres el paso de confirmación.

```text
tú      ¿Por qué falló la sesión run-001?

agente  Ejecutando: agenteye --json events --session-id run-001 --all
        El checkout-agent encontró un TimeoutError en su tercera llamada a herramienta.
```

Las lecturas son instantáneas, y cada escritura se pausa para que confirmes:

```text
tú      Dale a CI una clave que solo pueda enviar eventos.

agente  Esto crea una clave de API. Ejecutaré:
          agenteye keys create ci --add events:add
        ¿Procedo?

tú      sí

agente  Listo. Clave "ci" creada solo con events:add. El secreto se muestra una vez, así que guárdalo ahora.
```
