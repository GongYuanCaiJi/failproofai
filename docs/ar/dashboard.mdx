لوحة التحكم failproofai هي تطبيق ويب محلي لمراقبة جلسات وكيل الذكاء الاصطناعي الخاص بك وإدارة السياسات. شاهد ما فعله وكلاؤك أثناء غيابك.

---

## بدء لوحة التحكم

```bash
failproofai
```

تفتح على `http://localhost:8020`.

تقرأ لوحة التحكم مباشرة من نظام الملفات - مجلدات مشاريع Claude Code وملفات إعدادات failproofai. لا يتم كتابة أي شيء إلى خدمة بعيدة.

---

## الصفحات

### المشاريع

تسرد جميع مشاريع Claude Code و OpenAI Codex و GitHub Copilot CLI _(بيتا)_ و Cursor Agent _(بيتا)_ و OpenCode _(بيتا)_ و Pi _(بيتا)_ و Gemini CLI _(بيتا)_ الموجودة على جهازك. يتم اكتشاف مشاريع Claude من `~/.claude/projects/` (أو المسار المحدد بواسطة `CLAUDE_PROJECTS_PATH`); يتم اكتشاف مشاريع Codex من خلال مسح كل نسخة احتياطية تحت `~/.codex/sessions/<YYYY>/<MM>/<DD>/*.jsonl` والتجميع حسب `cwd` المسجل في السجل الأول من كل جلسة; يتم اكتشاف مشاريع Copilot CLI من خلال مسح كل `~/.copilot/session-state/<sessionId>/workspace.yaml` (قابل للتكوين عبر `COPILOT_HOME`) والتجميع حسب حقل `cwd` الخاص به; يتم اكتشاف مشاريع Cursor Agent من خلال مسح بيانات وصفية لكل جلسة تحت `~/.cursor/agent-sessions/<sessionId>/` (قابل للتكوين عبر `CURSOR_HOME`، مع اختبار `conversations/` و `sessions/` كبدائل) بحثاً عن `cwd` عددي في `meta.json` / `session.json` / `workspace.yaml`; يتم اكتشاف مشاريع OpenCode من خلال الاستعلام عن قاعدة بيانات SQLite الخاصة به في `~/.local/share/opencode/opencode.db` عبر `opencode db --format json` (نقرأ جداول `session` و `project` ونجمعها حسب `project_id`); يتم اكتشاف مشاريع Pi من خلال مسح نسخ احتياطية JSONL لكل جلسة تحت `~/.pi/agent/sessions/<encoded-cwd>/<timestamp>_<uuid>.jsonl` (قابل للتكوين عبر `PI_SESSIONS_DIR`) وسحب `cwd` من السجل الأول لكل جلسة; يتم اكتشاف مشاريع Gemini CLI من خلال مسح `~/.gemini/tmp/<basename>/chats/session-<timestamp>-<uuid-prefix>.jsonl` (قابل للتكوين عبر `GEMINI_SESSIONS_DIR`) واسترجاع cwd الأساسي من علامة النص الشقيقة `.project_root`. يتم عرض المشروع الذي تم استخدامه بواسطة أدوات CLI متعددة كصف واحد مع جميع الشارات المطابقة. استخدم القائمة المنسدلة **CLI** أعلى الجدول للتصفية حسب وكيل CLI محدد; يحافظ عنوان URL على اختيارك كـ `?cli=claude|codex|copilot|cursor|opencode|pi|gemini`.

يعرض كل مشروع:
- اسم المشروع (مشتق من مسار المجلد)
- شارة CLI — `Claude Code` (برتقالي)، `OpenAI Codex` (بنفسجي)، `GitHub Copilot` (أزرق)، `Cursor Agent` (زمردي)، `OpenCode` (كهرماني)، `Pi` (وردي)، و/أو `Gemini CLI` (سماوي)
- تاريخ نشاط الجلسة الأخيرة

انقر على مشروع لرؤية جلساته.

### الجلسات

تسرد جميع الجلسات داخل مشروع. تعرض كل جلسة:
- معرف الجلسة
- طوابع البداية والنهاية
- عدد استدعاءات الأدوات
- عدد نشاط الخطاف (السياسات التي تم تفعيلها)

استخدم مرشح نطاق التاريخ والبحث عن معرف الجلسة لتضييق القائمة. يتم تقسيم الجلسات إلى صفحات.

انقر على جلسة لفتح عارض الجلسة.

### عارض الجلسة

يجيب عارض الجلسة على السؤال الرئيسي للوكلاء المستقلين: ماذا فعل الوكيل، وهل بقي على المسار الصحيح؟ تشير شارة CLI بجانب الرأس إلى ما إذا كانت الجلسة نسخة احتياطية من Claude Code أو OpenAI Codex أو GitHub Copilot CLI أو Cursor Agent أو OpenCode أو Pi أو Gemini CLI. يعرض خط زمني لكل ما حدث في الجلسة:

- **الرسائل** - ردود نصوص Claude وطلبات المستخدم
- **استدعاءات الأدوات** - كل أداة استدعاها Claude، مع مدخلاتها ومخرجاتها
- **نشاط السياسة** - لكل استدعاء أداة، السياسات التي تم تفعيلها والقرار الذي أعادته

يعرض شريط الإحصائيات في الأعلى مدة الجلسة وإجمالي استدعاءات الأدوات وملخص قرارات الخطاف (السماح / الرفض / عدد التعليمات).

انقر على زر **تنزيل السجلات** لتصدير الجلسة. بالنسبة لجلسات Claude Code و Codex و Copilot و Cursor و Pi و Gemini تحصل على نسخة احتياطية JSONL على القرص الأصلية بنسبة بايت لبايت; لـ OpenCode (التي توجد جلساتها في SQLite وليس على القرص) تحصل على مستند JSON يعكس جداول `session` / `messages` / `parts` الأساسية.

صفحة بتبويبين لإدارة السياسات ومراجعة النشاط.

  - توسيع سياسة لتكوين معاملات المدخلات (للسياسات التي تدعم `policyParams`)
  - تثبيت أو إزالة الخطافات لنطاق معين

  </Tab>

## التحديث التلقائي

تحتوي لوحة التحكم على تبديل التحديث التلقائي في الملاحة العلوية. عند التفعيل، يتم تحديث الصفحة الحالية بشكل دوري لعرض الجلسات الجديدة ونشاط السياسة كما تظهر. ضروري لمراقبة جلسات الوكلاء المستقلين طويلة المدى.

---

## تعطيل الصفحات

إذا كنت تحتاج فقط إلى بعض أجزاء لوحة التحكم، قم بتعيين `FAILPROOFAI_DISABLE_PAGES` إلى قائمة مفصولة بفواصل من أسماء الصفحات:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

القيم الصحيحة: `policies`، `projects`.

---

## تكوين مسار المشاريع

بشكل افتراضي، تقرأ لوحة التحكم من دليل مشاريع Claude Code القياسي. تجاوز هذا الإعداد للأنظمة المخصصة:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## الوصول من مضيف غير localhost

عند تشغيل لوحة التحكم في **وضع التطوير** (`npm run dev`) والوصول إليها من اسم مضيف بخلاف `localhost` - على سبيل المثال، مجال مخصص أو IP بعيد أو عنوان URL مرفق - قد تراجع تحذيراً مثل:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

هذا هو Next.js يحظر الوصول عبر الأصول إلى مورد HMR (إعادة تحميل الوحدات الساخنة) الخاص به، وهي ميزة للتطوير فقط. للسماح بمضيفك، استخدم علم `--allowed-origins`:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

لعدة مضيفات أو عناوين IP، مرر قائمة مفصولة بفواصل:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

يمكنك أيضاً تعيين متغير البيئة `FAILPROOFAI_ALLOWED_DEV_ORIGINS`:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
هذا ينطبق فقط على وضع التطوير. عند تشغيل `failproofai` (وضع الإنتاج)، لا توجد مقبس HMR ولا توجد مشكلة مورد تطوير عبر الأصول.
</Note>
