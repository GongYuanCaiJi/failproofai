## الحزمة الرسمية

الحزمة npm الأساسية هي **`failproofai`**:

```bash
npm install -g failproofai
# or
bun add -g failproofai
```

---

## لماذا نمتلك أسماء الاستعارات

هجوم Typosquatting هو هجوم على سلسلة التوريد الشائعة حيث يقوم جهة فاعلة خبيثة بتسجيل اسم حزمة يبعد ضربة مفاتيح واحدة عن حزمة شهيرة. المستخدمون غير المدركين الذين يخطئون في كتابة أمر التثبيت ينتهي بهم الحال بتشغيل رمز يتحكم به المهاجم مع وصول كامل للنظام - وهذا هو بالضبط نوع التهديد الذي تم تصميم Failproof AI للدفاع ضده.

لاستبعاد هذا الخطر، **نمتلك بشكل استباقي جميع الأخطاء الإملائية الشائعة والمتغيرات الصيغية** لـ `failproofai` على npm. لا يمكن تسجيل أي من هذه الأسماء من قبل طرف ثالث. كل واحد منها هو وكيل رقيق يثبت ويفوض إلى حزمة `failproofai` الحقيقية.

## الأسماء المستعارة المسجلة

**متغيرات الصيغة** - طرق مختلفة لكتابة "failproof ai":

| الحزمة | الحالة |
|---------|--------|
| `failproof` | ✅ منشورة |
| `failproof-ai` | ⏳ قيد انتظار دعم npm |
| `fail-proof-ai` | ⏳ قيد انتظار دعم npm |
| `failproof_ai` | ⏳ قيد انتظار دعم npm |
| `fail_proof_ai` | ⏳ قيد انتظار دعم npm |
| `fail-proofai` | ⏳ قيد انتظار دعم npm |

**أخطاء `failprof*`** - حرف `o` واحد ناقص من "proof":

| الحزمة | الحالة |
|---------|--------|
| `failprof` | ✅ منشورة |
| `failprof-ai` | ✅ منشورة |
| `failprofai` | ⏳ قيد انتظار دعم npm |
| `fail-prof-ai` | ⏳ قيد انتظار دعم npm |
| `failprof_ai` | ⏳ قيد انتظار دعم npm |

**أخطاء `faliproof*`** - `a` و `i` معكوسة:

| الحزمة | الحالة |
|---------|--------|
| `faliproof` | ✅ منشورة |
| `faliproof-ai` | ✅ منشورة |
| `faliproofai` | ⏳ قيد انتظار دعم npm |

> **لماذا قيد الانتظار؟** سياسة منع الرسائل غير المرغوب فيها في npm تحظر الأسماء التي يتم تطبيعها على نفس السلسلة النصية كحزمة موجودة بعد إزالة علامات الترقيم وتشغيل فحوصات التشابه. لقد اتصلنا بدعم npm لحجز هذه الأسماء لأغراض مكافحة التزوير. سيتم تفعيلها بمجرد الموافقة عليها.

يمكنك التحقق من أن أي اسم مستعار منشور مملوك لنا:

```bash
npm info failproof
# ابحث عن: "ExosphereHost Inc." في حقل المشرفين
```

## كيفية عمل الأسماء المستعارة

كل حزمة مستعارة:

1. تدرج `failproofai` كتبعية - بحيث تعمل الحزمة الحقيقية (بما في ذلك إعداد خطاف `postinstall`) عند التثبيت
2. تكشف ملف تنفيذي يطابق اسمها الخاص (مثل `failproof-ai`) يوجه جميع المعاملات إلى ملف `failproofai` التنفيذي

الوكيل عبارة عن سكريبت Node من سطرين؛ لا توجد منطق، ولا استدعاءات شبكة، ولا جمع بيانات يتجاوز ما تقوم به `failproofai` نفسها.
