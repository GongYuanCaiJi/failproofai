## الحزمة الرسمية

حزمة npm الأساسية هي **`failproofai`**:

```bash
npm install -g failproofai
# or
bun add -g failproofai
```

---

## لماذا نمتلك أسماء الحزم البديلة

محاكاة التهجئة (Typosquatting) هي هجمة شائعة على سلسلة التوريد حيث يقوم جهة فاعلة خبيثة بتسجيل اسم حزمة يبعد ضغطة مفتاح واحدة عن حزمة شهيرة. المستخدمون غير المنتبهون الذين يخطئون في كتابة أمر التثبيت ينتهي بهم الحال إلى تشغيل كود يتحكم به المهاجم بإمكانية الوصول الكامل إلى النظام - وهذا هو بالضبط نوع التهديد الذي تم تصميم Failproof AI للدفاع ضده.

لحذف هذا السطح الضعيف، **نمتلك بشكل استباقي جميع الأخطاء الإملائية الشائعة والمتغيرات الصيغية** لـ `failproofai` على npm. لا يمكن لأي جهة خارجية تسجيل أي من هذه الأسماء. كل واحد منها هو بمثابة وسيط رقيق يثبت الحزمة الحقيقية ويفوض المهام إليها.

## الأسماء البديلة المسجلة

**متغيرات الصيغة** - طرق مختلفة لكتابة "failproof ai":

| الحزمة | الحالة |
|---------|--------|
| `failproof` | ✅ منشورة |
| `failproof-ai` | ⏳ في انتظار دعم npm |
| `fail-proof-ai` | ⏳ في انتظار دعم npm |
| `failproof_ai` | ⏳ في انتظار دعم npm |
| `fail_proof_ai` | ⏳ في انتظار دعم npm |
| `fail-proofai` | ⏳ في انتظار دعم npm |

**أخطاء `failprof*`** - حرف `o` واحد مفقود من "proof":

| الحزمة | الحالة |
|---------|--------|
| `failprof` | ✅ منشورة |
| `failprof-ai` | ✅ منشورة |
| `failprofai` | ⏳ في انتظار دعم npm |
| `fail-prof-ai` | ⏳ في انتظار دعم npm |
| `failprof_ai` | ⏳ في انتظار دعم npm |

**أخطاء `faliproof*`** - حرفا `a` و `i` معكوسان:

| الحزمة | الحالة |
|---------|--------|
| `faliproof` | ✅ منشورة |
| `faliproof-ai` | ✅ منشورة |
| `faliproofai` | ⏳ في انتظار دعم npm |

> **لماذا في انتظار الموافقة؟** سياسة منع البريد المزعج في npm تحظر الأسماء التي يتم تطبيعها بحيث تصبح نفس سلسلة نص الحزمة الموجودة بعد إزالة علامات الترقيم وتشغيل فحوصات التشابه. لقد تواصلنا مع دعم npm لحجز هذه الأسماء لأغراض مكافحة المحاكاة. سيتم تفعيلها بمجرد الموافقة عليها.

يمكنك التحقق من أن أي اسم بديل منشور يملكه:

```bash
npm info failproof
# Look for: "ExosphereHost Inc." in the maintainers field
```

## كيفية عمل الأسماء البديلة

كل حزمة بديلة:

1. تدرج `failproofai` كتبعية - بحيث تعمل الحزمة الحقيقية (بما في ذلك إعداد `postinstall` hook الخاص بها) عند التثبيت
2. تعرّض ملف تنفيذي مطابق لاسمها الخاص (على سبيل المثال `failprof-ai`) يفوض جميع المعاملات إلى ملف failproofai التنفيذي

الوسيط هو نص Node بسطرين فقط؛ لا توجد منطق، لا استدعاءات شبكة، ولا جمع بيانات يتجاوز ما تقوم به failproofai نفسها.
