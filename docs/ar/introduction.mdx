خطافات وسياسات لـ **معالجة فشل الذكاء الاصطناعي**، **استرجاع الأخطاء**، و **موثوقية نماذج اللغة الكبيرة**. حافظ على أن تبقى وكلاء الذكاء الاصطناعي الخاصة بك موثوقة وتعمل بشكل مستقل عبر **Claude Code**، **OpenAI Codex**، **GitHub Copilot**، **Cursor Agent**، **OpenCode**، **Pi**، **Hermes**، **OpenClaw**، **Factory Droid**، **Devin CLI**، **Antigravity CLI**، و **Agents SDK**.

وكلاء الذكاء الاصطناعي يفشلون بطرق يمكن التنبؤ بها. إنهم ينفذون أوامر تدميرية، ويسربون الأسرار، ويحيدون عن المهمة، ويعلقون في حلقات، أو يدفعون مباشرة إلى الفرع الرئيسي. إذا تركت دون رقابة، فإن الأخطاء الصغيرة تتسبب في انقطاعات الخدمة وتسرب بيانات الاعتماد وفقدان العمل.

FailproofAI يحل هذا باستخدام **السياسات**. هذه القواعد تتصل بكل استدعاء أداة للوكيل لـ **الكشف عن الأخطاء**، **تخفيفها** (الحظر، التعليمات، التعقيم)، و **تنبيهك** عندما يحتاج شيء ما إلى انتباه. لوحة معلومات محلية تسمح لك بمراجعة كل استدعاء أداة وفشل وكيل وإجراء استرجاع بعد ذلك.

النصوص وتقييم السياسات تبقى على جهازك. يتم إرسال البيانات فقط عندما تستخدم بشكل صريح ميزة عبر الإنترنت، مثل تذكيرات التدقيق المصرح بها أو الدعوات.

## البدء السريع

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

```bash
failproofai policies --install   # enable policies (or skip — `failproofai` will offer to set them up on first run)
failproofai                      # launch the dashboard
```
