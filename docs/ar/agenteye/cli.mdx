قُد كل مراقبة Failproof AI من الطرفية أو نص برمجي: لا توجد رحلات لوحة التحكم. تستعلم واجهة `agenteye` للسطر عن بياناتك (الجلسات، سجلات الأحداث، التقييمات) وتدير منظمتك (مفاتيح API، المستخدمون، الإعدادات، التنبيهات، الحوادث، الاستعلامات المحفوظة)، لذا استخدمها عندما تريد أتمتة فحص أو دمج المراقبة في CI أو السماح لوكيل ترميز فحص الإنتاج. كل أمر يدعم علامة `--json`، لذا يعمل بنفس السهولة لك في المحطة أو لوكيل ترميز (Claude Code، Cursor) ينفذ أوامر shell ويحلل النتيجة.

باستخدام ملف تنفيذي واحد يمكنك:

- **قراءة بياناتك**: `sessions`، `events`، `evals`، `errors` (تصفية حسب الوقت، الوكيل، البيئة، الدرجة).
- **إدارة منظمتك**: `keys`، `users`، `settings`، `alerts`، `incidents`.
- **تشغيل التحليلات**: SQL المحفوظ ومشغل الاستعلام المخصص (`query`).
- **اسأل مساعد الذكاء الاصطناعي**: نفس المحلل للقراءة فقط الذي تتحدث معه في لوحة التحكم (`agent`).

> **ملاحظة:** هذه هي واجهة `agenteye` للسطر، أداة مختلفة عن عفريت المجمّع (`agenteye-collector`). تتحدث واجهة السطر مع لوحة التحكم الخاصة بك؛ يرسل المجمّع الأحداث إلى الخادم.

## البدء السريع

من لا شيء إلى نتيجتك الأولى في أربعة أسطر. وجّه واجهة السطر إلى لوحة التحكم الخاصة بك، وسجّل دخولك، وأكّد هويتك، ثم اسحب آخر يوم من التشغيلات:

```bash
pipx install agenteye
agenteye --base-url https://agenteye.example.com login --email you@example.com   # emailed 6-digit code
agenteye whoami                                                                  # confirm user + active org
agenteye --json sessions --since 24h                                             # one row per agent run, last 24h
```

يطبع هذا الأمر الأخير كائن JSON بأحدث الجلسات (الأحدث أولاً، محدود بـ 50 افتراضياً). أنابيه إلى `jq` لتقطيعه، أو أسقط `--json` للحصول على جدول مربع وملون. يحمل كل صف حالة التشغيل وإذا قيّمها المقيِّم، درجات المقياس الخاصة به (مختصرة هنا):

```json
{
  "sessions": [
    {
      "session_id": "run-8f2a",
      "agent_id": "checkout-bot",
      "environment": "prod",
      "status": "error",
      "scores": { "helpfulness": 0.42, "tool_efficiency": 0.55 },
      "event_count": 37,
      "started_at": "2026-07-16T09:14:02Z",
      "last_event_at": "2026-07-16T09:14:48Z"
    }
  ],
  "next_cursor": null
}
```

يشرح باقي هذه الصفحة كل جزء: [التثبيت](#installation) بعزلة، [تسجيل الدخول](#authentication)، [الإعدادات](#configuration)، [الاتفاقيات العامة](#global-options--conventions) التي تشاركها كل أمر، و[مرجع الأوامر الكامل](#command-reference).

## التثبيت

واجهة السطر هي حزمة عامة PyPI باسم **`agenteye`**. ثبتها في بيئة معزولة حتى تكون دائماً لديها اعتماداتها الخاصة:

```bash
pipx install agenteye
# or
uv tool install agenteye
```

تتطلب Python 3.10+. الأمر المثبت هو **`agenteye`**:

```bash
agenteye --version
agenteye --help
```

> **ملاحظة:** يستخدم Failproof AI Observability Python SDK أيضاً اسم توزيع `agenteye`. يحافظ التثبيت مع `pipx` أو `uv tool` (بدلاً من `pip install` في virtualenv مشترك) على عدم اصطدامهما. `pip install agenteye` بسيط فقط إذا لم يتم تثبيت SDK في نفس البيئة.

## المصادقة

تتحقق واجهة السطر من صحة **لوحة التحكم** باستخدام رمز لمرة واحدة مرسل بالبريد الإلكتروني:

```bash
agenteye login --email you@example.com
# A 6-digit code is emailed to you; paste it at the prompt.
```

يتم تخزين رمز الجلسة في `~/.agenteye/cli.json` (قابل للقراءة فقط بواسطتك، الوضع `0600`) وصالح لمدة 24 ساعة افتراضياً. عند انتهاء صلاحيته، شغّل `agenteye login` مرة أخرى.

```bash
agenteye whoami     # show the current user, active org, and permissions
agenteye logout     # revoke the session and clear the stored token
```

`whoami` لا يخطئ أبداً على جلسة مفقودة أو منتهية الصلاحية؛ بدلاً من ذلك يبلّغ `logged_in: false`، لذا يمكن لنص برمجي أو وكيل اختبار حالة المصادقة بأمان (يمكنه لا يزال الخروج برمز غير صفري إذا لم يتم تعيين عنوان أساسي أو كانت لوحة التحكم غير قابلة للوصول).

**المتطلبات:** يجب أن يكون بريدك الإلكتروني مصرح له بتسجيل الدخول إلى لوحة التحكم (اسأل مسؤول Failproof AI Observability)، ويجب أن تكون لوحة التحكم قابلة للوصول على عنوانها الأساسي (انظر [الإعدادات](#configuration)). إذا طلبت رمزاً ولم يصل، فمن المحتمل أن بريدك الإلكتروني لم يتم تفعيله بعد لوصول لوحة التحكم.

## اختيار منظمتك (متعدد المستأجرين)

إذا كان حسابك ينتمي إلى أكثر من منظمة واحدة، فاختر المنظمة النشطة **عند تسجيل الدخول**؛ يتم حفظها واستخدامها لكل أمر لاحق:

```bash
agenteye login --org acme           # authenticate and set the active tenant in one step
agenteye orgs list                  # the orgs you can access (the active one is marked)
agenteye orgs switch globex         # change the saved default
agenteye --org globex sessions      # override for a single command
```

إذا كنت تنتمي إلى منظمة واحدة بالضبط يتم تحديدها تلقائياً ويمكنك تجاهل `--org` تماماً. إذا كنت تنتمي إلى عدة منظمات ولم تختر واحدة، تسرد واجهة السطر الخيارات وتطلب منك إعادة التشغيل مع `--org <slug>`. يتم إرسال المنظمة النشطة إلى لوحة التحكم على كل طلب، وتتم معالجة أذوناتك **لكل منظمة**؛ يعرض `agenteye whoami` المنظمة النشطة وأذوناتك فيها وجميع عضويتك.

## الإعدادات

| الإعداد | العلامة | متغير البيئة | الافتراضي |
|---|---|---|---|
| عنوان لوحة التحكم الأساسي | `--base-url` | `AGENTEYE_DASHBOARD_URL` | **مطلوب** (لا يوجد افتراضي) |
| المنظمة/المستأجر النشط | `--org` | `AGENTEYE_ORG` | تم اختياره عند تسجيل الدخول؛ محفوظ في `~/.agenteye/cli.json` |
| رمز الجلسة | `--token` | `AGENTEYE_CLI_TOKEN` | من `~/.agenteye/cli.json` |
| مخرجات JSON | `--json` | `AGENTEYE_CLI_JSON` | معطّل |
| تخطي التحقق من TLS | `--insecure` / `--secure` | `AGENTEYE_INSECURE` | معطّل (محفوظ عند تسجيل الدخول) |
| انتظار الطلب (بالثواني) | `--timeout` | _(none)_ | 30 |
| تعطيل القياس الإحصائي للاستخدام | _(none)_ | `AGENTEYE_ANALYTICS_DISABLED` (أو `DO_NOT_TRACK`) | يتم تعطيل القياس الإحصائي حالياً؛ لا يتم إرسال شيء |

ترتيب الحل هو **العلامة → متغير البيئة → ملف الإعدادات**. لا يوجد افتراضي؛ يجب عليك توجيه واجهة السطر إلى لوحة التحكم الخاصة بك، إما لكل أمر (`--base-url https://agenteye.example.com`) أو مرة واحدة عبر البيئة (يتم حفظه أيضاً بعد أول `login` الخاص بك):

```bash
export AGENTEYE_DASHBOARD_URL=https://agenteye.example.com
```

يحترم دليل الإعدادات `AGENTEYE_HOME` (نفس الاتفاقية المستخدمة من قبل SDK والمجمّع)؛ إذا تم تعيينه، `cli.json` يعيش في `$AGENTEYE_HOME/cli.json`.

### TLS موقّع ذاتياً أو داخلي

إذا تم تقديم لوحة التحكم الخاصة بك عبر HTTPS باستخدام شهادة موقّعة ذاتياً أو داخلية (على سبيل المثال، اسم مضيف موازن التحميل الخام)، يرفضها التحقق من TLS مع خطأ `CERTIFICATE_VERIFY_FAILED`. مرر `--insecure` لتخطي التحقق من الشهادة:

```bash
agenteye --base-url https://agenteye.internal --insecure login
```

يتم **حفظ `--insecure` إلى `cli.json` عند تسجيل دخولك**، لذا تتخطى الأوامر اللاحقة التحقق تلقائياً؛ لا تضطر إلى تكرار العلامة. مرر `--secure` للاتصال المحقق من مرة واحدة، أو لحفظ التحقق مرة أخرى عند تسجيل دخولك التالي. تطبع واجهة السطر تحذيراً إلى stderr قبل أي أمر يتواصل مع لوحة التحكم بينما يكون التحقق معطّلاً. يزيل تخطي التحقق الحماية من هجمات man-in-the-middle؛ تأكد من ثقتك بمسار الشبكة إلى لوحة التحكم الخاصة بك (VPN، شبكة خاصة، وما إلى ذلك) قبل الاعتماد عليه.

## القياس الإحصائي والخصوصية

> **ملاحظة:** واجهة السطر المُرسلة **لا تُرسل أي قياس إحصائي للاستخدام اليوم.** مفتاح إيقاف رئيسي قيد التشغيل، لذا لا يتم نقل شيء بغض النظر عن بيئتك. يصف القسم أدناه القدرة على عدم الاشتراك إذا وعندما يتم تفعيل القياس الإحصائي.

حتى عند تفعيله، سيكون القياس الإحصائي **تحليلات الاستخدام المجهولة فقط**، لا بيانات وكيلك أو جلستك أو حدثك:

- **لا تغادر بيانات الوكيل أو الجلسة أو الحدث أبداً بنيتك الأساسية.** سيتم الإبلاغ عن استخدام واجهة السطر فقط: اسم الأمر والأمر الفرعي (مثل `keys create`)، **الأسماء** للعلامات التي استخدمتها (لا قيمتها)، حالة النجاح/الخروج، والمدة، بالإضافة إلى حدث لكل إجراء للطفرات (مثل `api_key_created`، `query_run`) يحمل فقط أسماء/enums ثابتة وعدادات خشنة. عنوان لوحة التحكم الخاص بك، رمز الجلسة، بريدك الإلكتروني، slug المنظمة، معرّفات الموارد، SQL، أسرار المفتاح، وعوامل تصفية الاستعلام **لا يتم نقلها أبداً**. يتم تحديد المشغلين فقط برقم معرّف داخلي معتم، لا بريد إلكتروني.
- **لا تشترك مقدماً** بتعيين `AGENTEYE_ANALYTICS_DISABLED=1` في بيئة واجهة السطر (تحترم واجهة السطر أيضاً اتفاقية `DO_NOT_TRACK=1` عبر الأدوات). يدخل حيز التنفيذ في اللحظة التي يتم فيها تفعيل القياس الإحصائي، لذا يمكن لبيئة واعية بالخصوصية البقاء معطلة بشكل دائم.
- إذا تم تفعيل القياس الإحصائي، ستُرسل واجهة السطر مباشرة إلى PostHog (`https://us.i.posthog.com`)؛ ستصمت آلة بها هذا المضيف محجوب وستترك واجهة السطر دون تأثر.

## الخيارات العامة والاتفاقيات

اقرأ هذا مرة واحدة؛ ينطبق على كل أمر.

- **تذهب الخيارات العامة قبل الأمر.** `agenteye --json sessions` صحيح؛ `agenteye sessions --json` خطأ استخدام. القيم العامة هي `--json`، `--base-url`، `--org`، `--token`، `--insecure`/`--secure`، `--timeout`، `--quiet`، و`--no-color`.
- **`--json` يطبع JSON نقي إلى stdout، وشيء آخر فقط.** تذهب سطور حالة الإنسان والتحذيرات والأخطاء إلى **stderr**، لذا يبقى التقاط مخرجات `--json` نظيفاً لأنابيه إلى `jq` حتى عندما يتم عرض سطر حالة. بدون `--json` تحصل على عرض مربع وملون لعيون الإنسان.
- **اكتشف مع `--help`.** لكل أمر وأمر فرعي `--help` (والاسم المستعار `-h`): `agenteye -h`، `agenteye sessions -h`، `agenteye keys create -h`. يسرد المساعدة على المستوى الأعلى أيضاً رموز الخروج والخيارات العامة. لا يوجد تفريغ سطح موحد قابل للقراءة من قبل الآلة؛ استخدم لكل أمر `--help`، بالإضافة إلى `agenteye query schema` و`agenteye settings schema` الخاصة بالمجال لتلك السجلات الاثنين.
- **التأكيدات قافزة تلقائياً للنصوص والوكلاء.** تطالب أوامر الإنشاء/التحديث/الحذف هل أنت متأكد؟ في محطة تفاعلية، لكن **تتخطى هذا الموجه تلقائياً تحت `--json` أو عندما لا تكون stdin TTY** (TTY هي جلسة محطة تفاعلية؛ الأنبوب أو عامل CI ليس كذلك)، لذا لا تتعطل النصوص والوكلاء. مرر `--yes`/`-y` لتخطيه بوضوح. نظراً لأن الموجه لن ينطلق لوكيل، يجب أن يؤكد الوكيل الإجراءات المدمرة مع الإنسان أولاً.
- **الترقيم:** النتائج هي الأحدث أولاً والمؤشر المرقم (كل صفحة تعيد رمزاً تستخدمه لجلب التالي). `--limit N` (الاسم المستعار `-n`) يحد الصفوف و**افتراضيات إلى 50**؛ `--all` ترقيم تلقائي (في أجزاء 200 صف) **حتى `--limit`**، لذا `--all` بسيط لا يزال يتوقف عند 50. للحصول على فرس كامل مرر حد واضح عالي: `--all --limit 1000`. `--page-size N` يتحكم في الجزء لكل طلب (الحد الأقصى 200)؛ `--cursor <id>` يستأنف من `next_cursor` للصفحة السابقة.
- **عوامل تصفية الوقت:** `--since` يأخذ نافذة نسبية: `15m`، `1h`، `6h`، `24h`، `7d`، أو `all` (إعدادات لوحة التحكم). لنطاق أطول أو مخصص (مثل آخر 30 يوماً)، استخدم `--from`/`--to`: طوابع زمنية ISO-8601 UTC صريحة **مع `T` ومنطقة زمنية** (مثل `2026-06-01T00:00:00Z`) التي تتجاوز `--since`. قيمة مفصولة بمسافة أو بدون منطقة زمنية خطأ استخدام.
- **`--fields a,b,c`** (على `events`، `sessions`، `evals`، `errors`) يقيد المخرجات إلى تلك المفاتيح، لكل من الجدول و`--json`. يتم رفض الأسماء غير المعروفة بالقائمة الصحيحة، وسيلة رخيصة لاكتشاف أسماء المجالات.
- **`--file payload.json`** (أو `--file -` لقراءة stdin) يوفر نص جسم الطلب JSON كاملاً حيث يكون للمورد شكل معقد (على `alerts create/update`، `settings set`، و`users create/update`). يستخدم SQL المحفوظ-استعلام `--sql @file.sql` بدلاً من ذلك.
- **عوامل تصفية متعددة القيم** مفصولة بفاصلات → تم مطابقتها كمجموعة (الاتحاد داخل عامل واحد، AND عبر عوامل): `--event-type tool_use,tool_result`. خيارات النقرات ليست variadic، لذا `--add a b` ينقطع. استخدم `--add a,b`، كرر العلامة (`--add a --add b`)، أو اقتبس (`--add "a b"`).

## مرجع الأوامر

### ستستخدم هذه الأوامر الخمسة في الغالب

يتم تشغيل معظم العمل اليومي من خلال حفنة من أوامر القراءة. ابدأ هنا، ثم استخدم السطح الكامل أدناه عندما تحتاج إليه:

| الأمر | ما يفعله | جربه |
|---|---|---|
| `sessions` | صف واحد لكل تشغيل وكيل: الوقت، البيئة، الوكيل، الحالة، أحدث درجة. | `agenteye --json sessions --since 24h --status error` |
| `events` | مسار خام لكل خطوة داخل التشغيل (أضف `--full` للحمولات). | `agenteye --json events --session-id run-001 --all` |
| `evals` | نتائج التقييم والدرجات؛ `--aggregate` يلفها. | `agenteye --json evals --aggregate --since 7d --env prod` |
| `errors` | فقط الأحداث المخطئة؛ `--aggregate` للعدادات حسب النوع. | `agenteye --json errors --since 24h --aggregate` |
| `list` | اكتشف قيم العامل الصحيحة (الوكلاء، البيئات، النماذج، …). | `agenteye list agents` |

### كل شيء يمكن لواجهة السطر أن تفعله

يتبع السطح الكامل. واجهة السطر لها **18 أمر على المستوى الأعلى**. تقبل جميع أوامر القراءة `--json` والخيارات العامة أعلاه؛ شغّل `agenteye <command> -h` (أو `<command> <subcommand> -h`) لقائمة العلامات الشاملة وشكل JSON لأي واحد.

### الهوية: `login` · `logout` · `whoami` · `orgs` · `version` · `help`

```bash
agenteye login --email you@example.com [--org acme]   # emailed one-time code; saves the session
agenteye logout                                       # clear the saved session on this machine
agenteye whoami                                       # current user, active org, permissions
agenteye version                                      # print the CLI version (same as --version)
agenteye help                                         # top-level help (same as --help)
```

`orgs` تفتش وتبدل المستأجر النشط:

```bash
agenteye orgs list        # your orgs + your role in each (active one marked)
agenteye orgs switch acme # change the saved active org (omit the slug to pick from a list on a TTY)
agenteye orgs current     # identity card for the active org
agenteye orgs perms       # your permissions in the active org, grouped by resource
```

### مراقبة (قراءة فقط): `events` · `sessions` · `evals` · `errors` · `list`

لا أحد من هؤلاء يحتاج تأكيد. الفلاتر المشتركة: `--session-id`، `--agent-id`، `--env` (**ليس** `--environment`)، ونطاق الوقت (`--since` / `--from` / `--to`).

```bash
# events (alias: the raw per-step trail), newest first
agenteye --json events --session-id run-001 --event-type tool_use,tool_result --all --limit 1000
agenteye --json events --since 1h --search timeout --all | jq '.events[].payload'

# sessions: one row per agent run (time/env/agent/session/status; no score filtering)
agenteye --json sessions --since 24h --status error
agenteye --json sessions --agent-id checkout-bot --env prod --all --limit 1000

# evals: evaluation results + scores; --score filters by metric, --aggregate rolls up
agenteye --json evals --score helpfulness:0.5..0.8 --score tool_efficiency:..0.3
agenteye --json evals --aggregate --since 7d --env prod        # status mix + per-key score stats

# errors: errored events; --aggregate for counts/sessions/agents/last-seen
agenteye --json errors --since 24h --aggregate
agenteye --json errors --since 24h --error-type timeout --all --limit 1000

# list: discover valid filter values before you filter
agenteye list envs        # also: agents event_types score_filters models hooks tools error_types
```

`--score KEY:MIN..MAX` (على **`evals`**، ليس `sessions`) قابل للتكرار و AND-مدمج؛ يكون أي حد اختياري (`..0.5` يعني ≤ 0.5، `0.9..` يعني ≥ 0.9). حتى 20 عامل تصفية درجة لكل طلب. `evals --scores-full` هو علم العرض فقط للجدول **للإنسان فقط**؛ يعرض كل زوج درجة بدلاً من الأول كم زائد `+N` العد. ليس له تأثير تحت `--json`، الذي يعيد دائماً كائن الدرجة الكامل. لقراءة **جلسة واحدة من البداية إلى النهاية**، اجمع مسار الحدث مع تقييمه:

```bash
agenteye --json events --session-id run-001 --all --limit 1000 | jq '.events[] | {ts, event_type}'
agenteye --json evals  --session-id run-001                       # its scores + status
```

### إدارة (مبوابة الإذن): `keys` · `users` · `settings` · `alerts` · `incidents`

**`keys`**: مفاتيح API. يتم إنشاء السر محلياً، وإرساله إلى الخادم (الذي يخزن فقط hash)، و**يُظهر مرة واحدة** عند الإنشاء/إعادة الإنشاء؛ احبسه بعد ذلك. مع `--json` يظهر فقط في حقل `key`. المشار إليه بـ **الاسم**.

```bash
agenteye keys list                                   # active keys first, then revoked
agenteye keys show ci-bot
agenteye keys create ci-bot --add events:read.add    # scope to what you need; prints the secret ONCE
agenteye keys create ops --permission-set standard --remove queries:run   # seed a preset, then trim
agenteye keys update ci-bot --add evaluations:read --yes
agenteye keys regenerate ci-bot --yes                # rotate the secret (the old one stops working)
agenteye keys disable ci-bot --yes                   # revoke
```

تعمل الأذونات كـ `(permission-set ∪ --add) − --remove`. الرموز هي `slug:action` (مثل `events:read`) أو `slug:action.action` للتوسيع على نفس المورد (`events:read.add` → `events:read`، `events:add`). الإعدادات المسبقة: `read-only`، `standard`، `admin`. الأذونات الخاصة بالإنسان فقط (`keys:update`) لا يمكن منحها لمفتاح.

**`users`**: أعضاء المنظمة، المشار إليهم بـ **البريد الإلكتروني** (يُقبل أيضاً معرّف UUID).

```bash
agenteye users list [--active-only]
agenteye users show dev@corp.com
agenteye users create dev@corp.com --permission-set standard
agenteye users update dev@corp.com --add alerts:write --remove queries:delete   # predicts + confirms
agenteye users disable dev@corp.com --yes            # has protected/self guards
agenteye users enable dev@corp.com
```

**`settings`**: سجل ثابت (تقرأ وتغيّر المفاتيح الموجودة؛ لا يمكنك إنشاء مفاتيح جديدة).

```bash
agenteye settings list                               # key · value · type · updated (secrets masked)
agenteye settings schema                             # what each key accepts (type · range · description)
agenteye settings set session_ttl_secs --value 86400 --yes
```

**`alerts`**: تعريفات التنبيه، المشار إليها بـ **الاسم**. يأخذ `create` NAME موضعي بالإضافة إلى العلامات أو نص JSON كامل عبر `--file`.

```bash
agenteye alerts list
agenteye alerts show high-errors
agenteye alerts create high-errors --file alert.json          # NAME is required (positional)
agenteye alerts update high-errors --severity critical --yes
agenteye alerts test high-errors --yes                        # fire a test notification
agenteye alerts delete high-errors --yes
```

**`incidents`**: حوادث التنبيه، المشار إليها بـ id (تُقبل الرموز القصيرة). `show` يطبع سجل النشاط الكامل؛ اقرأه قبل التصرف.

```bash
agenteye incidents list --state firing                # also: acknowledged, resolved
agenteye incidents count
agenteye incidents show <id>
agenteye incidents ack <id>
agenteye incidents assign <id> you@corp.com           # assignee must be an operator
agenteye incidents resolve <id> --yes
agenteye incidents open --alert-id <id> --severity critical    # open one manually against an alert
agenteye incidents comment-add <id> "root cause: upstream 5xx"
agenteye incidents comment-list <id> ; agenteye incidents comment-delete <id> <comment-id>
agenteye incidents subscribe <id> ; agenteye incidents unsubscribe <id> ; agenteye incidents subscribers <id>
```

### التحليلات والمساعد: `query` · `agent`

**`query`**: SQL محفوظ ضد متجر التحليلات الخاص بك بالإضافة إلى مشغل حر. يتم الإشارة إلى الاستعلامات المحفوظة بـ **الاسم**؛ يتم التحقق من SQL من جانب الخادم (SELECT/WITH فقط، انتهاء الوقت المسموح به، حد الصف).

```bash
agenteye query schema [TABLE]                         # column layout of the analytics views
agenteye query run --sql "select count(*) from analytics.events"
agenteye query run errs --arg prod --limit 100        # run a saved query + a positional $1
agenteye query list ; agenteye query show errs
agenteye query create errs --sql @errs.sql --description "errored events (24h)"
agenteye query update errs --sql @errs.sql --yes ; agenteye query delete errs --yes
```

**`agent`**: يتحدث مع **مساعد الذكاء الاصطناعي** المدمج (نفس محلل القراءة فقط الذي يمكنك الدردشة معه في لوحة التحكم). يتم الإشارة إلى الدردشات برقم chat-id قصير (prefix-resolved).

```bash
agenteye agent health                                 # is the AI assistant configured/reachable
agenteye agent models                                 # models you can pass to --model (default marked)
agenteye agent ask "which agents errored most in the last day?"    # starts a chat; prints its short id
agenteye agent ask --chat <short-id> "and which tools did they call?"   # continue that chat
agenteye agent chats ; agenteye agent show <short-id>
agenteye agent rename <short-id> --title "error triage" ; agenteye agent delete <short-id>
```

## رموز الخروج

| الرمز | المعنى |
|---|---|
| 0 | نجاح |
| 1 | خطأ غير متوقع (مثل لوحة التحكم التي أرجعت 5xx) |
| 2 | خطأ الاستخدام (حجج غير صحيحة، أمر/علامة غير معروفة، اصطدام الاسم) |
| 3 | لا يمكن الوصول إلى لوحة التحكم |
| 4 | لم تسجل دخولك أو انتهت جلستك؛ شغّل `agenteye login` |
| 5 | تم المصادقة، لكن حسابك يفتقد الإذن المطلوب (تسميها الرسالة) |
| 6 | لم يتم العثور على المورد المطلوب (مثل معرّف جلسة أو حادث غير معروف) |

هذه تجعل واجهة السطر آمنة للنصوص: يمكن لوكيل ترميز فرع على `4` لمطالبتك بإعادة المصادقة، أو `5` لسطح الإذن المفقود. انظر [وصفات CLI للوكلاء](/ar/agenteye/cli-recipes) لأنماط معالجة رمز الخروج وأشكال مخرجات JSON.
