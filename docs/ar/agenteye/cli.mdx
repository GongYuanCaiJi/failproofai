تشغيل كل ميزات FailproofAI Observability من المحطة الطرفية أو نص برمجي: بدون الحاجة للعودة إلى لوحة التحكم. يستعلم CLI `agenteye` عن بيانات القائمة (الجلسات وسجلات الأحداث والتقييمات) ويدير المؤسسة (مفاتيح API والمستخدمين والإعدادات والتنبيهات والحوادث والاستعلامات المحفوظة)، لذا استخدمه عندما تريد أتمتة فحص أو دمج Observability في CI أو السماح لوكيل ترميز بفحص الإنتاج. يدعم كل أمر العلم `--json`، لذا يعمل بشكل متساوٍ سواء كنت في المحطة الطرفية أو لوكيل ترميز (Claude Code وCursor) ينفذ أوامر خارجية ويحلل النتائج.

باستخدام ملف ثنائي واحد يمكنك:

- **قراءة البيانات**: `sessions` و `events` و `evals` و `errors` (تصفية حسب الوقت والوكيل والبيئة والنقاط).
- **إدارة المؤسسة**: `keys` و `users` و `settings` و `alerts` و `incidents`.
- **تشغيل التحليلات**: SQL المحفوظة ومشغل استعلام مخصص (`query`).
- **السؤال مساعد ذكي**: نفس محلل القراءة فقط الذي تتحدث معه في لوحة التحكم (`agent`).

> **ملاحظة:** هذه واجهة سطر الأوامر `agenteye`، وهي أداة مختلفة عن مراقب جمع البيانات (`agenteye-collector`). تتحدث واجهة سطر الأوامر إلى لوحة التحكم الخاصة بك؛ المجمع يرسل الأحداث إلى الخادم.

## البدء السريع

من لا شيء إلى أول نتيجة في أربعة أسطر. وجّه واجهة سطر الأوامر نحو لوحة التحكم الخاصة بك، وسجّل الدخول، وأكّد هويتك، ثم اسحب آخر يوم من التشغيلات:

```bash
pipx install agenteye
agenteye --base-url https://agenteye.example.com login --email you@example.com   # emailed 6-digit code
agenteye whoami                                                                  # confirm user + active org
agenteye --json sessions --since 24h                                             # one row per agent run, last 24h
```

يطبع الأمر الأخير كائن JSON لأحدث الجلسات (الأحدث أولاً، محدود بـ 50 افتراضياً). مرّره إلى `jq` لتقسيمه، أو احذف `--json` للحصول على جدول مربع وملون. يحمل كل صف حالة التشغيل وإذا قيّمها المقيّم، درجات المقاييس الخاصة به (مختصرة هنا):

```json
{
  "sessions": [
    {
      "session_id": "run-8f2a",
      "agent_id": "checkout-bot",
      "environment": "prod",
      "status": "error",
      "scores": { "helpfulness": 0.42, "tool_efficiency": 0.55 },
      "event_count": 37,
      "started_at": "2026-07-16T09:14:02Z",
      "last_event_at": "2026-07-16T09:14:48Z"
    }
  ],
  "next_cursor": null
}
```

يشرح بقية هذه الصفحة كل جزء: [التثبيت](#installation) بمعزل عن الآخرين، [تسجيل الدخول](#authentication)، [الإعدادات](#configuration)، [الخيارات العامة والاتفاقيات](#global-options--conventions) التي يشاركها كل أمر، و[مرجع الأوامر الكامل](#command-reference).

## التثبيت

واجهة سطر الأوامر عبارة عن حزمة PyPI عامة تسمى **`agenteye`**. ثبّتها في بيئة معزولة بحيث يكون لها دائماً مكتباتها الخاصة:

```bash
pipx install agenteye
# or
uv tool install agenteye
```

تتطلب Python 3.10 أو إصدار أحدث. الأمر المثبت هو **`agenteye`**:

```bash
agenteye --version
agenteye --help
```

> **ملاحظة:** مكتبة FailproofAI Observability Python SDK أيضاً تستخدم اسم التوزيع `agenteye`. يمنع التثبيت عبر `pipx` أو `uv tool` (بدلاً من `pip install` في virtualenv مشترك) تصادماً بينهما. استخدام `pip install agenteye` البسيط جيد فقط إذا لم تكن SDK مثبتة في نفس البيئة.

## المصادقة

تقوم واجهة سطر الأوامر بالمصادقة إلى **لوحة التحكم** برمز لمرة واحدة يتم إرساله عبر البريد الإلكتروني:

```bash
agenteye login --email you@example.com
# A 6-digit code is emailed to you; paste it at the prompt.
```

يتم تخزين رمز الجلسة في `~/.agenteye/cli.json` (قابل للقراءة فقط من قبلك، الوضع `0600`) وصالح لمدة 24 ساعة افتراضياً. عند انتهاء الصلاحية، قم بتشغيل `agenteye login` مرة أخرى.

```bash
agenteye whoami     # show the current user, active org, and permissions
agenteye logout     # revoke the session and clear the stored token
```

لن يرمي `whoami` أخطاء عند وجود جلسة مفقودة أو منتهية الصلاحية؛ بدلاً من ذلك يبلّغ عن `logged_in: false`، لذا يمكن للنص البرمجي أو الوكيل استكشاف حالة المصادقة بأمان (يمكنه لا يزال الخروج بقيمة غير صفرية إذا لم يتم تعيين عنوان URL الأساسي أو كانت لوحة التحكم غير قابلة للوصول).

**المتطلبات:** يجب السماح لبريدك الإلكتروني بتسجيل الدخول إلى لوحة التحكم (استأذن من مسؤول FailproofAI Observability)، ويجب أن تكون لوحة التحكم قابلة للوصول على عنوان URL الأساسي الخاص بها (انظر [الإعدادات](#configuration)). إذا طلبت رمزاً ولم يصل، فعلى الأرجح لم يتم تفعيل بريدك الإلكتروني للوصول إلى لوحة التحكم بعد.

## اختيار المؤسسة (متعدد المستأجرين)

إذا كان حسابك ينتمي إلى أكثر من مؤسسة واحدة، فاختر النشطة **عند تسجيل الدخول**؛ يتم حفظها واستخدامها في كل أمر لاحق:

```bash
agenteye login --org acme           # authenticate and set the active tenant in one step
agenteye orgs list                  # the orgs you can access (the active one is marked)
agenteye orgs switch globex         # change the saved default
agenteye --org globex sessions      # override for a single command
```

إذا كنت تنتمي إلى مؤسسة واحدة بالضبط، يتم تحديدها تلقائياً ويمكنك تجاهل `--org` تماماً. إذا كنت تنتمي إلى عدة وليم تختر واحدة، تعرض واجهة سطر الأوامر قائمة وتطلب منك إعادة التشغيل مع `--org <slug>`. يتم إرسال المؤسسة النشطة إلى لوحة التحكم في كل طلب، وتتم حل الأذونات **لكل مؤسسة**؛ يعرض `agenteye whoami` المؤسسة النشطة والأذونات الخاصة بك فيها وجميع عضويتك.

## الإعدادات

| الإعداد | العلم | متغير البيئة | القيمة الافتراضية |
|---|---|---|---|
| عنوان URL الأساسي للوحة التحكم | `--base-url` | `AGENTEYE_DASHBOARD_URL` | **مطلوب** (بدون افتراضي) |
| المؤسسة/المستأجر النشط | `--org` | `AGENTEYE_ORG` | مختار عند تسجيل الدخول؛ محفوظ في `~/.agenteye/cli.json` |
| رمز الجلسة | `--token` | `AGENTEYE_CLI_TOKEN` | من `~/.agenteye/cli.json` |
| مخرجات JSON | `--json` | `AGENTEYE_CLI_JSON` | معطّل |
| تخطي التحقق من TLS | `--insecure` / `--secure` | `AGENTEYE_INSECURE` | معطّل (محفوظ عند تسجيل الدخول) |
| مهلة الطلب (ثواني) | `--timeout` | _(بدون)_ | 30 |
| تعطيل قياس الاستخدام | _(بدون)_ | `AGENTEYE_ANALYTICS_DISABLED` (أو `DO_NOT_TRACK`) | قياس الاستخدام معطّل حالياً؛ لا شيء يتم إرساله |

ترتيب الحل هو **العلم → متغير البيئة → ملف الإعدادات**. لا توجد قيمة افتراضية؛ يجب عليك توجيه واجهة سطر الأوامر نحو لوحة التحكم، إما لكل أمر (`--base-url https://agenteye.example.com`) أو مرة واحدة عبر البيئة (يتم حفظها أيضاً بعد أول `login` لك):

```bash
export AGENTEYE_DASHBOARD_URL=https://agenteye.example.com
```

يحترم دليل الإعدادات `AGENTEYE_HOME` (نفس الاتفاقية المستخدمة من قبل SDK والمجمع)؛ إذا تم تعيينها، يعيش `cli.json` في `$AGENTEYE_HOME/cli.json`.

### شهادات TLS موقّعة ذاتياً أو داخلية

إذا تم توفير لوحة التحكم الخاصة بك عبر HTTPS بشهادة موقّعة ذاتياً أو داخلية (على سبيل المثال، اسم مضيف موازن تحميل خام)، يرفضها التحقق من TLS برسالة خطأ `CERTIFICATE_VERIFY_FAILED`. مرّر `--insecure` لتخطي التحقق من الشهادة:

```bash
agenteye --base-url https://agenteye.internal --insecure login
```

يتم **حفظ `--insecure` إلى `cli.json` عند تسجيل الدخول**، لذا تتخطى الأوامر اللاحقة التحقق تلقائياً؛ لا تضطر إلى تكرار العلم. مرّر `--secure` لاستدعاء موثّق لمرة واحدة، أو لحفظ التحقق مرة أخرى عند تسجيل الدخول التالي. تطبع واجهة سطر الأوامر تحذيراً إلى stderr قبل أي أمر يتصل بلوحة التحكم أثناء تعطيل التحقق. يزيل تخطي التحقق الحماية من هجمات الوسيط؛ تأكد من أنك تثق في مسار الشبكة إلى لوحة التحكم (VPN أو شبكة فرعية خاصة وما إلى ذلك) قبل الاعتماد عليها.

## قياس الاستخدام والخصوصية

> **ملاحظة:** واجهة سطر الأوامر المرسلة **لا تُرسل أي قياس استخدام اليوم.** مفتاح إيقاف رئيسي قيد التشغيل، لذا لا يتم نقل أي شيء بغض النظر عن بيئتك. يصف القسم أدناه إمكانية الاستبعاد في حالة تفعيل قياس الاستخدام في المستقبل.

حتى عند التفعيل، سيكون قياس الاستخدام **تحليلات الاستخدام المجهولة فقط**، لا أبداً بيانات الوكيل أو الجلسة أو الحدث:

- **بيانات الوكيل أو الجلسة أو الحدث لا تغادر البنية الأساسية أبداً.** سيتم الإبلاغ عن استخدام واجهة سطر الأوامر فقط: اسم الأمر والأمر الفرعي (على سبيل المثال `keys create`)، و**أسماء** الأعلام التي استخدمتها (لا تبلغ عن قيمها أبداً)، حالة النجاح/الخروج والمدة، بالإضافة إلى حدث لكل إجراء للتغييرات (على سبيل المثال `api_key_created` و `query_run`) يحمل فقط أسماء/تعدادات ثابتة وعدادات غير دقيقة. عنوان URL لوحة التحكم والرمز ورمز الجلسة والبريد الإلكتروني وشعار المؤسسة ومعرفات الموارد و SQL وأسرار المفاتيح وعوامل تصفية الاستعلام **لن** يتم إرسالها أبداً. سيتم تحديد المشغلين فقط بمعرف داخلي معتم، لا أبداً بالبريد الإلكتروني.
- **استبعد مقدماً** بتعيين `AGENTEYE_ANALYTICS_DISABLED=1` في بيئة واجهة سطر الأوامر (تحترم واجهة سطر الأوامر أيضاً اتفاقية `DO_NOT_TRACK=1` متعددة الأدوات). يسري هذا من اللحظة التي يتم فيها تفعيل قياس الاستخدام أبداً، لذا يمكن للبيئة الواعية بالخصوصية البقاء مستبعدة بشكل دائم.
- إذا تم تفعيل قياس الاستخدام، ستُرسل واجهة سطر الأوامر مباشرة إلى PostHog (`https://us.i.posthog.com`)؛ ستُسكت آلة محظور فيها هذا المضيف بصمت ولن تتأثر واجهة سطر الأوامر.

## الخيارات العامة والاتفاقيات

اقرأ هذا مرة واحدة؛ ينطبق على كل أمر.

- **الخيارات العامة تسبق الأمر.** `agenteye --json sessions` صحيح؛ `agenteye sessions --json` خطأ في الاستخدام. الخيارات العامة هي `--json` و `--base-url` و `--org` و `--token` و `--insecure`/`--secure` و `--timeout` و `--quiet` و `--no-color`.
- **`--json` يطبع JSON نقي إلى stdout ولا شيء آخر.** تذهب أسطر الحالة البشرية والتحذيرات والأخطاء إلى **stderr**، لذا يبقى التقاط مخرجات `--json` نظيفاً للأنابيب إلى `jq` حتى عند ظهور سطر الحالة. بدون `--json` تحصل على عرض مربع وملون للعيون البشرية.
- **اكتشف مع `--help`.** كل أمر وأمر فرعي له `--help` (و اسم مستعار `-h`): `agenteye -h` و `agenteye sessions -h` و `agenteye keys create -h`. يسرد الملخص العلوي أيضاً رموز الخروج والخيارات العامة. لا توجد سطح موحد قابل للقراءة آلياً عمومياً؛ استخدم `--help` لكل أمر، بالإضافة إلى `agenteye query schema` و `agenteye settings schema` لتلك السجلات المخصصة للنطاق.
- **التأكيدات تتخطى تلقائياً للنصوص البرمجية والوكلاء.** تطلب أوامر الإنشاء/التحديث/الحذف تأكيد "هل أنت متأكد؟" في المحطة الطرفية التفاعلية، لكن **تتخطى التأكيد تلقائياً تحت `--json` أو عندما لا يكون stdin TTY** (TTY عبارة عن جلسة محطة طرفية تفاعلية؛ الأنابيب أو تشغيل CI ليست كذلك)، لذا النصوص البرمجية والوكلاء لا تتعلق أبداً. مرّر `--yes`/`-y` لتخطيها صراحة. لأن التأكيد لن ينطلق لوكيل، يجب على الوكيل تأكيد الإجراءات المدمرة مع الإنسان أولاً.
- **الترقيم:** النتائج هي الأحدث أولاً والترقيم المستند إلى المؤشر (تُرجع كل صفحة رمزاً تستخدمه لجلب التالي). يقبّع `--limit N` (اسم مستعار `-n`) الصفوف و **يافتراضي 50**؛ `--all` ترقيم تلقائي (في أجزاء 200 صف) **حتى `--limit`**، لذا `--all` بسيط يتوقف بعد 50. لعملية مسح كاملة مرّر حد أعلى صريح عالٍ: `--all --limit 1000`. `--page-size N` يتحكم في الجزء لكل طلب (أقصى 200)؛ `--cursor <id>` يستأنف من `next_cursor` لصفحة سابقة.
- **عوامل تصفية الوقت:** `--since` يأخذ نافذة نسبية: `15m` و `1h` و `6h` و `24h` و `7d` أو `all` (الإعدادات المسبقة للوحة التحكم). لنطاق أطول أو مخصص (على سبيل المثال آخر 30 يوماً)، استخدم `--from`/`--to`: الطوابع الزمنية UTC الصريحة بصيغة ISO-8601 **مع `T` ومنطقة زمنية** (على سبيل المثال `2026-06-01T00:00:00Z`) التي تتجاوز `--since`. قيمة مفصولة بمسافة أو بدون منطقة زمنية هي خطأ في الاستخدام.
- **`--fields a,b,c`** (على `events` و `sessions` و `evals` و `errors`) يقيّد المخرجات إلى تلك المفاتيح، لكل من الجدول و `--json`. يتم رفض الأسماء المجهولة برسالة القائمة الصحيحة، طريقة رخيصة لاكتشاف أسماء الحقول.
- **`--file payload.json`** (أو `--file -` لقراءة stdin) يوفر جسم طلب JSON كامل حيث يحمل المورد شكلاً معقداً (على `alerts create/update` و `settings set` و `users create/update`). استخدم SQL الاستعلام المحفوظ `--sql @file.sql` بدلاً من ذلك.
- **عوامل التصفية ذات القيم المتعددة** مفصولة بفاصلات → مطابقة كمجموعة (اتحاد داخل عامل تصفية واحد، و بين عوامل التصفية): `--event-type tool_use,tool_result`. خيارات النقر ليست متغيرة، لذا `--add a b` ينقطع. استخدم `--add a,b` أو كرّر العلم (`--add a --add b`) أو اقتباس (`--add "a b"`).

## مرجع الأوامر

### ستستخدم هذه الأوامر الخمسة أكثر

يتم تشغيل معظم العمل اليومي من خلال حفنة من أوامر القراءة. ابدأ هنا، ثم توصّل بالسطح الكامل أدناه عندما تحتاجه:

| الأمر | ماذا يفعل | جرّبه |
|---|---|---|
| `sessions` | صف واحد لكل تشغيل وكيل: الوقت والبيئة والوكيل والحالة وأحدث نقاط. | `agenteye --json sessions --since 24h --status error` |
| `events` | مسار خطوة بخطوة خام داخل تشغيل (أضف `--full` للأحمال). | `agenteye --json events --session-id run-001 --all` |
| `evals` | نتائج التقييم والدرجات؛ `--aggregate` يجمعها. | `agenteye --json evals --aggregate --since 7d --env prod` |
| `errors` | أحداث مخطئة فقط؛ `--aggregate` للعدد حسب النوع. | `agenteye --json errors --since 24h --aggregate` |
| `list` | اكتشف القيم الصحيحة للتصفية (وكلاء وبيئات ونماذج وما إلى ذلك). | `agenteye list agents` |

### كل شيء يمكن لـ CLI أن يفعله

يتبع السطح الكامل. تحتوي واجهة سطر الأوامر على **18 أمر من المستوى الأعلى**. تقبل جميع أوامر القراءة `--json` والخيارات العامة أعلاه؛ قم بتشغيل `agenteye <command> -h` (أو `<command> <subcommand> -h`) للقائمة الشاملة للأعلام وشكل JSON لأي واحد.

### الهوية: `login` · `logout` · `whoami` · `orgs` · `version` · `help`

```bash
agenteye login --email you@example.com [--org acme]   # emailed one-time code; saves the session
agenteye logout                                       # clear the saved session on this machine
agenteye whoami                                       # current user, active org, permissions
agenteye version                                      # print the CLI version (same as --version)
agenteye help                                         # top-level help (same as --help)
```

يفحص `orgs` ويبدّل المستأجر النشط:

```bash
agenteye orgs list        # your orgs + your role in each (active one marked)
agenteye orgs switch acme # change the saved active org (omit the slug to pick from a list on a TTY)
agenteye orgs current     # identity card for the active org
agenteye orgs perms       # your permissions in the active org, grouped by resource
```

### ملاحظة (قراءة فقط): `events` · `sessions` · `evals` · `errors` · `list`

لا يحتاج أي من هذه إلى تأكيد. عوامل تصفية مشتركة: `--session-id` و `--agent-id` و `--env` (**ليس** `--environment`)، ونطاق الوقت (`--since` / `--from` / `--to`).

```bash
# events (alias: the raw per-step trail), newest first
agenteye --json events --session-id run-001 --event-type tool_use,tool_result --all --limit 1000
agenteye --json events --since 1h --search timeout --all | jq '.events[].payload'

# sessions: one row per agent run (time/env/agent/session/status; no score filtering)
agenteye --json sessions --since 24h --status error
agenteye --json sessions --agent-id checkout-bot --env prod --all --limit 1000

# evals: evaluation results + scores; --score filters by metric, --aggregate rolls up
agenteye --json evals --score helpfulness:0.5..0.8 --score tool_efficiency:..0.3
agenteye --json evals --aggregate --since 7d --env prod        # status mix + per-key score stats

# errors: errored events; --aggregate for counts/sessions/agents/last-seen
agenteye --json errors --since 24h --aggregate
agenteye --json errors --since 24h --error-type timeout --all --limit 1000

# list: discover valid filter values before you filter
agenteye list envs        # also: agents event_types score_filters models hooks tools error_types
```

`--score KEY:MIN..MAX` (على **`evals`** وليس `sessions`) قابل للتكرار ومجموعة AND؛ كلا الحدين اختياري (`..0.5` يعني ≤ 0.5 و `0.9..` يعني ≥ 0.9). حتى 20 عامل تصفية نقاط لكل طلب. `evals --scores-full` هو علم العرض **للجدول البشري فقط**؛ يعرض كل زوج نقاط بدلاً من الأولى قليلة بالإضافة إلى عد `+N`. ليس له تأثير تحت `--json` الذي يُرجع دائماً كائن النقاط الكامل. لقراءة **جلسة واحدة من البداية إلى النهاية**، دمّج مسار الحدث مع تقييمه:

```bash
agenteye --json events --session-id run-001 --all --limit 1000 | jq '.events[] | {ts, event_type}'
agenteye --json evals  --session-id run-001                       # its scores + status
```

### إدارة (ممنوعة بحسب الأذونات): `keys` · `users` · `settings` · `alerts` · `incidents`

**`keys`**: مفاتيح API. يتم إنشاء السر محلياً وإرساله إلى الخادم (الذي يخزن فقط التجزئة) و**يعرض مرة واحدة** عند الإنشاء/إعادة التوليد؛ التقطه بعد ذلك. مع `--json` يظهر فقط في حقل `key`. المرجع حسب **الاسم**.

```bash
agenteye keys list                                   # active keys first, then revoked
agenteye keys show ci-bot
agenteye keys create ci-bot --add events:read.add    # scope to what you need; prints the secret ONCE
agenteye keys create ops --permission-set standard --remove queries:run   # seed a preset, then trim
agenteye keys update ci-bot --add evaluations:read --yes
agenteye keys regenerate ci-bot --yes                # rotate the secret (the old one stops working)
agenteye keys disable ci-bot --yes                   # revoke
```

تعمل الأذونات كـ `(permission-set ∪ --add) − --remove`. الرموز هي `slug:action` (على سبيل المثال `events:read`) أو `slug:action.action` لتوسيع عدة على مورد واحد (`events:read.add` → `events:read` و `events:add`). الإعدادات المسبقة: `read-only` و `standard` و `admin`. الأذونات البشرية فقط (`keys:update`) لا يمكن منحها لمفتاح.

**`users`**: أعضاء المؤسسة، المرجع بـ **البريد الإلكتروني** (معرّف UUID يُقبل أيضاً).

```bash
agenteye users list [--active-only]
agenteye users show dev@corp.com
agenteye users create dev@corp.com --permission-set standard
agenteye users update dev@corp.com --add alerts:write --remove queries:delete   # predicts + confirms
agenteye users disable dev@corp.com --yes            # has protected/self guards
agenteye users enable dev@corp.com
```

**`settings`**: سجل ثابت (تقرأ وتغيّر المفاتيح الموجودة؛ لا يمكنك إنشاء مفاتيح جديدة).

```bash
agenteye settings list                               # key · value · type · updated (secrets masked)
agenteye settings schema                             # what each key accepts (type · range · description)
agenteye settings set session_ttl_secs --value 86400 --yes
```

**`alerts`**: تعريفات التنبيه، المرجع حسب **الاسم**. `create` يأخذ NAME موضعي بالإضافة إلى أعلام أو جسم JSON كامل عبر `--file`.

```bash
agenteye alerts list
agenteye alerts show high-errors
agenteye alerts create high-errors --file alert.json          # NAME is required (positional)
agenteye alerts update high-errors --severity critical --yes
agenteye alerts test high-errors --yes                        # fire a test notification
agenteye alerts delete high-errors --yes
```

**`incidents`**: حوادث التنبيهات، المرجع بـ ID (تُقبل الـ IDs القصيرة). `show` يطبع سجل النشاط الكامل؛ اقرأه قبل التصرف.

```bash
agenteye incidents list --state firing                # also: acknowledged, resolved
agenteye incidents count
agenteye incidents show <id>
agenteye incidents ack <id>
agenteye incidents assign <id> you@corp.com           # assignee must be an operator
agenteye incidents resolve <id> --yes
agenteye incidents open --alert-id <id> --severity critical    # open one manually against an alert
agenteye incidents comment-add <id> "root cause: upstream 5xx"
agenteye incidents comment-list <id> ; agenteye incidents comment-delete <id> <comment-id>
agenteye incidents subscribe <id> ; agenteye incidents unsubscribe <id> ; agenteye incidents subscribers <id>
```

### التحليلات والمساعد: `query` · `agent`

**`query`**: SQL محفوظة ضد متجر التحليلات الخاص بك بالإضافة إلى مشغل مخصص. تُشار الاستعلامات المحفوظة بـ **الاسم**؛ يتم التحقق من SQL من قِبل الخادم (SELECT/WITH فقط وانتهاء الجلسة وغطاء الصفوف).

```bash
agenteye query schema [TABLE]                         # column layout of the analytics views
agenteye query run --sql "select count(*) from analytics.events"
agenteye query run errs --arg prod --limit 100        # run a saved query + a positional $1
agenteye query list ; agenteye query show errs
agenteye query create errs --sql @errs.sql --description "errored events (24h)"
agenteye query update errs --sql @errs.sql --yes ; agenteye query delete errs --yes
```

**`agent`**: يتحدث إلى **مساعد ذكي** مدمج (نفس محلل القراءة فقط الذي يمكنك الدردشة معه في لوحة التحكم). تُشار الدردشات بـ chat-id قصير (دقة البادئة).

```bash
agenteye agent health                                 # is the AI assistant configured/reachable
agenteye agent models                                 # models you can pass to --model (default marked)
agenteye agent ask "which agents errored most in the last day?"    # starts a chat; prints its short id
agenteye agent ask --chat <short-id> "and which tools did they call?"   # continue that chat
agenteye agent chats ; agenteye agent show <short-id>
agenteye agent rename <short-id> --title "error triage" ; agenteye agent delete <short-id>
```

## رموز الخروج

| الرمز | المعنى |
|---|---|
| 0 | نجاح |
| 1 | خطأ غير متوقع (على سبيل المثال لوحة التحكم أرجعت 5xx) |
| 2 | خطأ في الاستخدام (أوامر أو أعلام غير صحيحة وأسماء غير معروفة أو تصادم) |
| 3 | لا يمكن الوصول إلى لوحة التحكم |
| 4 | غير مسجل دخول أو انتهت صلاحية الجلسة؛ قم بتشغيل `agenteye login` |
| 5 | موثّق لكن حسابك يفتقد الأذن المطلوبة (تذكرها الرسالة) |
| 6 | لم يتم العثور على المورد المطلوب (على سبيل المثال معرّف جلسة أو حادثة غير معروف) |

هذا يجعل واجهة سطر الأوامر آمنة للنص البرمجي: يمكن لوكيل ترميز أن يتفرع على `4` لمطالبتك بإعادة المصادقة أو `5` لعرض الأذن المفقودة. انظر [CLI وصفات للوكلاء](/ar/agenteye/cli-recipes) لأنماط معالجة رموز الخروج وأشكال مخرجات JSON.
