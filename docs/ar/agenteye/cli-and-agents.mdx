نشرك الكامل في FailproofAI Observability، على بعد أمر واحد فقط. تحقق من الإنتاج، أنشئ مفتاح API، أو أقِر حادثة دون الحاجة لترك محطتك الطرفية، ثم أتمتِ أي منها في CI أو دع وكيل ترميز يفعلها نيابة عنك بلغة إنجليزية بسيطة.

```bash
pipx install agenteye
agenteye login --email you@example.com      # a 6-digit code lands in your inbox
agenteye --json sessions --since 24h         # every agent run from the last day, newest first
```

*واجهة سطر الأوامر `agenteye` تتحدث إلى لوحة التحكم الخاصة بك. وهي أداة مختلفة عن المجمِّع الذي يرسل الأحداث إلى الخادم.*

## نشرك الكامل، أمر واحد فقط

توقف عن القفز بين عدة نوافذ للإجابة عن سؤال سريع. واجهة سطر الأوامر `agenteye` تقرأ بيانتك وتدير منظمتك من ملف ثنائي واحد، بحيث يصبح الفحص الذي كان يتطلب النقر عبر لوحة التحكم سطراً واحداً يمكنك تشغيله مجدداً أو إنشاء اختصار له أو لصقه في دليل التشغيل. تحصل على أربع واجهات:

- **اقرأ بيانتك:** `sessions` و `events` و `evals` و `errors`، مُصفّاة حسب الوقت والوكيل والبيئة.
- **أدِر منظمتك:** `keys` و `users` و `settings` و `alerts` و `incidents`.
- **شغِّل التحليلات:** SQL محفوظة بالإضافة إلى محرك استعلام مخصص `query` على بيانات أحداثك.
- **اسأل المساعد:** `agent ask` يصل إلى نفس محلل القراءة فقط الذي تتحدث معه في لوحة التحكم.

ثبتها مرة واحدة مع `pipx`، وسجل الدخول باستخدام رمز 6 أرقام يصل بالبريد الإلكتروني، وأنت جاهز. الجلسة تستمر حوالي يوم واحد؛ شغِّل `agenteye login` مجدداً عند انتهائها. استخدمه للتحقق السريع من الإنتاج أو توفير مفتاح أو فحص حادثة حرجة، وكل ذلك دون فتح متصفح:

```bash
agenteye errors --since 24h --aggregate      # what is breaking, grouped by error type
agenteye incidents list --state firing       # what is on fire right now
agenteye keys create ci --add events:add     # a key that can only push events, secret shown once
```

عادة واحدة يجب معرفتها: الخيارات العامة مثل `--json` تأتي قبل الأمر. `agenteye --json sessions` صحيح؛ `agenteye sessions --json` غير صحيح.

## أتمتِها ووصِّلها في CI

كل أمر يأخذ `--json`، وهذا يغير كل شيء. يذهب JSON نظيف إلى stdout بينما تحذيرات الحالة البشرية تذهب إلى stderr، لذا يمكن لالتقاط `--json` أن ينقل مباشرة إلى `jq` بدون سطر شاذ لإزالته. هذا ما يجعل واجهة سطر الأوامر جيدة بنفس القدر لك في موجه وللمحلل الذي يحلل الإخراج:

```bash
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'
```

وهي مبنية للعمل دون مراقبة. تخطي رموز التأكيد تلقائياً عند عدم وجود محطة طرفية مرفقة، لذا لا شيء يتوقف في خط أنابيب، وكل أمر يُرجع رمز خروج ذو مغزى: `0` نجح، `4` لم تسجل الدخول، `5` تفتقد إذن (الرسالة تسميه، على سبيل المثال `alerts:write`)، `3` لوحة التحكم غير متاحة. يمكن للسكريبت أن يتفرع على `4` للمصادقة مجدداً أو على `5` لإخبارك بالضبط ما الذي تطلبه من مسؤول، بدلاً من الفشل العمياء.

## دع وكيل ترميز يقودها بلغة إنجليزية بسيطة

بل الأفضل من ذلك أنه يجب ألا تضطر لتذكر أي من هذه الأعلام على الإطلاق. **مهارة CLI** هي مجلد مهارة وكيل صغير يسمى `agenteye-cli` يعلم وكيل ترميز مثل Claude Code أو Codex كيفية قيادة واجهة سطر الأوامر من طلبات باللغة الإنجليزية البسيطة. اسأل "هل حدث أي شيء خاطئ اليوم؟" والوكيل يختار الأمر وينفذه كأنك أنت ويجيب بصيغة نثرية.

بالنسبة إلى Claude Code، ألق مجلد `agenteye-cli` في `~/.claude/skills/` وسيتم اكتشافه تلقائياً. توفر FailproofAI Observability المجلد؛ لا توجد أي عمليات تثبيت إضافية، لأنه يقود واجهة سطر الأوامر التي ثبتتها بالفعل فقط. سجل الدخول بنفسك أولاً: المهارة لا يمكنها إكمال تسجيل الدخول الذي يأتي عبر البريد الإلكتروني نيابة عنك.

لأن الوكيل يشغل واجهة سطر الأوامر كأنك أنت، يمكنه فعل كل شيء يسمح به تسجيل الدخول الخاص بك، القراءة والكتابة على حد سواء: إنشاء مفاتيح وتغيير الإعدادات وحل الحوادث. رمز التأكيد "هل أنت متأكد؟" في واجهة سطر الأوامر لا ينطلق للوكيل، لذا تمت كتابة المهارة لتوضيح الأمر الدقيق والانتظار لموافقتك قبل أي تغيير. أنت خطوة التأكيد.

```text
you    Why did session run-001 fail?

agent  Running: agenteye --json events --session-id run-001 --all
       The checkout-agent hit a TimeoutError on its third tool call.
```

القراءات تبقى فورية، وكل عملية كتابة تتوقف انتظاراً لك:

```text
you    Give CI a key that can only push events.

agent  This creates an API key. I'll run:
         agenteye keys create ci --add events:add
       OK to proceed?

you    yes

agent  Done. Key "ci" created with events:add only. The secret is shown once, so store it now.
```
