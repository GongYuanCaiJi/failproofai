نشر Failproof AI Observability كاملاً، بأمر واحد فقط. تحقق من الإنتاج، أنشئ مفتاح API، أو أقرّ حادثة دون مغادرة محطة العمل لديك، ثم أتمتة أي من ذلك في CI، أو اترك وكيل ترميز يفعل ذلك نيابة عنك باللغة الطبيعية.

```bash
pipx install agenteye
agenteye login --email you@example.com      # a 6-digit code lands in your inbox
agenteye --json sessions --since 24h         # every agent run from the last day, newest first
```

*يتحدث CLI `agenteye` إلى لوحة المعلومات لديك. وهي أداة مختلفة عن جامع البيانات، الذي يرسل الأحداث إلى الخادم.*

## نشرك كاملاً، بأمر واحد فقط

توقف عن الانتقال بين التبويبات للإجابة عن سؤال سريع. يقرأ CLI `agenteye` بيانات الإدارة الخاصة بك من ملف ثنائي واحد، لذلك يصبح الفحص الذي كان يتطلب النقر عبر لوحة المعلومات سطراً واحداً يمكنك إعادة تشغيله أو تحديثه أو إدراجه في دليل العمليات. تحصل على أربع واجهات:

- **اقرأ بياناتك:** `sessions`، و`events`، و`evals`، و`errors`، مرشحة حسب الوقت والوكيل والبيئة.
- **إدارة مؤسستك:** `keys`، و`users`، و`settings`، و`alerts`، و`incidents`.
- **تشغيل التحليلات:** SQL محفوظة بالإضافة إلى عداء `query` مخصص على بيانات الأحداث الخاصة بك.
- **اسأل المساعد:** `agent ask` يتصل بنفس محلل القراءة فقط الذي تتحدث معه في لوحة المعلومات.

ثبّته مرة واحدة باستخدام `pipx`، وسجل الدخول باستخدام رمز 6 أرقام تم إرساله عبر البريد الإلكتروني، وأنت جاهز. تستغرق الجلسة حوالي يوم واحد؛ أعد تشغيل `agenteye login` عند انتهاء صلاحيتها. استخدمه للتحقق من الإنتاج أو توفير مفتاح أو تصنيف حادثة حريق، كل ذلك دون فتح متصفح:

```bash
agenteye errors --since 24h --aggregate      # what is breaking, grouped by error type
agenteye incidents list --state firing       # what is on fire right now
agenteye keys create ci --add events:add     # a key that can only push events, secret shown once
```

عادة واحدة يجب معرفتها: الخيارات العامة مثل `--json` تأتي قبل الأمر. `agenteye --json sessions` صحيح؛ `agenteye sessions --json` غير صحيح.

## أتمتة ذلك، واربطه في CI

كل أمر يأخذ `--json`، وهذا يغير كل شيء. JSON نظيفة تذهب إلى stdout بينما تحذيرات الحالة والإنسان تذهب إلى stderr، لذلك يمر التقاط `--json` مباشرة إلى `jq` بدون سطر ضال لتجريده. هذا ما يجعل CLI مفيداً لك عند الموجه وللوكيل الترميز الذي يحلل الإخراج:

```bash
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'
```

تم بناؤه للعمل دون إشراف. تخطي طلبات التأكيد تلقائياً عند عدم إرفاق محطة عمل، لذا لا شيء معلق في خط أنابيب، وكل أمر يعيد رمز خروج معنى: `0` نجاح، `4` لم تسجل الدخول، `5` تفتقد إذن (الرسالة تسميها، على سبيل المثال `alerts:write`)، `3` لوحة المعلومات غير قابلة للوصول. يمكن للبرنامج النصي أن يتفرع على `4` لإعادة المصادقة أو `5` لإخبارك بالضبط ما تطلبه من المسؤول، بدلاً من الفشل الأعمى.

## دع وكيل ترميز يقودها باللغة الإنجليزية العادية

الأفضل من ذلك، لا يجب عليك تذكر أي من هذه الأعلام على الإطلاق. **مهارة CLI** هي مجلد مهارة وكيل صغير يسمى `agenteye-cli` يعلم وكيل ترميز مثل Claude Code أو Codex قيادة CLI من طلبات باللغة الإنجليزية العادية. اسأل "هل هناك أي شيء معطوب اليوم؟" ويختار الوكيل الأمر، ويشغله باسمك، ويجيب بنثر.

بالنسبة إلى Claude Code، أسقط مجلد `agenteye-cli` في `~/.claude/skills/` وسيتم اكتشافه تلقائياً. توفر Failproof AI Observability المجلد؛ لا يوجد شيء إضافي للتثبيت، لأنه يقود فقط CLI الذي ثبتته بالفعل. سجل الدخول بنفسك أولاً: لا يمكن للمهارة إكمال تسجيل الدخول برمز البريد الإلكتروني نيابة عنك.

لأن الوكيل يشغل CLI كأنت، فيمكنه فعل كل شيء تسمح به بيانات اعتماداتك، قراءة وكتابة على حد سواء: إنشاء مفاتيح، تغيير الإعدادات، حل الحوادث. لا يتم تشغيل موجه "هل أنت متأكد؟" للوكيل، لذلك تمت كتابة المهارة لتوضيح الأمر الدقيق والانتظار للموافقة قبل أي تغيير. أنت خطوة التأكيد.

```text
you    Why did session run-001 fail?

agent  Running: agenteye --json events --session-id run-001 --all
       The checkout-agent hit a TimeoutError on its third tool call.
```

البقاء سريعة، وكل كتابة توقف تنتظرك:

```text
you    Give CI a key that can only push events.

agent  This creates an API key. I'll run:
         agenteye keys create ci --add events:add
       OK to proceed?

you    yes

agent  Done. Key "ci" created with events:add only. The secret is shown once, so store it now.
```
