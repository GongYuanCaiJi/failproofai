اسحب بيانات الجلسة والحدث والتقييم (وشغّل إعادة التقييمات) مباشرة من نص برمجي أو وكيل ترميز، مع JSON نظيف على stdout يتدفق مباشرة إلى `jq`. تحول هذه الوصفات بيانات Failproof AI Observability إلى شيء يمكن لمستخدم الطرفية أو وكيل ترميز AI (Claude Code، Cursor) أن يستعلم عنه ويأتمته، بدون الضغط من خلال لوحة التحكم.

الأنماط أدناه جاهزة للنسخ واللصق لواجهة سطر أوامر Failproof AI Observability (`agenteye`). للتثبيت والمصادقة وقائمة الخيارات الكاملة، انظر [CLI](/ar/agenteye/cli)؛ شغّل `agenteye -h` أو `agenteye <command> -h` للمساعدة المضمنة.

## القواعد الذهبية

1. **تذهب الخيارات العامة *قبل* الأمر.** `agenteye --json sessions` صحيح؛ `agenteye sessions --json` ليس كذلك. الخيارات العامة هي `--json`, `--base-url`, `--org`, `--token`, `--insecure`/`--secure`, `--timeout`, `--quiet`, `--no-color`.
2. **مرر `--json` كلما كنت تحلل الإخراج.** البيانات تذهب إلى **stdout** كـ JSON؛ حالة الإنسان والأخطاء تذهب إلى **stderr**، لذا يبقى stdout نظيفًا للتدفق إلى `jq`.
3. **افرع على رمز الخروج**، وليس على نص stderr: `0` حسناً · `1` خطأ غير متوقع · `2` وسيطات سيئة · `3` لا يمكن الوصول إلى لوحة التحكم · `4` لم تسجل الدخول أو انتهت الصلاحية · `5` إذن مفقود · `6` المورد غير موجود.
4. **اكتشف باستخدام `-h`.** كل أمر يوثّق عوامل التصفية وتنسيقات القيم وشكل JSON.

## إعداد لمرة واحدة

```bash
export AGENTEYE_DASHBOARD_URL=https://agenteye.example.com   # حتى لا تكرر --base-url
agenteye login --email you@example.com                       # الصق الكود المُرسل بالبريد الإلكتروني؛ صالح ~24h
```

## تأكد من المصادقة قبل العمل

`whoami` لا يخطئ أبدًا في جلسة مفقودة أو منتهية الصلاحية؛ بدلاً من ذلك يُبلّغ عن `logged_in:false`، لذا يمكن لوكيل أن يستكشف حالة المصادقة بأمان. (لا يزال يمكنه الخروج برمز غير صفري إذا لم يتم تعيين عنوان URL أساسي أو كانت لوحة التحكم غير قابلة للوصول.)

```bash
if [ "$(agenteye --json whoami | jq -r .logged_in)" != "true" ]; then
  echo "Not authenticated. Run: agenteye login" >&2; exit 1
fi
```

## ابحث عن الجلسات الفاشلة أو منخفضة الدرجات

```bash
# جلسات في آخر 24 ساعة حيث حدث خطأ في التقييم
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'

# التقييمات التي تسجل <= 0.5 على المفيدية، لوكيل واحد
agenteye --json evals --agent-id checkout-bot --score helpfulness:..0.5 \
  | jq '.evaluations[] | {session_id, scores}'
```

تصفية الدرجات تعيش على **`evals`**، وليس `sessions`. `--score KEY:MIN..MAX` قابل للتكرار ومجموع AND؛ أي حد اختياري (`..0.5` يعني ≤ 0.5، `0.9..` يعني ≥ 0.9). يمكنك تمرير ما يصل إلى 20 مرشح درجة لكل طلب؛ المزيد يرجع HTTP 400. `sessions` يشارك مرشحات `--env`، `--status`، `--agent-id`، `--session-id`، ونطاق الوقت مع `evals`، لكن ليس لديه `--score`.

## اقرأ جلسة واحدة من البداية إلى النهاية

لا يوجد أمر `session show` واحد. ادمج مسار الحدث مع تقييم الجلسة:

```bash
# أحدث تقييم للجلسة (الحالة + الدرجات)
agenteye --json evals --session-id run-001 | jq '.evaluations[0] | {status, scores}'

# كل حدث في التشغيل (ارفع --limit للمسح الكامل)
agenteye --json events --session-id run-001 --all --limit 1000 | jq '.events[] | {ts, event_type}'

# فقط استدعاءات الأدوات في جلسة (--full مطلوب للحصول على الحمول الخام)
agenteye --json events --full --session-id run-001 --event-type tool_use,tool_result --all \
  | jq '.events[].payload'
```

> **ملاحظة:** بشكل افتراضي، `events` يقرأ خلاصة سريعة خالية من الحمول. كل حدث يحمل ملخص `summary` محسوب من الخادم بسطر واحد بالإضافة إلى علامات مثل `is_error` وعدادات الرموز، لكن `payload` يعود كـ `{}`. لسحب الحمول الخام، أضف `--full` (أو `--fields payload`). الخلاصة الكاملة أبطأ على نطاق واسع، لذا احبسها: ادمج `--full` مع `--session-id` واحد.

## جلب كل شيء (الترقيم)

النتائج الأحدث أولاً والترقيم عن طريق المؤشر.

```bash
# لقطة واحدة: جلب ما يصل إلى 500 صف في صفحات 200 صف
agenteye --json events --session-id run-001 --limit 500 --all > events.json

# الترقيم اليدوي: أرجع المؤشر التالي
page=$(agenteye --json events --limit 100)
cursor=$(echo "$page" | jq -r '.next_cursor // empty')
[ -n "$cursor" ] && agenteye --json events --limit 100 --cursor "$cursor"
```

## قلل الإخراج باستخدام --fields

اقصر المفاتيح (في الجدول و`--json`) لتقليل ما يجب على الوكيل قراءته.

```bash
agenteye --json sessions --since 7d --fields session_id,status,scores | jq -c '.sessions[]'
agenteye --json events --session-id run-001 --fields ts,event_type --all
```

أسماء الحقول غير المعروفة يتم رفضها (خروج `2`) مع القائمة الصحيحة، طريقة رخيصة لاكتشاف أسماء الحقول.

## اكتشف قيم المرشح الصحيحة

```bash
agenteye --json list envs | jq -r '.values[]'           # قيم --env
agenteye --json list tools | jq -r '.values[]'          # أسماء الأدوات؛ أيضًا الوكلاء والنماذج وأنواع الأحداث، ...
agenteye --json list score_filters | jq -r '.values[]'  # KEY صحيح لـ --score KEY:MIN..MAX
```

## اختر منظمتك (متعدد المستأجرين)

إذا كنت تنتمي إلى أكثر من منظمة واحدة، اختر المستأجر النشط عند تسجيل الدخول (يتم حفظه):

```bash
agenteye login --org acme --email you@corp.com   # عيّن المستأجر في نفس خطوة تسجيل الدخول
agenteye --json orgs list | jq -r '.orgs[].org_slug'
agenteye --org globex --json sessions --since 24h   # تجاوز لأمر واحد
```

تسجيل دخول متعدد المنظمات بدون `--org` ينتهي برمز غير صفري ويطبع المنظمات للاختيار من بينها.

## توفير مفتاح API للـ SDK/المجمّع

```bash
# يُطبع السر مرة واحدة، مع --json إنه الحقل .key
key=$(agenteye --json keys create ci-bot --add events:read.add | jq -r '.key')
agenteye keys regenerate ci-bot --yes    # استدر؛ agenteye keys disable ci-bot --yes لإلغاء الإذن
```

## شغّل استعلام محفوظ أو مخصص

```bash
agenteye --json query run --sql "select count(*) from analytics.events" | jq '.rows'
agenteye --json query run errs --arg prod | jq '.rows'   # استعلام محفوظ + وسيط موضعي $1
```

## فرز حادثة بدون تفاعل

```bash
id=$(agenteye --json incidents list --state firing | jq -r '.incidents[0].id')
agenteye incidents ack "$id"
agenteye incidents assign "$id" --assignee you@corp.com
agenteye incidents resolve "$id" --yes
```

> **ملاحظة:** الطفرات تتجاوز تلقائيًا موجه التأكيد الخاص بها تحت `--json` أو عندما لا يكون stdin TTY، لذا لا تتعلق الوكلاء؛ مرر `--yes`/`-y` لتخطيه بشكل صريح في مكان آخر.

## التعامل مع رمز الخروج في نص برمجي

```bash
out=$(agenteye --json sessions --since 1h) || code=$?
case "${code:-0}" in
  0) echo "$out" | jq '.sessions | length' ;;
  4) echo "Session expired - run 'agenteye login'." >&2 ;;
  5) echo "Missing permission (ask an admin for evaluations:read)." >&2 ;;
  3) echo "Dashboard unreachable - check the URL." >&2 ;;
  *) echo "Unexpected error (exit ${code})." >&2 ;;
esac
```

## أشكال إخراج JSON

| الأمر | stdout JSON (مع `--json`) |
|---|---|
| `whoami` | `{"logged_in": true, "id", "email", "is_instance_admin", "active_org", "permissions": [...], "memberships": [...]}` أو `{"logged_in": false}` |
| `orgs list` | `{"active_org", "orgs": [{"org_slug","org_name","permission_set","permissions"}]}` |
| `events` | `{"events": [...], "next_cursor": <cursor or null>}` |
| `evals` | `{"evaluations": [...], "next_cursor": <cursor or null>}` |
| `sessions` | `{"sessions": [...], "next_cursor": <cursor or null>}` |
| `errors` | `{"errors": [...], "next_cursor": <cursor or null>}` |
| `list <kind>` | `{"kind", "values": [...]}` |
| `keys list` / `keys create` | `{"keys": [...]}` / `{id, name, permissions, created_at, key}` (عُرض `key` مرة واحدة) |
| `query run` | `{columns: [{name,type}], rows: [[...]], truncated, elapsed_ms}` |
| `users list` / `settings list` | `{"users": [...]}` / `{"settings": [...]}` |
| `alerts list` / `incidents list` | `{"alerts": [...]}` / `{"incidents": [...]}` |
| إنشاء/تحديث/حذف (أي) | كائن المورد، أو `{"deleted": true, "id"}` للحذف |
| فشل (أي، مع `--json`) | `{"error": "...", "exit_code": <n>, "status"?: <http>, "hint"?: "..."}` على stdout |

- كل عنصر **حدث** (`events`): `id, session_id, agent_id, event_type, ts, payload, environment, summary, is_error, error_type, output_tokens, context_window, context_fill`. لاحظ أن `payload` هو `{}` إلا إذا طلبت الخلاصة الكاملة باستخدام `--full` (أو `--fields payload`).
- كل عنصر **تقييم** (`evals`): `id, session_id, agent_id, environment, status, scores, reasoning, summary, error, attempt_count, duration_ms, completed_at, created_at`.
- كل عنصر **جلسة** (`sessions`): `session_id, agent_id, environment, status, scores, event_count, started_at, last_event_at, first_event_id, last_event_id, latest_evaluation`.

يقبل `--fields` لكل أمر بالضبط أسماء حقول عنصره الخاص. تختلف المجموعة بين `sessions` و`evals`، لذا قد يُرفض اسم صحيح لأحدهما من قبل الآخر.
