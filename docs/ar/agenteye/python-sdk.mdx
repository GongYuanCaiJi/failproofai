شاهد بالضبط ما قامت به وكلاء الذكاء الاصطناعي الخاصة بك في الإنتاج: كل تشغيل للعامل، واستدعاء الأداة، وطلب النموذج، والخطاف، والتدخل البشري. يسجل FailproofAI Observability Python SDK هذا المسار من داخل كود العامل الخاص بك حتى تتمكن من تصحيح الأخطاء والتدقيق وتقييم ما حدث. استخدمه كلما كنت تريد أن تراقب FailproofAI Observability وكلاءك.

## البداية السريعة

```python
import agenteye

agenteye.configure(environment="production")

agenteye.event.agent_start(session_id="run-001", agent_id="planner", goal="answer user query")

agenteye.event.tool_use(
    session_id="run-001",
    agent_id="planner",
    tool_name="web_search",
    tool_call_id="toolu_01",
    input={"query": "latest AI research"},
)

agenteye.event.tool_result(
    session_id="run-001",
    agent_id="planner",
    tool_name="web_search",
    tool_call_id="toolu_01",
    output={"results": ["..."]},
)

agenteye.event.agent_end(session_id="run-001", agent_id="planner", outcome="success")
```

### جعل استدعاء فعلي يعمل

عملياً، تقوم بلف كود العامل الموجود لديك. ضع استدعاء نموذج بين `model_request` قبل و `model_response` بعده، بحيث يغطي الحدثان الطلب الفعلي ويمكن لـ FailproofAI Observability اقتران الاثنين:

```python
import anthropic
import agenteye

agenteye.configure(environment="production")
client = anthropic.Anthropic()

messages = [{"role": "user", "content": "Summarise today's incidents."}]

agenteye.event.model_request(
    session_id="run-001",
    agent_id="planner",
    model="claude-sonnet-4-6",
    messages=messages,
)

reply = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=512,
    messages=messages,
)

agenteye.event.model_response(
    session_id="run-001",
    agent_id="planner",
    model=reply.model,
    stop_reason=reply.stop_reason,
    input_tokens=reply.usage.input_tokens,
    output_tokens=reply.usage.output_tokens,
    content=[block.model_dump() for block in reply.content],
)
```

قم بلف استدعاءات الأداة بنفس الطريقة باستخدام `tool_use` و `tool_result`، وأعد استخدام `tool_call_id` واحد عبر الزوج.

إليك كيف تبدو هذه الأحداث بمجرد وصولها إلى لوحة المعلومات، مع ألوان حسب النوع وقابلة للتصفية حسب البيئة والعامل والجلسة:

## configure()

```python
agenteye.configure(
    base_dir=None,           # Path | str | None. الافتراضي: $AGENTEYE_HOME أو ~/.agenteye
    flush_interval=0.5,      # float، ثواني بين دورات التفريغ
    environment=None,        # str | None. تسمية بيئة النشر
)
```

استدعِ مرة واحدة قبل أي استدعاء `event.*`. آمن للحذف؛ الافتراضيات تعمل بدون مشاكل. جميع الوسائط عبارة عن كلمات رئيسية فقط؛ مررها حسب الاسم كما هو موضح أعلاه.

عندما يكون `base_dir` هو `None` (الافتراضي)، يقرأ SDK `$AGENTEYE_HOME` إن كانت محددة،
وإلا يرجع إلى `~/.agenteye`. هذا يتطابق مع حل المجمِّع الخاص به،
لذا متغير بيئة `AGENTEYE_HOME` واحد يعد ملف الحدث المشترك لكل من
SDK والمجمِّع.

## البيئة

ضع علامة على كل حدث ببيئة النشر (`production`, `staging`, `qa`, `canary`, إلخ). اضبطه مرة واحدة؛ يرفقه SDK بكل حدث تلقائياً.

**الخيار 1: عبر `configure()`:**

```python
agenteye.configure(environment="production")
```

**الخيار 2: عبر متغير البيئة:**

```bash
export AGENTEYE_ENVIRONMENT=production
```

**الأولوية:** يفوز `configure(environment=...)` على متغير البيئة. إذا لم يتم تعيين أحدهما، فسيتم التخلف عن القيمة الافتراضية `"dev"`.

تظهر قيمة البيئة كمرشح من الدرجة الأولى في لوحة المعلومات وتُخزن على الخادم للاستعلامات السريعة.

> **تحذير:** قيم البيئة يجب ألا تحتوي على فاصلة `,` حرفية. تستخدم مرشحات لوحة المعلومات المحددة متعددة مفصولة بفواصل على السلك (`?environment=prod,staging`)، لذا البيئة المسماة `prod,blue` ستُقسم إلى قيمتين. الأحداث التي تحتوي على بيئات تحتوي على فواصل يتم رفضها في وقت البيانات.

## البيانات والخصوصية

يسجل SDK فقط الحقول التي تمررها بشكل صريح. الأسلحة والرسائل والمدخلات والمخرجات والمحتوى النموذجي يتم التقاطها لأنك تسلمها لاستدعاء `event.*`. لا يتم قراءة أي شيء من العملية الخاصة بك أو التقاطه بشكل ضمني. أي حقل تتركه غير محدد يُحذف من الحدث بالكامل؛ لا يُكتب إلى القرص.

هذا يجعل الحذف خيارك ومسؤوليتك. إذا كان الطلب أو حمل الأداة يحتوي على معلومات تعريف شخصية أو أسرار لا تريد تخزينها، فقم بإزالتها أو إخفاؤها قبل تمريرها إلى طريقة الحدث.

## مرجع الأحداث

تأتي معظم الأحداث في أزواج بداية/نهاية تشترك في معرّف الارتباط: `tool_use` و `tool_result` تشترك في `tool_call_id`، `hook_triggered` و `hook_completed` تشترك في `hook_id`، و `human_wait` و `human_input` تشترك في `input_id`. أصدر حدث البداية، افعل العمل، ثم أصدر حدث النهاية بنفس المعرّف. تتطابق FailproofAI Observability مع الزوج وتحسب `duration_ms` لك، لذا لا تمرر `duration_ms` بنفسك.

![رسم بياني لتنفيذ نمط git للجلسة بجانب الخط الزمني للأحداث، أُعيد بناؤه من الأحداث المقترنة، مع جزء تصنيف الأداة/النموذج/الخطاف](/agenteye/images/session-detail.png)

تتطلب جميع طرق الأحداث هذين الحقلين:

| الحقل | النوع | الوصف |
|---|---|---|
| `session_id` | `str` | يحدد تشغيل العامل من المستوى الأعلى |
| `agent_id` | `str` | يحدد أي عامل داخل الجلسة أصدر الحدث |

جميع الطرق أيضاً تقبل `**kwargs` تعسفية للبيانات الوصفية المخصصة (انظر [الحقول المخصصة](#custom-fields)).

---

### `event.agent_start()`

يتم إصداره عندما يبدأ العامل العمل.

```python
agenteye.event.agent_start(
    session_id="run-001",
    agent_id="planner",
    goal="answer user query",   # str | None
    parent_id=None,             # str | None - معرّف العامل الأب للعوامل المتداخلة
)
```

---

### `event.agent_end()`

يتم إصداره عندما ينتهي العامل من العمل.

```python
agenteye.event.agent_end(
    session_id="run-001",
    agent_id="planner",
    outcome="success",          # str | None
    summary="Answered query",   # str | None
)
```

---

### `event.tool_use()`

يتم إصداره عندما يستدعي العامل أداة. اقترن مع `tool_result`؛ يحسب SDK `duration_ms` تلقائياً.

```python
agenteye.event.tool_use(
    session_id="run-001",
    agent_id="planner",
    tool_name="web_search",     # str، مطلوب
    tool_call_id="toolu_01",    # str، مطلوب - مفتاح الارتباط لـ tool_result المطابق
    input={"query": "..."},     # dict | None
)
```

---

### `event.tool_result()`

يتم إصداره عندما تعيد أداة النتيجة. يرتبط مع `tool_use` عبر `tool_call_id`.

```python
agenteye.event.tool_result(
    session_id="run-001",
    agent_id="planner",
    tool_name="web_search",
    tool_call_id="toolu_01",        # يجب أن يطابق tool_use السابق
    output={"results": ["..."]},    # Any | None
    error=None,                     # str | None - اضبط إذا أطلقت الأداة
    # يتم حساب duration_ms تلقائياً - لا تمرره
)
```

---

### `event.model_request()`

يتم إصداره قبل إرسال طلب إلى نموذج لغة.

```python
agenteye.event.model_request(
    session_id="run-001",
    agent_id="planner",
    model="claude-sonnet-4-6", # str | None - أي سلسلة مزود/نموذج؛ غير مقيم
    messages=[                   # list[dict] | None - أدوار المحادثة
        {"role": "user", "content": "..."},
    ],
    system="You are helpful.",   # Any | None - str أو قائمة كتل المحتوى
    tools=[                      # list[dict] | None - مخططات الأدوات المقدمة للنموذج
        {"name": "search", "input_schema": {"type": "object"}},
    ],
)
```

تقبل إدخالات `messages` إما `content` سلسلة عادية أو `content` قائمة كتل نمط Anthropic. يمكن تمرير معاملات العينة (`temperature`, `max_tokens`, إلخ) كـ kwargs إضافية.

---

### `event.model_response()`

يتم إصداره عندما يعيد نموذج اللغة استجابة.

```python
agenteye.event.model_response(
    session_id="run-001",
    agent_id="planner",
    model="claude-sonnet-4-6", # str | None - أي سلسلة مزود/نموذج؛ غير مقيم
    stop_reason="end_turn",     # str | None
    input_tokens=1024,          # int | None
    output_tokens=256,          # int | None
    content=[                    # Any | None - str، أو قائمة كتل المحتوى
        {"type": "text", "text": "..."},
    ],
    role="assistant",            # str | None
)
```

يقبل `content` إما سلسلة عادية (موردون عامة) أو قائمة كتل محتوى نمط Anthropic. استدعاءات الأداة تعيش داخل `content` كـ كتل `{"type": "tool_use", ...}`، بدون حقل `tool_calls` منفصل.

---

### `event.hook_triggered()`

يتم إصداره عندما ينطلق خطاف. اقترن مع `hook_completed`؛ يحسب SDK `duration_ms` تلقائياً.

```python
agenteye.event.hook_triggered(
    session_id="run-001",
    agent_id="planner",
    hook_name="pre_tool_use",   # str، مطلوب
    hook_id="hook-abc",         # str، مطلوب - مفتاح الارتباط
    trigger_event="tool_use",   # str | None
    input={"tool": "search"},   # Any | None
)
```

---

### `event.hook_completed()`

يتم إصداره عندما ينتهي الخطاف. يرتبط مع `hook_triggered` عبر `hook_id`.

```python
agenteye.event.hook_completed(
    session_id="run-001",
    agent_id="planner",
    hook_name="pre_tool_use",
    hook_id="hook-abc",         # يجب أن يطابق hook_triggered السابق
    outcome="allow",            # str | None
    output=None,                # Any | None
    error=None,                 # str | None
    # يتم حساب duration_ms تلقائياً - لا تمرره
)
```

---

### `event.error()`

يتم إصداره عندما يحدث خطأ لم يتم التعامل معه.

```python
agenteye.event.error(
    session_id="run-001",
    agent_id="planner",
    error_type="TimeoutError",  # str، مطلوب
    message="timed out",        # str، مطلوب
    traceback="Traceback...",   # str | None
)
```

## أحداث التفاعل البشري

توفر أحداث التفاعل البشري إشرافاً على اللحظات التي يتدخل فيها شخص ما في تنفيذ العامل (انتظار الموافقة، توفير المدخلات، الإيقاف المؤقت، أو إيقاف العامل). تسمح لك بقياس المدة التي يستغرقها الإنسان للاستجابة (يحسب SDK `duration_ms` تلقائياً على الأحداث المقترنة)، والتدقيق في من وقف العامل أو قاطعه، وبناء سير عمل الموافقة والإشراف التي تظهر في لوحة المعلومات.

### `event.human_wait()`

يتم إصداره عندما يوقف العامل التنفيذ في انتظار مدخلات إنسانية. اقترن مع `human_input`؛ يحسب SDK `duration_ms` تلقائياً (كم من الوقت استغرق الإنسان للاستجابة).

```python
agenteye.event.human_wait(
    session_id="run-001",
    agent_id="planner",
    input_id="inp-abc",                          # str، مطلوب - مفتاح الارتباط لـ human_input المطابق
    prompt="Do you approve this action?",        # str | None - السؤال المعروض للإنسان
    options=["approve", "reject", "defer"],      # list[str] | None - الخيارات المقدمة للإنسان
    reason="approval_required",                  # str | None - لماذا ينتظر العامل
)
```

### `event.human_input()`

يتم إصداره عندما يوفر الإنسان مدخلات ويستأنف العامل. يرتبط مع `human_wait` عبر `input_id`. يتم حساب `duration_ms` تلقائياً ولا يجب أن يمرره المتصل.

```python
agenteye.event.human_input(
    session_id="run-001",
    agent_id="planner",
    input_id="inp-abc",    # str، مطلوب - يجب أن يطابق human_wait السابق
    response="approve",    # str | None - إجابة الإنسان (نص حر أو خيار محدد)
    # يتم حساب duration_ms تلقائياً - لا تمرره
)
```

### `event.human_pause()`

يتم إصداره عندما يوقف الإنسان العامل بفعالية (على سبيل المثال عبر تحكم لوحة المعلومات). العامل معلق لكن لم ينته.

```python
agenteye.event.human_pause(
    session_id="run-001",
    agent_id="planner",
    reason="user_requested",  # str | None
    user_id="usr_42",         # str | None - من وقف العامل
)
```

### `event.human_interrupt()`

يتم إصداره عندما يوقف الإنسان العامل في منتصف التنفيذ. بخلاف `human_pause`، يتم إنهاء عمل العامل بدلاً من تعليقه.

```python
agenteye.event.human_interrupt(
    session_id="run-001",
    agent_id="planner",
    reason="output_incorrect",       # str | None
    user_id="usr_42",                # str | None - من قاطع العامل
    at_step="tool_use:web_search",   # str | None - ما كان يفعله العامل عند التوقف
)
```

## الحقول المخصصة

أي وسائط كلمة رئيسية إضافية يتم إلحاقها بالحدث بعد الحقول القياسية:

```python
agenteye.event.tool_use(
    session_id="run-001",
    agent_id="planner",
    tool_name="db_query",
    tool_call_id="toolu_02",
    tenant_id="acme",           # حقل مخصص
    region="us-east-1",         # حقل مخصص
)
```

`timestamp`, `type`, و `environment` محجوزة وتثير `ValueError` (`لا يمكن استخدام أسماء الحقول المحجوزة كحقول مخصصة: [...]`) إذا تم تمريرها كحقول مخصصة. `session_id` و `agent_id` معاملات مطلوبة على كل طريقة حدث ولا يمكن توفيرها مرة ثانية؛ Python يثير `TypeError` إذا فعلت. اضبط البيئة باستخدام `configure(environment=...)` (أو متغير `AGENTEYE_ENVIRONMENT`) بدلاً من ذلك.

## كيف يتم كتابة الأحداث

يتم تخزين الأحداث مؤقتاً في العملية ويتم تفريغها إلى القرص كل `flush_interval` ثانية (الافتراضي 500 مللي ثانية). يكتب كل تفريغ ملف JSONL واحد:

```text
~/.agenteye/events/event-2026-04-01T12-00-00-000Z.jsonl
```

يراقب المجمِّع هذا الدليل ويرفع الملفات تلقائياً. لا تحتاج إلى إدارة هذه الملفات مباشرة.

يتم كتابة كل ملف بشكل ذري: يكتب SDK إلى ملف مؤقت ثم يعيد تسميته في مكانه، لذا لا يرى المجمِّع ملفاً نصف مكتوب. يتم أيضاً تشغيل تفريغ نهائي عند خروج العملية الخاصة بك، لذا الأحداث المخزنة مؤقتاً في الفترة الأخيرة لا تُفقد. إذا كان المجمِّع غير متصل، تتراكم الأحداث ببساطة كملفات على القرص وتُرسل بمجرد عودته.
