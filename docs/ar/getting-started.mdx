- **Node.js** >= 20.9.0

## البدء السريع

<Steps>
  <Step title="تفعيل السياسات">
    السياسات هي قواعد تعمل قبل وبعد كل استدعاء أداة للوكيل. تلتقط الأوامر المدمرة، وتسريب الأسرار، وأنماط الفشل الأخرى قبل أن تسبب أضراراً.

    ```bash
    failproofai policies --install
    ```

    يكتب هذا إدخالات الخطاف في واجهات سطر الأوامر المثبتة (`~/.claude/settings.json` للـ Claude Code، `~/.codex/hooks.json` لـ OpenAI Codex، `~/.copilot/hooks/failproofai.json` لـ GitHub Copilot CLI، `~/.cursor/hooks.json` لـ Cursor Agent، وحدة التغليف المتولدة في `~/.config/opencode/plugins/failproofai.mjs` لـ OpenCode بالإضافة إلى إدخال التسجيل في `~/.config/opencode/opencode.json` في صفيف `plugin`، `~/.pi/agent/settings.json` لـ Pi، أو `~/.gemini/settings.json` لـ Gemini CLI). عند وجود أكثر من واحد، سيُطلب منك الاختيار؛ مرر `--cli claude codex copilot cursor opencode pi gemini` (أي مجموعة جزئية) لتخطي الطلب.

    يتمتع GitHub Copilot CLI و Cursor Agent و OpenCode و Pi و Gemini CLI بدعم **تجريبي** — ثبّت باستخدام `--cli copilot` أو `--cli cursor` أو `--cli opencode` أو `--cli pi` أو `--cli gemini`.

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install --cli copilot --scope project
    failproofai policies --install --cli cursor --scope project
    failproofai policies --install --cli opencode --scope project
    failproofai policies --install --cli pi --scope project
    failproofai policies --install --cli gemini --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="التحقق">
    ```bash
    failproofai policies
    ```

    يعرض كل سياسة، سواء كانت مفعّلة، وأي معاملات مكونة.
  </Step>
  <Step title="تشغيل لوحة التحكم">
    ```bash
    failproofai
    ```

    يفتح لوحة تحكم محلية في `http://localhost:8020` حيث يمكنك استعراض الجلسات، وفحص استدعاءات الأداة، وإدارة السياسات.
  </Step>
  <Step title="تشغيل وكيلك">
    شغّل Claude Code كالمعتاد. إذا حاول الوكيل فعل شيء محفوف بالمخاطر، يعترضه failproofai تلقائياً. اتركه يعمل بلا مراقبة واستعرض ما حدث في لوحة التحكم.
  </Step>
</Steps>

## كيف تعمل السياسات

في كل مرة يشغّل الوكيل أداة، يستدعي Claude Code failproofai كعملية فرعية:

```text
Claude Code  →  failproofai --hook PreToolUse  →  reads stdin JSON
                                                 evaluates policies
                                                 writes decision to stdout
```

تُرجع كل سياسة واحداً من ثلاثة قرارات:

- **allow** - يستمر الوكيل بشكل طبيعي
- **deny** - يتم حظر الإجراء، ويُخبر الوكيل السبب
- **instruct** - يتم إضافة سياق إضافي إلى موجه الوكيل

<Note>
تعمل السياسات في عمليتك المحلية. لا يتم إرسال شيء إلى خدمة بعيدة.
</Note>

## إعداد سياسات الفريق باستخدام السياسات القائمة على الاتفاقية

أسرع طريقة لإنشاء معايير الجودة عبر فريقك هي اتفاقية `.failproofai/policies/`. ضع ملفات السياسات في هذا المجلد وسيتم تحميلها تلقائياً — بدون أعلام، بدون تغييرات الإعدادات، بدون أوامر التثبيت.

<Steps>
  <Step title="إنشاء مجلد السياسات">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="إضافة ملفات السياسات">
    انسخ أمثلة المبتدئين أو اكتب الخاصة بك:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    أو أنشئ واحداً جديداً:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Run tests before committing.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="الالتزام بـ git">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```

    كل عضو في الفريق لديه failproofai مثبت يلتقط هذه السياسات تلقائياً. لا توجد حاجة لإعداد منفصل لكل مطور.
  </Step>
</Steps>

<Tip>
التزم `.failproofai/policies/` إلى مستودعك حتى يشارك الفريق بأكمله نفس المعايير. عندما يكتشف فريقك أنماط فشل جديدة، أضف سياسات واضغط — يحصل الجميع على التحديث عند جلب `git pull` التالي. بمرور الوقت، تصبح هذه السياسات معياراً جودة حياً يستمر في التحسن.
</Tip>

## تخزين البيانات

تبقى جميع الإعدادات والسجلات على جهازك:

| المسار | ما يخزنه |
|------|----------------|
| `~/.failproofai/policies-config.json` | إعدادات السياسة العامة |
| `~/.failproofai/hook-activity.jsonl` | سجل تنفيذ الخطاف |
| `~/.failproofai/hook.log` | سجل تصحيح الأخطاء لأخطاء الخطاف المخصصة |
| `.failproofai/policies-config.json` | إعدادات كل مشروع (مرتكبة) |
| `.failproofai/policies-config.local.json` | التجاوزات الشخصية (مضافة إلى gitignore) |

---

## إلغاء التثبيت

```bash
failproofai policies --uninstall
```

يزيل إدخالات الخطاف من `~/.claude/settings.json`. يتم الاحتفاظ بملفات الإعدادات في `~/.failproofai/`.
