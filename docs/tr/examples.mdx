## Claude Code için hook'ları ayarlama

Failproof AI, Claude Code ile [hook'lar sistemi](https://docs.anthropic.com/en/docs/claude-code/hooks) aracılığıyla entegre olur. `failproofai policies --install` komutunu çalıştırdığınızda, her araç çağrısında tetiklenen hook komutlarını Claude Code'un `settings.json` dosyasına kaydeder.

<Steps>
  <Step title="failproofai'ı kurun">
    ```bash
    npm install -g failproofai
    ```
  </Step>
  <Step title="Tüm yerleşik politikaları etkinleştirin">
    ```bash
    failproofai policies --install
    ```
  </Step>
  <Step title="Hook'ların kaydedildiğini doğrulayın">
    ```bash
    cat ~/.claude/settings.json | grep failproofai
    ```

    `PreToolUse`, `PostToolUse`, `Notification` ve `Stop` olayları için hook girdilerini görmelisiniz.
  </Step>
  <Step title="Claude Code'u çalıştırın">
    ```bash
    claude
    ```

    Politikalar artık her araç çağrısında otomatik olarak çalışıyor. Claude'a `sudo rm -rf /` komutunu çalıştırmasını istemeyi deneyin - engellenecektir.
  </Step>
</Steps>

## Agents SDK için hook'ları ayarlama

[Agents SDK](https://docs.anthropic.com/en/docs/agents-sdk) ile geliştiriyorsanız, aynı hook sistemini programlama yoluyla kullanabilirsiniz.

<Steps>
  <Step title="failproofai'ı projenize kurun">
    ```bash
    npm install failproofai
    ```
  </Step>
  <Step title="Ajanınızda hook'ları yapılandırın">
    Ajan sürecinizi oluştururken hook komutlarını geçin. Hook'lar Claude Code'daki gibi çalışır - stdin/stdout JSON aracılığıyla:

    ```bash
    failproofai --hook PreToolUse   # her araçtan önce çağrılır
    failproofai --hook PostToolUse  # her araçtan sonra çağrılır
    ```
  </Step>
  <Step title="Ajanınız için özel bir politika yazın">
    ```javascript
    import { customPolicies, allow, deny } from "failproofai";

    customPolicies.add({
      name: "limit-to-project-dir",
      description: "Keep the agent inside the project directory",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        const path = String(ctx.toolInput?.file_path ?? "");
        if (path.startsWith("/") && !path.startsWith(ctx.session?.cwd ?? "")) {
          return deny("Agent is restricted to the project directory");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Özel politikayı kurun">
    ```bash
    failproofai policies --install --custom ./my-agent-policies.js
    ```
  </Step>
</Steps>

## Yıkıcı komutları engelle

En yaygın kurulum - ajanların geri alınamaz zararlar vermesini önleyin.

```bash
failproofai policies --install block-sudo block-rm-rf block-force-push block-curl-pipe-sh
```

Bunu yapan şeyler:
- `block-sudo` - tüm `sudo` komutlarını engeller
- `block-rm-rf` - özyinelemeli dosya silmeyi engeller
- `block-force-push` - `git push --force` komutunu engeller
- `block-curl-pipe-sh` - uzak betikleri shell'e borulamayı engeller

## Gizli bilgilerin sızmasını önleyin

Ajanların kimlik bilgilerini araç çıktısında görmesini veya sızmasını durdurun.

```bash
failproofai policies --install sanitize-api-keys sanitize-jwt sanitize-connection-strings sanitize-bearer-tokens
```

Bunlar `PostToolUse` üzerinde tetiklenir - bir araç çalıştıktan sonra, çıktıyı ajan görmeden önce temizlerler.

## Ajanlar dikkat gerektirdiğinde Slack uyarısı alın

Boşta olan uyarıları Slack'e iletmek için bildirim hook'unu kullanın.

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "slack-on-idle",
  description: "Alert Slack when the agent is waiting for input",
  match: { events: ["Notification"] },
  fn: async (ctx) => {
    const webhookUrl = process.env.SLACK_WEBHOOK_URL;
    if (!webhookUrl) return allow();

    const message = String(ctx.payload?.message ?? "Agent is waiting");
    const project = ctx.session?.cwd ?? "unknown";

    try {
      await fetch(webhookUrl, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          text: `*${message}*\nProject: \`${project}\``,
        }),
        signal: AbortSignal.timeout(5000),
      });
    } catch {
      // never block the agent if Slack is unreachable
    }

    return allow();
  },
});
```

Kurun:

```bash
SLACK_WEBHOOK_URL=https://hooks.slack.com/... failproofai policies --install --custom ./slack-alerts.js
```

## Ajanları bir dalda tutun

Ajanların dal değiştirmesini veya korumalı dallara itme yapmasını önleyin.

```javascript
import { customPolicies, allow, deny } from "failproofai";

customPolicies.add({
  name: "stay-on-branch",
  description: "Prevent the agent from checking out other branches",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+checkout\s+(?!-b)/.test(cmd)) {
      return deny("Stay on the current branch. Create a new branch with -b if needed.");
    }
    return allow();
  },
});
```

## Commit'ten önce testleri zorunlu kılın

Ajanları commit'ten önce test çalıştırmaya hatırlatın.

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "test-before-commit",
  description: "Remind the agent to run tests before committing",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+commit/.test(cmd)) {
      return instruct("Run tests before committing. Use `npm test` or `bun test` first.");
    }
    return allow();
  },
});
```

## Üretim deposunu kilitleyin

Takım üyelerinin hepsi aynı politikaları alması için projeye özgü bir yapılandırmayı commit edin.

Deponuzda `.failproofai/policies-config.json` dosyası oluşturun:

```json
{
  "enabledPolicies": [
    "block-sudo",
    "block-rm-rf",
    "block-force-push",
    "block-push-master",
    "block-env-files",
    "sanitize-api-keys",
    "sanitize-jwt"
  ],
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "release", "production"]
    }
  }
}
```

Ardından commit edin:

```bash
git add .failproofai/policies-config.json
git commit -m "Add failproofai team policies"
```

failproofai kurulu olan her takım üyesi bu kuralları otomatik olarak alacaktır.

## Convention politikaları ile kuruluş çapında kalite standardı oluşturun

En etkili kurulum: deponuza `.failproofai/policies/` commit edin ve projenize göre hazırlanmış politikalarını içerir. Her takım üyesi bunları otomatik olarak alır — kurulum komutu yok, yapılandırma değişikliği yok.

<Steps>
  <Step title="Dizini oluşturun ve politikaları ekleyin">
    ```bash
    mkdir -p .failproofai/policies
    ```

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    // Enforce your team's preferred package manager
    // (or enable the built-in prefer-package-manager policy instead)
    customPolicies.add({
      name: "enforce-bun",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        const cmd = String(ctx.toolInput?.command ?? "");
        if (/\bnpm\b/.test(cmd)) return deny("Use bun instead of npm.");
        return allow();
      },
    });

    // Remind the agent to run tests before committing
    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Run tests before committing.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Git'e commit edin">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```
  </Step>
  <Step title="İyileştirmeyi devam ettirin">
    Takımınız yeni başarısızlık modlarıyla karşılaştığında, politikaları ekleyin ve push yapın. Herkes bir sonraki `git pull`'da güncellemeyi alır. Bu politikalar, takımınızla birlikte büyüyen yaşayan bir kalite standardı haline gelir.
  </Step>
</Steps>
