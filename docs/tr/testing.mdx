failproofai iki test paketine sahiptir: **birim testleri** (hızlı, mock edilmiş) ve **uçtan uca testler** (gerçek subprocess çağrıları).

---

## Testleri çalıştırma

```bash
# Tüm birim testlerini bir kez çalıştır
bun run test:run

# Birim testlerini watch modunda çalıştır
bun run test

# E2E testlerini çalıştır (setup gerekir - aşağıya bakın)
bun run test:e2e

# Derleme olmadan type-check et
bunx tsc --noEmit

# Lint
bun run lint
```

## Birim testleri

Birim testleri `__tests__/` klasöründe yer alır ve [Vitest](https://vitest.dev) ile `happy-dom` kullanır.

```text

    builtin-policies.test.ts      # Her builtin için politika mantığı

```

### Politika birim testi yazma

```typescript
import { describe, it, expect, beforeEach } from "vitest";
import { getBuiltinPolicies } from "../../src/hooks/builtin-policies";
import { allow, deny } from "../../src/hooks/policy-types";

describe("block-sudo", () => {
  const policy = getBuiltinPolicies().find((p) => p.name === "block-sudo")!;

  it("denies sudo commands", () => {
    const ctx = {
      eventType: "PreToolUse" as const,
      payload: {},
      toolName: "Bash",
      toolInput: { command: "sudo apt install nodejs" },
      params: { allowPatterns: [] },
    };
    expect(policy.fn(ctx)).toEqual(deny("sudo command blocked by failproofai"));
  });

  it("allows non-sudo commands", () => {
    const ctx = {
      eventType: "PreToolUse" as const,
      payload: {},
      toolName: "Bash",
      toolInput: { command: "ls -la" },
      params: { allowPatterns: [] },
    };
    expect(policy.fn(ctx)).toEqual(allow());
  });

  it("allows patterns in allowPatterns", () => {
    const ctx = {
      eventType: "PreToolUse" as const,
      payload: {},
      toolName: "Bash",
      toolInput: { command: "sudo systemctl status nginx" },
      params: { allowPatterns: ["sudo systemctl status"] },
    };
    expect(policy.fn(ctx)).toEqual(allow());
  });
});
```

## Uçtan uca testler

E2E testleri gerçek `failproofai` binary dosyasını subprocess olarak çalıştırır, JSON yükünü stdin'e gönderir ve stdout çıktısı ile çıkış kodunu doğrular. Bu, Claude Code'un kullandığı tam entegrasyon yolunu test eder.

### Setup

E2E testleri binary dosyasını doğrudan repo kaynağından çalıştırır. İlk çalıştırmadan önce, özel hook dosyalarının `'failproofai'` den import ettiğinde kullandığı CJS paketini derleyin:

```bash
bun build src/index.ts --outdir dist --target node --format cjs
```

Daha sonra testleri çalıştırın:

```bash
bun run test:e2e
```

Ortak hook API'sini değiştirdiğinizde `dist/` klasörünü yeniden derleyin (`src/hooks/custom-hooks-registry.ts`, `src/hooks/policy-helpers.ts` veya `src/hooks/policy-types.ts`).

### E2E test yapısı

```text
__tests__/e2e/
  helpers/
    hook-runner.ts      # Binary dosyasını çalıştır, yükü JSON olarak gönder, çıkış kodu + stdout + stderr yakala
    fixture-env.ts      # Test başına izole geçici dizinler ve config dosyaları
    payloads.ts         # Her event türü için Claude-doğru yükü fabrikaları
  hooks/
    builtin-policies.e2e.test.ts   # Gerçek subprocess ile her builtin politika
    custom-hooks.e2e.test.ts       # Özel hook yükleme ve değerlendirme
    config-scopes.e2e.test.ts      # Proje/yerel/global arasında config birleştirme
    policy-params.e2e.test.ts      # Parametreli her politika için parametre injeksiyonu
```

### E2E yardımcılarını kullanma

**`FixtureEnv`** - test başına izole ortam:

```typescript
import { createFixtureEnv } from "../helpers/fixture-env";

const env = createFixtureEnv();
// env.cwd    - temp dir; .failproofai/policies-config.json'i almak için payload.cwd olarak geçin
// env.home   - izole home dir; gerçek ~/.failproofai sızıntısı yok

env.writeConfig({
  enabledPolicies: ["block-sudo"],
  policyParams: {
    "block-sudo": { allowPatterns: ["sudo systemctl status"] },
  },
});
```

`createFixtureEnv()` `afterEach` temizliğini otomatik olarak kaydeder.

**`runHook`** - binary dosyasını çalıştırın:

```typescript
import { runHook } from "../helpers/hook-runner";
import { Payloads } from "../helpers/payloads";

const result = await runHook(
  "PreToolUse",
  Payloads.preToolUse.bash("sudo apt install nodejs", env.cwd),
  { homeDir: env.home }
);

expect(result.exitCode).toBe(0);
expect(result.parsed?.hookSpecificOutput?.permissionDecision).toBe("deny");
```

**`Payloads`** - hazır yükü fabrikaları:

```typescript
Payloads.preToolUse.bash(command, cwd)
Payloads.preToolUse.write(filePath, content, cwd)
Payloads.preToolUse.read(filePath, cwd)
Payloads.postToolUse.bash(command, output, cwd)
Payloads.postToolUse.read(filePath, content, cwd)
Payloads.notification(message, cwd)
Payloads.stop(cwd)
```

### E2E testi yazma

```typescript
import { describe, it, expect } from "vitest";
import { createFixtureEnv } from "../helpers/fixture-env";
import { runHook } from "../helpers/hook-runner";
import { Payloads } from "../helpers/payloads";

describe("block-rm-rf (E2E)", () => {
  it("denies rm -rf", async () => {
    const env = createFixtureEnv();
    env.writeConfig({ enabledPolicies: ["block-rm-rf"] });

    const result = await runHook(
      "PreToolUse",
      Payloads.preToolUse.bash("rm -rf /", env.cwd),
      { homeDir: env.home }
    );

    expect(result.exitCode).toBe(0);
    expect(result.parsed?.hookSpecificOutput?.permissionDecision).toBe("deny");
  });

  it("allows non-recursive rm", async () => {
    const env = createFixtureEnv();
    env.writeConfig({ enabledPolicies: ["block-rm-rf"] });

    const result = await runHook(
      "PreToolUse",
      Payloads.preToolUse.bash("rm /tmp/file.txt", env.cwd),
      { homeDir: env.home }
    );

    expect(result.exitCode).toBe(0);
    expect(result.stdout).toBe("");  // allow → boş stdout
  });
});
```

### E2E yanıt şekilleri

| Karar | Çıkış kodu | stdout |
|----------|-----------|--------|
| `PreToolUse` deny | `0` | `{"hookSpecificOutput":{"permissionDecision":"deny","permissionDecisionReason":"..."}}` |
| `PostToolUse` deny | `0` | `{"hookSpecificOutput":{"additionalContext":"Blocked ... because: ..."}}` |
| Instruct (Stop olmayan) | `0` | `{"hookSpecificOutput":{"additionalContext":"Instruction from failproofai: ..."}}` |
| Stop instruct | `2` | boş stdout; neden stderr'de |
| Allow | `0` | boş string |

### Vitest config

E2E testleri `vitest.config.e2e.mts` ile kullanır:

- `environment: "node"` - tarayıcı globalleri gerekli değil
- `pool: "forks"` - gerçek process izolasyonu (testler subprocess'ler oluşturur)
- `testTimeout: 20_000` - test başına 20s (binary başlatma + hook değerlendirmesi)

`forks` pool önemlidir: thread tabanlı çalışanlar `globalThis` paylaşır, bu da subprocess oluşturan testlere müdahale edebilir. Process tabanlı forks bunu engeller.
