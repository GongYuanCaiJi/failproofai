## Resmi paket

Kanonik npm paketi **`failproofai`**:

```bash
npm install -g failproofai
# veya
bun add -g failproofai
```

---

## Neden takma ad adlarını biz kontrol ediyoruz

Yazım hatası saldırısı (typosquatting), kötü niyetli bir aktörün popüler bir pakete bir tuş mesafesinde olan paket adını kaydettiği yaygın bir tedarik zinciri saldırısıdır. Yükleme komutunu yanlış yazan şüphesiz kullanıcılar, tam sistem erişimi ile saldırgan tarafından kontrol edilen kodu çalıştırmaya maruz kalır - tam olarak Failproof AI'ın savunması için tasarlandığı tehdit türü.

Bu tehdidi ortadan kaldırmak için, **`failproofai`'ın tüm yaygın yazım hatalarını ve biçimlendirme varyantlarını önceden npm'de kontrol ediyoruz**. Bu adların hiçbiri üçüncü taraflar tarafından kaydedilemez. Her biri, gerçek `failproofai` paketine yükleyen ve delegasyon yapan ince bir vekil (proxy) sunucusudur.

## Kayıtlı takma adlar

**Biçimlendirme varyantları** - "failproof ai"yi yazmanın farklı yolları:

| Paket | Durum |
|---------|--------|
| `failproof` | ✅ Yayınlandı |
| `failproof-ai` | ⏳ npm desteği beklemede |
| `fail-proof-ai` | ⏳ npm desteği beklemede |
| `failproof_ai` | ⏳ npm desteği beklemede |
| `fail_proof_ai` | ⏳ npm desteği beklemede |
| `fail-proofai` | ⏳ npm desteği beklemede |

**`failprof*` yazım hataları** - "proof"tan bir `o` eksik:

| Paket | Durum |
|---------|--------|
| `failprof` | ✅ Yayınlandı |
| `failprof-ai` | ✅ Yayınlandı |
| `failprofai` | ⏳ npm desteği beklemede |
| `fail-prof-ai` | ⏳ npm desteği beklemede |
| `failprof_ai` | ⏳ npm desteği beklemede |

**`faliproof*` yazım hataları** - `a` ve `i` yer değiştirmesi:

| Paket | Durum |
|---------|--------|
| `faliproof` | ✅ Yayınlandı |
| `faliproof-ai` | ✅ Yayınlandı |
| `faliproofai` | ⏳ npm desteği beklemede |

> **Neden beklemede?** npm'nin spam önleme politikası, noktalama işaretleri kaldırıldıktan ve benzerlik kontrolleri yapıldıktan sonra mevcut bir pakete normal şekilde dönen adları engeller. npm desteği ile bu adları anti-squatting amaçlı ayırtmak için iletişime geçtik. Onay alındıktan sonra etkinleştirilecekler.

Yayınlanan herhangi bir takma adın bize ait olduğunu doğrulayabilirsiniz:

```bash
npm info failproof
# Bakınız: maintainers alanında "ExosphereHost Inc."
```

## Takma adlar nasıl çalışır

Her takma ad paketi:

1. `failproofai`'ı bir bağımlılık olarak listeler - böylece gerçek paket (kendi `postinstall` hook kurulumu dahil) yükleme sırasında çalışır
2. Kendi adıyla eşleşen bir ikili dosya sunar (örneğin `failprof-ai`) ve tüm argümanları `failproofai` ikili dosyasına aktarır

Vekil, iki satırlık bir Node betiğidir; hiçbir mantık, ağ çağrısı yok ve `failproofai`'ın kendisinin ötesinde veri toplama yoktur.
