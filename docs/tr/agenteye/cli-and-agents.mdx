Tüm FailproofAI Observability dağıtımınız, tek bir komut uzağınızda. Production'ı kontrol edin, bir API anahtarı oluşturun veya bir olayı onaylayın, terminalinizi asla terk etmeden, ardından bunları CI'a yazın veya bir kod aracısının bunu düz İngilizce olarak yapmasını sağlayın.

```bash
pipx install agenteye
agenteye login --email you@example.com      # a 6-digit code lands in your inbox
agenteye --json sessions --since 24h         # every agent run from the last day, newest first
```

*`agenteye` CLI, dashboard'unuzla iletişim kurar. Collector'dan (olayları sunucuya gönderen) farklı bir araçtır.*

## Tüm dağıtımınız, tek bir komut uzağında

Hızlı bir soruyu cevaplamak için sekme arasında geçmeyi durdurun. `agenteye` CLI verilerinizi okur ve kuruluşunuzu tek bir binary'den yönetir, böylece dashboard'da tıklama gerektiren bir kontrol tek satır haline gelir ve bunu yeniden çalıştırabilir, takma ad oluşturabilir veya runbook'a yapıştırabilirsiniz. Dört yüzeyiniz var:

- **Verilerinizi okuyun:** `sessions`, `events`, `evals`, ve `errors`, zaman, agent ve ortama göre filtrelenmiş.
- **Kuruluşunuzu yönetin:** `keys`, `users`, `settings`, `alerts`, ve `incidents`.
- **Analitiği çalıştırın:** kaydedilmiş SQL artı olay verileriniz üzerinde ad-hoc `query` çalıştırıcı.
- **Asistana sorun:** `agent ask` dashboard'da sohbet ettiğiniz aynı salt okunur analisite ulaşır.

`pipx` ile bir kez yükleyin, emaille alınan 6 haneli koduyla oturum açın ve hazırsınız. Oturum yaklaşık bir gün sürer; süresi dolduğunda `agenteye login` komutunu yeniden çalıştırın. Production'ı kontrol etmek, bir anahtar sağlamak veya bir firing olayını triage etmek için kullanın, tümü bir browser açmadan:

```bash
agenteye errors --since 24h --aggregate      # what is breaking, grouped by error type
agenteye incidents list --state firing       # what is on fire right now
agenteye keys create ci --add events:add     # a key that can only push events, secret shown once
```

Bilmeniz gereken bir alışkanlık: `--json` gibi global seçenekler komuttan önce gelir. `agenteye --json sessions` doğru; `agenteye sessions --json` değildir.

## Scriptle, CI'a bağla

Her komut `--json` alır ve bu her şeyi değiştirir. Temiz JSON stdout'a giderken insan durumu ve uyarıları stderr'e gider, böylece `--json` yakalama doğrudan `jq`'ye boşa harcanacak satır olmadan bağlanır. Bu, CLI'ı hem promptta siz için hem de çıktıyı ayrıştıran bir kod aracısı için eşit derecede iyi yapar:

```bash
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'
```

İşsiz çalışmak için tasarlanmıştır. Onay istemlerine terminał bağlı olmadığında otomatik olarak atlanır, böylece pipeline'da hiçbir şey askıda kalmaz ve her komut anlamlı bir çıkış kodu döndürür: `0` başarı, `4` oturum açılmadı, `5` izin eksik (mesaj adını verir, örneğin `alerts:write`), `3` dashboard ulaşılamaz. Bir script, kodu yeniden doğrulamak için `4` veya bir yöneticiden tam olarak ne isteyeceğinizi söylemek için `5` üzerinde branş yapabilir, körü körüne başarısız olmak yerine.

## Bir kod aracısının düz İngilizce olarak bunu sürücü yapmasına izin verin

Daha iyi bir şekilde, tüm bu bayrakları hatırlamanız gerekmemelidir. **CLI skill** `agenteye-cli` adlı küçük bir Agent Skill klasörüdür ve Claude Code veya Codex gibi bir kod aracısını düz İngilizce isteklerinden CLI'ı sürücü yapmayı öğretir. "Bugün hiçbir şey bozuk mu?" diye sorun ve agent komutu seçer, sizin olarak çalıştırır ve prozayla cevaplar.

Claude Code için, `agenteye-cli` klasörünü `~/.claude/skills/` içine bırakın ve otomatik olarak keşfedilir. FailproofAI Observability klasörü sağlar; kurulacak ekstra bir şey yoktur, çünkü yalnızca zaten yüklediğiniz CLI'ı sürücü yapar. Kendiniz oturum açın: skill, emaille alınan kodu tamamlayamaz.

Agent CLI'ı sizin olarak çalıştırdığından, oturum açmışken yaptığınız her şeyi yapabilir, okuma ve yazma gibi: anahtar oluşturun, ayarları değiştirin, olayları çözün. CLI'ın "emin misiniz?" istemi bir agent için ateşlenmez, bu nedenle skill tam komutu ifade etmek ve herhangi bir değişiklik yapılmadan önce onayınızı beklemek için yazılmıştır. Siz onaylama adımısınız.

```text
you    Why did session run-001 fail?

agent  Running: agenteye --json events --session-id run-001 --all
       The checkout-agent hit a TimeoutError on its third tool call.
```

Okumalar anında kalır ve her yazma sizin için duraklar:

```text
you    Give CI a key that can only push events.

agent  This creates an API key. I'll run:
         agenteye keys create ci --add events:add
       OK to proceed?

you    yes

agent  Done. Key "ci" created with events:add only. The secret is shown once, so store it now.
```
