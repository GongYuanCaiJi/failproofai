Terminali veya bir betiğin içinden FailproofAI Observability'nin tümünü yönetin: pano turlarına gerek yoktur. `agenteye` CLI'si verilerinizi (oturumlar, etkinlik günlükleri, değerlendirmeler) sorgular ve kuruluşunuzu yönetir (API anahtarları, kullanıcılar, ayarlar, uyarılar, olaylar, kaydedilmiş sorgular). Bir kontrolü otomatikleştirmek, Observability'yi CI'ye entegre etmek veya bir kodlama aracısının üretimi incelemesini istediğinizde başvurun. Her komut `--json` bayrağını destekler, bu yüzden sizin komut isteminde veya bir kodlama aracısının (Claude Code, Cursor) sonucu kabuğa alarak ayrıştırmasında eşit şekilde iyi çalışır.

Tek bir ikili ile şunları yapabilirsiniz:

- **Verilerinizi okuyun**: `sessions`, `events`, `evals`, `errors` (zaman, ajan, ortam, puanla filtreleyin).
- **Kuruluşunuzu yönetin**: `keys`, `users`, `settings`, `alerts`, `incidents`.
- **Analitik çalıştırın**: kaydedilmiş SQL ve geçici sorgu çalıştırıcı (`query`).
- **AI asistanına sorun**: panoda sohbet ettiğiniz aynı salt okunur analist (`agent`).

> **Not:** Bu `agenteye` CLI'sidir, toplayıcı daemon'u (`agenteye-collector`) ile farklı bir araçtır. CLI panonuza bağlanır; toplayıcı olayları sunucuya gönderir.

## Hızlı Başlangıç

Hiçbir şeyden ilk sonuca dört satırda ulaşın. CLI'yi panonuza yönlendirin, oturum açın, kimliğinizi doğrulayın, sonra son günün çalışmalarını çekin:

```bash
pipx install agenteye
agenteye --base-url https://agenteye.example.com login --email you@example.com   # gönderilen 6 haneli kod
agenteye whoami                                                                  # kullanıcı + etkin kuruluş
agenteye --json sessions --since 24h                                             # ajan çalışması başına bir satır, son 24s
```

Son komut en son oturumların JSON nesnesini yazdırır (en yeniler ilk, varsayılan olarak 50 ile sınırlı). Bunu kesmek için `jq` içine aktarın veya bir kutu halinde, renklendirilmiş tablo için `--json` bırakın. Her satır çalışmanın durumunu ve değerlendiricinin puanladığı metrik puanlarını içerir (burada kısaltılmış):

```json
{
  "sessions": [
    {
      "session_id": "run-8f2a",
      "agent_id": "checkout-bot",
      "environment": "prod",
      "status": "error",
      "scores": { "helpfulness": 0.42, "tool_efficiency": 0.55 },
      "event_count": 37,
      "started_at": "2026-07-16T09:14:02Z",
      "last_event_at": "2026-07-16T09:14:48Z"
    }
  ],
  "next_cursor": null
}
```

Bu sayfanın geri kalanı her parçayı açıklar: izole edilmiş [kurulum](#installation), [oturum açma](#authentication), [yapılandırma](#configuration), her komutun paylaştığı [genel kurallar](#global-options--conventions) ve [tam komut referansı](#command-reference).

## Kurulum

CLI, **`agenteye`** adlı genel bir PyPI paketidir. Her zaman kendi bağımlılıklarına sahip olması için izole bir ortamda kurun:

```bash
pipx install agenteye
# veya
uv tool install agenteye
```

Python 3.10+ gerektirir. Yüklenen komut **`agenteye`** dir:

```bash
agenteye --version
agenteye --help
```

> **Not:** FailproofAI Observability Python SDK'sı da `agenteye` dağıtım adını kullanır. CLI'yi `pipx` veya `uv tool` ile (paylaşılan bir virtualenv'e `pip install` yerine) yüklemek ikisinin çarpışmasını önler. Basit bir `pip install agenteye` yalnızca SDK aynı ortamda yüklü değilse uygundur.

## Kimlik Doğrulama

CLI, **pano** ile e-postayla gönderilen bir kerelik kod kullanarak kimlik doğrulaması yapar:

```bash
agenteye login --email you@example.com
# Size 6 haneli bir kod gönderilir; soruda yapıştırın.
```

Oturum belirteci `~/.agenteye/cli.json` içinde depolanır (yalnızca sizin tarafınızdan okunabilir, mod `0600`) ve varsayılan olarak 24 saat geçerlidir. Süresi dolduğunda, `agenteye login` komutunu yeniden çalıştırın.

```bash
agenteye whoami     # mevcut kullanıcı, etkin kuruluş ve izinleri göster
agenteye logout     # oturumu iptal et ve depolanmış belirteci temizle
```

`whoami` eksik veya süresi dolmuş bir oturumda hata vermez; bunun yerine `logged_in: false` bildirir, bu yüzden bir betik veya ajan kimlik doğrulama durumunu güvenli bir şekilde araştırabilir (taban URL'si ayarlanmamışsa veya pano ulaşılamazsa halen sıfır olmayan bir şekilde çıkabilir).

**Gereksinimler:** e-postanız panoda oturum açmak için izin verilmelidir (FailproofAI Observability yöneticinize sorun) ve pano, taban URL'sinde erişilebilir olmalıdır (bkz. [Yapılandırma](#configuration)). Kod talep ederseniz ve hiçbiri gelmezse, e-postanız muhtemelen henüz pano erişimi için etkin değildir.

## Kuruluşunuzu seçin (çok kiracılı)

Hesabınız birden fazla kuruluşa aitse, **oturum açmada** etkin olanı seçin; kaydedilir ve daha sonraki her komut için kullanılır:

```bash
agenteye login --org acme           # kimlik doğrula ve etkin kiracıyı bir adımda ayarla
agenteye orgs list                  # erişebileceğiniz kuruluşlar (etkin olan işaretlenir)
agenteye orgs switch globex         # kaydedilmiş varsayılanı değiştir
agenteye --org globex sessions      # tek komut için geçersiz kıl
```

Tam olarak bir kuruluşa aitse otomatik olarak seçilir ve `--org` tamamen yoksayabilirsiniz. Birkaç kuruluşa aitseniz ve bir tane seçmezseniz, CLI bunları listeler ve `--org <slug>` ile yeniden çalıştırmanızı ister. Etkin kuruluş her istekte pano'ya gönderilir ve izinleriniz **kuruluş başına** çözümlenir; `agenteye whoami` etkin kuruluşu, içindeki izinlerinizi ve tüm üyeliklerinizi gösterir.

## Yapılandırma

| Ayar | Bayrak | Ortam değişkeni | Varsayılan |
|---|---|---|---|
| Pano taban URL'si | `--base-url` | `AGENTEYE_DASHBOARD_URL` | **gerekli** (varsayılan yok) |
| Etkin kuruluş/kiracı | `--org` | `AGENTEYE_ORG` | oturum açmada seçilir; `~/.agenteye/cli.json` içinde kaydedilir |
| Oturum belirteci | `--token` | `AGENTEYE_CLI_TOKEN` | `~/.agenteye/cli.json` içinden |
| JSON çıkışı | `--json` | `AGENTEYE_CLI_JSON` | kapalı |
| TLS doğrulamasını atla | `--insecure` / `--secure` | `AGENTEYE_INSECURE` | kapalı (oturum açmada kaydedilir) |
| İstek zaman aşımı (saniye) | `--timeout` | _(yok)_ | 30 |
| Kullanım telemetrisini devre dışı bırak | _(yok)_ | `AGENTEYE_ANALYTICS_DISABLED` (veya `DO_NOT_TRACK`) | telemetri şu anda devre dışıdır; hiçbir şey gönderilmez |

Çözüm sırası **bayrak → ortam değişkeni → yapılandırma dosyasıdır**. Varsayılan değer yoktur; CLI'yi panonuza yönlendirmelisiniz, komut başına (`--base-url https://agenteye.example.com`) veya ortam aracılığıyla bir kez (ilk `login` sonrasında da kaydedilir):

```bash
export AGENTEYE_DASHBOARD_URL=https://agenteye.example.com
```

Yapılandırma dizini `AGENTEYE_HOME` 'ı onurlandırır (SDK ve toplayıcı tarafından kullanılan aynı kural); ayarlanırsa, `cli.json` `$AGENTEYE_HOME/cli.json` içinde bulunur.

### Kendi imzalı veya iç TLS

Panonuz kendi imzalı veya iç bir sertifika ile HTTPS üzerinden sunuluyorsa (örneğin, ham bir yük dengeleyici adı), TLS doğrulaması bunu `CERTIFICATE_VERIFY_FAILED` hatasıyla reddeder. Sertifika doğrulamasını atlamak için `--insecure` iletişim:

```bash
agenteye --base-url https://agenteye.internal --insecure login
```

`--insecure` **oturum açmada `cli.json` içinde kaydedilir**, bu nedenle daha sonraki komutlar doğrulamayı otomatik olarak atlar; bayrağı tekrarlamanız gerekmez. Tek seferlik doğrulanmış bir çağrı için `--secure` ilettir veya doğrulamayı bir sonraki oturum açmada geri açmak için. CLI, doğrulama devre dışı bırakıldığında pano'ya bağlanan herhangi bir komuttan önce stderr'ye bir uyarı yazdırır. Doğrulamayı atlamak ortadaki saldırılara karşı korumayı kaldırır; pano'ya ağ yoluna güvenmeden önce ona güvendiğinizden emin olun (VPN, özel alt ağ vb.).

## Telemetri ve Gizlilik

> **Not:** Gönderilen CLI, bugün **hiçbir kullanım telemetrisi göndermez.** Ana bir devre kesici açık olduğu için, ortamınızdan bağımsız olarak hiçbir şey iletilmez. Aşağıdaki bölüm, telemetri etkinleştirilirse vazgeçme yeteneğini açıklar.

Etkinleştirilse bile, telemetri **yalnızca anonim kullanım analitikleri** olacaktır, asla ajan, oturum veya etkinlik verileriniz:

- **Ajan, oturum veya etkinlik verisi asla altyapınızdan dışarı çıkmaz.** Yalnızca CLI kullanımı bildirilir: komut ve alt komut adı (örn. `keys create`), kullandığınız bayrakların **adları** (asla değerleri), başarı/çıkış durumu ve süre, ayrıca mutasyonlar için komut başına olay (örn. `api_key_created`, `query_run`) yalnızca statik adlar/enum'lar ve kaba sayıları taşır. Pano URL'niz, oturum belirteci, e-postanız, kuruluş slug'unuz, kaynak kimlikleri, SQL, anahtar sırları ve sorgu filtreleri **asla** gönderilmez. Operatörler yalnızca donuk bir iç kimlik tarafından tanımlanacaktır, asla e-posta tarafından değil.
- **Önceden devre dışı bırakın** CLI ortamında `AGENTEYE_ANALYTICS_DISABLED=1` ayarlayarak (CLI ayrıca araçlar arası `DO_NOT_TRACK=1` kuralını onurlandırır). Bu, telemetri etkinleştirildiği anda etkili olur, bu yüzden gizliliğe duyarlı bir ortam sonsuza kadar devre dışı kalabilir.
- Telemetri etkinleştirilirse, CLI doğrudan PostHog'a gönderir (`https://us.i.posthog.com`); bu ana bilgisayar engellenmiş bir makine sessizce hiçbir şey gönderisi ve CLI etkilenmemiş olacaktır.

## Genel Seçenekler ve Kurallar

Bunu bir kez okuyun; her komut için geçerlidir.

- **Genel seçenekler komutun ÖNCESİNE gider.** `agenteye --json sessions` doğru; `agenteye sessions --json` kullanım hatası. Globaller `--json`, `--base-url`, `--org`, `--token`, `--insecure`/`--secure`, `--timeout`, `--quiet` ve `--no-color` dir.
- **`--json` saf JSON'u stdout'a ve başka hiçbir şeyi yazdırır.** İnsan durum satırları, uyarılar ve hatalar **stderr'ye** gider, bu yüzden `--json` stdout yakalaması `jq` içine gönderilirse temiz kalır, hatta bir durum satırı gösterilirken. `--json` olmadan kutu halinde, insan gözleri için renklendirilmiş bir görünüm alırsınız.
- **`--help` ile keşfedin.** Her komut ve alt komutun `--help` (ve `-h` takma adı) vardır: `agenteye -h`, `agenteye sessions -h`, `agenteye keys create -h`. Üst düzey yardım ayrıca çıkış kodlarını ve genel seçenekleri listeler. Genel makine tarafından okunabilir bir yüzey dökümü yoktur; komut başına `--help` artı alan adına özgü `agenteye query schema` ve `agenteye settings schema` kullanın.
- **Onaylar betikler ve aracılar için otomatik atlanır.** Oluşturma/güncelleme/silme komutları etkileşimli bir terminalde "emin misin?" ister, ancak **`--json` altında veya stdin bir TTY olmadığında otomatik olarak bu istem atlanır** (TTY etkileşimli bir terminal oturumudur; bir boru veya CI çalıştırıcı değildir), bu yüzden betikler ve aracılar hiçbir zaman askıya almaz. Açıkça atlamak için `--yes`/`-y` ilettir. İstem bir ajan için ateşlenmeyeceğinden, ajan yıkıcı işlemleri insan ile ilk olarak doğrulamalıdır.
- **Sayfalandırma:** sonuçlar en yeniler ilk ve imleçli sayfalandırma (her sayfa sonraki sayfa almak için kullandığınız bir belirteci döner). `--limit N` (takma ad `-n`) satırları sınırlar ve **varsayılan olarak 50**; `--all` otomatik sayfalandırır (200 satırlık parçalar içinde) **`--limit` kadar**, bu yüzden çıplak `--all` yine de 50'de durur. Tam bir tarama için yüksek açık bir kapak ilettir: `--all --limit 1000`. `--page-size N` istekleri başına parçayı kontrol eder (maksimum 200); `--cursor <id>` önceki sayfanın `next_cursor` değerinden devam eder.
- **Zaman filtreleri:** `--since` göreli bir pencere alır: `15m`, `1h`, `6h`, `24h`, `7d` veya `all` (pano'nun hazır ayarları). Daha uzun veya özel bir aralık için (30 günü söyleyelim), `--from`/`--to` kullanın: `T` ve bir saat dilimi içeren açık ISO-8601 UTC zaman damgaları (örn. `2026-06-01T00:00:00Z`) `--since` geçersiz kılarlar. Boşlukla ayrılmış veya saat dilimi olmayan bir değer kullanım hatasıdır.
- **`--fields a,b,c`** (`events`, `sessions`, `evals`, `errors` üzerinde) çıkışı tablo ve `--json` için bu anahtarlarla kısıtlar. Bilinmeyen adlar geçerli listenin bulunması ile reddedilir, alan adlarını keşfetmenin ucuz bir yolu.
- **`--file payload.json`** (veya stdin okumak için `--file -`) bir kaynağın karmaşık bir şekle sahip olması halinde tam JSON istek gövdesi sağlar (`alerts create/update`, `settings set` ve `users create/update` üzerinde). Kaydedilmiş sorgu SQL'i bunun yerine `--sql @file.sql` kullanır.
- **Çok değerli filtreler** virgülle ayrılmış → bir set olarak eşleştirilir (bir filtre içinde birleşim, filtreler arasında VE): `--event-type tool_use,tool_result`. Tıkla seçenekleri variadic değildir, bu yüzden `--add a b` kırılır. `--add a,b` kullanın, bayrağı tekrarlayın (`--add a --add b`) veya alıntı yapın (`--add "a b"`).

## Komut Referansı

### En çok bu 5 komutu kullanacaksınız

Çoğu günlük çalışma bir avuç okuma komutu ile çalışır. Buradan başlayın, sonra daha sonraki tam yüzeyine ihtiyacınız olduğunda ulaşın:

| Komut | Ne yaptığı | Deneyin |
|---|---|---|
| `sessions` | Ajan çalışması başına bir satır: zaman, ortam, ajan, durum, son puan. | `agenteye --json sessions --since 24h --status error` |
| `events` | Bir çalışmanın içindeki ham adım adım izi (yükleri için `--full` ekleyin). | `agenteye --json events --session-id run-001 --all` |
| `evals` | Değerlendirme sonuçları ve puanlar; `--aggregate` bunları toplayıp kesmek için. | `agenteye --json evals --aggregate --since 7d --env prod` |
| `errors` | Yalnızca hata verilen etkinlikler; `--aggregate` tipe göre sayıları içindir. | `agenteye --json errors --since 24h --aggregate` |
| `list` | Geçerli filtre değerlerini keşfedin (ajanlar, ortamlar, modeller, …). | `agenteye list agents` |

### CLI'nin yapabileceği her şey

Tam yüzey izler. CLI **18 üst düzey komuta** sahiptir. Tüm okuma komutları `--json` ve yukarıdaki genel seçenekleri kabul eder; herhangi biri için kapsamlı bayrak listesi ve JSON şekli için `agenteye <command> -h` (veya `<command> <subcommand> -h`) çalıştırın.

### Kimlik: `login` · `logout` · `whoami` · `orgs` · `version` · `help`

```bash
agenteye login --email you@example.com [--org acme]   # gönderilen tek seferlik kod; oturumu kaydeder
agenteye logout                                       # bu makinede kaydedilmiş oturumu temizle
agenteye whoami                                       # mevcut kullanıcı, etkin kuruluş, izinler
agenteye version                                      # CLI sürümünü yazdır (--version ile aynı)
agenteye help                                         # üst düzey yardım (--help ile aynı)
```

`orgs` etkin kiracıyı inceler ve değiştirir:

```bash
agenteye orgs list        # kuruluşlar + her birindeki rolü (etkin olan işaretlenir)
agenteye orgs switch acme # kaydedilmiş etkin kuruluşu değiştir (TTY'de bir listeden seçmek için slug'u atlayın)
agenteye orgs current     # etkin kuruluş için kimlik kartı
agenteye orgs perms       # kaynağa göre gruplandırılmış etkin kuruluşta izinleriniz
```

### Gözlemle (salt okunur): `events` · `sessions` · `evals` · `errors` · `list`

Bunların hiçbiri onay gerektirmez. Paylaşılan filtreler: `--session-id`, `--agent-id`, `--env` (**değil** `--environment`) ve zaman aralığı (`--since` / `--from` / `--to`).

```bash
# events (takma ad: ham adım adım izi), en yeniler ilk
agenteye --json events --session-id run-001 --event-type tool_use,tool_result --all --limit 1000
agenteye --json events --since 1h --search timeout --all | jq '.events[].payload'

# sessions: ajan çalışması başına bir satır (zaman/ortam/ajan/oturum/durum; puan filtresi yok)
agenteye --json sessions --since 24h --status error
agenteye --json sessions --agent-id checkout-bot --env prod --all --limit 1000

# evals: değerlendirme sonuçları + puanlar; --score metriğe göre filtreler, --aggregate kesmek için
agenteye --json evals --score helpfulness:0.5..0.8 --score tool_efficiency:..0.3
agenteye --json evals --aggregate --since 7d --env prod        # durum karışımı + anahtar başına puan istatistikleri

# errors: hata verilen etkinlikler; --aggregate sayı/oturumlar/ajanlar/son görülen
agenteye --json errors --since 24h --aggregate
agenteye --json errors --since 24h --error-type timeout --all --limit 1000

# list: filtreleme öncesi geçerli filtre değerlerini keşfet
agenteye list envs        # ayrıca: agents event_types score_filters models hooks tools error_types
```

`--score KEY:MIN..MAX` (**`evals` üzerinde**, `sessions` değil) tekrarlanabilir ve VE-birleştirilir; her sınır isteğe bağlıdır (`..0.5` ≤ 0.5 anlamı, `0.9..` ≥ 0.9 anlamı). İstek başına en fazla 20 puan filtresi. `evals --scores-full` **insan tablosu için** bir görüntü bayrağıdır; ilk birkaçı yerine her puan çiftini artı `+N` sayısını gösterir. `--json` altında etkisi olmaz, her zaman tam puan nesnesini döner. **Bir oturumu uçtan uca okumak** için etkinlik izini değerlendirmesi ile birleştirin:

```bash
agenteye --json events --session-id run-001 --all --limit 1000 | jq '.events[] | {ts, event_type}'
agenteye --json evals  --session-id run-001                       # puanları + durumu
```

### Yönet (izin geçidi): `keys` · `users` · `settings` · `alerts` · `incidents`

**`keys`**: API anahtarları. Gizli dizi yerel olarak oluşturulur, sunucuya gönderilir (yalnızca bir karması depolanır) ve oluştur/yeniden oluştur'da **bir kez gösterilir**; o zaman yakala. `--json` ile yalnızca `key` alanında görünür. **Ad** ile başvurulur.

```bash
agenteye keys list                                   # etkin anahtarlar ilk, sonra iptal edildi
agenteye keys show ci-bot
agenteye keys create ci-bot --add events:read.add    # ihtiyacınıza kapsamla; gizli dizisi yazdırır BİR KEZ
agenteye keys create ops --permission-set standard --remove queries:run   # hazır ayarla başla, sonra trim
agenteye keys update ci-bot --add evaluations:read --yes
agenteye keys regenerate ci-bot --yes                # gizli diziyi döndür (eski bir durur)
agenteye keys disable ci-bot --yes                   # iptal et
```

İzinler `(permission-set ∪ --add) − --remove` olarak çalışır. Belirteçler `slug:action` (örn. `events:read`) veya bir kaynakta birkaçını genişletmek için `slug:action.action` (`events:read.add` → `events:read`, `events:add`). Hazır ayarlar: `read-only`, `standard`, `admin`. İnsan yalnızca izinleri (`keys:update`) bir anahtara verilemez.

**`users`**: kuruluş üyeleri, **e-posta** tarafından başvurulur (UUID kimliği de kabul edilir).

```bash
agenteye users list [--active-only]
agenteye users show dev@corp.com
agenteye users create dev@corp.com --permission-set standard
agenteye users update dev@corp.com --add alerts:write --remove queries:delete   # tahmin + doğrula
agenteye users disable dev@corp.com --yes            # korumalı/kendi korumalar vardır
agenteye users enable dev@corp.com
```

**`settings`**: sabit bir kayıt defteri (mevcut anahtarları okur ve değiştirirsiniz; yeni olanlar oluşturamazsınız).

```bash
agenteye settings list                               # anahtar · değer · tür · güncellendi (gizliler maskelenir)
agenteye settings schema                             # her anahtarın ne kabul ettiği (tür · aralık · açıklama)
agenteye settings set session_ttl_secs --value 86400 --yes
```

**`alerts`**: uyarı tanımları, **ad** ile başvurulur. `create` pozisyonel NAME artı bayraklar veya `--file` aracılığıyla tam JSON gövdesi alır.

```bash
agenteye alerts list
agenteye alerts show high-errors
agenteye alerts create high-errors --file alert.json          # NAME gerekli (pozisyonel)
agenteye alerts update high-errors --severity critical --yes
agenteye alerts test high-errors --yes                        # test bildirimi ateşle
agenteye alerts delete high-errors --yes
```

**`incidents`**: uyarı olayları, kimlik tarafından başvurulur (kısa kimlikler kabul edilir). `show` tam etkinlik günlüğünü yazdırır; davranmadan önce oku.

```bash
agenteye incidents list --state firing                # ayrıca: acknowledged, resolved
agenteye incidents count
agenteye incidents show <id>
agenteye incidents ack <id>
agenteye incidents assign <id> you@corp.com           # atanan bir operatör olmalı
agenteye incidents resolve <id> --yes
agenteye incidents open --alert-id <id> --severity critical    # bir uyarı'ya karşı manuel olarak bir tane aç
agenteye incidents comment-add <id> "kök neden: üst akış 5xx"
agenteye incidents comment-list <id> ; agenteye incidents comment-delete <id> <comment-id>
agenteye incidents subscribe <id> ; agenteye incidents unsubscribe <id> ; agenteye incidents subscribers <id>
```

### Analitik & Asistan: `query` · `agent`

**`query`**: analitik mağazanıza karşı kaydedilmiş SQL artı geçici bir çalıştırıcı. Kaydedilmiş sorgular **ad** tarafından başvurulur; SQL sunucu tarafı doğrulanır (yalnızca SELECT/WITH, ifade zaman aşımı, satır sınırı).

```bash
agenteye query schema [TABLE]                         # analitik görünümlerin sütun düzeni
agenteye query run --sql "select count(*) from analytics.events"
agenteye query run errs --arg prod --limit 100        # kaydedilmiş sorgu + pozisyonel $1 çalıştır
agenteye query list ; agenteye query show errs
agenteye query create errs --sql @errs.sql --description "hata verilen etkinlikler (24s)"
agenteye query update errs --sql @errs.sql --yes ; agenteye query delete errs --yes
```

**`agent`**: yerleşik **AI asistanı** ile konuşur (panoda sohbet edebileceğiniz aynı salt okunur analist). Sohbetler kısa bir sohbet kimliği ile başvurulur (ön ek çözümlü).

```bash
agenteye agent health                                 # AI asistanı yapılandırılmış/ulaşılabilir mi
agenteye agent models                                 # --model'e ilettebileceğiniz modeller (varsayılan işaretlenir)
agenteye agent ask "hangi ajanlar son günde hata verdi?"    # bir sohbet başlatır; kısa kimliğini yazdırır
agenteye agent ask --chat <short-id> "ve hangi araçları çağırdılar?"   # bu sohbeti devam ettir
agenteye agent chats ; agenteye agent show <short-id>
agenteye agent rename <short-id> --title "hata sınıflandırması" ; agenteye agent delete <short-id>
```

## Çıkış Kodları

| Kod | Anlamı |
|---|---|
| 0 | Başarı |
| 1 | Beklenmeyen hata (örn. pano bir 5xx döndü) |
| 2 | Kullanım hatası (geçersiz argümanlar, bilinmeyen komut/bayrak, ad çarpışması) |
| 3 | Panoya ulaşılamaz |
| 4 | Oturum açmadı veya oturum süresi doldu; `agenteye login` çalıştırın |
| 5 | Kimlik doğrulandı, ancak hesabınız gerekli izne sahip değil (ileti onu adlandırır) |
| 6 | İstenen kaynak bulunamadı (örn. bilinmeyen oturum veya olay kimliği) |

Bunlar CLI'yi komut dosyası yazmak için güvenli kılar: bir kodlama aracısı yeniden kimlik doğrulama istemi için `4`'e dallandırabilir veya eksik izni yüzey için `5`'e. Bkz. [Aracılar için CLI Tarifler](/tr/agenteye/cli-recipes) çıkış kodu işleme desenleri ve JSON çıkış şekilleri için kodlama aracıları CLI'yi sürüyor.
