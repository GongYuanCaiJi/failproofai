Terminali veya bir betiğinden Failproof AI Observability'nin tüm özelliklerini yönet: pano gezi yok. `agenteye` CLI'si verilerinizi (oturumlar, etkinlik günlükleri, değerlendirmeler) sorgular ve kuruluşunuzu yönetir (API anahtarları, kullanıcılar, ayarlar, uyarılar, olaylar, kaydedilmiş sorgular). Bir kontrol otomatikleştirmek, Observability'i CI'ye entegre etmek veya bir kodlama aracısının üretimi incelemesini sağlamak istediğinizde buna başvurun. Her komut `--json` bayrağını destekler, bu sayede komut satırında veya bir kodlama aracısının (Claude Code, Cursor) komut çalıştırıp sonucu ayrıştırması için eşit ölçüde iyi çalışır.

Tek bir binary ile şunları yapabilirsiniz:

- **Verilerinizi okuyun**: `sessions`, `events`, `evals`, `errors` (zaman, ajan, ortam, puan ile filtreleme).
- **Kuruluşunuzu yönetin**: `keys`, `users`, `settings`, `alerts`, `incidents`.
- **Analitik çalıştırın**: kaydedilmiş SQL ve ad-hoc sorgu çalıştırıcısı (`query`).
- **AI asistanına sorun**: panoda sohbet ettiğiniz aynı salt-okunur analist (`agent`).

> **Not:** Bu `agenteye` CLI'sidir, toplayıcı daemon'dan (`agenteye-collector`) farklı bir araçtır. CLI panonuzla konuşur; toplayıcı etkinlikleri sunucuya gönderir.

## Hızlı Başlangıç

Sıfırdan ilk sonuca dört satırda ulaşın. CLI'yi panonuza işaret edin, oturum açın, kimliğinizi onaylayın, sonra son günün çalıştırmalarını çekin:

```bash
pipx install agenteye
agenteye --base-url https://agenteye.example.com login --email you@example.com   # emailed 6-digit code
agenteye whoami                                                                  # confirm user + active org
agenteye --json sessions --since 24h                                             # one row per agent run, last 24h
```

Bu son komut en son oturumların JSON nesnesini yazdırır (en yeni ilk, varsayılan olarak 50 ile sınırlı). `jq`'ye akış göndererek dilimleyin veya salt-okunur tablo için `--json`'i bırakın. Her satır çalıştırmanın durumunu ve bir değerlendirici tarafından puanlanmışsa metrik puanlarını içerir (burada kısaltılmış):

```json
{
  "sessions": [
    {
      "session_id": "run-8f2a",
      "agent_id": "checkout-bot",
      "environment": "prod",
      "status": "error",
      "scores": { "helpfulness": 0.42, "tool_efficiency": 0.55 },
      "event_count": 37,
      "started_at": "2026-07-16T09:14:02Z",
      "last_event_at": "2026-07-16T09:14:48Z"
    }
  ],
  "next_cursor": null
}
```

Bu sayfanın geri kalanı her parçayı açıklar: [yükleme](#installation), [oturum açma](#authentication), [yapılandırma](#configuration), [global kurallar](#global-options--conventions) ve [tam komut referansı](#command-reference).

## Yükleme

CLI, **`agenteye`** adlı halka açık PyPI paketidir. Her zaman kendi bağımlılıklarına sahip olması için izole bir ortamda kurun:

```bash
pipx install agenteye
# veya
uv tool install agenteye
```

Python 3.10+ gerektirir. Yüklenmiş komut **`agenteye`**'dir:

```bash
agenteye --version
agenteye --help
```

> **Not:** Failproof AI Observability Python SDK de `agenteye` dağıtım adını kullanır. CLI'yi `pipx` veya `uv tool` ile kurmak (paylaşılan virtualenv'e `pip install` yapmak yerine) ikisinin çakışmasını engeller. Düz bir `pip install agenteye`, SDK aynı ortamda yüklü olmadığı takdirde uygundur.

## Kimlik Doğrulama

CLI, **pano** ile e-postayla gönderilen tek seferlik bir kod kullanarak kimlik doğrulaması yapar:

```bash
agenteye login --email you@example.com
# 6 haneli bir kod e-postayla gönderilir; komut satırına yapıştırın.
```

Oturum jetonu `~/.agenteye/cli.json`'de depolanır (sadece sizin tarafından okunabilir, mod `0600`) ve varsayılan olarak 24 saat geçerlidir. Süresi dolduğunda `agenteye login` komutunu tekrar çalıştırın.

```bash
agenteye whoami     # geçerli kullanıcıyı, etkin kuruluşu ve izinleri göster
agenteye logout     # oturumu iptal et ve depolanmış jetonu temizle
```

`whoami` eksik veya süresi dolan oturum üzerinde hata vermez; bunun yerine `logged_in: false` rapor eder, bu nedenle bir betik veya ajan kimlik doğrulama durumunu güvenle sondayabilir (pano URL'si ayarlanmamışsa veya erişilemiyorsa yine de sıfırdan farklı bir kod döndürebilir).

**Gereklilikler:** e-postanız panoda oturum açabilmek için izinli olmalıdır (Failproof AI Observability yöneticinize sorun) ve pano temel URL'sinde erişilebilir olmalıdır (bkz. [Yapılandırma](#configuration)). Bir kod talep eder ve hiçbiri gelmezse, e-postanız muhtemelen henüz pano erişimi için etkinleştirilmemiştir.

## Kuruluşunuzu seçin (çok kiracılı)

Hesabınız birden fazla kuruluşa aitse, etkin olanı **oturum açarken** seçin; kaydedilir ve sonraki her komut için kullanılır:

```bash
agenteye login --org acme           # kimlik doğrula ve etkin kiracıyı tek adımda ayarla
agenteye orgs list                  # erişebileceğiniz kuruluşlar (etkin olan işaretlenmiş)
agenteye orgs switch globex         # kaydedilmiş varsayılanı değiştir
agenteye --org globex sessions      # tek bir komut için geçersiz kıl
```

Tam olarak bir kuruluşa ait olursanız otomatik seçilir ve `--org`'u tamamen göz ardı edebilirsiniz. Birkaçına ait olur ve birini seçmezseniz, CLI bunları listeler ve `--org <slug>` ile yeniden çalıştırmanızı ister. Etkin kuruluş her istekte panoya gönderilir ve izinleriniz **kuruluş başına** çözülür; `agenteye whoami` etkin kuruluşu, içindeki izinlerinizi ve tüm üyeliklerinizi gösterir.

## Yapılandırma

| Ayar | Bayrak | Ortam değişkeni | Varsayılan |
|---|---|---|---|
| Pano temel URL | `--base-url` | `AGENTEYE_DASHBOARD_URL` | **gerekli** (varsayılan yok) |
| Etkin kuruluş/kiracı | `--org` | `AGENTEYE_ORG` | oturum açarken seçilmiş; `~/.agenteye/cli.json`'de kaydedilmiş |
| Oturum jetonu | `--token` | `AGENTEYE_CLI_TOKEN` | `~/.agenteye/cli.json`'den |
| JSON çıktısı | `--json` | `AGENTEYE_CLI_JSON` | kapalı |
| TLS doğrulamasını atla | `--insecure` / `--secure` | `AGENTEYE_INSECURE` | kapalı (oturum açarken kaydedilmiş) |
| İstek zaman aşımı (saniye) | `--timeout` | _(yok)_ | 30 |
| Kullanım telemetrisini devre dışı bırak | _(yok)_ | `AGENTEYE_ANALYTICS_DISABLED` (veya `DO_NOT_TRACK`) | telemetri şu anda devre dışı; hiçbir şey gönderilmez |

Çözümleme sırası **bayrak → ortam değişkeni → yapılandırma dosyası**'dır. Varsayılan yoktur; CLI'yi panonuza işaret etmelisiniz, komut başına (`--base-url https://agenteye.example.com`) veya ortam aracılığıyla bir kez (ilk `login`'den sonra da kaydedilir):

```bash
export AGENTEYE_DASHBOARD_URL=https://agenteye.example.com
```

Yapılandırma dizini `AGENTEYE_HOME`'u onurlandırır (SDK ve toplayıcı tarafından kullanılan aynı kural); ayarlanmışsa, `cli.json` `$AGENTEYE_HOME/cli.json`'de bulunur.

### Kendi imzalı veya dahili TLS

Panonuz kendi imzalı veya dahili bir sertifika ile HTTPS üzerinden sunuluyorsa (örneğin, ham yük dengeleyici adı), TLS doğrulaması `CERTIFICATE_VERIFY_FAILED` hatası ile reddeder. Sertifika doğrulamasını atlamak için `--insecure` geçirin:

```bash
agenteye --base-url https://agenteye.internal --insecure login
```

`--insecure` **oturum açarken `cli.json`'ye kaydedilir**, bu nedenle sonraki komutlar doğrulamayı otomatik olarak atlar; bayrağı tekrar etmemeniz gerekir. Tek seferlik doğrulanmış bir çağrı için `--secure`'ı geçirin veya bir sonraki oturum açarken doğrulamayı geri açmak için. CLI, doğrulama devre dışıyken panoya bağlanan herhangi bir komuttan önce stderr'e bir uyarı yazdırır. Doğrulamayı atlamak ortadaki adam saldırılarına karşı korumayı kaldırır; panonuza olan ağ yolunun güvenilir olduğundan (VPN, özel alt ağ vb.) emin olun.

## Telemetri ve Gizlilik

> **Not:** Gönderilen CLI **bugün hiçbir kullanım telemetrisi göndermez.** Master bir kill switch açıktır, bu nedenle ortamınız ne olursa olsun hiçbir şey iletilmez. Aşağıdaki bölüm, telemetri hiç etkinleştirilirse opt-out yeteneğini açıklar.

Etkinleştirilse bile, telemetri **yalnızca anonim kullanım analitiği** olacak, asla ajan, oturum veya etkinlik verileriniz olmayacak:

- **Ajan, oturum veya etkinlik verisi asla altyapınızdan ayrılmaz.** Yalnızca CLI kullanımı rapor edilir: komut ve alt komut adı (ör. `keys create`), kullandığınız bayrakların **adları** (hiçbir zaman değerleri), başarı/çıkış durumu ve süresi, artı mutasyonlar için eylem başına etkinlik (ör. `api_key_created`, `query_run`) yalnızca statik adlar/enums ve kaba sayımlar ile. Pano URL'niz, oturum jetonu, e-posta, kuruluş slug, kaynak kimlikleri, SQL, anahtar gizlilikleri ve sorgu filtreleri **asla** gönderilmeyecektir. Operatörler yalnızca donuk bir dahili kimlik tarafından tanımlanacak, hiçbir zaman e-posta tarafından değil.
- **Önceden opt out edin** CLI'nin ortamında `AGENTEYE_ANALYTICS_DISABLED=1` ayarlayarak (CLI ayrıca araç arası `DO_NOT_TRACK=1` kuralına uyar). Bu, telemetri hiç açılırsa, gizliliğe duyarlı bir ortam kalıcı olarak opt out kalabilir.
- Telemetri etkinleştirilirse, CLI doğrudan PostHog'a gönderir (`https://us.i.posthog.com`); bu ana sahip engellenen bir makine sessizce hiçbir şey göndermez ve CLI etkilenmez.

## Global seçenekler ve kurallar

Bunu bir kez okuyun; her komuta uygulanır.

- **Global seçenekler komuttan ÖNCE gider.** `agenteye --json sessions` doğru; `agenteye sessions --json` kullanım hatasıdır. Global'ler `--json`, `--base-url`, `--org`, `--token`, `--insecure`/`--secure`, `--timeout`, `--quiet` ve `--no-color`'dür.
- **`--json` stdout'a saf JSON yazdırır ve başka hiçbir şey yazdırmaz.** İnsan durum satırları, uyarılar ve hatalar **stderr**'e gider, bu nedenle `--json` stdout yakalama `jq`'ye aktarmak için temiz kalır, bir durum satırı gösterilse bile. `--json` olmadan insan gözleri için kutulu, renkli bir görüntü alırsınız.
- **`--help` ile keşfedin.** Her komut ve alt komutun `--help` vardır (ve `-h` takma adı): `agenteye -h`, `agenteye sessions -h`, `agenteye keys create -h`. Üst düzey yardım ayrıca çıkış kodlarını ve global seçenekleri listeler. Hiç global makine tarafından okunabilir yüzey dökümü yoktur; komut başına `--help`, artı alana özgü `agenteye query schema` ve `agenteye settings schema`'yı kullanın.
- **Onaylar etkileşimli olmayan durumlarda otomatik atlanır.** Oluştur/güncelle/sil komutları etkileşimli bir terminalde "emin misiniz?" sorar, ancak **`--json` altında veya stdin bir TTY olmadığında otomatik olarak bu istemi atlarlar** (TTY etkileşimli bir terminal oturumudur; bir pipe veya CI çalıştırıcısı değildir), bu nedenle betikler ve ajanlar asla takılmaz. Bunu açıkça atlamak için `--yes`/`-y` geçirin. İstem bir ajan için ateşlenmeyeceği için, bir ajan yıkıcı eylemleri insan ile önce onaylamalıdır.
- **Sayfalandırma:** sonuçlar en yeni ilk ve cursor tabanlı (her sayfa sonraki sayfayı getirmek için kullanılan bir jeton döner). `--limit N` (takma ad `-n`) satırları sınırlar ve **varsayılan olarak 50**; `--all` otomatik sayfalandırma (**200 satır parçalarda**) `--limit`'e kadar, bu nedenle düz bir `--all` yine 50'de durur. Tam bir tarama için yüksek bir açık sınır geçin: `--all --limit 1000`. `--page-size N` istek başına parçayı kontrol eder (maksimum 200); `--cursor <id>` önceki sayfanın `next_cursor`'undan devam eder.
- **Zaman filtreleri:** `--since` göreceli bir pencere alır: `15m`, `1h`, `6h`, `24h`, `7d` veya `all` (panunun ön ayarları). Daha uzun veya özel bir aralık için (son 30 gün deyin), `--from`/`--to` kullanın: açık ISO-8601 UTC zaman damgaları **`T` ve saat dilimi** (ör. `2026-06-01T00:00:00Z`) ile `--since`'i geçersiz kılar. Boşluk ayrılmış veya saat dilimi olmayan bir değer kullanım hatasıdır.
- **`--fields a,b,c`** (`events`, `sessions`, `evals`, `errors`'de), çıktıyı bu anahtarlara kısıtlar, hem tablo hem de `--json` için. Bilinmeyen adlar geçerli listeyle reddedilir, alan adlarını keşfetmenin ucuz bir yolu.
- **`--file payload.json`** (veya stdin'i okumak için `--file -`) kaynak karmaşık bir şekle sahip olduğunda tam bir JSON istek gövdesini sağlar (`alerts create/update`, `settings set` ve `users create/update`'de). Kaydedilmiş sorgu SQL'i `--sql @file.sql` kullanır.
- **Çok değerli filtreler** virgülle ayrılmıştır → bir set olarak eşleştirilir (bir filtre içinde birleşim, filtreler arasında AND): `--event-type tool_use,tool_result`. Click seçenekleri değişken değildir, bu nedenle `--add a b` kırılır. `--add a,b`'yi, bayrağı tekrarlayın (`--add a --add b`) veya alıntı yapın (`--add "a b"`).

## Komut Referansı

### En sık kullanacağınız 5 komut

Günlük işlerin çoğu birkaç okuma komutu aracılığıyla çalışır. Buradan başlayın, sonra aşağıdaki tam yüzeye ihtiyaç duyduğunuzda ulaşın:

| Komut | Ne yaptığı | Deneyin |
|---|---|---|
| `sessions` | Ajan çalıştırması başına bir satır: zaman, ortam, ajan, durum, en son puan. | `agenteye --json sessions --since 24h --status error` |
| `events` | Çalıştırmanın içindeki ham adım-adım iz (yükler için `--full` ekleyin). | `agenteye --json events --session-id run-001 --all` |
| `evals` | Değerlendirme sonuçları ve puanlar; `--aggregate` bunları toplar. | `agenteye --json evals --aggregate --since 7d --env prod` |
| `errors` | Yalnızca hata etkinlikleri; türe göre sayımlar için `--aggregate`. | `agenteye --json errors --since 24h --aggregate` |
| `list` | Geçerli filtre değerlerini keşfedin (ajanlar, ortamlar, modeller vb.). | `agenteye list agents` |

### CLI'nin yapabileceği her şey

Tam yüzey aşağıdadır. CLI **18 üst düzey komuta** sahiptir. Tüm okuma komutları `--json` ve yukarıdaki global seçenekleri kabul eder; herhangi birinin kapsamlı bayrak listesi ve JSON şekli için `agenteye <command> -h` (veya `<command> <subcommand> -h`) çalıştırın.

### Kimlik: `login` · `logout` · `whoami` · `orgs` · `version` · `help`

```bash
agenteye login --email you@example.com [--org acme]   # emailed one-time code; saves the session
agenteye logout                                       # clear the saved session on this machine
agenteye whoami                                       # current user, active org, permissions
agenteye version                                      # print the CLI version (same as --version)
agenteye help                                         # top-level help (same as --help)
```

`orgs` etkin kiracıyı inceler ve değiştirir:

```bash
agenteye orgs list        # your orgs + your role in each (active one marked)
agenteye orgs switch acme # change the saved active org (omit the slug to pick from a list on a TTY)
agenteye orgs current     # identity card for the active org
agenteye orgs perms       # your permissions in the active org, grouped by resource
```

### Gözlemle (salt okunur): `events` · `sessions` · `evals` · `errors` · `list`

Bunların hiçbirinin onaya ihtiyacı yoktur. Paylaşılan filtreler: `--session-id`, `--agent-id`, `--env` (**değil** `--environment`) ve zaman aralığı (`--since` / `--from` / `--to`).

```bash
# events (takma ad: ham adım-adım iz), en yeni ilk
agenteye --json events --session-id run-001 --event-type tool_use,tool_result --all --limit 1000
agenteye --json events --since 1h --search timeout --all | jq '.events[].payload'

# sessions: ajan çalıştırması başına bir satır (zaman/ortam/ajan/oturum/durum; puan filtreleme yok)
agenteye --json sessions --since 24h --status error
agenteye --json sessions --agent-id checkout-bot --env prod --all --limit 1000

# evals: değerlendirme sonuçları + puanlar; --score metrik tarafından filtreler, --aggregate toplar
agenteye --json evals --score helpfulness:0.5..0.8 --score tool_efficiency:..0.3
agenteye --json evals --aggregate --since 7d --env prod        # status mix + per-key score stats

# errors: hata etkinlikleri; sayımlar/oturumlar/ajanlar/son görülme için --aggregate
agenteye --json errors --since 24h --aggregate
agenteye --json errors --since 24h --error-type timeout --all --limit 1000

# list: filtrelemeden önce geçerli filtre değerlerini keşfedin
agenteye list envs        # also: agents event_types score_filters models hooks tools error_types
```

`--score KEY:MIN..MAX` (`evals`'de, `sessions`'de değil) tekrarlanabilir ve AND-birleştirilir; her sınır isteğe bağlıdır (`..0.5` ≤ 0.5, `0.9..` ≥ 0.9 anlamına gelir). İstek başına 20'ye kadar puan filtresi. `evals --scores-full` **yalnızca insan tablosu** için bir görüntüleme bayrağıdır; ilk birkaçın yerine her puan çiftini ve `+N` sayısını gösterir. `--json`'un altında hiçbir etkisi yoktur, bu her zaman tam puan nesnesini döner. **Bir oturumu uçtan uca okumak** için etkinlik izini değerlendirmesi ile birleştirin:

```bash
agenteye --json events --session-id run-001 --all --limit 1000 | jq '.events[] | {ts, event_type}'
agenteye --json evals  --session-id run-001                       # its scores + status
```

### Yönet (izin kapılı): `keys` · `users` · `settings` · `alerts` · `incidents`

**`keys`**: API anahtarları. Gizli yerel olarak oluşturulur, sunucuya gönderilir (yalnızca hash depolar) ve oluştur/yeniden oluştur'da **bir kez gösterilir**; o zaman yakala. `--json`'un altında yalnızca `key` alanında görünür. **Ad** tarafından referans alınır.

```bash
agenteye keys list                                   # active keys first, then revoked
agenteye keys show ci-bot
agenteye keys create ci-bot --add events:read.add    # scope to what you need; prints the secret ONCE
agenteye keys create ops --permission-set standard --remove queries:run   # seed a preset, then trim
agenteye keys update ci-bot --add evaluations:read --yes
agenteye keys regenerate ci-bot --yes                # rotate the secret (the old one stops working)
agenteye keys disable ci-bot --yes                   # revoke
```

İzinler `(permission-set ∪ --add) − --remove` olarak çalışır. Jetonlar `slug:action` (ör. `events:read`) veya bir kaynakta birkaçını genişletmek için `slug:action.action` (`events:read.add` → `events:read`, `events:add`). Ön ayarlar: `read-only`, `standard`, `admin`. İnsan izinleri (`keys:update`) bir anahtara verilemez.

**`users`**: kuruluş üyeleri, **e-posta** tarafından referans alınır (bir UUID kimliği de kabul edilir).

```bash
agenteye users list [--active-only]
agenteye users show dev@corp.com
agenteye users create dev@corp.com --permission-set standard
agenteye users update dev@corp.com --add alerts:write --remove queries:delete   # predicts + confirms
agenteye users disable dev@corp.com --yes            # has protected/self guards
agenteye users enable dev@corp.com
```

**`settings`**: sabit bir kayıt defteri (mevcut anahtarları okur ve değiştirir; yeni olanlar oluşturamazsınız).

```bash
agenteye settings list                               # key · value · type · updated (secrets masked)
agenteye settings schema                             # what each key accepts (type · range · description)
agenteye settings set session_ttl_secs --value 86400 --yes
```

**`alerts`**: uyarı tanımları, **ad** tarafından referans alınır. `create` konumsal bir NAME artı bayraklar veya `--file` aracılığıyla tam JSON gövdesi alır.

```bash
agenteye alerts list
agenteye alerts show high-errors
agenteye alerts create high-errors --file alert.json          # NAME is required (positional)
agenteye alerts update high-errors --severity critical --yes
agenteye alerts test high-errors --yes                        # fire a test notification
agenteye alerts delete high-errors --yes
```

**`incidents`**: uyarı olayları, kimlik tarafından referans alınır (kısa kimlikler kabul edilir). `show` tam etkinlik günlüğünü yazdırır; hareket etmeden önce okuyun.

```bash
agenteye incidents list --state firing                # also: acknowledged, resolved
agenteye incidents count
agenteye incidents show <id>
agenteye incidents ack <id>
agenteye incidents assign <id> you@corp.com           # assignee must be an operator
agenteye incidents resolve <id> --yes
agenteye incidents open --alert-id <id> --severity critical    # open one manually against an alert
agenteye incidents comment-add <id> "root cause: upstream 5xx"
agenteye incidents comment-list <id> ; agenteye incidents comment-delete <id> <comment-id>
agenteye incidents subscribe <id> ; agenteye incidents unsubscribe <id> ; agenteye incidents subscribers <id>
```

### Analitik ve asistan: `query` · `agent`

**`query`**: analitik mağazanıza karşı kaydedilmiş SQL artı ad-hoc çalıştırıcı. Kaydedilmiş sorgular **ad** tarafından referans alınır; SQL sunucu tarafı tarafından doğrulanır (SEÇ/İLE yalnızca, ifade zaman aşımı, satır sınırı).

```bash
agenteye query schema [TABLE]                         # column layout of the analytics views
agenteye query run --sql "select count(*) from analytics.events"
agenteye query run errs --arg prod --limit 100        # run a saved query + a positional $1
agenteye query list ; agenteye query show errs
agenteye query create errs --sql @errs.sql --description "errored events (24h)"
agenteye query update errs --sql @errs.sql --yes ; agenteye query delete errs --yes
```

**`agent`**: yerleşik **AI asistanı** ile konuşur (panonuzda sohbet edebileceğiniz aynı salt-okunur analist). Sohbetler kısa bir sohbet kimliği (ön ek çözümlendi) ile referans alınır.

```bash
agenteye agent health                                 # is the AI assistant configured/reachable
agenteye agent models                                 # models you can pass to --model (default marked)
agenteye agent ask "which agents errored most in the last day?"    # starts a chat; prints its short id
agenteye agent ask --chat <short-id> "and which tools did they call?"   # continue that chat
agenteye agent chats ; agenteye agent show <short-id>
agenteye agent rename <short-id> --title "error triage" ; agenteye agent delete <short-id>
```

## Çıkış kodları

| Kod | Anlamı |
|---|---|
| 0 | Başarı |
| 1 | Beklenmeyen hata (ör. pano 5xx döndürdü) |
| 2 | Kullanım hatası (geçersiz argümanlar, bilinmeyen komut/bayrak, ad çakışması) |
| 3 | Panoya ulaşılamaz |
| 4 | Oturum açmadı veya oturum süresi doldu; `agenteye login` çalıştırın |
| 5 | Kimlik doğrulanmış, ancak hesabınızda gerekli izin yok (ileti bunu adlandırır) |
| 6 | İstenen kaynak bulunamadı (ör. bilinmeyen oturum veya olay kimliği) |

Bunlar CLI'yi komut yapmak için güvenli kılar: bir kodlama aracısı `4`'te yeniden kimlik doğrulamaya yönlendirmek veya eksik izni yüzeylemek için `5`'e dallanabilir. Çıkış kodu işleme desenleri ve JSON çıktı şekilleri için [Ajanlar için CLI tarifleri](/tr/agenteye/cli-recipes)'ne bakın.
