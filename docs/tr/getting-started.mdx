## Gereksinimler

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (isteğe bağlı - yalnızca kaynaktan derleme için gerekli)

## Kurulum

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

## Hızlı başlangıç

<Steps>
  <Step title="Politikaları etkinleştirin">
    Politikalar, her aracı araç çağrısından önce ve sonra çalışan kurallardır. Zararlı komutları, gizli dizi sızıntılarını ve diğer hata modlarını hasar oluşturmadan önce yakalar.

    ```bash
    failproofai policies --install
    ```

    Bu, yüklü aracı CLI'lerinize hook girdileri yazar (Claude Code'un `~/.claude/settings.json`, OpenAI Codex'in `~/.codex/hooks.json`, GitHub Copilot CLI'ın `~/.copilot/hooks/failproofai.json`, Cursor Agent'ın `~/.cursor/hooks.json`, OpenCode'un `~/.config/opencode/plugins/failproofai.mjs` konumundaki oluşturulan eklenti dolgusu ve `~/.config/opencode/opencode.json` dosyasının `plugin` dizisindeki bir kayıt girişi, Pi'nin `~/.pi/agent/settings.json` veya Gemini CLI'ın `~/.gemini/settings.json`). Birden fazla varsa size sorulacaktır; istemini atlamak için `--cli claude codex copilot cursor opencode pi gemini` (herhangi bir alt küme) geçirin.

    GitHub Copilot CLI, Cursor Agent, OpenCode, Pi ve Gemini CLI desteği **beta** aşamasındadır — `--cli copilot`, `--cli cursor`, `--cli opencode`, `--cli pi` veya `--cli gemini` ile yükleyin.

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install --cli copilot --scope project
    failproofai policies --install --cli cursor --scope project
    failproofai policies --install --cli opencode --scope project
    failproofai policies --install --cli pi --scope project
    failproofai policies --install --cli gemini --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="Doğrulayın">
    ```bash
    failproofai policies
    ```

    Tüm politikaları, bunların etkinleştirilip etkinleştirilmediğini ve yapılandırılan parametreleri gösterir.
  </Step>
  <Step title="Panoyu başlatın">
    ```bash
    failproofai
    ```

    `http://localhost:8020` konumundaki yerel bir panoyu açar; burada oturumları göz atabilir, araç çağrılarını inceleyebilir ve politikaları yönetebilirsiniz.
  </Step>
  <Step title="Aracınızı çalıştırın">
    Claude Code'u normal şekilde başlatın. Aracı riskli bir şey denerse, failproofai otomatik olarak bunu engeller. Onu çalışır durumda bırakın ve panoda ne olduğunu gözden geçirin.
  </Step>
</Steps>

## Politikalar nasıl çalışır

Bir aracı her araç çalıştırdığında, Claude Code failproofai'yi bir alt işlem olarak çağırır:

```text
Claude Code  →  failproofai --hook PreToolUse  →  stdin JSON'u okur
                                                 politikaları değerlendirir
                                                 stdout'a karar yazar
```

Her politika üç karardan birini döndürür:

- **allow** - aracı normal şekilde devam eder
- **deny** - işlem engellenir, aracıya neden engellendi söylenir
- **instruct** - aracının isteminde ek bağlam eklenir

<Note>
Politikalar yerel işleminizde çalışır. Hiçbir şey uzak bir hizmete gönderilmez.
</Note>

## Kural tabanlı politikalarla takım politikaları ayarlayın

Takımınız genelinde kalite standartları oluşturmanın en hızlı yolu `.failproofai/policies/` kuralıdır. Politika dosyalarını bu dizine bırakın ve otomatik olarak yüklenir — bayrak yok, yapılandırma değişikliği yok, kurulum komutu yok.

<Steps>
  <Step title="Politikalar dizinini oluşturun">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="Politika dosyaları ekleyin">
    Başlangıç örneklerini kopyalayın veya kendi politikalarınızı yazın:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    Veya yeni bir tane oluşturun:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Commit yapmadan önce testleri çalıştırın.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Git'e yürütün">
    ```bash
    git add .failproofai/policies/
    git commit -m "Takım kalite politikalarını ekle"
    ```

    failproofai'nin yüklü olduğu her takım üyesi bu politikaları otomatik olarak alır. Geliştirici başına hiçbir kurulum gerekmez.
  </Step>
</Steps>

<Tip>
`.failproofai/policies/` dizinini deponuza yürütün ki bütün takım aynı standartları paylaşsın. Takımınız yeni hata modları keşfettikçe, politikalar ekleyin ve gönderin — herkes bir sonraki `git pull` üzerinde güncellemeleri alır. Zamanla bu politikalar, gelişmeyi sürdüren bir kalite standardı haline gelir.
</Tip>

## Veri depolaması

Tüm yapılandırma ve günlükler makinenizde kalır:

| Yol | Neyi saklar |
|------|----------------|
| `~/.failproofai/policies-config.json` | Küresel politika yapılandırması |
| `~/.failproofai/hook-activity.jsonl` | Hook yürütme geçmişi |
| `~/.failproofai/hook.log` | Özel hook hataları için hata ayıklama günlüğü |
| `.failproofai/policies-config.json` | Proje başına yapılandırma (yürütülmüş) |
| `.failproofai/policies-config.local.json` | Kişisel geçersiz kılmalar (gitignored) |

---

## Kaldırma

```bash
failproofai policies --uninstall
```

`~/.claude/settings.json` dosyasından hook girdilerini kaldırır. `~/.failproofai/` içindeki yapılandırma dosyaları tutulur.
