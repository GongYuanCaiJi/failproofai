מדיניויות מותאמות אישית מאפשרות לך לכתוב כללים לכל התנהגות של סוכן: אכוף קונבנציות פרויקט, מנע סטייה, חסום פעולות הרסניות, גלה סוכנים תקועים, או התאם עם Slack, זרימות אישור, ועוד. הן משתמשות באותו מערכת אירועי hook ובהחלטות `allow`, `deny`, `instruct` כמו מדיניויות מובנות.

---

## דוגמה מהירה

```js
// my-policies.js
import { customPolicies, allow, deny, instruct } from "failproofai";

customPolicies.add({
  name: "no-production-writes",
  description: "Block writes to paths containing 'production'",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Write" && ctx.toolName !== "Edit") return allow();
    const path = ctx.toolInput?.file_path ?? "";
    if (path.includes("production")) {
      return deny("Writes to production paths are blocked");
    }
    return allow();
  },
});
```

התקן אותה:

```bash
failproofai policies --install --custom ./my-policies.js
```

## שתי דרכים לטעון מדיניויות מותאמות אישית

### אפשרות 1: מבוססת קונבנציה (מומלצת)

שים קבצים `*policies.{js,mjs,ts}` ב-`.failproofai/policies/` והם נטענים באופן אוטומטי — אין צורך בדגלים או שינויי הגדרה. זה עובד כמו git hooks: שם קובץ, זה פשוט עובד.

```
# Project level — committed to git, shared with the team
.failproofai/policies/security-policies.mjs
.failproofai/policies/workflow-policies.mjs

# User level — personal, applies to all projects
~/.failproofai/policies/my-policies.mjs
```

**איך זה עובד:**
- שני הספריות של פרויקט ומשתמש נסרקות (union — לא first-scope-wins)
- קבצים נטענים בסדר אלפביתי בתוך כל ספרייה. הוסף קידומת `01-`, `02-` כדי לשלוט בסדר
- רק קבצים התואמים `*policies.{js,mjs,ts}` נטענים; קבצים אחרים מתעלמים
- כל קובץ נטען באופן עצמאי (fail-open לכל קובץ)
- עובד לצד מדיניויות `--custom` מפורשות ומדיניויות מובנות

<Tip>
מדיניויות קונבנציה הן הדרך הקלה ביותר לבנות תקן איכות לארגון שלך. Commit את `.failproofai/policies/` ל-git וכל חבר בצוות מקבל את אותם כללים באופן אוטומטי — אין צורך בהתקנה לכל מפתח. כשהצוות שלך מגלה מצבי כשל חדשים, הוסף מדיניות ודחוף. עם הזמן אלה הופכות לתקן איכות חי שיתמשך ויעתיק עם כל תרומה.
</Tip>

### אפשרות 2: נתיב קובץ מפורש

```bash
# Install with a custom policies file
failproofai policies --install --custom ./my-policies.js

# Replace the policies file path
failproofai policies --install --custom ./new-policies.js

# Remove the custom policies path from config
failproofai policies --uninstall --custom
```

הנתיב המוחלט שנפתר מאוחסן ב-`policies-config.json` כ-`customPoliciesPath`. הקובץ נטען מחדש בכל אירוע hook - אין שום caching בין אירועים.

### שימוש בשניהם יחד

מדיניויות קונבנציה וקובץ `--custom` המפורש יכולים להתקיים זה לצד זה. סדר טעינה:

1. קובץ `customPoliciesPath` מפורש (אם מוגדר)
2. קבצי קונבנציה של פרויקט (`{cwd}/.failproofai/policies/`, אלפביתי)
3. קבצי קונבנציה של משתמש (`~/.failproofai/policies/`, אלפביתי)

## API

### Import

```js
import { customPolicies, allow, deny, instruct } from "failproofai";
```

### `customPolicies.add(hook)`

רושם מדיניות. קרא לזה כמה פעמים שצריך לעבור מדיניויות מרובות באותו קובץ.

```ts
customPolicies.add({
  name: string;                         // required - unique identifier
  description?: string;                 // shown in `failproofai policies` output
  match?: { events?: HookEventType[] }; // filter by event type; omit to match all
  fn: (ctx: PolicyContext) => PolicyResult | Promise<PolicyResult>;
});
```

### עוזרי החלטה

| פונקציה | אפקט | משתמש כאשר |
|----------|--------|----------|
| `allow()` | התר את הפעולה בשקט | הפעולה בטוחה, אין צורך בהודעה |
| `deny(message)` | חסום את הפעולה | הסוכן לא צריך לנקוט בפעולה זו |
| `instruct(message)` | הוסף הקשר ללא חסימה | תן לסוכן הקשר נוסף כדי להישאר במסלול |

`deny(message)` - ההודעה מופיעה ל-Claude עם קידומת `"Blocked by failproofai:"`. `deny` יחיד חוצה כל הערכה נוספת.

`instruct(message)` - ההודעה מצורפת להקשר של Claude לקריאת הכלי הנוכחית. כל הודעות `instruct` מצטברות ומועברות ביחד.

<Tip>
אתה יכול להוסיף הנחיות נוסף לכל הודעת `deny` או `instruct` על ידי הוספת שדה `hint` ב-`policyParams` — אין צורך בשינוי קוד. זה עובד עבור מדיניויות מותאמות אישית (`custom/`), קונבנציה של פרויקט (`.failproofai-project/`), וקונבנציה של משתמש (`.failproofai-user/`) גם כן. ראה [Configuration → hint](/he/configuration#hint-cross-cutting) לפרטים.
</Tip>

### הודעות allow מידע

`allow(message)` מתיר את הפעולה **וגם** שולח הודעת מידע חזרה ל-Claude. ההודעה מועברת כ-`additionalContext` בתגובת stdout של ה-hook handler — אותו מנגנון בשימוש על ידי `instruct`, אך שונה מבחינה סמנטית: זה עדכון מצב, לא התראה.

| פונקציה | אפקט | משתמש כאשר |
|----------|--------|----------|
| `allow(message)` | התר ושלח הקשר ל-Claude | אשר בדיקה עברה, או הסבר למה בדיקה הושמטה |

מקרי שימוש:
- **אישורי סטטוס:** `allow("All CI checks passed.")` — אומר ל-Claude שהכל ירוק
- **הסברי fail-open:** `allow("GitHub CLI not installed, skipping CI check.")` — אומר ל-Claude למה בדיקה הושמטה כדי שיהיה לו הקשר מלא
- **מספר הודעות מצטברות:** אם מדיניויות מרובות כל אחת מחזירה `allow(message)`, כל ההודעות מחוברות עם newlines ומועברות ביחד

```js
customPolicies.add({
  name: "confirm-branch-status",
  match: { events: ["Stop"] },
  fn: async (ctx) => {
    const cwd = ctx.session?.cwd;
    if (!cwd) return allow("No working directory, skipping branch check.");

    // ... check branch status ...
    if (allPushed) {
      return allow("Branch is up to date with remote.");
    }
    return deny("Unpushed changes detected.");
  },
});
```

### שדות `PolicyContext`

| שדה | סוג | תיאור |
|-------|------|-------------|
| `eventType` | `string` | `"PreToolUse"`, `"PostToolUse"`, `"Notification"`, `"Stop"` |
| `toolName` | `string \| undefined` | הכלי שמקבל קריאה (למשל `"Bash"`, `"Write"`, `"Read"`) |
| `toolInput` | `Record<string, unknown> \| undefined` | פרמטרי קלט של הכלי |
| `payload` | `Record<string, unknown>` | כל payload האירוע הגולמי מ-Claude Code |
| `session` | `SessionMetadata \| undefined` | הקשר הסשן (ראה למטה) |

### שדות `SessionMetadata`

| שדה | סוג | תיאור |
|-------|------|-------------|
| `sessionId` | `string` | Claude Code session identifier |
| `cwd` | `string` | Working directory of the Claude Code session |
| `transcriptPath` | `string` | Path to the session's JSONL transcript file |

### סוגי אירוע

| אירוע | מתי זה כורה | תוכן `toolInput` |
|-------|--------------|----------------------|
| `PreToolUse` | לפני Claude מריץ כלי | קלט הכלי (למשל `{ command: "..." }` עבור Bash) |
| `PostToolUse` | אחרי כלי משלים | קלט הכלי + `tool_result` (הפלט) |
| `Notification` | כשל Claude שולח הודעה | `{ message: "...", notification_type: "idle" \| "permission_prompt" \| ... }` - hooks חייבים להחזיר תמיד `allow()`, הם לא יכולים לחסום הודעות |
| `Stop` | כשהסשן Claude מסתיים | ריק |

## סדר הערכה

מדיניויות מוערכות בסדר זה:

1. מדיניויות מובנות (בסדר הגדרה)
2. מדיניויות מותאמות אישית מפורשות מ-`customPoliciesPath` (בסדר `.add()`)
3. מדיניויות קונבנציה מ-`.failproofai/policies/` של פרויקט (קבצים אלפביתיים, סדר `.add()` בתוך)
4. מדיניויות קונבנציה מ-`~/.failproofai/policies/` של משתמש (קבצים אלפביתיים, סדר `.add()` בתוך)

<Note>
ה-`deny` הראשון חוצה את כל המדיניויות הבאות. כל הודעות `instruct` מצטברות ומועברות ביחד.
</Note>

## Transitive imports

קבצי מדיניות מותאמות אישית יכולים לייבא מודולים מקומיים בשימוש בנתיבים יחסיים:

```js
// my-policies.js
import { isBlockedPath } from "./utils.js";
import { checkApproval } from "./approval-client.js";

customPolicies.add({
  name: "approval-gate",
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const approved = await checkApproval(ctx.toolInput?.command, ctx.session?.sessionId);
    return approved ? allow() : deny("Approval required for this command");
  },
});
```

כל היבואים יחסיים שניתן להגיע אליהם מקובץ הקלט מופתרים. זה מיושם על ידי כתיבה מחדש של `from "failproofai"` יבואים לנתיב dist בפועל ויצירת קבצי `.mjs` זמניים כדי להבטיח תאימות ESM.

## סינון סוג אירוע

השתמש ב-`match.events` כדי להגביל מתי מדיניות כורה:

```js
customPolicies.add({
  name: "require-summary-on-stop",
  match: { events: ["Stop"] },
  fn: async (ctx) => {
    // Only fires when the session ends
    // ctx.session.transcriptPath contains the full session log
    return allow();
  },
});
```

השמט את `match` לחלוטין כדי להשרות על כל סוג אירוע.

## טיפול בשגיאות וומצבי כשל

מדיניויות מותאמות אישית הן **fail-open**: שגיאות לעולם לא חוסמות מדיניויות מובנות או קורסות את ה-hook handler.

| כשל | התנהגות |
|---------|----------|
| `customPoliciesPath` לא מוגדר | אין מדיניויות מותאמות אישית מפורשות; מדיניויות קונבנציה ומובנות ממשיכות בעדינות |
| קובץ לא נמצא | התראה מתועדת ל-`~/.failproofai/hook.log`; מובנות ממשיכות |
| Syntax/import error (explicit) | שגיאה מתועדת ל-`~/.failproofai/hook.log`; מדיניויות מותאמות אישית מפורשות מדולגות |
| Syntax/import error (convention) | שגיאה מתועדת; קובץ הזה מדולג, קבצי קונבנציה אחרים עדיין נטענים |
| `fn` זורק בזמן ריצה | שגיאה מתועדת; hook הזה מעובד כ-`allow`; hooks אחרים ממשיכים |
| `fn` לוקח יותר מ-10 שניות | Timeout מתועד; מעובד כ-`allow` |
| ספריית קונבנציה חסרה | אין מדיניויות קונבנציה; אין שגיאה |

<Tip>
לעבור שגיאות מדיניות מותאמות אישית, צפה בקובץ הרישום:

```bash
tail -f ~/.failproofai/hook.log
```
</Tip>

## דוגמה מלאה: מדיניויות מרובות

```js
// my-policies.js
import { customPolicies, allow, deny, instruct } from "failproofai";

// Prevent agent from writing to secrets/ directory
customPolicies.add({
  name: "block-secrets-dir",
  description: "Prevent agent from writing to secrets/ directory",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (!["Write", "Edit"].includes(ctx.toolName ?? "")) return allow();
    const path = ctx.toolInput?.file_path ?? "";
    if (path.includes("secrets/")) return deny("Writing to secrets/ is not permitted");
    return allow();
  },
});

// Keep the agent on track: verify tests before committing
customPolicies.add({
  name: "remind-test-before-commit",
  description: "Keep the agent on track: verify tests pass before committing",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = ctx.toolInput?.command ?? "";
    if (/git\s+commit/.test(cmd)) {
      return instruct("Verify all tests pass before committing. Run `bun test` if you haven't already.");
    }
    return allow();
  },
});

// Prevent unplanned dependency changes during freeze
customPolicies.add({
  name: "dependency-freeze",
  description: "Prevent unplanned dependency changes during freeze period",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = ctx.toolInput?.command ?? "";
    const isInstall = /^(npm install|yarn add|bun add|pnpm add)\s+\S/.test(cmd);
    if (isInstall && process.env.DEPENDENCY_FREEZE === "1") {
      return deny("Package installs are frozen. Unset DEPENDENCY_FREEZE to allow.");
    }
    return allow();
  },
});

export { customPolicies };
```

```bash
failproofai policies --install --custom ./examples/policies-basic.js
```

### שימוש בדוגמאות מבוססות קונבנציה

```bash
# Copy to project level
mkdir -p .failproofai/policies
cp examples/convention-policies/*.mjs .failproofai/policies/

# Or copy to user level
mkdir -p ~/.failproofai/policies
cp examples/convention-policies/*.mjs ~/.failproofai/policies/
```

אין צורך בפקודת התקנה — הקבצים נבחרים באופן אוטומטי באירוע ה-hook הבא.
