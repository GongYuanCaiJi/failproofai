מדיניויות מותאם אישית מאפשרות לך לכתוב כללים לכל התנהגות של agent: אכוף קונבנציות פרויקט, מנע סטייה, חסום פעולות הרסניות, גלה agents תקועים, או התשלב עם Slack, תהליכי אישור ועוד. הם משתמשים באותה מערכת hook events וטכסי ההחלטה `allow`, `deny`, `instruct` כמו במדיניויות מובנות.

---

## דוגמה מהירה

```js
// my-policies.js
import { customPolicies, allow, deny, instruct } from "failproofai";

customPolicies.add({
  name: "no-production-writes",
  description: "Block writes to paths containing 'production'",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Write" && ctx.toolName !== "Edit") return allow();
    const path = ctx.toolInput?.file_path ?? "";
    if (path.includes("production")) {
      return deny("Writes to production paths are blocked");
    }
    return allow();
  },
});
```

התקן אותה:

```bash
failproofai policies --install --custom ./my-policies.js
```

## שתי דרכים לטעון מדיניויות מותאם אישית

### אפשרות 1: מבוססת קונבנציה (מומלצת)

שים קבצים `*policies.{js,mjs,ts}` ב-`.failproofai/policies/` והם יטענו באופן אוטומטי — אין צורך בדגלים או שינויי הגדרות. זה עובד כמו git hooks: שים קובץ, זה פשוט עובד.

```
# רמת פרויקט — מעוד ב-git, משותף עם הצוות
.failproofai/policies/security-policies.mjs
.failproofai/policies/workflow-policies.mjs

# רמת משתמש — אישי, חל על כל הפרויקטים
~/.failproofai/policies/my-policies.mjs
```

**איך זה עובד:**
- שני הספריות של פרויקט ומשתמש סורקות (איחוד — לא first-scope-wins)
- קבצים טוענים בסדר אלפאביתי בתוך כל ספרייה. הקדם עם `01-`, `02-` לשליטה בסדר
- רק קבצים התואמים `*policies.{js,mjs,ts}` טוענים; קבצים אחרים מתעלמים
- כל קובץ טוען בנפרד (fail-open לכל קובץ)
- עובד לצד `--custom` מפורש ומדיניויות מובנות

<Tip>
מדיניויות קונבנציה הם הדרך הקלה ביותר לבנות תקן איכות לארגון שלך. אלחק `.failproofai/policies/` ל-git וכל חברי הצוות מקבלים את אותם כללים באופן אוטומטי — אין צורך בהגדרה לכל מפתח. כאשר הצוות שלך מגלה אופנויים כשל חדשים, הוסף מדיניות ודחוף. עם הזמן אלה הופכים לתקן איכות חי שמשתפר עם כל תרומה.
</Tip>

### אפשרות 2: נתיב קובץ מפורש

```bash
# התקן עם קובץ מדיניויות מותאם אישית
failproofai policies --install --custom ./my-policies.js

# החלף את נתיב קובץ המדיניויות
failproofai policies --install --custom ./new-policies.js

# הסר את נתיב המדיניויות המותאם אישית מתוך הגדרות
failproofai policies --uninstall --custom
```

נתיב מוחלט שנפתר מאוחסן ב-`policies-config.json` כ-`customPoliciesPath`. הקובץ טוען טרי בכל hook event - אין caching בין events.

### שימוש בשניהם ביחד

מדיניויות קונבנציה וקובץ `--custom` מפורש יכולים להתקיים. סדר טעינה:

1. קובץ `customPoliciesPath` מפורש (אם מוגדר)
2. קבצי קונבנציה של פרויקט (`{cwd}/.failproofai/policies/`, אלפאביתי)
3. קבצי קונבנציה של משתמש (`~/.failproofai/policies/`, אלפאביתי)

## API

### ייבוא

```js
import { customPolicies, allow, deny, instruct } from "failproofai";
```

### `customPolicies.add(hook)`

רושם מדיניות. קרא לזה כמה פעמים שצריך עבור מדיניויות מרובות באותו קובץ.

```ts
customPolicies.add({
  name: string;                         // דרוש - מזהה ייחודי
  description?: string;                 // מוצג בפלט `failproofai policies`
  match?: { events?: HookEventType[] }; // סנן לפי סוג event; השמט לתאימה לכל
  fn: (ctx: PolicyContext) => PolicyResult | Promise<PolicyResult>;
});
```

### עוזרי החלטה

| פונקציה | השפעה | השתמש כאשר |
|----------|--------|----------|
| `allow()` | אפשר את הפעולה בשקט | הפעולה בטוחה, אין צורך בהודעה |
| `deny(message)` | חסום את הפעולה | ה-agent לא צריך לבצע פעולה זו |
| `instruct(message)` | הוסף הקשר ללא חסימה | תן ל-agent הקשר נוסף להישאר בעקבות |

`deny(message)` - ההודעה מופיעה ל-Claude עם קידומת `Blocked by failproofai:`. `deny` יחיד מקצר את כל ההערכה הנוספת.

`instruct(message)` - ההודעה מצורפת להקשר של Claude עבור קריאת הכלי הנוכחית. כל הודעות `instruct` מצטברות ומסופקות ביחד.

<Tip>
אתה יכול לצרף הנחיות נוספות לכל הודעת `deny` או `instruct` על ידי הוספת שדה `hint` ב-`policyParams` — אין צורך בשינוי קוד. זה עובד גם עבור מדיניויות מותאם אישית (`custom/`), קונבנציה של פרויקט (`.failproofai-project/`), וקונבנציה של משתמש (`.failproofai-user/`). ראה [הגדרות → hint](/he/configuration#hint-cross-cutting) לפרטים.
</Tip>

### הודעות allow אינפורמטיביות

`allow(message)` מאפשר את הפעולה **וגם** שולח הודעה אינפורמטיבית חזרה ל-Claude. ההודעה מסופקת כ-`additionalContext` בתגובת stdout של מטפל ה-hook — אותה מנגנון שמשמש `instruct`, אבל סמנטית שונה: זה עדכון סטטוס, לא אזהרה.

| פונקציה | השפעה | השתמש כאשר |
|----------|--------|----------|
| `allow(message)` | אפשר וקבל הקשר ל-Claude | אשר שבדיקה עברה, או הסבר למה בדיקה דלגה |

מקרי שימוש:
- **אישורי סטטוס:** `allow("All CI checks passed.")` — אומר ל-Claude שהכל ירוק
- **הסברי fail-open:** `allow("GitHub CLI not installed, skipping CI check.")` — אומר ל-Claude למה בדיקה דלגה כדי שיהיה לו הקשר מלא
- **הודעות מרובות מצטברות:** אם מספר מדיניויות כל אחת מחזירה `allow(message)`, כל ההודעות מצורפות עם newlines ומסופקות ביחד

```js
customPolicies.add({
  name: "confirm-branch-status",
  match: { events: ["Stop"] },
  fn: async (ctx) => {
    const cwd = ctx.session?.cwd;
    if (!cwd) return allow("No working directory, skipping branch check.");

    // ... check branch status ...
    if (allPushed) {
      return allow("Branch is up to date with remote.");
    }
    return deny("Unpushed changes detected.");
  },
});
```

### שדות `PolicyContext`

| שדה | סוג | תיאור |
|-------|------|-------------|
| `eventType` | `string` | `"PreToolUse"`, `"PostToolUse"`, `"Notification"`, `"Stop"` |
| `toolName` | `string \| undefined` | הכלי המצוי בקריאה (למשל `"Bash"`, `"Write"`, `"Read"`) |
| `toolInput` | `Record<string, unknown> \| undefined` | פרמטרי הקלט של הכלי |
| `payload` | `Record<string, unknown>` | payload גלם מלא מ-Claude Code |
| `session` | `SessionMetadata \| undefined` | הקשר session (ראה מטה) |

### שדות `SessionMetadata`

| שדה | סוג | תיאור |
|-------|------|-------------|
| `sessionId` | `string` | Claude Code session identifier |
| `cwd` | `string` | ספריית העבודה של Claude Code session |
| `transcriptPath` | `string` | נתיב לקובץ תמליל JSONL של ה-session |

### סוגי events

| Event | מתי זה מופעל | תוכן `toolInput` |
|-------|--------------|----------------------|
| `PreToolUse` | לפני Claude מריץ כלי | קלט הכלי (למשל `{ command: "..." }` עבור Bash) |
| `PostToolUse` | לאחר כלי מסתיים | קלט הכלי + `tool_result` (הפלט) |
| `Notification` | כאשר Claude שולח הודעה | `{ message: "...", notification_type: "idle" \| "permission_prompt" \| ... }` - hooks חייבים להחזיר תמיד `allow()`, הם לא יכולים לחסום הודעות |
| `Stop` | כאשר Claude session מסתיים | ריק |

## סדר הערכה

מדיניויות מוערכות בסדר זה:

1. מדיניויות מובנות (בסדר הגדרה)
2. מדיניויות מותאם אישית מפורשות מ-`customPoliciesPath` (בסדר `.add()`)
3. מדיניויות קונבנציה מ-project `.failproofai/policies/` (קבצים אלפאביתיים, סדר `.add()` בתוך)
4. מדיניויות קונבנציה מ-user `~/.failproofai/policies/` (קבצים אלפאביתיים, סדר `.add()` בתוך)

<Note>
ה-`deny` הראשון מקצר את כל המדיניויות הבאות. כל הודעות `instruct` מצטברות ומסופקות ביחד.
</Note>

## ייבוא ריבוני

קבצי מדיניויות מותאם אישית יכולים לייבא מודולים מקומיים באמצעות נתיבים יחסיים:

```js
// my-policies.js
import { isBlockedPath } from "./utils.js";
import { checkApproval } from "./approval-client.js";

customPolicies.add({
  name: "approval-gate",
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const approved = await checkApproval(ctx.toolInput?.command, ctx.session?.sessionId);
    return approved ? allow() : deny("Approval required for this command");
  },
});
```

כל הייבואים היחסיים הניתנים להשגה מקובץ הכניסה מתפתרים. זה מיושם על ידי כתיבה מחדש של ייבואים `from "failproofai"` לנתיב dist בפועל ויצירת קבצי `.mjs` זמניים כדי להבטיח תאימות ESM.

## סינון סוג event

השתמש ב-`match.events` כדי להגביל מתי מדיניות מופעלת:

```js
customPolicies.add({
  name: "require-summary-on-stop",
  match: { events: ["Stop"] },
  fn: async (ctx) => {
    // מופעל רק כאשר ה-session מסתיים
    // ctx.session.transcriptPath מכיל את יומן ה-session המלא
    return allow();
  },
});
```

השמט את `match` לחלוטין כדי להופעל בכל סוג event.

## טיפול בשגיאות ומצבי כשל

מדיניויות מותאם אישית הם **fail-open**: שגיאות לעולם לא חוסמות מדיניויות מובנות או מייצרות קריסה של מטפל ה-hook.

| כשל | התנהגות |
|---------|----------|
| `customPoliciesPath` לא מוגדר | אין מדיניויות מותאם אישית מפורשות שרצות; מדיניויות קונבנציה ומובנות המשיכו בדרך כלל |
| קובץ לא נמצא | אזהרה רשומה ל-`~/.failproofai/hook.log`; מובנים ממשיכים |
| שגיאת תחביר/ייבוא (מפורש) | שגיאה רשומה ל-`~/.failproofai/hook.log`; מדיניויות מותאם אישית מפורשות דלגים |
| שגיאת תחביר/ייבוא (קונבנציה) | שגיאה רשומה; קובץ זה דלג, קבצי קונבנציה אחרים עדיין טוענים |
| `fn` זורק בזמן ריצה | שגיאה רשומה; hook זה מטופל כ-`allow`; hooks אחרים ממשיכים |
| `fn` לוקח יותר מ-10 שניות | timeout רשום; מטופל כ-`allow` |
| ספריית קונבנציה חסרה | אין מדיניויות קונבנציה שרץ; אין שגיאה |

<Tip>
כדי לתקן שגיאות מדיניות מותאם אישית, עקוב אחרי קובץ היומן:

```bash
tail -f ~/.failproofai/hook.log
```
</Tip>

## דוגמה מלאה: מדיניויות מרובות

```js
// my-policies.js
import { customPolicies, allow, deny, instruct } from "failproofai";

// מנע מ-agent לכתוב לספרייה secrets/
customPolicies.add({
  name: "block-secrets-dir",
  description: "Prevent agent from writing to secrets/ directory",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (!["Write", "Edit"].includes(ctx.toolName ?? "")) return allow();
    const path = ctx.toolInput?.file_path ?? "";
    if (path.includes("secrets/")) return deny("Writing to secrets/ is not permitted");
    return allow();
  },
});

// שמור את ה-agent בעקבות: אמת בדיקות לפני commit
customPolicies.add({
  name: "remind-test-before-commit",
  description: "Keep the agent on track: verify tests pass before committing",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = ctx.toolInput?.command ?? "";
    if (/git\s+commit/.test(cmd)) {
      return instruct("Verify all tests pass before committing. Run `bun test` if you haven't already.");
    }
    return allow();
  },
});

// מנע שינויים בתלויות לא מתוכננים במהלך freeze
customPolicies.add({
  name: "dependency-freeze",
  description: "Prevent unplanned dependency changes during freeze period",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = ctx.toolInput?.command ?? "";
    const isInstall = /^(npm install|yarn add|bun add|pnpm add)\s+\S/.test(cmd);
    if (isInstall && process.env.DEPENDENCY_FREEZE === "1") {
      return deny("Package installs are frozen. Unset DEPENDENCY_FREEZE to allow.");
    }
    return allow();
  },
});

export { customPolicies };
```

### שימוש בדוגמאות מבוססות קונבנציה

```bash
# העתק לרמת פרויקט
mkdir -p .failproofai/policies
cp examples/convention-policies/*.mjs .failproofai/policies/

mkdir -p ~/.failproofai/policies
cp examples/convention-policies/*.mjs ~/.failproofai/policies/
```

אין צורך בפקודת התקנה — הקבצים נמסרים באופן אוטומטי בהאירוע ה-hook הבא.
