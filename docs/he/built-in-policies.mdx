failproofai משלח עם 39 מדיניות מובנות הלוכדות מצבי כשל נפוצים של agents. כל מדיניות מופעלת על סוג אירוע hook ספציפי ושם כלי. תשע עשרה מדיניות מקבלות פרמטרים המאפשרים לך כיוונון את ההתנהגות שלהן ללא כתיבת קוד. חמש מדיניות זרימת עבודה אוכפות pipeline של commit → push → PR → CI לפני ש-Claude עוצר.

---

## סקירה כללית

המדיניות מקובצות לקטגוריות:

| קטגוריה | מדיניות | סוג Hook |
|----------|---------|-----------|
| [פקודות מסוכנות](#dangerous-commands) | block-sudo, block-rm-rf, block-curl-pipe-sh, block-failproofai-commands | PreToolUse |
| [פקודות תשתית](#infra-commands) | block-kubectl, block-terraform, block-aws-cli, block-gcloud, block-az-cli, block-helm, block-gh-pipeline | PreToolUse |
| [סודות (sanitizers)](#secrets-sanitizers) | sanitize-jwt, sanitize-api-keys, sanitize-connection-strings, sanitize-private-key-content, sanitize-bearer-tokens | PostToolUse |
| [סביבה](#environment) | block-env-files, protect-env-vars | PreToolUse |
| [גישת קובץ](#file-access) | block-read-outside-cwd, block-secrets-write | PreToolUse |
| [Git](#git) | block-push-master, block-work-on-main, block-force-push, warn-git-amend, warn-git-stash-drop, warn-all-files-staged | PreToolUse |
| [בסיס נתונים](#database) | warn-destructive-sql, warn-schema-alteration | PreToolUse |
| [אזהרות](#warnings) | warn-large-file-write, warn-package-publish, warn-background-process, warn-global-package-install | PreToolUse |
| [מנהלי חבילות](#package-managers) | prefer-package-manager | PreToolUse |
| [זרימת עבודה](#workflow) | require-commit-before-stop, require-push-before-stop, require-pr-before-stop, require-no-conflicts-before-stop, require-ci-green-before-stop | Stop |

- **`block-`** — עצור את ה-agent מהמשך.
- **`warn-`** — תן ל-agent הקשר נוסף כדי שיוכל להתקן בעצמו.
- **`sanitize-`** — נקה נתונים רגישים מפלט הכלי לפני שה-agent רואה זאת.

### Namespaces

כל מדיניות נמצאת בחריץ `<namespace>/<name>`. מדיניות מובנות שייכות ל-namespace **`exospherehost/`** — לדוגמה, `exospherehost/sanitize-jwt`. ה-namespace מונע התנגשויות כאשר אתה גם טוען מדיניות מותאמות או של צד שלישי עם שמות קצרים דומים.

בתצורה שלך תוכל להתייחס למדיניות מובנית על ידי שמה הקצר או שמה המלא; שתי הצורות מסתכמות לאותה מדיניות:

```json
{
  "enabledPolicies": [
    "sanitize-jwt",
    "exospherehost/block-rm-rf"
  ]
}
```

אם לשם אין `/`, failproofai מטפל בו כשייך ל-namespace ברירת המחדל `exospherehost`. שמות המכילים כבר `/` (לדוגמה `myorg/foo`, `custom/my-hook`) נשמרים כמו שהם.
- **`require-`** — חסום את אירוע ה-Stop עד שתנאים מתקיימים.

---

<Tip>
כל מדיניות תומכת בשדה `hint` אופציונלי ב-`policyParams`. ה-hint מצורף להודעת deny או instruct שה-Claude רואה, מה שנותן הנחיה פעולותית ללא שינוי קוד המדיניות. עובד עם מדיניות מובנות, מותאמות וקונבנציה. ראה [Configuration → hint](/he/configuration#hint-cross-cutting) לפרטים.
</Tip>

## פקודות מסוכנות

מנע agents מהפעלת פעולות שקשה להשיג או שיכולות להוציא נזק למערכת המארח.

### `block-sudo`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** מונע כל פקודת `sudo`.

חוסם הזעקות שכוללות את המילה-כיד `sudo`. התאמת דפוסים נעשית על אסימונים מנותחים של פקודה, לא על המחרוזת הגולמית, כדי למנוע עקיפה באמצעות הזרקת מפעיל shell.

**פרמטרים:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | בדיוק קידומות פקודה מותרות. כל כניסה משווה לעומת אסימוני argv מנותחים. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    }
  }
}
```

עם תצורה זו, `sudo systemctl status nginx` מותר, אך `sudo rm /etc/hosts` מונע.

<Note>
דפוסים משווים לעומת אסימונים מנותחים, לא מחרוזת הפקודה הגולמית. זה מונע עקיפה באמצעות מפעילי shell מצורפים (לדוגמה, `sudo systemctl status x; rm -rf /` אינו תואם `sudo systemctl status *`).
</Note>

## פקודות תשתית

עצור coding agents מהפעלת CLIs של תשתית או הוא טיגריג של CI/CD pipelines. כל המדיניות בקטגוריה זו הם **opt-in** (`defaultEnabled: false`) — agents שצריכים לגיטימציה לקרוא `kubectl`, `terraform`, וכו' לא יופר אלא אם אתה מפעיל את המדיניות. כשהוא מופעל, כל הזעקה של ה-CLI המתאים מונעת אלא אם הפקודה תואמת ערך ב-`allowPatterns`.

דקדוק הדפוס זהה ל-[`block-sudo`](#block-sudo): אסימונים משווים לעומת argv מנותח, `*` הוא wildcard לאסימון אחד, וכל פקודה המכילה מפעיל shell עצמאי (`&&`, `||`, `|`, `;`) או אסימון עם metacharacters shell משובצים נדחה לפני התאמת allowlist כדי למנוע עקיפות הזרקה.

### `block-kubectl`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** מונע כל הזעקת `kubectl`.

**פרמטרים:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | קידומות פקודה kubectl שמותר להן. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-kubectl": {
      "allowPatterns": ["kubectl get *", "kubectl describe *", "kubectl logs *"]
    }
  }
}
```

עם תצורה זו, `kubectl get pods` מותר אך `kubectl apply -f deploy.yaml` מונע.

### `block-gh-pipeline`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** מונע את תת-הפקודות `gh` CLI הבאות המשנות מצב או מפעילות pipelines:

- `gh workflow run`, `gh workflow enable`, `gh workflow disable`
- `gh run rerun`, `gh run cancel`
- `gh pr merge`
- `gh release create`, `gh release delete`
- `gh cache delete`
- `gh secret set`, `gh secret delete`

תת-פקודות `gh` שאינן read-only כמו `gh pr view`, `gh pr list`, `gh run list`, `gh release view`, ו-`gh api repos/.../...` **אינן** משווקות על ידי מדיניות זו — הן נדרשות באופן שגרתי לבדיקות זרימת עבודה (כולל `require-ci-green-before-stop` של failproofai).

**פרמטרים:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `allowPatterns` | `string[]` | `[]` | הזעקות scripted ספציפיות שיש לאפשר למרות שהם ממיד יימנעו. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-gh-pipeline": {
      "allowPatterns": ["gh run rerun *"]
    }
  }
}
```

## סודות (sanitizers)

עצור agents משום דליפת אישורים להקשרם או פלט. מדיניות Sanitizer מופעלת על אירועי **PostToolUse**. כאשר Claude הוא עובד פקודת Bash, קורא קובץ, או קורא לכל כלי, המדיניות הללו בודקות את הפלט לפני שהוא מוחזר ל-Claude. אם דפוס סודי מתגלה, המדיניות מחזירה החלטת deny המונעת את הפלט מהעברה חזרה.

### `sanitize-jwt`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מחבוא JWT tokens (שלוש שכבות base64url מופרדות ב-`.`).

ללא פרמטרים.

---

### `sanitize-api-keys`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מחבוא פורמטי מפתח API נפוצים: Anthropic (`sk-ant-`), OpenAI (`sk-`), GitHub PATs (`ghp_`), AWS access keys (`AKIA`), Stripe keys (`sk_live_`, `sk_test_`), ומפתחות Google API (`AIza`).

**פרמטרים:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `additionalPatterns` | `{ regex: string; label: string }[]` | `[]` | דפוסי regex נוספים לעיתול כסודות. |

**דוגמה:**

```json
{
  "policyParams": {
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo internal API key" },
        { "regex": "pat_[0-9a-f]{40}", "label": "Internal PAT" }
      ]
    }
  }
}
```

---

### `sanitize-connection-strings`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מחבוא מחרוזות חיבור בסיס נתונים המכילות אישורים משובצים (לדוגמה, `postgresql://user:password@host/db`).

ללא פרמטרים.

---

### `sanitize-private-key-content`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מחבוא בלוקי PEM (`-----BEGIN PRIVATE KEY-----`, `-----BEGIN RSA PRIVATE KEY-----`, וכו').

ללא פרמטרים.

---

### `sanitize-bearer-tokens`

**אירוע:** PostToolUse (כל הכלים)  
**ברירת מחדל:** מחבוא `Authorization: Bearer <token>` headers כאשר הטוקן הוא 20 או יותר תווים.

ללא פרמטרים.

## סביבה

הגן על תצורת סביבה רגישה מקריאה או חשיפה על ידי agents.

### `block-env-files`

**אירוע:** PreToolUse (Bash, Read)  
**ברירת מחדל:** מונע קריאת קבצי `.env` דרך `cat .env`, שיחות כלי Read עם `.env` כנתיב הקובץ, וכו'.

לא חוסם `.envrc` או קבצים אחרים סמוכים לסביבה - רק קבצים בשם בדיוק `.env`.

ללא פרמטרים.

**ברירת מחדל:** מונע פקודות המדפיסות משתנים סביבתיים: `printenv`, `env`, `echo $VAR`.

ללא פרמטרים.

## גישת קובץ

שמור agents עובדים בתוך גבולות פרויקט וחיוניים מקבצים רגישים.

### `block-read-outside-cwd`

**אירוע:** PreToolUse (Read, Bash)  
**ברירת מחדל:** מונע קריאת קבצים מחוץ לשורש הפרויקט. הגבול הוא `CLAUDE_PROJECT_DIR` (מוגדר פעם אחת לכל הפעלה על ידי Claude Code), עם fallback לתיקייה העבודה הנוכחית של ההפעלה כאשר משתנה זה אינו מוגדר. שימוש בשורש הפרויקט ולא ב-`cwd` החי פירושו הגבול נשאר יציב גם אחרי Claude `cd`s לתת-ספריה.

**פרמטרים:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `allowPaths` | `string[]` | `[]` | קידומות נתיב מוחלטות שמותר להן גם אם מחוץ לשורש הפרויקט. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company/config"]
    }
  }
}
```

---

### `block-secrets-write`

**אירוע:** PreToolUse (Write, Edit)  
**ברירת מחדל:** מונע כתיבה לקבצים בשימוש נפוץ עבור מפתחות פרטיים ותעודות: `id_rsa`, `id_ed25519`, `*.key`, `*.pem`, `*.p12`, `*.pfx`.

**פרמטרים:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `additionalPatterns` | `string[]` | `[]` | דפוסי שם קובץ נוסף (בסגנון glob) לחסום. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-secrets-write": {
      "additionalPatterns": [".token", ".secret"]
    }
  }
}
```

## Git

מנע pushes אקראיים, force-pushes, וטעויות ענף שקשה להשיג.

### `block-push-master`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** מונע `git push origin main` ו-`git push origin master`.

**פרמטרים:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | שמות ענפים שלא ניתן לדחוף אליהם ישירות. |

**דוגמה:**

```json
{
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "master", "release", "prod"]
    }
  }
}
```

<Tip>
כדי לאפשר דחיפה לכל ענפים (למעשה השבתת מדיניות זו ללא הסרתה מ-`enabledPolicies`), עיין `protectedBranches: []`.
</Tip>

---

### `block-work-on-main`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** מונע `git commit`, `git merge`, `git rebase`, ו-`git cherry-pick` בזמן שעץ העבודה פועל על `main` או `master`. יצירת וחלופה של ענף (`git checkout`, `git checkout -b`, `git switch`, `git switch -c`) אינם מושפעים.

**פרמטרים:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `protectedBranches` | `string[]` | `["main", "master"]` | שמות ענפים שעליהם commit/merge/rebase/cherry-pick מונע. |

---

### `block-force-push`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** מונע `git push --force` ו-`git push -f`.

ללא פרמטרים ספציפיים למדיניות. השתמש בחוצה [`hint`](/he/configuration#hint-cross-cutting) כדי להציע חלופות:

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "Create a new branch from your current HEAD (e.g. `git checkout -b <new-branch>`) and push that instead."
    }
  }
}
```

---

## מנהלי חבילות

אכוף איזה מנהלי חבילות ה-agent מותר להשתמש.

### `prefer-package-manager`

**אירוע:** PreToolUse (Bash)  
**ברירת מחדל:** מנוטרל. כאשר מופעל, חוסם כל פקודת מנהל חבילות לא ברשימת `allowed` ומספר ל-Claude לשכתב את הפקודה באמצעות מנהל מותר.

מגלה: pip, pip3, python -m pip, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `allowed` | string[] | `[]` | שמות מנהל חבילות מותר. כל מנהל מתגלה לא ברשימה זו חוסם. כאשר ריק, המדיניות הוא לא-פעולה. |
| `blocked` | string[] | `[]` | שמות מנהל נוסף לחסום מעבר לרשימה המובנית (לדוגמה, `['pdm', 'pipx']`). |

רשימת ה-block המובנית מכסה: pip, pip3, npm, npx, yarn, pnpm, pnpx, bun, bunx, uv, poetry, pipenv, conda, cargo. השתמש `blocked` כדי להוסיף מנהלים לא ברשימה זו.

**דוגמה תצורה:**

```json
{
  "enabledPolicies": ["prefer-package-manager"],
  "policyParams": {
    "prefer-package-manager": {
      "allowed": ["uv", "bun"],
      "blocked": ["pdm", "pipx"]
    }
  }
}
```

עם תצורה זו, `pip install flask` ו-`pdm install flask` שניהם מונעים עם הודעה לתוך Claude כדי להשתמש `uv` או `bun` במקום. פקודות כמו `uv pip install flask` מותרות כי `uv` נמצא ברשימת allowlist ונבדק קודם.

## זרימת עבודה

אכוף זרימת עבודה משמעת של סיום הפעלה. מדיניות אלה מופעלת על אירוע **Stop** ודחיה Claude מעצירה עד שכל תנאי מתקיימים. הם עוקבים אחר chain של תלות טבעית: commit → push → PR → CI. אם מדיניות מונעת, מדיניות אחרות בשרשרת דלוקות (deny short-circuits).

כל מדיניות זרימת עבודה הוא **fail-open**: אם הכלי הנדרש אינו זמין (לדוגמה, `gh` לא מותקן, אין git remote), המדיניות מאפשרת עם הודעה מידע המסבירה מדוע הבדיקה דלוקה.

### `require-commit-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** מונע עצירה כאשר יש שינויים לא committed (שונה, מבוצע, או קבצים לא עוקבים). מחזיר הודעת מידע כאשר ספריית העבודה נקייה.

ללא פרמטרים.

---

### `require-push-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** מונע עצירה כאשר יש commits שלא דחופים או כאשר הענף הנוכחי אין remote tracking branch. מציע `git push -u` כדי ליצור tracking branch אם נדרש. נפל פתוח אם אין remote מוגדר.

**פרמטרים:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `remote` | `string` | `"origin"` | שם remote לדחיפה אליו. |

**דוגמה:**

```json
{
  "policyParams": {
    "require-push-before-stop": {
      "remote": "upstream"
    }
  }
}
```

---

### `require-pr-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** מונע עצירה כאשר לא קיימת בקשת pull לענף הנוכחי, או כאשר ה-PR הקיים סגור ללא merging. מנחה את Claude ליצור PR עם `gh pr create`. כאשר ה-PR הוא **merged**, המדיניות מאפשרת (העבודה חונה) והודעה רמזים לחלופה מהענף (`git checkout main && git pull`).

ללא פרמטרים.

<Note>
מדיניות זו דורשת [GitHub CLI](https://cli.github.com/) (`gh`) להתקין ואומתות.
הרץ `gh auth login` עם personal access token בעל `repo` scope לקרוא גישה ל-
pull requests. אם `gh` לא מותקן או לא מאומת, המדיניות נופלת פתוח ודוחה את הסיבה ל-Claude.
</Note>

---

### `require-no-conflicts-before-stop`

**אירוע:** Stop  
**ברירת מחדל:** מונע עצירה כאשר הענף הנוכחי לא יכול לmerge בניקיון לענף הבסיס. המדיניות קודם מאשרת יש `OPEN` PR על GitHub לענף — ללא אחד, אין merge target להאכוף, כדי כל המדיניות short-circuits לאפשר. פעם `OPEN` PR מאומת, שני probes עצמאיים הרצים:

1. **Local** — `git merge-tree --write-tree --name-only origin/<baseBranch> HEAD`. בסכסוך, ההודעה deny שמה קבצי בסכסוך כדי Claude יודע בדיוק מה לפתור.
2. **GitHub** — reuses את `gh pr view --json mergeable,state` תוצאה כבר מאוחסנת ב-precheck. תופסים סכסוכים שחד stale local `origin/<baseBranch>` היה להשמיט (לדוגמה, מישהו נחתמו סכסוך PR על `main` מאז החיפוש האחרון). תוצאה `CONFLICTING` מונעת. תוצאה `UNKNOWN` גם מונעת ומנחה את Claude כדי להמתין ~10 שניות ובדוק-שנית לפני ניסיון לעצור שנית — זה מונע false negatives בזמן GitHub recomputes.

דלוקות כל גמור (מאפשרות) כאשר: `gh` אינו מותקן, אין PR קיימת לענף, מצב ה-PR אינו `OPEN` (לדוגמה, `MERGED`, `CLOSED`), או `gh pr view` מחזיר פלט unparseable. גם נופל פתוח כאשר `origin/<baseBranch>` חסר locally או כאשר אין commits קדימה של base — אלה Layer 1 fall-throughs עדיין בעצרת המטמן PR mergeability לפני אפשור.

**פרמטרים:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `baseBranch` | `string` | `"main"` | בענף בסיס לבדוק סכסוכים נגד. |

<Note>
GitHub CLI (`gh`) נדרש למדיניות זו. המדיניות משתמשת `gh pr view` כדי לאשר
`OPEN` PR קיימת לפני הפעלת כל סכסוך probe — ללא `gh`, המדיניות
short-circuits לאפשר. הרץ `gh auth login` עם personal access token בעל
`repo` scope לקרוא גישה pull requests.
</Note>

ללא פרמטרים.

</Note>

## השבתת מדיניות בודדות

הסר מדיניות ספציפית מ-`enabledPolicies` בתצורה שלך, או לחצן אותו בכרטיסיה Policies של הלוח.

```json
{
  "enabledPolicies": [
    "block-rm-rf",
    "sanitize-api-keys"
  ]
}
```

מדיניות לא רשומות ב-`enabledPolicies` לא הרצות, גם אם כניסות `policyParams` קיימות עבורם.
