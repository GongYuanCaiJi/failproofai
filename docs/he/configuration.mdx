failproofai משתמש בקובצי JSON להגדרת אילו מדיניויות פעילות, כיצד הן מתנהגות, ומהיכן מדיניויות מותאמות נטענות. התצורה מעוצבת להיות קלה לשיתוף עם הצוות - בצרו אותה לתוך המאגר והכל פיתוח קבל את אותה רשת בטיחות לאגנט.

---

## היקפי תצורה

ישנם שלושה היקפי תצורה, המוערכים בסדר עדיפויות:

| היקף | נתיב קובץ | מטרה |
|-------|-----------|---------|
| **project** | `.failproofai/policies-config.json` | הגדרות לכל מאגר, מחויבות לשליטה בגרסה |
| **local** | `.failproofai/policies-config.local.json` | דריסות אישיות לכל מאגר, מתעלמות |
| **global** | `~/.failproofai/policies-config.json` | ברירות מחדל ברמת משתמש בכל הפרויקטים |

כאשר failproofai מקבל אירוע hook, הוא טוען ומזוג את כל שלושת הקובצים הקיימים עבור ספריית העבודה הנוכחית.

### כללי מיזוג

**`enabledPolicies`** - האיחוד של כל שלושת ההיקפים. מדיניות המופעלת בכל רמה היא פעילה.

```text
project:  ["block-sudo"]
local:    ["block-rm-rf"]
global:   ["block-sudo", "sanitize-api-keys"]

resolved: ["block-sudo", "block-rm-rf", "sanitize-api-keys"]  ← איחוד שהוסר כפלים
```

**`policyParams`** - ההיקף הראשון המגדיר פרמטרים למדיניות מסוימת מנצח לחלוטין. אין מיזוג עמוק של ערכים בתוך פרמטרים של מדיניות.

```text
project:  block-sudo → { allowPatterns: ["sudo apt-get update"] }
global:   block-sudo → { allowPatterns: ["sudo systemctl status"] }

resolved: { allowPatterns: ["sudo apt-get update"] }   ← project מנצח, global מתעלם
```

```text
project:  (ללא ערך block-sudo)
local:    (ללא ערך block-sudo)
global:   block-sudo → { allowPatterns: ["sudo systemctl status"] }

resolved: { allowPatterns: ["sudo systemctl status"] }  ← נופל לגלובל
```

**`customPoliciesPath`** - ההיקף הראשון המגדיר אותו מנצח.

**`llm`** - ההיקף הראשון המגדיר אותו מנצח.

## פורמט קובץ התצורה

```json
{
  "enabledPolicies": [
    "block-sudo",
    "block-rm-rf",
    "block-push-master",
    "sanitize-api-keys",
    "sanitize-jwt",
    "block-env-files",
    "block-read-outside-cwd"
  ],
  "policyParams": {
    "block-sudo": {
      "allowPatterns": ["sudo systemctl status", "sudo journalctl"]
    },
    "block-push-master": {
      "protectedBranches": ["main", "release", "prod"]
    },
    "block-rm-rf": {
      "allowPaths": ["/tmp"]
    },
    "block-read-outside-cwd": {
      "allowPaths": ["/shared/data", "/opt/company"]
    },
    "sanitize-api-keys": {
      "additionalPatterns": [
        { "regex": "myco_[A-Za-z0-9]{32}", "label": "MyCo API key" }
      ]
    },
    "warn-large-file-write": {
      "thresholdKb": 512
    }
  },
  "customPoliciesPath": "/home/alice/myproject/my-policies.js"
}
```

## התייחסות שדה

### `enabledPolicies`

סוג: `string[]`

רשימת שמות מדיניויות להפעלה. שמות חייבים להתאים בדיוק למזהי המדיניויות המוצגים על ידי `failproofai policies`. ראו [Built-in Policies](/he/built-in-policies) לרשימה המלאה.

מדיניויות שאינן ב-`enabledPolicies` אינן פעילות, גם אם יש להן ערכים ב-`policyParams`.

### `policyParams`

סוג: `Record<string, Record<string, unknown>>`

דריסות פרמטרים לכל מדיניות. המפתח החיצוני הוא שם המדיניות; המפתחות הפנימיים הם ספציפיים למדיניות. כל מדיניות תעד את הפרמטרים הזמינים שלה ב-[Built-in Policies](/he/built-in-policies).

אם למדיניות יש פרמטרים אך אתה לא מציין אותם, משתמשים בברירות המחדל המובנות של המדיניות. משתמשים שלא מגדירים את `policyParams` כלל מקבלים התנהגות זהה לגרסאות קודמות.

מפתחות לא ידועים בתוך בלוק הפרמטרים של מדיניות מתעלמים בשקט בזמן ירי hook אך מסומנים כאזהרות כאשר אתה מריץ `failproofai policies`.

#### `hint` (חוצה-חיתוך)

סוג: `string` (אופציונלי)

הודעה שנוספת לסיבה כאשר מדיניות מחזירה `deny` או `instruct`. השתמשו בה כדי לתן Claude הוראות שימושיות מבלי לשנות את המדיניות עצמה.

עובד עם כל סוג מדיניות - מובנה, מותאמת (`custom/`), קונבנציה של פרויקט (`.failproofai-project/`), או קונבנציה של משתמש (`.failproofai-user/`).

```json
{
  "policyParams": {
    "block-force-push": {
      "hint": "נסה ליצור ענף חדש במקום זאת."
    },
    "block-sudo": {
      "allowPatterns": ["sudo apt-get"],
      "hint": "השתמש ב-apt-get ישירות ללא sudo."
    },
    "custom/my-policy": {
      "hint": "בקש מהמשתמש אישור ראשון."
    }
  }
}
```

כאשר `block-force-push` דוחה, Claude רואה: *"Force-pushing חסום. נסה ליצור ענף חדש במקום זאת."*

ערכים שאינם מחרוזות ומחרוזות ריקות מתעלמים בשקט. אם `hint` לא מוגדר, ההתנהגות אינה משתנה (תואם לאחור).

### `customPoliciesPath`

סוג: `string` (נתיב מוחלט)

נתיב לקובץ JavaScript המכיל מדיניויות hook מותאמות. זה מוגדר באופן אוטומטי על ידי `failproofai policies --install --custom <path>` (הנתיב מתורגם לתוך מוחלט לפני שמירה).

הקובץ נטען מחדש בכל אירוע hook - אין קאשינג. ראו [Custom Policies](/he/custom-policies) לפרטי יצירה.

### מדיניויות מבוססות קונבנציה

בנוסף ל-`customPoliciesPath` המפורש, failproofai גילוי ופטיח אוטומטית קובצי מדיניויות מתוך ספריות `.failproofai/policies/`:

| רמה | ספרייה | היקף |
|-------|-----------|-------|
| פרויקט | `.failproofai/policies/` | משותף עם צוות דרך שליטה בגרסה |
| משתמש | `~/.failproofai/policies/` | אישי, חל על כל הפרויקטים |

**התאמת קובצים:** רק קובצים התואמים `*policies.{js,mjs,ts}` נטענים (למשל `security-policies.mjs`, `workflow-policies.js`). קובצים אחרים בספרייה מתעלמים.

**אין צורך בתצורה:** מדיניויות קונבנציה לא דורשות ערכים ב-`policies-config.json`. רק זרוק קובצים לתוך הספרייה והם בחרו אותו בעת האירוע hook הבא.

**טעינת איחוד:** גם ספריות הקונבנציה של פרויקט וגם משתמש סרוקות. כל הקובצים התואמים משני הרמות נטענים (בשונה מ-`customPoliciesPath` שמשתמש בניצחון בהיקף ראשון).

ראו [Custom Policies](/he/custom-policies) לפרטים נוספים ודוגמאות.

### `llm`

סוג: `object` (אופציונלי)

תצורת לקוח LLM למדיניויות שמבצעות קריאות AI. לא נדרש לרוב ההגדרות.

```json
{
  "llm": {
    "model": "claude-sonnet-4-6",
    "apiKey": "sk-ant-..."
  }
}
```

## ניהול תצורה מה-CLI

פקודות `policies --install` ו-`policies --uninstall` כותבות לקובץ הגדרות hook של ה-CLI של הסוכן שלך (נקודות הכניסה של hook), בעוד `policies-config.json` הוא הקובץ שאתה מנהל ישירות. השניים נפרדים:

- **הגדרות CLI של סוכן** — אומר לסוכן להתקשר `failproofai --hook <event>` בכל שימוש בכלי:
  - **Claude Code**: `~/.claude/settings.json` (משתמש), `<cwd>/.claude/settings.json` (פרויקט), `<cwd>/.claude/settings.local.json` (local)
  - **OpenAI Codex**: `~/.codex/hooks.json` (משתמש), `<cwd>/.codex/hooks.json` (פרויקט) — Codex אין לו היקף `local`
  - **GitHub Copilot CLI _(beta)_**: `~/.copilot/hooks/failproofai.json` (משתמש), `<cwd>/.github/hooks/failproofai.json` (פרויקט) — Copilot אין לו היקף `local`. ערכי Hook משתמשים בשדות הפקודה `bash`/`powershell` בעלי קלידים של Copilot עם `timeoutSec`; הקובץ נושא סימן `version: 1` ברמה העליונה. תמיכת Copilot CLI היא **beta** בזמן שאנו מאמתים את סכמת רשומות `events.jsonl` (אשר המסמכים הציבוריים לא מציינים) מול יותר סשנים בעולם האמיתי.
  - **Cursor Agent _(beta)_**: `~/.cursor/hooks.json` (משתמש), `<cwd>/.cursor/hooks.json` (פרויקט) — Cursor אין לו היקף `local`. ערכי Hook משתמשים בצורה `{type, command, timeout}` בצורת Claude (ללא פיצול `bash`/`powershell`), אך מאוחסנים תחת קלידים בstyle camelCase (`preToolUse`, `beforeSubmitPrompt`, …) במערך שטוח לפי [סכמת hooks של Cursor](https://cursor.com/docs/hooks); הקובץ נושא סימן `version: 1` ברמה העליונה. המטפל standardizes camelCase → PascalCase דרך `CURSOR_EVENT_MAP` כך מדיניויות מובנות קיימות מופעלות ללא שינוי. תמיכת Cursor Agent היא **beta** בזמן שאנו מאמתים פורמט הטרנסקריפט של Cursor בדיסק (לא מצוין בתעודות הציבור) מול יותר התקנות בעולם האמיתי.
  - **OpenCode _(beta)_**: `~/.config/opencode/opencode.json` + `~/.config/opencode/plugins/failproofai.mjs` (משתמש), `<cwd>/.opencode/opencode.json` + `<cwd>/.opencode/plugins/failproofai.mjs` (פרויקט) — OpenCode אין לו היקף `local`. בשונה מששת ה-CLIs האחרות, OpenCode יש **ללא מערכת hook פקודה חיצונית**: הוא טוען בתהליך JS/TS plugins שנרשמו בצורה מפורשת דרך מערך `plugin: []` ב-`opencode.json` (auto-discovery מ-`.opencode/plugins/` היא **לא** כיצד plugins טוענים ב-opencode v1.14.33). ההתקנה מורידה shim plugin קטן שנוצר שקוראות subprocess לבינארי failproofai ותורגם התגובה JSON בצורת Claude של הבינארי בחזרה לסמנטיקה של plugin: `throw new Error()` עבור tool-event deny (מבטל את קריאת הכלי), `client.session.prompt(...)` עבור instruct וגם עבור `Stop` / `SubagentStop` deny (שולח את סיבת ה-deny כהודעה היוזר הבאה — ערוץ ה-force-retry היחיד מכיוון ש-`session.idle` הוא notification-only וזריקה מזה היא no-op), ו-no-op עבור allow. סשנים חיים בבסיס נתונים של SQLite של opencode ב-`~/.local/share/opencode/opencode.db`; צופה הסשנים של הלוח קורא אותם דרך `opencode db --format json` ו-`opencode export <id>`. תמיכת OpenCode היא **beta** בזמן שאנו מאמתים התנהגות בין גרסאות ומול יותר סשנים בעולם האמיתי. ראו את [מסמכי OpenCode plugins](https://opencode.ai/docs/plugins/).
  - **Pi _(beta)_**: `~/.pi/agent/settings.json` (משתמש), `<cwd>/.pi/settings.json` (פרויקט) — Pi אין לו היקף `local`. Pi טוען חבילות TypeScript extension בעת ההפעלה; קובץ הגדרות הוא מערך מחרוזת שטוח `{"packages": ["./relative/path", …]}`. failproofai כותבת ערך packages-array יחיד המצביע על ספרייה `pi-extension/` שלה bundled. התוך extension בתוך subscribes לאירועי `tool_call` / `user_bash` / `input` / `session_start` של Pi ו-shells out ל-`failproofai --hook <Event> --cli pi`; המטפל standardizes underscore_lower_snake_case → PascalCase דרך `PI_EVENT_MAP` כך מדיניויות מובנות קיימות מופעלות ללא שינוי. תמיכת Pi היא **beta** בזמן ש-Pi's extension API וממד session-log stabilize.
  - **Gemini CLI _(beta)_**: `~/.gemini/settings.json` (משתמש), `<cwd>/.gemini/settings.json` (פרויקט) — Gemini אין לו היקף `local` (זה תעודות היקף `system` בכתובת `/etc/gemini-cli/settings.json` שאת failproofai לא חושף). Hook ערכים משתמשים בצורת `{type, command, timeout}` של Claude wrapped בسכמה מתאימה של Gemini `{matcher, hooks: [...]}` עם `matcher: "*"` כברירת מחדל. אירועים הם PascalCase (`SessionStart`, `BeforeAgent`, `AfterAgent`, `BeforeModel`, `AfterModel`, `BeforeToolSelection`, `BeforeTool`, `AfterTool`, `PreCompress`, `Notification`, `SessionEnd`); המטפל מוקד אל שמות Claude canonical דרך `GEMINI_EVENT_MAP`. שמות כלים הם snake_case (`run_shell_command`, `read_file`, `write_file`, `replace`, …) — המטפל standardizes דרך `GEMINI_TOOL_MAP` כך מדיניויות מובנות קיימות מופעלות ללא שינוי. המעריך מדיניות פולט צורה שטוחה של Gemini `{decision: "deny", reason}` (מעדיף לפי Gemini's "Golden Rule" exit-0 חוזה), `{hookSpecificOutput: {hookEventName, additionalContext}}` לתוך הקשר ב-BeforeAgent / AfterTool / SessionStart, ו-`{decision: "block", reason}` ב-AfterAgent עבור force-retry סמנטיקה. תמיכת Gemini CLI היא **beta** בזמן שאנו מרחיבים כיסוי בעולם האמיתי. ראו את [מסמכי Gemini CLI hooks](https://geminicli.com/docs/hooks/).
- **`policies-config.json`** — אומר ל-failproofai אילו מדיניויות להעריך וכן עם איזה פרמטרים (משותף בכל ה-CLIs של סוכן)

עברו `--cli claude|codex|copilot|cursor|opencode|pi|gemini` למטרת סוכן ספציפי (מופרדים בחלל או חוזרים עבור כל תת-קבוצה):

```bash
failproofai policies --install --cli codex --scope project
failproofai policies --install --cli copilot --scope project
failproofai policies --install --cli cursor --scope project
failproofai policies --install --cli opencode --scope project
failproofai policies --install --cli pi --scope project
failproofai policies --install --cli gemini --scope project
failproofai policies --install --cli claude codex copilot cursor opencode pi gemini
```

כאשר `--cli` מושמט, `failproofai` מגלה אילו CLIs סוכן מותקנים (`which claude` / `which codex` / `which copilot` / `which cursor-agent` / `which opencode` / `which pi` / `which gemini`):

- **CLI אחד גילה** — auto-בוחר CLI זה ללא הנמקה.
- **CLIs מרובים גילו** בטרמינל אינטרקטיבי — מציג תצוגה מקדימה עם בחירה יחידה חץ-מפתח מקובצת לתוך סעיף `Detected (N)` (עם שורה צבירת `Install for all N detected` + כל CLI גילה בנפרד) וסעיף `Not installed (M) · install hooks ahead of time` רשימות כל CLI לא גילה כאפשרות התקנה קדימה (↑↓ להזיז, Enter לבחור, ^C להקטין). זרימת ההסרה מציגה רק את סעיף ה-Detected.
- **CLIs מרובים גילו** בהפעלה לא-אינטרקטיבית (CI, ללא TTY) — מתקין עבור כל CLIs גילו ללא הנמקה.
- **ללא גילוי** — חוזר ל-`claude`, עם אזהרה שלא בינארי סוכן נמצא ב-PATH; פקודת ה-hook עדיין כתובה כך היא מופעלת ברגע שתתקין אחד.

אתה יכול לערוך `policies-config.json` ישירות בכל זמן; שינויים נכנסים לתוקף בחודש אירוע hook הבא ללא צורך בהפעלה מחדש.

## דוגמה: תצורה ברמת פרויקט עם ברירות מחדל של צוות

בצרו `.failproofai/policies-config.json` למאגר שלכם:

```json
{
  "enabledPolicies": [
    "block-sudo",
    "block-rm-rf",
    "block-push-master",
    "sanitize-api-keys",
    "block-env-files"
  ],
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "release", "hotfix"]
    }
  }
}
```

כל מפתח יכול אז ליצור `.failproofai/policies-config.local.json` (מתעלם) עבור דריסות אישיות ללא השפעה על חברי צוות.
