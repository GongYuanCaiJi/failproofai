כל פריסת Failproof AI Observability שלך, בפקודה אחת. בדוק את הייצור, צור API key, או אשר תקרית מבלי לעזוב את הטרמינל שלך, ואז תוכל לכתוב סקריפט לכל אלה ל-CI, או לתן לסוכן קוד לעשות זאת עבורך בשפה טבעית.

```bash
pipx install agenteye
agenteye login --email you@example.com      # a 6-digit code lands in your inbox
agenteye --json sessions --since 24h         # every agent run from the last day, newest first
```

*ה-CLI של `agenteye` מתחבר לדשבורד שלך. זהו כלי שונה מהמאסף, המשלח אירועים לשרת.*

## כל הפריסה שלך, בפקודה אחת

הפסק לקפוץ בין כרטיסיות כדי לענות על שאלה מהירה. ה-CLI של `agenteye` קורא את הנתונים שלך ומנהל את הארגון שלך מ-binary בודד, כך שבדיקה שהייתה פירושה לחיצה על דשבורד הופכת לשורה אחת שתוכל להפעיל מחדש, ליצור עבורה alias, או להדביק לתוך runbook. אתה מקבל ארבע ממשקים:

- **קרא את הנתונים שלך:** `sessions`, `events`, `evals`, ו-`errors`, מסוננים לפי זמן, סוכן וסביבה.
- **נהל את הארגון שלך:** `keys`, `users`, `settings`, `alerts`, ו-`incidents`.
- **הרץ ניתוחים:** SQL שמור בתוספת רץ `query` ad-hoc על נתוני האירועים שלך.
- **שאל את העוזר:** `agent ask` מגיע לאותו אנליסט קריאה-בלבד שאתה משוחח איתו בדשבורד.

התקן אותו פעם אחת עם `pipx`, היכנס עם קוד 6 ספרות ישלח בדוא"ל, ואתה מוכן. ההפעלה נמשכת בערך יום; הפעל מחדש את `agenteye login` כשהיא פוקעת. השתמש בה כדי לבדוק את הייצור, לספק key, או לעשות triage לתקרית דולקת, הכל ללא פתיחת דפדפן:

```bash
agenteye errors --since 24h --aggregate      # what is breaking, grouped by error type
agenteye incidents list --state firing       # what is on fire right now
agenteye keys create ci --add events:add     # a key that can only push events, secret shown once
```

נוהג אחד שכדאי לדעת: אפשרויות גלובליות כמו `--json` בא לפני הפקודה. `agenteye --json sessions` נכון; `agenteye sessions --json` לא.

## כתוב סקריפט, חבר אותו ל-CI

כל פקודה לוקחת `--json`, וזה משנה הכל. JSON נקי הולך ל-stdout בעוד שיתרויות וקבלות מצב בני אדם הולכות לstderr, כך שתיקייה `--json` piped ישר לתוך `jq` ללא שורה סטורית לפס. זה מה שהופך את ה-CLI להיות טוב באותה מידה לך בפרומפט ולסוכן קוד שמפרש פלט:

```bash
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'
```

זה בנוי להרוץ ללא פיקוח. בקשות אישור skip אוטומטי כשאין טרמינל מחובר, כך שלא כלום תלוי בצינור, וכל פקודה מחזירה קוד יציאה משמעותי: `0` הצלחה, `4` לא מחובר, `5` חסר הרשאה (ההודעה מתייג אותה, למשל `alerts:write`), `3` דשבורד לא מעל. סקריפט יכול לסעוף על `4` כדי להתחבר מחדש או `5` כדי לומר לך בדיוק למה לשאול מנהל, במקום להיכשל בעיוורון.

## תן לסוכן קוד להסיע אותו בשפה טבעית

אפילו יותר טוב, לא צריך לזכור כל אחד מהדגלים הללו בכלל. **CLI skill** היא תיקייה קטנה של Agent Skill בשם `agenteye-cli` שמלמדת סוכן קוד כמו Claude Code או Codex להסיע את ה-CLI מבקשות בשפה טבעית. שאל "האם משהו שבור היום?" והסוכן בוחר את הפקודה, מריץ אותה בתורך, ועונה בפרוזה.

עבור Claude Code, זרוק את תיקיית `agenteye-cli` לתוך `~/.claude/skills/` וזה discoverd אוטומטי. Failproof AI Observability מספק את התיקייה; אין כלום נוסף להתקנה, כי זה רק נוהג את ה-CLI שכבר התקנת. היכנס בעצמך קודם: הכישור לא יכול להשלים את ההתחברות לקוד משלח בדוא"ל עבורך.

כי הסוכן מריץ את ה-CLI בתורך, הוא יכול לעשות הכל שההתחברות שלך מתיר, קריאות וכתיבות כאחד: ליצור keys, לשנות הגדרות, לפתור תקריות. ה-"האם אתה בטוח?" prompt של ה-CLI לא כועס עבור סוכן, כך שהכישור כתוב כדי לקבוע את הפקודה המדויקת ולחכות לאישור שלך לפני כל שינוי. אתה שלב האישור.

```text
you    Why did session run-001 fail?

agent  Running: agenteye --json events --session-id run-001 --all
       The checkout-agent hit a TimeoutError on its third tool call.
```

קריאות נשארות מיידיות, וכל כתיבה מתעצרת לך:

```text
you    Give CI a key that can only push events.

agent  This creates an API key. I'll run:
         agenteye keys create ci --add events:add
       OK to proceed?

you    yes

agent  Done. Key "ci" created with events:add only. The secret is shown once, so store it now.
```
