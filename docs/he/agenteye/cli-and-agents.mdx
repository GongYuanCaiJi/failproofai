כל הפריסה של FailproofAI Observability שלך, בפקודה אחת. בדוק את הייצור, צור מפתח API, או אשר אירוע ללא צורך להשאיר את הטרמינל שלך, ואז כתוב סקריפט של כל זה ל-CI, או תן לסוכן קידוד לעשות זאת בעברית פשוטה.

```bash
pipx install agenteye
agenteye login --email you@example.com      # a 6-digit code lands in your inbox
agenteye --json sessions --since 24h         # every agent run from the last day, newest first
```

*ה-CLI של `agenteye` מתקשר עם הלוח שלך. זהו כלי שונה מהאספן, המשדר אירועים לשרת.*

## כל הפריסה שלך, בפקודה אחת

הפסק להקפיד על טאבים כדי לענות לשאלה מהירה. ה-CLI של `agenteye` קורא את הנתונים שלך ומנהל את הארגון שלך מקובץ בודד, כך שבדיקה שהייתה בעבר משמעותית של לחיצות על הלוח הופכת לשורה אחת שאתה יכול להריץ שוב, להגדיר כשם קצר, או להדביק לתוך runbook. יש לך ארבע אפשרויות:

- **קרא את הנתונים שלך:** `sessions`, `events`, `evals`, ו-`errors`, מסוננים לפי זמן, סוכן וסביבה.
- **נהל את הארגון שלך:** `keys`, `users`, `settings`, `alerts`, ו-`incidents`.
- **הרץ ניתוח:** SQL שמור בתוספת רץ `query` ad-hoc על נתוני האירועים שלך.
- **שאל את העוזר:** `agent ask` מגיע לאותו אנליסט קריאה-בלבד שאתה משוחח איתו בלוח.

התקן אותו פעם אחת עם `pipx`, היכנס עם קוד 6 ספרות שנשלח במייל, ואתה מוכן. ההפעלה נמשכת בערך יום; הרץ `agenteye login` שוב כשזה יפוג. גש אליו כדי לבדוק את הייצור, לספק מפתח, או לבדוק תקלה שמתרחשת, כל זה ללא פתיחת דפדפן:

```bash
agenteye errors --since 24h --aggregate      # what is breaking, grouped by error type
agenteye incidents list --state firing       # what is on fire right now
agenteye keys create ci --add events:add     # a key that can only push events, secret shown once
```

הרגל אחד שכדאי לדעת: אפשרויות גלובליות כמו `--json` מגיעות לפני הפקודה. `agenteye --json sessions` זה נכון; `agenteye sessions --json` לא.

## כתוב סקריפט, חבר זאת ל-CI

כל פקודה לוקחת `--json`, וזה משנה הכל. JSON נקי הולך ל-stdout בזמן שמצב אנושי והזהרות הולכות ל-stderr, כך שתיאום `--json` מתפרק ישירות ל-`jq` ללא שורה תועה שצריך להסיר. זה מה שהופך את ה-CLI לטוב באותה מידה עבורך בהנמקה וגם לסוכן קידוד המנתח פלט:

```bash
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'
```

זה בנוי לרוץ ללא פיקוח. הנמקות אישור מדלגות באופן אוטומטי כאשר אין טרמינל מצורף, כך שלא משהו תלוי בצינור, וכל פקודה מחזירה קוד יציאה משמעותי: `0` הצלחה, `4` לא מחובר, `5` הרשאה חסרה (ההודעה קובעת זאת, לדוגמה `alerts:write`), `3` לוח בלתי נגיש. סקריפט יכול להיות סניף בקוד `4` להחלת אימות מחדש או קוד `5` כדי לומר לך בדיוק מה לבקש ממנהל, במקום להיכשל ללא ראות.

## תן לסוכן קידוד לנהוג בו בעברית פשוטה

אפילו יותר טוב, אתה לא צריך לזכור את כל הדגלונים הללו בכלל. **CLI skill** היא תיקיית Agent Skill קטנה בשם `agenteye-cli` המלמדת סוכן קידוד כמו Claude Code או Codex לנהוג בـ CLI מבקשות בעברית פשוטה. שאל "האם משהו שבור היום?" והסוכן בוחר את הפקודה, מריץ אותה בתור אתה, וענה בפרוזה.

עבור Claude Code, שחרר את תיקייה `agenteye-cli` ל-`~/.claude/skills/` וזה מתגלה באופן אוטומטי. FailproofAI Observability מספק את התיקייה; אין שום דבר נוסף להתקנה, כי זה רק מנהל את ה-CLI שכבר התקנת. היכנס בעצמך קודם: הכישור לא יכול להשלים את ההתחברות של קוד מייל עבורך.

מכיוון שהסוכן מריץ את ה-CLI בתור אתה, הוא יכול לעשות הכל שההתחברות שלך מתירה, קריאות וכתיבה כאחד: ליצור מפתחות, לשנות הגדרות, לפתור אירועים. הנמקה "האם אתה בטוח?" של ה-CLI לא כוללת סוכן, כך שהכישור כתוב כדי לציין את הפקודה המדויקת ולהמתין לאישורך לפני כל שינוי. אתה שלב האישור.

```text
you    Why did session run-001 fail?

agent  Running: agenteye --json events --session-id run-001 --all
       The checkout-agent hit a TimeoutError on its third tool call.
```

הקריאות נשארות תיכף, וכל כתיבה מתעצרת עבורך:

```text
you    Give CI a key that can only push events.

agent  This creates an API key. I'll run:
         agenteye keys create ci --add events:add
       OK to proceed?

you    yes

agent  Done. Key "ci" created with events:add only. The secret is shown once, so store it now.
```
