משכו נתוני session, event והערכה (וזעזעו הערכות מחדש) ישירות מתוך סקריפט או סוכן coding, עם JSON נקי ב-stdout שזורם ישירות ל-`jq`. המתכונים הללו הופכים את הנתונים של FailproofAI Observability למשהו שמשתמש terminal או סוכן coding בעל AI (Claude Code, Cursor) יכול לשאול בשאילתא ולהחליף ללא לחיצה דרך ה-dashboard.

הדפוסים להלן מוכנים להעתקה-הדבקה עבור CLI של FailproofAI Observability (`agenteye`). להתקנה, אימות וקבוצת האפשרויות המלאה, ראה [CLI](/he/agenteye/cli); הרץ `agenteye -h` או `agenteye <command> -h` לעזרה מובנית.

## כללים זהב

1. **אפשרויות גלובליות הולכות *לפני* הפקודה.** `agenteye --json sessions` נכון; `agenteye sessions --json` לא. הגלובליים הם `--json`, `--base-url`, `--org`, `--token`, `--insecure`/`--secure`, `--timeout`, `--quiet`, `--no-color`.
2. **העבר `--json` בכל פעם שאתה מחלץ פלט.** נתונים הולכים ל-**stdout** כ-JSON; מצב אנושי ושגיאות הולכים ל-**stderr**, אז stdout נשאר נקי לזרימה ל-`jq`.
3. **הסתעף על קוד ההוצאה, לא על טקסט stderr**: `0` בסדר · `1` שגיאה בלתי צפויה · `2` ארגומנטים רעים · `3` לא יכול להגיע ל-dashboard · `4` לא מחובר או פג תוקף · `5` הרשאה חסרה · `6` משאב לא נמצא.
4. **גלה עם `-h`.** כל פקודה תיעד את הסננים שלה, פורמטי ערך וצורת JSON.

## התקנה חד-פעמית

```bash
export AGENTEYE_DASHBOARD_URL=https://agenteye.example.com   # כדי שלא תחזור על --base-url
agenteye login --email you@example.com                       # הדבק את הקוד שנשלח בדוא"ל; תקף ~24h
```

## אישור אימות לפני ביצוע עבודה

`whoami` לעולם לא שגוי על session חסר או פג תוקף; הוא דווח `logged_in:false` במקום זאת, כך שסוכן יכול לחקור מצב אימות בבטחה. (זה עדיין יכול לצאת עם קוד שאינו אפס אם לא הוגדר base URL או אם ה-dashboard לא ניתן להשגה.)

```bash
if [ "$(agenteye --json whoami | jq -r .logged_in)" != "true" ]; then
  echo "Not authenticated. Run: agenteye login" >&2; exit 1
fi
```

## מצא sessions שנכשלים או בעלי ניקוד נמוך

```bash
# sessions ב-24 שעות האחרונות שלהן הערכה נכשלה
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'

# הערכות עם ניקוד <= 0.5 על helpfulness, לסוכן אחד
agenteye --json evals --agent-id checkout-bot --score helpfulness:..0.5 \
  | jq '.evaluations[] | {session_id, scores}'
```

סינון ניקוד חי על **`evals`**, לא `sessions`. `--score KEY:MIN..MAX` ניתן לחזרה וו-AND מוכלל; כל גבול אופציונלי (`..0.5` פירושו ≤ 0.5, `0.9..` פירושו ≥ 0.9). אתה יכול להעביר עד 20 סיננים ניקוד לכל בקשה; יותר מחזיר HTTP 400. `sessions` משתפת את `--env`, `--status`, `--agent-id`, `--session-id`, ומסננים טווח זמן עם `evals`, אבל אין `--score`.

## קרא session אחד מקצה לקצה

אין פקודת `session show` יחידה. שלב את שביל ה-event עם הערכת ה-session:

```bash
# ההערכה העדכנית ביותר של ה-session (מצב + ניקודים)
agenteye --json evals --session-id run-001 | jq '.evaluations[0] | {status, scores}'

# כל event בהרצה (הגדל --limit לסיורת מלאה)
agenteye --json events --session-id run-001 --all --limit 1000 | jq '.events[] | {ts, event_type}'

# רק קריאות הכלים ב-session (--full נדרש כדי לקבל את payload גולמי)
agenteye --json events --full --session-id run-001 --event-type tool_use,tool_result --all \
  | jq '.events[].payload'
```

> **הערה:** כברירת מחדל, `events` קורא feed מהיר וללא payload. כל event נושא summary בעל חישוב שרת בשורה אחת בתוספת דגלים כמו `is_error` וספירות טוקן, אבל `payload` חוזר כ-`{}`. כדי למשוך את ה-payload הגולמי, הוסף `--full` (או `--fields payload`). ה-feed המלא איטי בקנה מידה, אז שמור עליו מוגבל: צמד `--full` עם `--session-id` יחיד.

## אחזור הכל (pagination)

התוצאות הן newest-first ו-cursor-paginated.

```bash
# shot אחד: אחזור עד 500 שורות בעמודים של 200 שורות
agenteye --json events --session-id run-001 --limit 500 --all > events.json

# עמודים ידניים: הזן את next_cursor חזרה
page=$(agenteye --json events --limit 100)
cursor=$(echo "$page" | jq -r '.next_cursor // empty')
[ -n "$cursor" ] && agenteye --json events --limit 100 --cursor "$cursor"
```

## הדק את הפלט עם --fields

הגבל את המפתחות (בטבלה ובـ `--json`) כדי להקטין את מה שסוכן חייב לקרוא.

```bash
agenteye --json sessions --since 7d --fields session_id,status,scores | jq -c '.sessions[]'
agenteye --json events --session-id run-001 --fields ts,event_type --all
```

שמות שדות לא ידועים נדחים (יציאה `2`) עם הרשימה התקפה, דרך זולה לגלות שמות שדות.

## גלה ערכי סיננים תקפים

```bash
agenteye --json list envs | jq -r '.values[]'           # ערכים עבור --env
agenteye --json list tools | jq -r '.values[]'          # שמות כלים; גם agents, models, event_types, …
agenteye --json list score_filters | jq -r '.values[]'  # KEY תקף עבור --score KEY:MIN..MAX
```

## בחר את ה-org שלך (multi-tenant)

אם אתה משתייך ליותר מ-org אחד, בחר את ה-tenant הפעיל בהתחברות (זה נשמר):

```bash
agenteye login --org acme --email you@corp.com   # הגדר את ה-tenant באותו הצעד כמו התחברות
agenteye --json orgs list | jq -r '.orgs[].org_slug'
agenteye --org globex --json sessions --since 24h   # דרוס לפקודה אחת
```

התחברות multi-org ללא `--org` יוצאה עם קוד שאינו אפס והדפיס את ה-orgs לבחירה.

## הנפק מפתח API עבור ה-SDK/collector

```bash
# הסוד מודפס פעם אחת, עם --json זה ה-.key שדה
key=$(agenteye --json keys create ci-bot --add events:read.add | jq -r '.key')
agenteye keys regenerate ci-bot --yes    # סובב; agenteye keys disable ci-bot --yes כדי לשלול
```

## הרץ שאילתה שמורה או ad-hoc

```bash
agenteye --json query run --sql "select count(*) from analytics.events" | jq '.rows'
agenteye --json query run errs --arg prod | jq '.rows'   # שאילתה שמורה + $1 positional
```

## Triage תקרית בצורה לא-אינטראקטיבית

```bash
id=$(agenteye --json incidents list --state firing | jq -r '.incidents[0].id')
agenteye incidents ack "$id"
agenteye incidents assign "$id" --assignee you@corp.com
agenteye incidents resolve "$id" --yes
```

> **הערה:** Mutations דילוג אוטומטי על הודעת האישור שלהם תחת `--json` או כאשר stdin אינו TTY, כך שסוכנים לעולם לא נתקעים; עבור `--yes`/`-y` כדי לדלג עליו במפורש במקום אחר.

## קוד ממשק ביציאה בסקריפט

```bash
out=$(agenteye --json sessions --since 1h) || code=$?
case "${code:-0}" in
  0) echo "$out" | jq '.sessions | length' ;;
  4) echo "Session expired - run 'agenteye login'." >&2 ;;
  5) echo "Missing permission (ask an admin for evaluations:read)." >&2 ;;
  3) echo "Dashboard unreachable - check the URL." >&2 ;;
  *) echo "Unexpected error (exit ${code})." >&2 ;;
esac
```

## צורות פלט JSON

| פקודה | stdout JSON (עם `--json`) |
|---|---|
| `whoami` | `{"logged_in": true, "id", "email", "is_instance_admin", "active_org", "permissions": [...], "memberships": [...]}` או `{"logged_in": false}` |
| `orgs list` | `{"active_org", "orgs": [{"org_slug","org_name","permission_set","permissions"}]}` |
| `events` | `{"events": [...], "next_cursor": <cursor or null>}` |
| `evals` | `{"evaluations": [...], "next_cursor": <cursor or null>}` |
| `sessions` | `{"sessions": [...], "next_cursor": <cursor or null>}` |
| `errors` | `{"errors": [...], "next_cursor": <cursor or null>}` |
| `list <kind>` | `{"kind", "values": [...]}` |
| `keys list` / `keys create` | `{"keys": [...]}` / `{id, name, permissions, created_at, key}` (`key` הוצג פעם אחת) |
| `query run` | `{columns: [{name,type}], rows: [[...]], truncated, elapsed_ms}` |
| `users list` / `settings list` | `{"users": [...]}` / `{"settings": [...]}` |
| `alerts list` / `incidents list` | `{"alerts": [...]}` / `{"incidents": [...]}` |
| create/update/delete (כל) | אובייקט המשאב, או `{"deleted": true, "id"}` למחיקות |
| כשל (כל, עם `--json`) | `{"error": "...", "exit_code": <n>, "status"?: <http>, "hint"?: "..."}` ב-stdout |

- כל פריט **event** (`events`): `id, session_id, agent_id, event_type, ts, payload, environment, summary, is_error, error_type, output_tokens, context_window, context_fill`. שים לב ש-`payload` הוא `{}` אלא אם אתה מבקש את ה-feed המלא עם `--full` (או `--fields payload`).
- כל פריט **evaluation** (`evals`): `id, session_id, agent_id, environment, status, scores, reasoning, summary, error, attempt_count, duration_ms, completed_at, created_at`.
- כל פריט **session** (`sessions`): `session_id, agent_id, environment, status, scores, event_count, started_at, last_event_at, first_event_id, last_event_id, latest_evaluation`.

`--fields` של כל פקודה מקבל בדיוק שמות שדה של הפריט שלה. הסט שונה בין `sessions` ו-`evals`, ולכן שם תקף לאחד עשוי להידחות על ידי השני.
