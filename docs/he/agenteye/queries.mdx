שאל כל שאלה על נתוני הסוכן שלך וקבל תשובה תוך שניות. Failproof AI Observability נותן לך ספריה של שאילתות שמורות ומוכנות להפעלה על האירועים וההערכות שלך, כדי שתתחיל מדוגמה שעובדת במקום מ-SQL עורך ריק.

![ספריית השאילתות השמורות: רשת של שאילתות בניתנות לשימוש חוזר, הן עם הנתונים המובנים והן עם אלה החדשות](/agenteye/images/queries.png)

*ספריית השאילתות השמורות שלך ב-`/<org-slug>/queries`: עם נתונים מובנים יושבים ליד השאילתות שהצוות שלך שמר.*

## התחל מתבנית, לא מעמוד ריק

אתה לא צריך לזכור שמות טבלאות או לכתוב SQL מאפס. הספריה נפתחת עם תבניות מובנות לשאלות שהצוותים שואלים הכי הרבה, יושבות ממש ליד השאילתות שהצוות שלך שמר וקרא להן שם. בחר אחת שקרובה למה שאתה רוצה ואתה כמעט במחצית הדרך לתשובה.

כל שאילתה שמורה היא בהיקף ארגון ומשותפת, כך שהשאילתות השימושיות שהחברים שלך כותבים הופכות גם לשלך. תן שם לשאילתה וכתוב עבורה תיאור פעם אחת, וכל אחד בארגון שלך יכול למצוא אותה, להפעיל אותה או להצמיד את התוצאות שלה לדוח מחבורת לימים קדימה.

מצא אותו ב-`/<org-slug>/queries`.

## התאם אותו והפעל אותו ב-SQL composer

פתח כל שאילתה והיא תנחת ב-SQL composer, כאשר אתה יכול להתאים אותה ולראות את התשובה מיד: ללא ייצוא, ללא הקפה, ללא המתנה למישהו אחר.

![SQL query composer שמפעיל שאילתה שמורה, עם סרגל צד של סכימה וגריד תוצאה חי](/agenteye/images/query-lab.png)

*ה-SQL composer: השאילתה שלך משמאל, סרגל צד של סכימה כדי שלעולם לא תנחש שם עמודה, וגריד תוצאה חי למטה.*

- **סרגל צד של סכימה** מפרט את טבלאות ניתוח הנתונים והעמודות שלהם, כדי שתוכל לעצב שאילתה ללא ציד על שמות שדה.
- **גריד תוצאה חי** מחזיר שורות ברגע שאתה מפעיל, כדי שתוכל לחזור על חידוש בתוך שניות ולא לנחש ולנחש שוב.
- **קריאה בלבד לפי עיצוב.** שאילתות רצות כנגד חנות האירועים שלך ומאומתות בשרת: רק הצהרות `SELECT` ו-`WITH` מורשות, עם משך זמן מקסימום וגבול שורה. שאילתה חקרנית לעולם לא יכולה לשנות את הנתונים שלך, ואחד שחרג מן השליטה יעצור בשבילך.

מרוצה מהתוצאה? שמור אותה חזרה לספריה כדי שהצוות כולו יורשה אותה, או הצמד את הפלט שלה לדוח כעמודה, עמודה, אזור או אריח עוגה.

## הפעל אותם מהטרמינל, או תן לעוזר לכתוב אותם

אותן שאילתות שמורות עוקבות אחריך לכל מקום בו אתה עובד:

- **מהטרמינל.** ה-CLI של `agenteye` מופיע, מפעיל ושומר את אותן שאילתות, כדי שתוכל להפיל תוצאה לתוך סקריפט, לחבר אותו ל-CI, או להעביר אותו לסוכן קידוד.

```bash
agenteye query list                 # the same saved queries, from your terminal
agenteye query run errs --arg prod  # run one and print the rows (add --json to pipe it)
```

  ראה [CLI and agents](/he/agenteye/cli-and-agents) לסט הפקודות המלא.

- **מהעוזר של AI.** לא בטוח איך לנסח את SQL? שאל את [עוזר ה-AI](/he/agenteye/assistant) שבתוך הדוח באנגלית פשוטה והוא יעצור את השאילתה ויחסוך אותה לספריה בשבילך.

הפעלה של שאילתה שמורה היא מסווגת לפי ההרשאה `queries:run`, שנשמרה בנפרד מההרשאות ליצור או למחוק שאילתות, כדי שתוכל להעניק גישה קריאה ללא הרשאה לכל אחד לכתוב מחדש את הספריה.
