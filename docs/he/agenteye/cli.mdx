שלוט בכל FailproofAI Observability מהטרמינל או מסקריפט: ללא סיבובים בדashboard. ה-CLI של `agenteye` שואל על הנתונים שלך (sessions, event logs, evaluations) וממנהל את הארגון שלך (API keys, users, settings, alerts, incidents, saved queries), אז השתמש בו כשאתה רוצה להפעיל בדיקה אוטומטית, לשלב Observability ב-CI, או להתיר לcoding agent לבדוק את ה-production. כל פקודה תומכת בדגל `--json`, כך שהיא עובדת באותה מידה עבורך בשורת הפקודה או עבור coding agent (Claude Code, Cursor) שמפעיל shell וחוזר על התוצאה.

עם בינארי אחד אתה יכול:

- **קרא את הנתונים שלך**: `sessions`, `events`, `evals`, `errors` (סנן לפי זמן, agent, env, score).
- **נהל את הארגון שלך**: `keys`, `users`, `settings`, `alerts`, `incidents`.
- **הפעל ניתוחים**: SQL שמור וrunner query ad-hoc (`query`).
- **שאל את עוזר ה-AI**: אותו אנליסט read-only שאתה משוחח איתו בdashboard (`agent`).

> **הערה:** זה ה-CLI של `agenteye`, כלי שונה מה-collector daemon (`agenteye-collector`). ה-CLI מדבר לdashboard שלך; ה-collector שולח events לשרת.

## התחלה מהירה

מלא חלל לתוצאה הראשונה שלך בארבע שורות. הפנה את ה-CLI לdashboard שלך, התחבר, אשר מי אתה, ואז משוך את היום האחרון של ריצות:

```bash
pipx install agenteye
agenteye --base-url https://agenteye.example.com login --email you@example.com   # emailed 6-digit code
agenteye whoami                                                                  # confirm user + active org
agenteye --json sessions --since 24h                                             # one row per agent run, last 24h
```

הפקודה האחרונה הזו מדפיסה אובייקט JSON של ה-sessions האחרונים ביותר (החדשים ביותר קודם, בעל cap של 50 כברירת מחדל). צנור אותו ל-`jq` כדי לחתוך אותו, או שמט `--json` בשביל טבלה עם קופסא וצבועה. כל שורה נושאת את status של הריצה ו, אם evaluator הבקיע אותה, את ה-metric scores שלה (מתוקצרים כאן):

```json
{
  "sessions": [
    {
      "session_id": "run-8f2a",
      "agent_id": "checkout-bot",
      "environment": "prod",
      "status": "error",
      "scores": { "helpfulness": 0.42, "tool_efficiency": 0.55 },
      "event_count": 37,
      "started_at": "2026-07-16T09:14:02Z",
      "last_event_at": "2026-07-16T09:14:48Z"
    }
  ],
  "next_cursor": null
}
```

שאר הדף הזה מסביר כל חלק: [התקנה](#installation) בבידוד, [התחברות](#authentication), [תצורה](#configuration), ה-[global conventions](#global-options--conventions) שכל פקודה חולקת, ו-[reference פקודה מלא](#command-reference).

## התקנה

ה-CLI הוא חבילת PyPI ציבורית בשם **`agenteye`**. התקן אותה בסביבה מבודדת כך שיהיה לה תמיד התלויות שלה:

```bash
pipx install agenteye
# or
uv tool install agenteye
```

זה דורש Python 3.10+. הפקודה המותקנת היא **`agenteye`**:

```bash
agenteye --version
agenteye --help
```

> **הערה:** ה-Python SDK של FailproofAI Observability משתמש גם בשם הפצה של `agenteye`. התקנת ה-CLI עם `pipx` או `uv tool` (במקום `pip install` לתוך virtualenv משותף) מונעת מהשניים להתנגש. plain `pip install agenteye` בסדר רק אם ה-SDK לא מותקן באותה סביבה.

## הנחה

ה-CLI מטמון לdashboard עם קוד חד-פעמי שנשלח בדואר:

```bash
agenteye login --email you@example.com
# A 6-digit code is emailed to you; paste it at the prompt.
```

token ה-session מאוחסן ב-`~/.agenteye/cli.json` (קריא רק על ידיך, mode `0600`) והוא תקף ל-24 שעות כברירת מחדל. כשהוא פג, הרץ `agenteye login` שוב.

```bash
agenteye whoami     # show the current user, active org, and permissions
agenteye logout     # revoke the session and clear the stored token
```

`whoami` לעולם לא שגיאות על session חסר או פג תוקף; הוא מדווח `logged_in: false` במקום זאת, כך שscript או agent יכול להתבדוק state auth בבטחה (הוא עדיין יכול לצאת non-zero אם לא הוגדר base URL או dashboard לא נגיש).

**דרישות:** ה-email שלך חייב להיות מאושר להתחבר לdashboard (שאל את ה-administrator של FailproofAI Observability שלך), וה-dashboard חייב להיות נגיש ב-base URL שלו (ראה [Configuration](#configuration)). אם אתה מבקש קוד וכלום לא מגיע, ה-email שלך כנראה עדיין לא מופעל לגישת dashboard.

## בחירת ה-org שלך (multi-tenant)

אם החשבון שלך שייך ליותר מ-org אחד, בחר את ה-active **בהתחברות**; הוא נשמר ומשמש לכל פקודה מאוחרת יותר:

```bash
agenteye login --org acme           # authenticate and set the active tenant in one step
agenteye orgs list                  # the orgs you can access (the active one is marked)
agenteye orgs switch globex         # change the saved default
agenteye --org globex sessions      # override for a single command
```

אם אתה שייך בדיוק ל-org אחד הוא נבחר באופן אוטומטי ותוכל להתעלם מ-`--org` לחלוטין. אם אתה שייך לכמה ולא בחרת אחד, ה-CLI מפרט אותם ומבקש ממך להריץ שוב עם `--org <slug>`. ה-org ה-active משלח לdashboard בכל בקשה, והרשאות שלך מוגדלות **לכל org**; `agenteye whoami` מציג את ה-org ה-active, ההרשאות שלך בו, וכל ה-memberships שלך.

## תצורה

| הגדרה | דגל | משתנה סביבה | ברירת מחדל |
|---|---|---|---|
| Dashboard base URL | `--base-url` | `AGENTEYE_DASHBOARD_URL` | **דרוש** (ללא ברירת מחדל) |
| Active org/tenant | `--org` | `AGENTEYE_ORG` | נבחר בהתחברות; שמור ב-`~/.agenteye/cli.json` |
| Session token | `--token` | `AGENTEYE_CLI_TOKEN` | מ-`~/.agenteye/cli.json` |
| JSON output | `--json` | `AGENTEYE_CLI_JSON` | כבוי |
| דלג על TLS verification | `--insecure` / `--secure` | `AGENTEYE_INSECURE` | כבוי (שמור בהתחברות) |
| Request timeout (seconds) | `--timeout` | _(none)_ | 30 |
| השבת telemetry שימוש | _(none)_ | `AGENTEYE_ANALYTICS_DISABLED` (או `DO_NOT_TRACK`) | telemetry כרגע מושבת; כלום לא נשלח |

סדר resolution הוא **flag → environment variable → config file**. אין ברירת מחדל; עליך להפנות את ה-CLI לdashboard שלך, או לפי-פקודה (`--base-url https://agenteye.example.com`) או פעם דרך הסביבה (היא גם שמורה לאחר ה-`login` הראשון שלך):

```bash
export AGENTEYE_DASHBOARD_URL=https://agenteye.example.com
```

ה-configuration directory מכבד את `AGENTEYE_HOME` (אותה convention המשמשת את ה-SDK וה-collector); אם הוגדר, `cli.json` חי ב-`$AGENTEYE_HOME/cli.json`.

### Self-signed או internal TLS

אם ה-dashboard שלך מוגש דרך HTTPS עם self-signed או internal certificate (לדוגמה, hostname load-balancer גולם), TLS verification דוחה אותו עם שגיאה `CERTIFICATE_VERIFY_FAILED`. עבור `--insecure` כדי לדלג על certificate verification:

```bash
agenteye --base-url https://agenteye.internal --insecure login
```

`--insecure` הוא **שמור ל-`cli.json` כשאתה מתחבר**, כך שפקודות מאוחרות יותר דולגות על verification באופן אוטומטי; אתה לא חייב לחזור על הדגל. עבור `--secure` כדי call מאומת חד-פעמי, או כדי לשמור חזרה on verification בהתחברות הבאה שלך. ה-CLI מדפיס אזהרה ל-stderr לפני כל פקודה שמתקשרת לdashboard בעת verification כבוי. דילוג verification מסיר הגנה נגד man-in-the-middle attacks; וודא שאתה סומך על network path לdashboard שלך (VPN, private subnet, וכו') לפני שאתה מסתמך עליו.

## טלמטריה ופרטיות

> **הערה:** ה-shipped CLI שלח **ללא telemetry שימוש היום.** master kill switch הוא ב-on, כך שכלום לא מועבר ללא קשר לסביבה שלך. הסעיף להלן מתאר את יכולת ה-opt-out בשביל אם ומתי telemetry אי פעם מופעל.

גם כשהוא מופעל, telemetry יהיה **analytics שימוש אנונימי בלבד**, לעולם לא ה-agent, session, או event data שלך:

- **לא agent, session, או event data לעולם עוזב את ה-infrastructure שלך.** רק CLI usage ידווח: שם ה-command וה-subcommand (לדוגמה `keys create`), **שמות** של הדגלים ששימשתם (לעולם לא הערכים שלהם), success/exit status, ודורך, בתוספת per-action event עבור mutations (לדוגמה `api_key_created`, `query_run`) נושא רק שמות/enums סטטיים וספירות גסות. ה-dashboard URL שלך, session token, email, org slug, resource ids, SQL, key secrets, וquery filters **לעולם** לא יישלחו. Operators יזוהו רק על ידי opaque internal id, לעולם לא על ידי email.
- **Opt out לפני הזמן** על ידי הגדרת `AGENTEYE_ANALYTICS_DISABLED=1` בסביבה של ה-CLI (ה-CLI גם מכבד את ה-cross-tool `DO_NOT_TRACK=1` convention). זה נכנס לתוקף ברגע שהטלמטריה מופעלת אי פעם, כך שסביבה privacy-conscious יכולה להישאר opted out לצמיתות.
- אם telemetry היו מופעלים, ה-CLI היה שולח ישירות ל-PostHog (`https://us.i.posthog.com`); מכונה עם host זה חסום היה שולח בשקט כלום וה-CLI לא היה מושפע.

## אפשרויות גלובליות ו-conventions

קרא את זה פעם אחת; זה חל לכל פקודה.

- **Global options הולכים לפני הפקודה.** `agenteye --json sessions` נכון; `agenteye sessions --json` היא שגיאת שימוש. ה-globals הם `--json`, `--base-url`, `--org`, `--token`, `--insecure`/`--secure`, `--timeout`, `--quiet`, וה-`--no-color`.
- **`--json` מדפיס pure JSON ל-stdout, ושום דבר אחר.** Human status lines, warnings, ו-errors הולכים ל-**stderr**, כך ש-`--json` stdout capture נשאר נקי כדי צנור ל-`jq` גם כשstatus line מוצג. בלי `--json` אתה מקבל תצוגה עם קופסא וצבועה לעיניים אנושיות.
- **גלה עם `--help`.** לכל פקודה וה-subcommand יש `--help` (וה-`-h` alias): `agenteye -h`, `agenteye sessions -h`, `agenteye keys create -h`. top-level help גם מפרט את ה-exit codes ו-global options. אין global machine-readable surface dump; השתמש per-command `--help`, בתוספת ה-domain-specific `agenteye query schema` ו-`agenteye settings schema` לאלה שתי registries.
- **Confirmations auto-skip לscripts וagents.** Create/update/delete commands prompt ״האם אתה בטוח?״ בטרמינל אינטראקטיבי, אך **auto-skip that prompt תחת `--json` או בכל פעם stdin לא TTY** (TTY הוא session טרמינל אינטראקטיבי; pipe או CI runner לא), כך שscripts וagents לעולם לא תלויים. עבור `--yes`/`-y` כדי לדלג עליו במפורש. כי ה-prompt לא יתקיים עבור agent, agent צריך לאשר actions הרסניים עם ה-human קודם לכן.
- **Pagination:** תוצאות הן newest-first וcursor-paginated (כל דף חוזר token שאתה משתמש בו כדי להביא את הבא). `--limit N` (alias `-n`) caps rows וברירת **מחדל ל-50**; `--all` auto-paginates (ב-200-row chunks) **עד `--limit`**, כך שbare `--all` עדיין מעצור ב-50. עבור full sweep תמסור cap גבוה מפורש: `--all --limit 1000`. `--page-size N` שומר על per-request chunk (max 200); `--cursor <id>` resumes מ-prior page's `next_cursor`.
- **Time filters:** `--since` לוקח window יחסי: `15m`, `1h`, `6h`, `24h`, `7d`, או `all` (ה-dashboard's presets). עבור ארוך יותר או custom range (אמור ה-30 ימים האחרון), השתמש ב-`--from`/`--to`: explicit ISO-8601 UTC timestamps **עם `T` וtimezone** (לדוגמה `2026-06-01T00:00:00Z`) שה-override `--since`. space-separated או timezone-less value היא usage error.
- **`--fields a,b,c`** (ב-`events`, `sessions`, `evals`, `errors`) מגביל את ה-output לאותם keys, עבור שני הטבלה וה-`--json`. Unknown names דחויות עם ה-valid list, way זול כדי לגלות field names.
- **`--file payload.json`** (או `--file -` כדי לקרוא stdin) מספק full JSON request body כאשר resource יש complex shape (על `alerts create/update`, `settings set`, וה-`users create/update`). Saved-query SQL משתמש ב-`--sql @file.sql` במקום זאת.
- **Multi-value filters** הם comma-separated → matched כ-set (union בתוך one filter, AND על פני filters): `--event-type tool_use,tool_result`. Click options לא variadic, כך `--add a b` כושל. השתמש ב-`--add a,b`, חזור על הדגל (`--add a --add b`), או הצע (`--add "a b"`).

## Command reference

### תשתמש בפקודות 5 אלה ביותר

רוב העבודה היומית-יומית מבוצעת דרך כמה read commands. התחל כאן, ואז הגע לה-full surface להלן כשאתה צריך זאת:

| פקודה | מה היא עושה | נסה את זה |
|---|---|---|
| `sessions` | שורה אחת לכל agent run: זמן, env, agent, status, latest score. | `agenteye --json sessions --since 24h --status error` |
| `events` | ה-raw per-step trail בתוך run (הוסף `--full` עבור payloads). | `agenteye --json events --session-id run-001 --all` |
| `evals` | תוצאות הערכה וציונים; `--aggregate` גלגלות אותם למעלה. | `agenteye --json evals --aggregate --since 7d --env prod` |
| `errors` | רק ה-errored events; `--aggregate` עבור counts לפי type. | `agenteye --json errors --since 24h --aggregate` |
| `list` | גלה את ה-valid filter values (agents, envs, models, …). | `agenteye list agents` |

### הכל ש-CLI יכול לעשות

ה-full surface עוקב. ל-CLI יש **18 top-level commands**. כל read commands קבל `--json` ו-global options למעלה; הרץ `agenteye <command> -h` (או `<command> <subcommand> -h`) עבור ה-exhaustive flag list וJSON shape של אחד כלשהו.

### Identity: `login` · `logout` · `whoami` · `orgs` · `version` · `help`

```bash
agenteye login --email you@example.com [--org acme]   # emailed one-time code; saves the session
agenteye logout                                       # clear the saved session on this machine
agenteye whoami                                       # current user, active org, permissions
agenteye version                                      # print the CLI version (same as --version)
agenteye help                                         # top-level help (same as --help)
```

`orgs` בודק ומעביר את ה-active tenant:

```bash
agenteye orgs list        # your orgs + your role in each (active one marked)
agenteye orgs switch acme # change the saved active org (omit the slug to pick from a list on a TTY)
agenteye orgs current     # identity card for the active org
agenteye orgs perms       # your permissions in the active org, grouped by resource
```

### Observe (read-only): `events` · `sessions` · `evals` · `errors` · `list`

אף אחד מאלה לא צריך confirmation. Shared filters: `--session-id`, `--agent-id`, `--env` (**לא** `--environment`), וה-time range (`--since` / `--from` / `--to`).

```bash
# events (alias: the raw per-step trail), newest first
agenteye --json events --session-id run-001 --event-type tool_use,tool_result --all --limit 1000
agenteye --json events --since 1h --search timeout --all | jq '.events[].payload'

# sessions: one row per agent run (time/env/agent/session/status; no score filtering)
agenteye --json sessions --since 24h --status error
agenteye --json sessions --agent-id checkout-bot --env prod --all --limit 1000

# evals: evaluation results + scores; --score filters by metric, --aggregate rolls up
agenteye --json evals --score helpfulness:0.5..0.8 --score tool_efficiency:..0.3
agenteye --json evals --aggregate --since 7d --env prod        # status mix + per-key score stats

# errors: errored events; --aggregate for counts/sessions/agents/last-seen
agenteye --json errors --since 24h --aggregate
agenteye --json errors --since 24h --error-type timeout --all --limit 1000

# list: discover valid filter values before you filter
agenteye list envs        # also: agents event_types score_filters models hooks tools error_types
```

`--score KEY:MIN..MAX` (ב-**`evals`**, לא `sessions`) ניתן לחזור וה-AND-combined; כל bound הוא optional (`..0.5` פירוש ≤ 0.5, `0.9..` פירוש ≥ 0.9). עד 20 score filters לכל בקשה. `evals --scores-full` הוא display flag לה-**human table בלבד**; זה מראה כל score pair במקום הראשון כמה בתוספת ספירה `+N`. זה אין השפעה תחת `--json`, אשר תמיד חוזר את ה-complete score object. כדי לקרוא **one session end-to-end**, שלב את ה-event trail עם ה-evaluation שלו:

```bash
agenteye --json events --session-id run-001 --all --limit 1000 | jq '.events[] | {ts, event_type}'
agenteye --json evals  --session-id run-001                       # its scores + status
```

### Manage (permission-gated): `keys` · `users` · `settings` · `alerts` · `incidents`

**`keys`**: API keys. ה-secret נוצר באופן מקומי, נשלח לשרת (אשר אוחסן רק hash), ו-**מוצג פעם** ב-create/regenerate; לתפוס אותו אחר כך. עם `--json` זה מופיע רק בתחום ה-`key`. Referenced by **name**.

```bash
agenteye keys list                                   # active keys first, then revoked
agenteye keys show ci-bot
agenteye keys create ci-bot --add events:read.add    # scope to what you need; prints the secret ONCE
agenteye keys create ops --permission-set standard --remove queries:run   # seed a preset, then trim
agenteye keys update ci-bot --add evaluations:read --yes
agenteye keys regenerate ci-bot --yes                # rotate the secret (the old one stops working)
agenteye keys disable ci-bot --yes                   # revoke
```

Permissions עבודה בתור `(permission-set ∪ --add) − --remove`. Tokens הם `slug:action` (לדוגמה `events:read`) או `slug:action.action` כדי להרחיב כמה ב-one resource (`events:read.add` → `events:read`, `events:add`). Presets: `read-only`, `standard`, `admin`. Human-only permissions (`keys:update`) לא יכול להיות מניח ל-key.

**`users`**: org members, referenced by **email** (UUID id גם מקובל).

```bash
agenteye users list [--active-only]
agenteye users show dev@corp.com
agenteye users create dev@corp.com --permission-set standard
agenteye users update dev@corp.com --add alerts:write --remove queries:delete   # predicts + confirms
agenteye users disable dev@corp.com --yes            # has protected/self guards
agenteye users enable dev@corp.com
```

**`settings`**: fixed registry (אתה קורא וחוזר על keys קיימים; אתה לא יכול ליצור חדשים).

```bash
agenteye settings list                               # key · value · type · updated (secrets masked)
agenteye settings schema                             # what each key accepts (type · range · description)
agenteye settings set session_ttl_secs --value 86400 --yes
```

**`alerts`**: alert definitions, referenced by **name**. `create` לוקח positional NAME בתוספת flags או full JSON body דרך `--file`.

```bash
agenteye alerts list
agenteye alerts show high-errors
agenteye alerts create high-errors --file alert.json          # NAME is required (positional)
agenteye alerts update high-errors --severity critical --yes
agenteye alerts test high-errors --yes                        # fire a test notification
agenteye alerts delete high-errors --yes
```

**`incidents`**: alert incidents, referenced by id (short ids מקובל). `show` מדפיס את ה-full activity log; קרא אותו לפני שפעלת.

```bash
agenteye incidents list --state firing                # also: acknowledged, resolved
agenteye incidents count
agenteye incidents show <id>
agenteye incidents ack <id>
agenteye incidents assign <id> you@corp.com           # assignee must be an operator
agenteye incidents resolve <id> --yes
agenteye incidents open --alert-id <id> --severity critical    # open one manually against an alert
agenteye incidents comment-add <id> "root cause: upstream 5xx"
agenteye incidents comment-list <id> ; agenteye incidents comment-delete <id> <comment-id>
agenteye incidents subscribe <id> ; agenteye incidents unsubscribe <id> ; agenteye incidents subscribers <id>
```

### Analytics & assistant: `query` · `agent`

**`query`**: saved SQL נגד ה-analytics store שלך בתוספת ad-hoc runner. Saved queries הם referenced by **name**; ה-SQL מאומת server-side (SELECT/WITH בלבד, statement timeout, row cap).

```bash
agenteye query schema [TABLE]                         # column layout of the analytics views
agenteye query run --sql "select count(*) from analytics.events"
agenteye query run errs --arg prod --limit 100        # run a saved query + a positional $1
agenteye query list ; agenteye query show errs
agenteye query create errs --sql @errs.sql --description "errored events (24h)"
agenteye query update errs --sql @errs.sql --yes ; agenteye query delete errs --yes
```

**`agent`**: מדבר לבנוי-ב-**AI assistant** (אותו read-only analyst שאתה יכול לשוחח איתו בdashboard). Chats הם referenced בקצר chat-id (prefix-resolved).

```bash
agenteye agent health                                 # is the AI assistant configured/reachable
agenteye agent models                                 # models you can pass to --model (default marked)
agenteye agent ask "which agents errored most in the last day?"    # starts a chat; prints its short id
agenteye agent ask --chat <short-id> "and which tools did they call?"   # continue that chat
agenteye agent chats ; agenteye agent show <short-id>
agenteye agent rename <short-id> --title "error triage" ; agenteye agent delete <short-id>
```

## Exit codes

| קוד | משמעות |
|---|---|
| 0 | הצלחה |
| 1 | Unexpected error (לדוגמה ה-dashboard חזר 5xx) |
| 2 | Usage error (invalid arguments, unknown command/flag, name collision) |
| 3 | לא יכול להגיע לdashboard |
| 4 | לא מחובר או session פג תוקף; הרץ `agenteye login` |
| 5 | מאומת, אך החשבון שלך חסר ההרשאה הנדרשת (ההודעה מתייגת אותה) |
| 6 | Resource המבוקש לא נמצא (לדוגמה unknown session או incident id) |

אלה עושים ל-CLI בטוח לscript: coding agent יכול branch ב-`4` כדי לבקש מחדש-להנחות אותך, או `5` כדי surface ההרשאה החסרה. ראה [CLI recipes לagents](/he/agenteye/cli-recipes) עבור exit-code-handling patterns וJSON output shapes.
