לוח הבקרה של failproofai היא יישומת ווב מקומית לעקוב אחר הפעלות סוכני AI שלך וניהול מדיניות. ראה מה עשו הסוכנים שלך בזמן שלא היית שם.

---

## הפעלת לוח הבקרה

```bash
failproofai
```

נפתח בכתובת `http://localhost:8020`.

לוח הבקרה קורא ישירות מקובץ המערכת - תיקיות הפרויקט של Claude Code וקובצי ההגדרות של failproofai. שום דבר לא נכתב לשירות רחוק.

---

## עמודים

### Projects

מפרט את כל פרויקטי Claude Code, OpenAI Codex, GitHub Copilot CLI _(beta)_, Cursor Agent _(beta)_, OpenCode _(beta)_, Pi _(beta)_, ו-Gemini CLI _(beta)_ שנמצאים במכונה שלך. פרויקטי Claude מתגלים מ-`~/.claude/projects/` (או הנתיב שמוגדר על ידי `CLAUDE_PROJECTS_PATH`); פרויקטי Codex מתגלים על ידי סריקת כל תמלול תחת `~/.codex/sessions/<YYYY>/<MM>/<DD>/*.jsonl` וקבוצה לפי `cwd` שנרשם בנתון הראשון של כל הפעלה; פרויקטי Copilot CLI מתגלים על ידי סריקת כל `~/.copilot/session-state/<sessionId>/workspace.yaml` (ניתן להגדרה דרך `COPILOT_HOME`) וקבוצה לפי שדה `cwd`; פרויקטי Cursor Agent מתגלים על ידי סריקת מטא-נתונים לפי הפעלה תחת `~/.cursor/agent-sessions/<sessionId>/` (ניתן להגדרה דרך `CURSOR_HOME`, עם `conversations/` ו-`sessions/` בדיקה כ-fallback) עבור סקלר `cwd` ב-`meta.json` / `session.json` / `workspace.yaml`; פרויקטי OpenCode מתגלים על ידי שאילתת מסד הנתונים SQLite שלו בכתובת `~/.local/share/opencode/opencode.db` דרך `opencode db --format json` (אנחנו קוראים את טבלאות `session` ו-`project` וקבוצה לפי `project_id`); פרויקטי Pi מתגלים על ידי סריקת תמליל JSONL לפי הפעלה תחת `~/.pi/agent/sessions/<encoded-cwd>/<timestamp>_<uuid>.jsonl` (ניתן להגדרה דרך `PI_SESSIONS_DIR`) והסקת ה-`cwd` מנתון הראשון של כל הפעלה; פרויקטי Gemini CLI מתגלים על ידי סריקת `~/.gemini/tmp/<basename>/chats/session-<timestamp>-<uuid-prefix>.jsonl` (ניתן להגדרה דרך `GEMINI_SESSIONS_DIR`) והחזרה של ה-cwd הקנוני מסימן טקסט `.project_root` השכן. פרויקט שהשתמש בו מספר CLIs יופיע כשורה יחידה עם כל התגים המתאימים. השתמש בתפריט **CLI** מעל הטבלה כדי לסנן לפי סוכן CLI ספציפי; כתובת ה-URL שמירה על הבחירה שלך כ-`?cli=claude|codex|copilot|cursor|opencode|pi|gemini`.

כל פרויקט מציג:
- שם הפרויקט (נגזר מנתיב התיקייה)
- תג CLI — `Claude Code` (כתום), `OpenAI Codex` (סגול), `GitHub Copilot` (כחול), `Cursor Agent` (אזמרגד), `OpenCode` (ענבר), `Pi` (ורוד), ו/או `Gemini CLI` (שמיים)
- תאריך של פעילות ההפעלה האחרונה

לחץ על פרויקט כדי לראות את הפעלותיו.

### Sessions

מפרט את כל ההפעלות בתוך פרויקט. כל הפעלה מציגה:
- מזהה הפעלה
- חותמות זמן של התחלה וסיום
- מספר קריאות כלים
- ספירת פעילות קונקציה (מדיניות שהופעלה)

השתמש בסנן טווח התאריכים וחיפוש מזהה הפעלה כדי לצמצם את הרשימה. הפעלות מחולקות לעמודים.

לחץ על הפעלה כדי לפתוח את מציג ההפעלה.

### Session viewer

מציג ההפעלה עונה על השאלה העיקרית לסוכנים אוטונומיים: מה עשה הסוכן, והאם הוא נשאר במסלול? תג CLI ליד הכותרת מציין אם ההפעלה היא תמלול של Claude Code, OpenAI Codex, GitHub Copilot CLI, Cursor Agent, OpenCode, Pi, או Gemini CLI. הוא מציג ציר זמן של כל מה שקרה בהפעלה:

- **Messages** - תגובות טקסט של Claude והנחות משתמש
- **Tool calls** - כל כלי שהעביר Claude, עם הקלט והפלט שלו
- **Policy activity** - לכל קריאת כלים, אילו מדיניויות הופעלו ואיזה החלטה הן החזירו

סרגל הסטטיסטיקה בחלק העליון מציג משך ההפעלה, סך הכל קריאות כלים, וסיכום של החלטות קונקציה (allow / deny / instruct counts).

לחץ על כפתור **Download Logs** כדי לייצא את ההפעלה. עבור הפעלות Claude Code, Codex, Copilot, Cursor, Pi, ו-Gemini תקבל את תמלול JSONL המקורי על הדיסק בצורה זהה לגמרי; עבור OpenCode (שהפעלותיו חיות ב-SQLite, לא על הדיסק) תקבל מסמך JSON המשקף את טבלאות `session` / `messages` / `parts` הבסיסיות.

### Policies

עמוד בשני לשוניות לניהול מדיניויות ובדיקת פעילות.

  - הרחב מדיניות כדי להגדיר את הפרמטרים שלה (למדיניויות שתומכות ב-`policyParams`)
  - התקן או הסר קונקציות עבור היקף נתון
  - הגדר נתיב קובץ מדיניויות מותאם אישית
  </Tab>
  <Tab title="Activity tab">
  - היסטוריה מחולקת לעמודים מלאה של כל אירוע קונקציה שהופעל בכל ההפעלות

  - כל שורה מציגה: חותם זמן, שם מדיניות, החלטה, תג CLI (כתום = Claude Code, סגול = OpenAI Codex, כחול = GitHub Copilot, אזמרגד = Cursor Agent, ענבר = OpenCode, ורוד = Pi, שמיים = Gemini CLI), שם כלי, מזהה הפעלה, והסיבה להחלטות deny/instruct

  </Tab>

## Auto-refresh

לוח הבקרה יש החלפת auto-refresh בניווט העליון. כשהוא מופעל, העמוד הנוכחי מרענן מחזוריות כדי להציג הפעלות חדשות ופעילות מדיניות כשהן מופיעות. חיוני למעקב אחרי הפעלות סוכן אוטונומיות ארוכות.

## השבתת עמודים

אם אתה זקוק רק לחלקים מסוימים של לוח הבקרה, הגדר את `FAILPROOFAI_DISABLE_PAGES` לרשימה מופרדת בפסיקים של שמות עמודים:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

ערכים תקפים: `policies`, `projects`.

## הגדרת נתיב הפרויקטים

כברירת מחדל, לוח הבקרה קורא מתיקיית הפרויקטים הסטנדרטית של Claude Code. עקוף אותה להגדרות מותאמות אישית:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## גישה מהוסט לא-localhost

כשמפעילים את לוח הבקרה ב-**dev mode** (`npm run dev`) ופונים אליה מ-hostname אחר מ-`localhost` - לדוגמה, דומיין מותאם אישית, IP מרוחק, או כתובת מנהרה - ייתכן שתראה אזהרה כמו:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

זה Next.js חוסם גישה cross-origin לא-websocket HMR (hot module reload) שלו, שהיא תכונה dev-only. כדי לאפשר את הוסט שלך, השתמש בדגל `--allowed-origins`:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

לאחד מרובים או IPs, העבר רשימה מופרדת בפסיקים:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

אתה יכול גם להגדיר את משתנה הסביבה `FAILPROOFAI_ALLOWED_DEV_ORIGINS` במקום:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
זה רק חל על dev mode. כשמפעילים `failproofai` (production mode), אין websocket HMR ואין בעיית משאב dev cross-origin.
</Note>
