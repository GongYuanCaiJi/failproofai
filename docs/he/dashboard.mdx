לוח הבקרה של failproofai היא יישומת אינטרנט מקומית לניטור הפעלות סוכני ה-AI שלך וניהול מדיניות. ראה מה עשו הסוכנים שלך בזמן שלא היית שם.

---

## הפעלת לוח הבקרה

```bash
failproofai
```

נפתח בכתובת `http://localhost:8020`.

לוח הבקרה קורא ישירות מערכת הקבצים - תיקיות Claude Code שלך וקבצי ה-failproofai config. לא נכתב כלום לשירות מרוחק.

---

## עמודים

### Projects

מפרט את כל Claude Code, OpenAI Codex, GitHub Copilot CLI _(beta)_, Cursor Agent _(beta)_, OpenCode _(beta)_, Pi _(beta)_, ו-Gemini CLI _(beta)_ פרויקטים שנמצאו במחשב שלך. פרויקטי Claude מתגלים מ-`~/.claude/projects/` (או הנתיב המוגדר על ידי `CLAUDE_PROJECTS_PATH`); פרויקטי Codex מתגלים על ידי סריקת כל תמליל תחת `~/.codex/sessions/<YYYY>/<MM>/<DD>/*.jsonl` וקבוצה לפי ה-`cwd` שנרשם בהקלטה הראשונה של כל הפעלה; פרויקטי Copilot CLI מתגלים על ידי סריקת כל `~/.copilot/session-state/<sessionId>/workspace.yaml` (ניתן להגדרה דרך `COPILOT_HOME`) וקבוצה לפי שדה ה-`cwd` שלו; פרויקטי Cursor Agent מתגלים על ידי סריקת מטא-נתונים לכל הפעלה תחת `~/.cursor/agent-sessions/<sessionId>/` (ניתן להגדרה דרך `CURSOR_HOME`, עם `conversations/` ו-`sessions/` שנבדקים כחלופות) עבור סקלר `cwd` בתוך `meta.json` / `session.json` / `workspace.yaml`; פרויקטי OpenCode מתגלים על ידי שאילתת SQLite DB שלו בכתובת `~/.local/share/opencode/opencode.db` דרך `opencode db --format json` (אנחנו קוראים את הטבלאות `session` ו-`project` וקבוצה לפי `project_id`); פרויקטי Pi מתגלים על ידי סריקת תמלילי JSONL לכל הפעלה תחת `~/.pi/agent/sessions/<encoded-cwd>/<timestamp>_<uuid>.jsonl` (ניתן להגדרה דרך `PI_SESSIONS_DIR`) ומשיכת ה-`cwd` מהקלטה הראשונה של כל הפעלה; פרויקטי Gemini CLI מתגלים על ידי סריקת `~/.gemini/tmp/<basename>/chats/session-<timestamp>-<uuid-prefix>.jsonl` (ניתן להגדרה דרך `GEMINI_SESSIONS_DIR`) וביצוע ה-cwd הקנוני מסימן הטקסט `project_root` הסמוך. פרויקט שהשתמשו בו מספר CLIs מוצג כשורה אחת עם כל התגים התואמים. השתמש בתפריט הנפתח **CLI** מעל הטבלה כדי לסנן לפי CLI סוכן ספציפי; ה-URL משמר את הבחירה שלך כ-`?cli=claude|codex|copilot|cursor|opencode|pi|gemini`.

כל פרויקט מציג:
- שם הפרויקט (נגזר מנתיב התיקייה)
- תג CLI — `Claude Code` (כתום), `OpenAI Codex` (סגול), `GitHub Copilot` (כחול), `Cursor Agent` (ירוק), `OpenCode` (ענבר), `Pi` (ורוד), ו/או `Gemini CLI` (שמיים)
- תאריך של פעילות ההפעלה האחרונה ביותר

לחץ על פרויקט כדי לראות את ההפעלות שלו.

### Sessions

מפרט את כל ההפעלות בתוך פרויקט. כל הפעלה מציגה:
- מזהה הפעלה
- חותמות זמן התחלה וסיום
- מספר קריאות כלים
- ספירת פעילות Hook (מדיניות שהופעלו)

השתמש בסנן טווח התאריכים וחיפוש מזהה הפעלה כדי לצמצם את הרשימה. הפעלות מופרדות לדפים.

לחץ על הפעלה כדי לפתוח את מציג ההפעלות.

### מציג ההפעלות

מציג ההפעלות עונה על השאלה העיקרית עבור סוכנים אוטונומיים: מה עשה הסוכן וחלפו מסלול? תג CLI ליד הכותרת מציין אם ההפעלה היא Claude Code, OpenAI Codex, GitHub Copilot CLI, Cursor Agent, OpenCode, Pi, או Gemini CLI תמליל. זה מציג ציר הזמן של כל מה שקרה בהפעלה:

- **הודעות** - תגובות טקסט של Claude והנחות למשתמשים
- **קריאות כלים** - כל כלי שהפעיל Claude, עם הקלט והפלט שלו
- **פעילות מדיניות** - לכל קריאת כלי, אילו מדיניות הופעלו ואילו החלטה הם החזירו

סרגל הנתונים בחלק העליון מציג משך הפעלה, סך הכל קריאות כלים וסיכום של החלטות hook (ספירות allow / deny / instruct).

לחץ על כפתור **Download Logs** כדי לייצא את ההפעלה. עבור Claude Code, Codex, Copilot, Cursor, Pi, ו-Gemini הפעלות אתה מקבל את תמליל JSONL המקורי בדיסק byte-for-byte; עבור OpenCode (שהפעלות שלו חיות ב-SQLite, לא בדיסק) אתה מקבל מסמך JSON המשקף את הטבלאות `session` / `messages` / `parts` הבסיסיות.

### Policies

דף שתי-כרטיסיות לניהול מדיניות וביקורת פעילות.

  - הרחב מדיניות כדי להגדיר את הפרמטרים שלה (עבור מדיניות התומכות ב-`policyParams`)
  - התקן או הסר hook עבור מתחם מסוים
  - הגדר נתיב קובץ מדיניות מותאם אישית

  <Tab title="כרטיסית Activity">

## רענון אוטומטי

לוח הבקרה כולל כפתור toggle רענון אוטומטי בניווט העליון. כאשר מופעל, העמוד הנוכחי רוענן כל מעת בכדי להציג הפעלות חדשות ופעילות מדיניות כשהם מופיעים. חיוני לניטור הפעלות סוכנים אוטונומיים שפועלים זמן רב.

## השבתת עמודים

אם אתה צריך רק חלקים מסוימים של לוח הבקרה, הגדר את `FAILPROOFAI_DISABLE_PAGES` לרשימה מופרדת בפסיקים של שמות עמודים:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

ערכים תקפים: `policies`, `projects`.

---

## הגדרת נתיב הפרויקטים

כברירת מחדל, לוח הבקרה קורא מדירקטוריון פרויקטי Claude Code הסטנדרטי. עקוף אותו עבור הגדרות מותאמות אישית:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## גישה מנתחנו שאינה localhost

כאשר מפעילים את לוח הבקרה ב-**dev mode** (`npm run dev`) ונגישים אליו מ-hostname אחר מ-`localhost` - לדוגמה, דומיין מותאם אישית, IP מרוחק או כתובת URL מנוקבת - ייתכן שתראה התרעה כמו:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

זהו Next.js חוסם גישה cross-origin ל-HMR (רענון חם של מודול) websocket שלו, שהוא תכונה dev-only. כדי לאפשר את הנתחנו שלך, השתמש בדגל `--allowed-origins`:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

עבור מספר נתחנו או IP, העבר רשימה מופרדת בפסיקים:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

אתה יכול גם להגדיר את משתנה הסביבה `FAILPROOFAI_ALLOWED_DEV_ORIGINS` במקום:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
זה חל רק על dev mode. כאשר מפעילים `failproofai` (production mode), אין HMR websocket ואין בעיית משאב dev cross-origin.
</Note>
