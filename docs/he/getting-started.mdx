## דרישות

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (אופציונלי - נדרש רק לבניה מקוד מקור)

## התקנה

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

## התחלה מהירה

<Steps>
  <Step title="הפעל מדיניות">
    מדיניות היא כללים שרצים לפני ואחרי כל קריאת כלי של סוכן. הם תופסים פקודות הרסניות, דליפות סודות, ומצבי כשל אחרים לפני שהם גורמים לנזק.

    ```bash
    failproofai policies --install
    ```

    זה כותב ערכי hook לתוך ה-CLI של הסוכן המותקן שלך (Claude Code's `~/.claude/settings.json`, OpenAI Codex's `~/.codex/hooks.json`, GitHub Copilot CLI's `~/.copilot/hooks/failproofai.json`, Cursor Agent's `~/.cursor/hooks.json`, OpenCode's generated plugin shim ב-`~/.config/opencode/plugins/failproofai.mjs` בתוספת ערך רישום ב-`~/.config/opencode/opencode.json`'s `plugin` array, Pi's `~/.pi/agent/settings.json`, או Gemini CLI's `~/.gemini/settings.json`). כאשר יותר מאחד קיים תתבקע בחירה; תמוך `--cli claude codex copilot cursor opencode pi gemini` (כל תת-קבוצה) כדי לדלג על הבקשה.

    GitHub Copilot CLI, Cursor Agent, OpenCode, Pi, ו-Gemini CLI support הם **beta** — התקן עם `--cli copilot`, `--cli cursor`, `--cli opencode`, `--cli pi`, או `--cli gemini`.

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install --cli copilot --scope project
    failproofai policies --install --cli cursor --scope project
    failproofai policies --install --cli opencode --scope project
    failproofai policies --install --cli pi --scope project
    failproofai policies --install --cli gemini --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="אמת">
    ```bash
    failproofai policies
    ```

    מציג כל מדיניות, האם היא מופעלת, וכל פרמטרים שהוגדרו.
  </Step>
  <Step title="הפעל את לוח הבקרה">
    ```bash
    failproofai
    ```

    פותח לוח בקרה מקומי ב-`http://localhost:8020` שבו אתה יכול לעיין בהפעלות, בדוק קריאות כלים, וניהול מדיניות.
  </Step>
  <Step title="הרץ את הסוכן שלך">
    התחל את Claude Code כרגיל. אם הסוכן מנסה משהו מסוכן, failproofai יירט אותו באופן אוטומטי. השאר אותו פועל ללא השגחה ובדוק מה קרה בלוח הבקרה.
  </Step>
</Steps>

## כיצד מדיניות פועלת

בכל פעם שסוכן מריץ כלי, Claude Code קורא ל-failproofai כתהליך משנה:

```text
Claude Code  →  failproofai --hook PreToolUse  →  reads stdin JSON
                                                 evaluates policies
                                                 writes decision to stdout
```

כל מדיניות מחזירה אחת משלוש החלטות:

- **allow** - הסוכן ממשיך בדרך רגילה
- **deny** - הפעולה חסומה, לסוכן מודיעים למה
- **instruct** - הקשר נוסף מתווסף להנחיה של הסוכן

<Note>
מדיניות רצות בתהליך המקומי שלך. כלום לא נשלח לשירות מרחוק.
</Note>

## הגדר מדיניות צוות עם מדיניות מבוססות קונוונציה

הדרך המהירה ביותר לבסס תקנים איכות בצוות שלך היא הקונוונציה `.failproofai/policies/`. זרוק קבצי מדיניות לתיקייה זו והם נטענים באופן אוטומטי — ללא דגלים, ללא שינויי תצורה, ללא פקודות התקנה.

<Steps>
  <Step title="צור את תיקיית המדיניות">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="הוסף קבצי מדיניות">
    העתק את הדוגמאות ההתחלה או כתוב את שלך:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    או צור אחד חדש:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Run tests before committing.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="בצע למאגר">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```

    כל חברת צוות שיש לה failproofai מותקן קוטף את המדיניות הללו באופן אוטומטי. אין צורך בהגדרה לכל מפתח.
  </Step>
</Steps>

<Tip>
בצע `.failproofai/policies/` לריפו שלך כדי שכל הצוות יחזיק באותם תקנים. כאשר הצוות שלך גילה מצבי כשל חדשים, הוסף מדיניות ודחוף — כולם מקבלים את העדכון ב-`git pull` הבא שלהם. עם הזמן המדיניות הללו הופכות לתקן איכות חי המשתפר כל הזמן.
</Tip>

## אחסון נתונים

כל התצורה והיומנים נשארים במכונה שלך:

| נתיב | מה זה מאחסן |
|------|------------|
| `~/.failproofai/policies-config.json` | תצורת מדיניות גלובלית |
| `~/.failproofai/hook-activity.jsonl` | היסטוריית ביצוע hook |
| `~/.failproofai/hook.log` | יומן ניפוי באגים לשגיאות hook מותאמות |
| `.failproofai/policies-config.json` | תצורה לכל פרויקט (מחויבת) |
| `.failproofai/policies-config.local.json` | כללים אישיים (gitignored) |

---

## הסרה

```bash
failproofai policies --uninstall
```

מסיר ערכי hook מ-`~/.claude/settings.json`. קבצי תצורה ב-`~/.failproofai/` נשמרים.
