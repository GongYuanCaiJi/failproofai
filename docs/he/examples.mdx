## הגדרת hooks עבור Claude Code

Failproof AI משתלבת עם Claude Code דרך [מערכת ה-hooks שלה](https://docs.anthropic.com/en/docs/claude-code/hooks). כאשר אתה מריץ `failproofai policies --install`, זה רושם פקודות hook ב-`settings.json` של Claude Code שמופעלות בכל קריאת כלי.

<Steps>
  <Step title="התקן את failproofai">
    ```bash
    npm install -g failproofai
    ```
  </Step>
  <Step title="הפעל את כל המדיניות המובנית">
    ```bash
    failproofai policies --install
    ```
  </Step>
  <Step title="אמת שה-hooks רשומים">
    ```bash
    cat ~/.claude/settings.json | grep failproofai
    ```

    אתה אמור לראות רשומות hook עבור אירועי `PreToolUse`, `PostToolUse`, `Notification` ו-`Stop`.
  </Step>
  <Step title="הרץ את Claude Code">
    ```bash
    claude
    ```

    המדיניויות מופעלות כעת אוטומטית בכל קריאת כלי. נסה לבקש מ-Claude להריץ `sudo rm -rf /` - זה יחסום.
  </Step>
</Steps>

## הגדרת hooks עבור Agents SDK

אם אתה בונה עם [Agents SDK](https://docs.anthropic.com/en/docs/agents-sdk), אתה יכול להשתמש באותה מערכת hook באופן תכנותי.

<Steps>
  <Step title="התקן את failproofai בפרויקט שלך">
    ```bash
    npm install failproofai
    ```
  </Step>
  <Step title="הגדר hooks בסוכן שלך">
    העבר פקודות hook כאשר יוצר את תהליך הסוכן שלך. ה-hooks מופעלים באותו אופן כמו ב-Claude Code - דרך JSON של stdin/stdout:

    ```bash
    failproofai --hook PreToolUse   # called before each tool
    failproofai --hook PostToolUse  # called after each tool
    ```
  </Step>
  <Step title="כתוב מדיניות מותאמת אישית עבור הסוכן שלך">
    ```javascript
    import { customPolicies, allow, deny } from "failproofai";

    customPolicies.add({
      name: "limit-to-project-dir",
      description: "Keep the agent inside the project directory",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        const path = String(ctx.toolInput?.file_path ?? "");
        if (path.startsWith("/") && !path.startsWith(ctx.session?.cwd ?? "")) {
          return deny("Agent is restricted to the project directory");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="התקן את המדיניות המותאמת אישית">
    ```bash
    failproofai policies --install --custom ./my-agent-policies.js
    ```
  </Step>
</Steps>

## חסום פקודות הרסניות

ההגדרה הנפוצה ביותר - מנע מסוכנים מלהיגרום לנזק בלתי הפיך.

```bash
failproofai policies --install block-sudo block-rm-rf block-force-push block-curl-pipe-sh
```

מה זה עושה:
- `block-sudo` - חוסם את כל פקודות ה-`sudo`
- `block-rm-rf` - חוסם מחיקת קבצים רקורסיבית
- `block-force-push` - חוסם את `git push --force`
- `block-curl-pipe-sh` - חוסם העברת סקריפטים מרחוקים ל-shell

## מנע דליפת סודות

עצור סוכנים מלראות או להדליף אישורים בפלט הכלי.

```bash
failproofai policies --install sanitize-api-keys sanitize-jwt sanitize-connection-strings sanitize-bearer-tokens
```

אלה מופעלים על `PostToolUse` - אחרי שכלי מופעל, הם מנקים את הפלט לפני שהסוכן רואה אותו.

## קבל התראות Slack כאשר סוכנים זקוקים לתשומת לב

השתמש ב-notification hook כדי להעביר התראות עצירה ל-Slack.

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "slack-on-idle",
  description: "Alert Slack when the agent is waiting for input",
  match: { events: ["Notification"] },
  fn: async (ctx) => {
    const webhookUrl = process.env.SLACK_WEBHOOK_URL;
    if (!webhookUrl) return allow();

    const message = String(ctx.payload?.message ?? "Agent is waiting");
    const project = ctx.session?.cwd ?? "unknown";

    try {
      await fetch(webhookUrl, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          text: `*${message}*\nProject: \`${project}\``,
        }),
        signal: AbortSignal.timeout(5000),
      });
    } catch {
      // never block the agent if Slack is unreachable
    }

    return allow();
  },
});
```

התקן זאת:

```bash
SLACK_WEBHOOK_URL=https://hooks.slack.com/... failproofai policies --install --custom ./slack-alerts.js
```

## שמור סוכנים על ענף

מנע מסוכנים להחליף ענפים או לדחוף לענפים מוגנים.

```javascript
import { customPolicies, allow, deny } from "failproofai";

customPolicies.add({
  name: "stay-on-branch",
  description: "Prevent the agent from checking out other branches",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+checkout\s+(?!-b)/.test(cmd)) {
      return deny("Stay on the current branch. Create a new branch with -b if needed.");
    }
    return allow();
  },
});
```

## דרוש בדיקות לפני commit

הזכר לסוכנים להריץ בדיקות לפני commit.

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "test-before-commit",
  description: "Remind the agent to run tests before committing",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+commit/.test(cmd)) {
      return instruct("Run tests before committing. Use `npm test` or `bun test` first.");
    }
    return allow();
  },
});
```

## נעול מאגר ייצור

commit קונפיגורציה ברמת פרויקט כדי שכל מפתח בצוות שלך יקבל את אותן המדיניויות.

צור `.failproofai/policies-config.json` במאגר שלך:

```json
{
  "enabledPolicies": [
    "block-sudo",
    "block-rm-rf",
    "block-force-push",
    "block-push-master",
    "block-env-files",
    "sanitize-api-keys",
    "sanitize-jwt"
  ],
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "release", "production"]
    }
  }
}
```

ואז commit זאת:

```bash
git add .failproofai/policies-config.json
git commit -m "Add failproofai team policies"
```

כל חברי הצוות שיש להם failproofai מותקן יאספו אוטומטית את הכללים הללו.

## בנה תקן איכות בקנה מידה ארגוני עם מדיניות קונבנציה

ההגדרה בעלת ההשפעה הגבוהה ביותר: commit `.failproofai/policies/` למאגר שלך עם מדיניויות המותאמות לפרויקט שלך. כל חברי הצוות מקבלים אותם אוטומטית — ללא פקודות התקנה, ללא שינויי תצורה.

<Steps>
  <Step title="צור את הספרייה והוסף מדיניויות">
    ```bash
    mkdir -p .failproofai/policies
    ```

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    // Enforce your team's preferred package manager
    // (or enable the built-in prefer-package-manager policy instead)
    customPolicies.add({
      name: "enforce-bun",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        const cmd = String(ctx.toolInput?.command ?? "");
        if (/\bnpm\b/.test(cmd)) return deny("Use bun instead of npm.");
        return allow();
      },
    });

    // Remind the agent to run tests before committing
    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Run tests before committing.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Commit ל-git">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```
  </Step>
  <Step title="המשך להשתפר">
    כאשר הצוות שלך נתקל במצבי כשל חדשים, הוסף מדיניויות ודחוף. כולם מקבלים את העדכון ב-`git pull` הבא שלהם. המדיניויות הללו הופכות לתקן איכות חי המתגדל עם הצוות שלך.
  </Step>
</Steps>
