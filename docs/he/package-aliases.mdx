## חבילה רשמית

החבילה npm הקנונית היא **`failproofai`**:

```bash
npm install -g failproofai
# or
bun add -g failproofai
```

---

## מדוע אנחנו בעלים של שמות הכינויים

Typosquatting היא התקפת supply-chain נפוצה שבה שחקן זדוני רושם שם חבילה שנמצא בהקשה אחת מחבילה פופולרית. משתמשים חסרי ספק שטועים בהקלדת פקודת ההתקנה בסופו של דבר מריצים קוד שנשלט על ידי התוקף עם גישה מלאה למערכת - בדיוק סוג ההאיום שFailproof AI מעוצבת להגן עליו.

כדי למנוע משטח זה, **אנחנו בעלי כל ההנמכות וגרסאות עיצוב נפוצות** של `failproofai` ב-npm. אף אחד מהשמות הללו לא יכול להיות רשום על ידי צד שלישי. כל אחד מהם הוא פרוקסי דק שמתקין ומעביר לחבילת `failproofai` האמיתית.

## כיצד הכינויים עובדים

כל חבילת כינוי:

1. מציינת את `failproofai` כתלות - כך החבילה האמיתית (כולל הגדרת ה-`postinstall` hook שלה) מתבצעת בעת ההתקנה
2. חושפת בינארי התואם לשם שלה (לדוגמה `failprof-ai`) שמעביר כל טיעונים לבינארי `failproofai`

ה-proxy הוא סקריפט Node של שתי שורות; אין לוגיקה, אין שיחות רשת, ואין אוסף נתונים מעבר לזה שעושה `failproofai` עצמה.
