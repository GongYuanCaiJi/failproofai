## חבילה רשמית

החבילה npm הקנונית היא **`failproofai`**:

```bash
npm install -g failproofai
# or
bun add -g failproofai
```

---

## למה אנחנו בעלים של שמות הכינויים

Typosquatting היא התקפת שרשרת אספקה נפוצה שבה שחקן זדוני רושם שם חבילה שהוא בדיוק מכת אחת משם חבילה פופולרי. משתמשים שלא חוזרים בהם שגם בפקודת ההתקנה מסתיימים בהרצת קוד שנשלט על ידי התוקף עם גישה מלאה למערכת - בדיוק סוג האיום שFailproof AI מעוצב להגנה נגדו.

כדי לחסל משטח זה, **אנחנו מקדימים בעלות על כל הטעויות הכתיב והווריאציות עיצוב נפוצות** של `failproofai` ב-npm. שום אחד מהשמות האלה לא יכול להיות רשום על ידי צד שלישי. כל אחד מהם הוא פרוקסי דק שמתקין ומעביר לחבילת ה-`failproofai` האמיתית.

| `faliproof-ai` | ✅ פורסם |

> **למה בהמתנה?** מדיניות מניעת ספאם של npm חוסמת שמות שמתנרמלים לאותו מחרוזת כחבילה קיימת לאחר הסרת סימני פיסוק וביצוע בדיקות דמיון. יצרנו קשר עם תמיכת npm כדי להסתיר שמות אלה למטרות אנטי-squat. הם יופעלו ברגע שיאושרו.

אתה יכול לאמת שכל כינוי שפורסם הוא בבעלותנו:

```bash
npm info failproof
# חפש: "ExosphereHost Inc." בשדה המתוחזקים
```

## כיצד הכינויים פועלים

כל חבילת כינוי:

1. מופעלת `failproofai` כתלות - כך שהחבילה האמיתית מותקנת והבינארי שלה הופך לזמין
2. חושפת בינארי התואם לשם שלה (למשל `failprof-ai`) שמעביר את כל הארגומנטים לבינארי `failproofai`

הפרוקסי הוא סקריפט Node בשתי שורות; אין לוגיקה, אין קריאות רשת, ואין איסוף נתונים מעבר למה ש-`failproofai` עצמו עושה.
