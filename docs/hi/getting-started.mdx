## त्वरित शुरुआत

<Steps>
  <Step title="Policies सक्षम करें">
    Policies ऐसे नियम हैं जो हर agent टूल कॉल से पहले और बाद में चलते हैं। वे विनाशकारी कमांड, गुप्त जानकारी का रिसाव, और अन्य विफलता के तरीकों को नुकसान पहुंचाने से पहले पकड़ते हैं।

    ```bash
    failproofai policies --install
    ```

    यह आपके इंस्टॉल किए गए agent CLIs में hook entries लिखता है (Claude Code का `~/.claude/settings.json`, OpenAI Codex का `~/.codex/hooks.json`, GitHub Copilot CLI का `~/.copilot/hooks/failproofai.json`, Cursor Agent का `~/.cursor/hooks.json`, OpenCode का `~/.config/opencode/plugins/failproofai.mjs` पर जनरेट की गई plugin shim प्लस `~/.config/opencode/opencode.json` के `plugin` array में एक registration entry, Pi का `~/.pi/agent/settings.json`, या Gemini CLI का `~/.gemini/settings.json`)। जब एक से अधिक मौजूद हों तो आपको prompt दिया जाएगा; `--cli claude codex copilot cursor opencode pi gemini` (कोई भी subset) पास करें prompt को छोड़ने के लिए।

    GitHub Copilot CLI, Cursor Agent, OpenCode, Pi, और Gemini CLI समर्थन **beta** में हैं — `--cli copilot`, `--cli cursor`, `--cli opencode`, `--cli pi`, या `--cli gemini` के साथ इंस्टॉल करें।

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install --cli copilot --scope project
    failproofai policies --install --cli cursor --scope project
    failproofai policies --install --cli opencode --scope project
    failproofai policies --install --cli pi --scope project
    failproofai policies --install --cli gemini --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="सत्यापित करें">
    ```bash
    failproofai policies
    ```

    हर policy, चाहे वह सक्षम है या नहीं, और कोई भी कॉन्फ़िगर किए गए पैरामीटर दिखाता है।
  </Step>
  <Step title="Dashboard लॉन्च करें">
    ```bash
    failproofai
    ```

    `http://localhost:8020` पर एक स्थानीय dashboard खोलता है जहां आप sessions को browse कर सकते हैं, tool calls को inspect कर सकते हैं, और policies को प्रबंधित कर सकते हैं।
  </Step>
  <Step title="अपना agent चलाएं">
    Claude Code को सामान्य तरीके से शुरू करें। यदि agent कुछ जोखिम भरा करने की कोशिश करता है, तो failproofai स्वचालित रूप से इसे intercept करता है। इसे unattended छोड़ें और dashboard में देखें कि क्या हुआ।
  </Step>
</Steps>

## Policies कैसे काम करती हैं

हर बार जब कोई agent एक tool चलाता है, Claude Code failproofai को subprocess के रूप में कॉल करता है:

```text
Claude Code  →  failproofai --hook PreToolUse  →  stdin JSON को पढ़ता है
                                                 policies का मूल्यांकन करता है
                                                 decision को stdout में लिखता है
```

प्रत्येक policy तीन में से एक decision रिटर्न करती है:

- **allow** - agent सामान्य रूप से आगे बढ़ता है
- **deny** - action को block किया जाता है, agent को बताया जाता है क्यों
- **instruct** - agent के prompt में अतिरिक्त context जोड़ा जाता है

<Note>
Policies आपकी स्थानीय प्रक्रिया में चलती हैं। कुछ भी remote service को नहीं भेजा जाता है।
</Note>

## Convention-based policies के साथ team policies सेट अप करें

आपकी team में गुणवत्ता के मानकों को स्थापित करने का सबसे तेज़ तरीका `.failproofai/policies/` convention है। इस directory में policy files डालें और वे स्वचालित रूप से लोड हो जाते हैं — कोई flags नहीं, कोई config परिवर्तन नहीं, कोई install commands नहीं।

<Steps>
  <Step title="Policies directory बनाएं">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="Policy files जोड़ें">
    स्टार्टर उदाहरणों को कॉपी करें या अपने स्वयं के लिखें:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    या एक नई बनाएं:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Run tests before committing.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Git में commit करें">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```

    हर team member जिसके पास failproofai इंस्टॉल है, ये policies स्वचालित रूप से उठाता है। कोई प्रति-developer setup की आवश्यकता नहीं है।
  </Step>
</Steps>

<Tip>
`.failproofai/policies/` को अपने repo में commit करें ताकि पूरी team एक ही मानकों को साझा करे। जैसे-जैसे आपकी team नई विफलताओं की खोज करती है, policies जोड़ें और push करें — हर किसी को अपने अगले `git pull` पर अपडेट मिलता है। समय के साथ ये policies एक जीवंत गुणवत्ता मानक बन जाती हैं जो लगातार सुधार करते रहते हैं।
</Tip>

## डेटा संग्रहण

सभी कॉन्फ़िगरेशन और logs आपकी मशीन पर रहते हैं:

| Path | यह क्या स्टोर करता है |
|------|----------------|
| `~/.failproofai/policies-config.json` | Global policy config |
| `~/.failproofai/hook-activity.jsonl` | Hook execution history |
| `~/.failproofai/hook.log` | Custom hook errors के लिए debug log |
| `.failproofai/policies-config.json` | Per-project config (committed) |
| `.failproofai/policies-config.local.json` | Personal overrides (gitignored) |

---

## अनइंस्टॉल करना

```bash
failproofai policies --uninstall
```

`~/.claude/settings.json` से hook entries को हटाता है। `~/.failproofai/` में config files को रखा जाता है।
