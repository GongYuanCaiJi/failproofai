आपकी संपूर्ण Failproof AI Observability डिप्लॉयमेंट, बस एक कमांड दूर। प्रोडक्शन की जांच करें, API की बनाएं, या घटना को स्वीकार करें — सब कुछ अपने टर्मिनल से ही करें। फिर इसे CI में स्क्रिप्ट करें, या किसी कोडिंग एजेंट को सादी अंग्रेजी में निर्देश दें।

```bash
pipx install agenteye
agenteye login --email you@example.com      # a 6-digit code lands in your inbox
agenteye --json sessions --since 24h         # every agent run from the last day, newest first
```

*`agenteye` CLI आपके डैशबोर्ड से बात करता है। यह कलेक्टर से अलग टूल है, जो सर्वर को इवेंट भेजता है।*

## आपकी पूरी डिप्लॉयमेंट, एक कमांड में

टैब के बीच भटकना बंद करें और जल्दी सवाल का जवाब पाएं। `agenteye` CLI आपके डेटा को पढ़ता है और एक ही बाइनरी से आपके संगठन को प्रबंधित करता है। जो जांच पहले डैशबोर्ड में क्लिक करने से होती थी, वह अब एक लाइन में हो जाती है — जिसे आप फिर से चला सकते हैं, एलियास बना सकते हैं, या runbook में पेस्ट कर सकते हैं। आपको चार इंटरफेस मिलते हैं:

- **अपने डेटा को पढ़ें:** `sessions`, `events`, `evals`, और `errors`, जिन्हें समय, एजेंट और वातावरण से फ़िल्टर करें।
- **अपने संगठन को प्रबंधित करें:** `keys`, `users`, `settings`, `alerts`, और `incidents`।
- **विश्लेषण चलाएं:** सेव किए हुए SQL और आपके इवेंट डेटा पर ad-hoc `query` रनर।
- **सहायक से पूछें:** `agent ask` वही read-only विश्लेषक तक पहुंचता है जिससे आप डैशबोर्ड में बात करते हैं।

`pipx` से एक बार इंस्टॉल करें, ईमेल किए गए 6-अंकीय कोड से साइन इन करें, और आप तैयार हैं। सत्र लगभग एक दिन तक चलता है; समय समाप्त होने पर `agenteye login` फिर से चलाएं। इसे प्रोडक्शन की जांच करने, कुंजी प्रदान करने, या एक विफल घटना को वर्गीकृत करने के लिए उपयोग करें — सब कुछ बिना ब्राउजर खोले:

```bash
agenteye errors --since 24h --aggregate      # what is breaking, grouped by error type
agenteye incidents list --state firing       # what is on fire right now
agenteye keys create ci --add events:add     # a key that can only push events, secret shown once
```

एक आदत जानने के लिए: वैश्विक विकल्प जैसे `--json` कमांड से पहले जाते हैं। `agenteye --json sessions` सही है; `agenteye sessions --json` नहीं है।

## इसे स्क्रिप्ट करें, CI में जोड़ें

हर कमांड `--json` लेता है, और यह सबकुछ बदल देता है। स्वच्छ JSON stdout पर जाता है जबकि मानवीय स्थिति और चेतावनियां stderr पर जाती हैं, इसलिए `--json` कैप्चर `jq` में सीधे पाइप होता है — कोई भी लाइन हटाने के लिए नहीं। यही कारण है कि CLI टर्मिनल प्रॉम्प्ट पर आपके लिए और कोडिंग एजेंट के लिए आउटपुट को पार्स करने के लिए समान रूप से अच्छा है:

```bash
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'
```

यह बिना निरीक्षण के चलने के लिए बनाया गया है। पुष्टि संकेत स्वचालित रूप से छोड़ दिए जाते हैं जब कोई टर्मिनल संलग्न नहीं होता, इसलिए पाइपलाइन में कुछ भी लटकता नहीं है, और हर कमांड एक सार्थक exit code देता है: `0` सफल, `4` लॉगिन नहीं किया गया, `5` अनुमति नहीं है (संदेश इसे नाम देता है, उदाहरण के लिए `alerts:write`), `3` डैशबोर्ड unreachable। एक स्क्रिप्ट `4` पर पुनः प्रमाणीकरण करने या `5` पर आपको यह बताने के लिए शाखा कर सकता है कि एक admin से क्या मांगना है, अंधे विफल होने के बजाय।

## एक कोडिंग एजेंट को सादी अंग्रेजी में ड्राइव करने दें

इससे भी बेहतर, आपको इन सभी flags को याद नहीं रखना चाहिए। **CLI skill** एक छोटा Agent Skill फोल्डर है जिसे `agenteye-cli` कहा जाता है जो Claude Code या Codex जैसे कोडिंग एजेंट को सादी-अंग्रेजी अनुरोधों से CLI चलाना सिखाता है। पूछें "क्या आज कुछ टूटा है?" और एजेंट कमांड चुनता है, इसे आपके रूप में चलाता है, और गद्य में उत्तर देता है।

Claude Code के लिए, `agenteye-cli` फोल्डर को `~/.claude/skills/` में ड्रॉप करें और यह स्वचालित रूप से खोजा जाता है। Failproof AI Observability फोल्डर प्रदान करता है; कुछ भी अतिरिक्त इंस्टॉल करने के लिए नहीं है, क्योंकि यह केवल CLI को चलाता है जो आप पहले से इंस्टॉल कर चुके हैं। पहले स्वयं लॉगिन करें: skill ईमेल किए गए कोड लॉगिन को पूरा नहीं कर सकता।

क्योंकि एजेंट आपके रूप में CLI चलाता है, यह सब कुछ कर सकता है जो आपकी लॉगिन अनुमति देती है, पढ़ना और लिखना दोनों: कुंजियां बनाएं, सेटिंग्स बदलें, घटनाओं को हल करें। CLI का "क्या आप सुनिश्चित हैं?" संकेत एजेंट के लिए नहीं आता, इसलिए skill ठीक कमांड बताने और किसी भी परिवर्तन से पहले आपकी ठीक होने का इंतजार करने के लिए लिखा गया है। आप पुष्टि चरण हैं।

```text
you    Why did session run-001 fail?

agent  Running: agenteye --json events --session-id run-001 --all
       The checkout-agent hit a TimeoutError on its third tool call.
```

पढ़ना तुरंत रहता है, और हर लेखन आपके लिए रुकता है:

```text
you    Give CI a key that can only push events.

agent  This creates an API key. I'll run:
         agenteye keys create ci --add events:add
       OK to proceed?

you    yes

agent  Done. Key "ci" created with events:add only. The secret is shown once, so store it now.
```
