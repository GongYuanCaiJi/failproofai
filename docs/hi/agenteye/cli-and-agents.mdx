आपकी पूरी FailproofAI Observability डिप्लॉयमेंट, एक कमांड की दूरी पर। प्रोडक्शन चेक करें, API की काटें, या अपने टर्मिनल को छोड़े बिना एक घटना को स्वीकार करें, फिर इसे CI में स्क्रिप्ट करें, या एक कोडिंग एजेंट को सादी अंग्रेजी में ऐसा करने दें।

```bash
pipx install agenteye
agenteye login --email you@example.com      # a 6-digit code lands in your inbox
agenteye --json sessions --since 24h         # every agent run from the last day, newest first
```

*`agenteye` CLI आपके डैशबोर्ड से बात करता है। यह कलेक्टर से एक अलग टूल है, जो डैशबोर्ड को इवेंट भेजता है।*

## आपकी पूरी डिप्लॉयमेंट, एक कमांड की दूरी पर

त्वरित प्रश्न का उत्तर देने के लिए टैब के बीच कूदना बंद करें। `agenteye` CLI आपका डेटा पढ़ता है और एक बाइनरी से अपने संगठन का प्रबंधन करता है, इसलिए एक जांच जो पहले डैशबोर्ड के माध्यम से क्लिक करने का मतलब था, अब एक लाइन बन जाती है जिसे आप फिर से चला सकते हैं, उपनाम दे सकते हैं, या रनबुक में पेस्ट कर सकते हैं। आपको चार इंटरफेस मिलते हैं:

- **अपना डेटा पढ़ें:** `sessions`, `events`, `evals`, और `errors`, समय, एजेंट और वातावरण द्वारा फ़िल्टर किए गए।
- **अपने संगठन का प्रबंधन करें:** `keys`, `users`, `settings`, `alerts`, और `incidents`।
- **विश्लेषण चलाएं:** सहेजा गया SQL प्लस आपके इवेंट डेटा पर एक तदर्थ `query` रनर।
- **असिस्टेंट से पूछें:** `agent ask` डैशबोर्ड में आपके साथ चैट करने वाले समान केवल-पढ़ने वाले विश्लेषक तक पहुंचता है।

इसे `pipx` के साथ एक बार इंस्टॉल करें, एक ईमेल किए गए 6-अंकीय कोड के साथ साइन इन करें, और आप तैयार हैं। सत्र लगभग एक दिन तक रहता है; जब यह समाप्त हो तो `agenteye login` को फिर से चलाएं। इसका उपयोग प्रोडक्शन को स्पॉट-चेक करने, एक की प्रदान करने, या एक फायरिंग घटना को ट्रिएज करने के लिए करें, सभी ब्राउजर खोले बिना:

```bash
agenteye errors --since 24h --aggregate      # what is breaking, grouped by error type
agenteye incidents list --state firing       # what is on fire right now
agenteye keys create ci --add events:add     # a key that can only push events, secret shown once
```

जानने वाली एक आदत: वैश्विक विकल्प जैसे `--json` कमांड से पहले जाते हैं। `agenteye --json sessions` सही है; `agenteye sessions --json` नहीं है।

## इसे स्क्रिप्ट करें, इसे CI में वायर करें

प्रत्येक कमांड `--json` लेता है, और यह सब कुछ बदल देता है। स्वच्छ JSON stdout पर जाता है जबकि मानव स्थिति और चेतावनियां stderr पर जाती हैं, इसलिए एक `--json` कैप्चर सीधे `jq` में पाइप करता है कोई आवश्यक लाइन के बिना। यही वह है जो CLI को एक प्रॉम्प्ट पर आपके लिए और एक कोडिंग एजेंट के लिए आउटपुट पार्सिंग के लिए समान रूप से अच्छा बनाता है:

```bash
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'
```

यह बिना निरीक्षण के चलाने के लिए बनाया गया है। पुष्टि प्रॉम्प्ट स्वचालित रूप से छोड़ देते हैं जब कोई टर्मिनल संलग्न नहीं होता है, इसलिए कुछ भी पाइपलाइन में हैंग नहीं होता है, और प्रत्येक कमांड एक अर्थपूर्ण निकास कोड लौटाता है: `0` सफलता, `4` लॉगिन नहीं किया गया, `5` एक अनुमति गायब है (संदेश इसका नाम देता है, उदाहरण के लिए `alerts:write`), `3` डैशबोर्ड अनुपलब्ध। एक स्क्रिप्ट `4` पर पुन: प्रमाणित करने के लिए या `5` पर आपको बिल्कुल क्या पूछने के लिए बताने के लिए शाखा कर सकता है, एक व्यवस्थापक से अंधे विफल होने के बजाय।

## एक कोडिंग एजेंट को इसे सादी अंग्रेजी में चलाने दें

बेहतर अभी भी, आपको इन सभी फ़्लैग्स को याद रखने की आवश्यकता नहीं होनी चाहिए। **CLI कौशल** एक छोटी एजेंट कौशल फ़ोल्डर है जिसका नाम `agenteye-cli` है जो Claude Code या Codex जैसे कोडिंग एजेंट को सादी-अंग्रेजी अनुरोधों से CLI चलाना सिखाता है। "क्या आज कुछ टूटा है?" पूछें और एजेंट कमांड चुनता है, इसे आपके रूप में चलाता है, और गद्य में उत्तर देता है।

Claude Code के लिए, `agenteye-cli` फ़ोल्डर को `~/.claude/skills/` में ड्रॉप करें और इसे स्वचालित रूप से खोजा जाता है। FailproofAI Observability फ़ोल्डर प्रदान करता है; इंस्टॉल करने के लिए कुछ अतिरिक्त नहीं है, क्योंकि यह केवल CLI को चलाता है जिसे आपने पहले से इंस्टॉल किया है। पहले स्वयं लॉगिन करें: कौशल ईमेल-कोड लॉगिन को आपके लिए पूरा नहीं कर सकता।

क्योंकि एजेंट आपके रूप में CLI चलाता है, यह सब कुछ कर सकता है जो आपका लॉगिन अनुमति देता है, पढ़ता और लिखता है: कुंजियां बनाएं, सेटिंग्स बदलें, घटनाओं को हल करें। CLI का "क्या आप सुनिश्चित हैं?" प्रॉम्प्ट एक एजेंट के लिए नहीं फायर करता है, इसलिए कौशल को सटीक कमांड बताने और कोई परिवर्तन से पहले आपकी ठीक है प्रतीक्षा करने के लिए लिखा गया है। आप पुष्टि चरण हैं।

```text
you    Why did session run-001 fail?

agent  Running: agenteye --json events --session-id run-001 --all
       The checkout-agent hit a TimeoutError on its third tool call.
```

पढ़ता तुरंत रहता है, और हर लिखना आपके लिए रुकता है:

```text
you    Give CI a key that can only push events.

agent  This creates an API key. I'll run:
         agenteye keys create ci --add events:add
       OK to proceed?

you    yes

agent  Done. Key "ci" created with events:add only. The secret is shown once, so store it now.
```
