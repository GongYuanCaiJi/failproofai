सेशन, इवेंट और इवैलुएशन डेटा (और री-इवैलुएशन ट्रिगर करें) सीधे स्क्रिप्ट या कोडिंग एजेंट से खींचें, stdout पर क्लीन JSON के साथ जो सीधे `jq` में पाइप होता है। ये रेसिपीज़ Failproof AI Observability के डेटा को टर्मिनल यूजर या AI कोडिंग एजेंट (Claude Code, Cursor) के लिए कुछ ऐसा बनाती हैं जिसे डैशबोर्ड से क्लिक किए बिना क्वेरी और ऑटोमेट किया जा सकता है।

नीचे दिए गए पैटर्न Failproof AI Observability CLI (`agenteye`) के लिए कॉपी-पेस्ट के लिए तैयार हैं। इंस्टॉलेशन, ऑथेंटिकेशन और पूरी ऑप्शन लिस्ट के लिए [CLI](/hi/agenteye/cli) देखें; बिल्ट-इन हेल्प के लिए `agenteye -h` या `agenteye <command> -h` चलाएँ।

## स्वर्णिम नियम

1. **ग्लोबल ऑप्शंस कमांड *से पहले* आते हैं।** `agenteye --json sessions` सही है; `agenteye sessions --json` नहीं है। ग्लोबल्स हैं `--json`, `--base-url`, `--org`, `--token`, `--insecure`/`--secure`, `--timeout`, `--quiet`, `--no-color`।
2. **जब भी आप आउटपुट पार्स करें तो `--json` पास करें।** डेटा **stdout** पर JSON के रूप में जाता है; मानव स्टेटस और errors stderr पर जाते हैं, तो stdout क्लीन रहता है `jq` में पाइप करने के लिए।
3. **stderr टेक्स्ट पर नहीं, exit कोड पर ब्रांच करें**: `0` ठीक है · `1` अप्रत्याशित error · `2` ख़राब arguments · `3` डैशबोर्ड तक नहीं पहुँच सकते · `4` लॉगिन नहीं है या एक्सपायरड है · `5` परमिशन नहीं है · `6` रिसोर्स नहीं मिला।
4. **`-h` से खोजें।** हर कमांड अपने फिल्टर्स, वैल्यू फॉर्मेट्स और JSON शेप को दस्तावेज़ित करता है।

## एक बार सेटअप

```bash
export AGENTEYE_DASHBOARD_URL=https://agenteye.example.com   # ताकि आप --base-url दोहराएँ नहीं
agenteye login --email you@example.com                       # ईमेल किया गया कोड पेस्ट करें; ~24h के लिए मान्य
```

## काम करने से पहले ऑथ की पुष्टि करें

`whoami` कभी missing या एक्सपायरड सेशन पर error नहीं करता; इसके बजाय `logged_in:false` रिपोर्ट करता है, तो एक एजेंट सुरक्षित रूप से ऑथ स्टेट प्रोब कर सकता है। (यदि कोई base URL सेट नहीं है या डैशबोर्ड अनरीचेबल है तो यह अभी भी non-zero exit कर सकता है।)

```bash
if [ "$(agenteye --json whoami | jq -r .logged_in)" != "true" ]; then
  echo "Not authenticated. Run: agenteye login" >&2; exit 1
fi
```

## विफल या कम-स्कोरिंग सेशन खोजें

```bash
# पिछले 24h में सेशन जिनकी evaluation errored
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'

# एक एजेंट के लिए helpfulness पर <= 0.5 स्कोर करने वाली evaluations
agenteye --json evals --agent-id checkout-bot --score helpfulness:..0.5 \
  | jq '.evaluations[] | {session_id, scores}'
```

स्कोर फिल्टरिंग `sessions` पर नहीं, **`evals`** पर रहती है। `--score KEY:MIN..MAX` दोहराने योग्य है और AND-कम्बाइनड है; कोई भी बाउंड ऑप्शनल है (`..0.5` का मतलब ≤ 0.5, `0.9..` का मतलब ≥ 0.9)। आप प्रति रिक्वेस्ट 20 तक स्कोर फिल्टर पास कर सकते हैं; अधिक HTTP 400 रिटर्न करता है। `sessions` `--env`, `--status`, `--agent-id`, `--session-id` और `evals` के साथ time-range फिल्टर शेयर करता है, लेकिन `--score` नहीं है।

## एक सेशन को एंड-टू-एंड पढ़ें

कोई single `session show` कमांड नहीं है। event ट्रेल को सेशन की evaluation के साथ कम्बाइन करें:

```bash
# सेशन की लेटेस्ट evaluation (status + scores)
agenteye --json evals --session-id run-001 | jq '.evaluations[0] | {status, scores}'

# रन में हर event (पूरी स्वीप के लिए --limit बढ़ाएँ)
agenteye --json events --session-id run-001 --all --limit 1000 | jq '.events[] | {ts, event_type}'

# सिर्फ एक सेशन में tool कॉल्स (--full raw payload पाने के लिए जरूरी है)
agenteye --json events --full --session-id run-001 --event-type tool_use,tool_result --all \
  | jq '.events[].payload'
```

> **नोट:** डिफॉल्ट रूप से, `events` फास्ट, payload-फ्री फीड पढ़ता है। हर event एक सर्वर-कम्प्यूटेड वन-लाइन `summary` प्लस `is_error` जैसे फ्लैग्स और टोकन काउंट्स कैरी करता है, लेकिन `payload` `{}` के रूप में वापस आता है। raw payload खींचने के लिए, `--full` (या `--fields payload`) जोड़ें। पूरी फीड स्केल पर धीमी है, तो इसे बाउंडेड रखें: `--full` को single `--session-id` के साथ पेयर करें।

## सब कुछ फेच करें (pagination)

रिजल्ट्स नेवेस्ट-फर्स्ट और कर्सर-पेजिनेटेड हैं।

```bash
# वन शॉट: 200-row पेजेज़ में 500 तक रो फेच करें
agenteye --json events --session-id run-001 --limit 500 --all > events.json

# मैनुअल पेजिंग: next_cursor को वापस फीड करें
page=$(agenteye --json events --limit 100)
cursor=$(echo "$page" | jq -r '.next_cursor // empty')
[ -n "$cursor" ] && agenteye --json events --limit 100 --cursor "$cursor"
```

## --fields के साथ आउटपुट स्लिम करें

कीज़ (टेबल और `--json` दोनों में) को प्रतिबंधित करें जो एक एजेंट को पढ़ना होगा उसे कम करने के लिए।

```bash
agenteye --json sessions --since 7d --fields session_id,status,scores | jq -c '.sessions[]'
agenteye --json events --session-id run-001 --fields ts,event_type --all
```

अज्ञात फील्ड नेम्स को रिजेक्ट किया जाता है (exit `2`) वैलिड लिस्ट के साथ, फील्ड नेम्स खोजने का सस्ता तरीका।

## वैलिड फिल्टर वैल्यूज़ खोजें

```bash
agenteye --json list envs | jq -r '.values[]'           # --env के लिए वैल्यूज़
agenteye --json list tools | jq -r '.values[]'          # टूल नेम्स; agents, models, event_types भी
agenteye --json list score_filters | jq -r '.values[]'  # --score KEY:MIN..MAX के लिए वैलिड KEY
```

## अपना ऑर्ग चुनें (multi-tenant)

यदि आप एक से अधिक ऑर्ग से संबंधित हैं, तो लॉगिन पर सक्रिय टेनेंट चुनें (यह सेव होता है):

```bash
agenteye login --org acme --email you@corp.com   # लॉगिन के समान स्टेप में टेनेंट सेट करें
agenteye --json orgs list | jq -r '.orgs[].org_slug'
agenteye --org globex --json sessions --since 24h   # एक कमांड के लिए ओवरराइड करें
```

`--org` के बिना multi-org लॉगिन non-zero exit करता है और चुनने के लिए ऑर्ग्स प्रिंट करता है।

## SDK/collector के लिए API कुंजी प्रोविजन करें

```bash
# सीक्रेट एक बार प्रिंट होता है, --json के साथ यह .key फील्ड है
key=$(agenteye --json keys create ci-bot --add events:read.add | jq -r '.key')
agenteye keys regenerate ci-bot --yes    # रोटेट करें; रिवोक करने के लिए agenteye keys disable ci-bot --yes
```

## सेव्ड या एड-हॉक क्वेरी चलाएँ

```bash
agenteye --json query run --sql "select count(*) from analytics.events" | jq '.rows'
agenteye --json query run errs --arg prod | jq '.rows'   # सेव्ड क्वेरी + पॉज़िशनल $1
```

## घटना को गैर-इंटरैक्टिवली ट्राइएज करें

```bash
id=$(agenteye --json incidents list --state firing | jq -r '.incidents[0].id')
agenteye incidents ack "$id"
agenteye incidents assign "$id" --assignee you@corp.com
agenteye incidents resolve "$id" --yes
```

> **नोट:** म्यूटेशंस `--json` के तहत या जब stdin TTY नहीं है तो अपनी कन्फर्मेशन प्रॉम्प्ट ऑटो-स्किप करते हैं, तो एजेंट्स कभी हैंग नहीं होते; दूसरी जगह स्पष्ट रूप से स्किप करने के लिए `--yes`/`-y` पास करें।

## स्क्रिप्ट में Exit-code हैंडलिंग

```bash
out=$(agenteye --json sessions --since 1h) || code=$?
case "${code:-0}" in
  0) echo "$out" | jq '.sessions | length' ;;
  4) echo "Session expired - run 'agenteye login'." >&2 ;;
  5) echo "Missing permission (ask an admin for evaluations:read)." >&2 ;;
  3) echo "Dashboard unreachable - check the URL." >&2 ;;
  *) echo "Unexpected error (exit ${code})." >&2 ;;
esac
```

## JSON आउटपुट शेप्स

| कमांड | stdout JSON (`--json` के साथ) |
|---|---|
| `whoami` | `{"logged_in": true, "id", "email", "is_instance_admin", "active_org", "permissions": [...], "memberships": [...]}` या `{"logged_in": false}` |
| `orgs list` | `{"active_org", "orgs": [{"org_slug","org_name","permission_set","permissions"}]}` |
| `events` | `{"events": [...], "next_cursor": <cursor or null>}` |
| `evals` | `{"evaluations": [...], "next_cursor": <cursor or null>}` |
| `sessions` | `{"sessions": [...], "next_cursor": <cursor or null>}` |
| `errors` | `{"errors": [...], "next_cursor": <cursor or null>}` |
| `list <kind>` | `{"kind", "values": [...]}` |
| `keys list` / `keys create` | `{"keys": [...]}` / `{id, name, permissions, created_at, key}` (`key` एक बार दिखाया गया) |
| `query run` | `{columns: [{name,type}], rows: [[...]], truncated, elapsed_ms}` |
| `users list` / `settings list` | `{"users": [...]}` / `{"settings": [...]}` |
| `alerts list` / `incidents list` | `{"alerts": [...]}` / `{"incidents": [...]}` |
| create/update/delete (कोई भी) | रिसोर्स ऑब्जेक्ट, या डिलीट्स के लिए `{"deleted": true, "id"}` |
| विफलता (कोई भी, `--json` के साथ) | `{"error": "...", "exit_code": <n>, "status"?: <http>, "hint"?: "..."}` stdout पर |

- हर **event** आइटम (`events`): `id, session_id, agent_id, event_type, ts, payload, environment, summary, is_error, error_type, output_tokens, context_window, context_fill`। नोट करें कि `payload` `{}` है जब तक आप `--full` (या `--fields payload`) के साथ पूरी फीड न मांगें।
- हर **evaluation** आइटम (`evals`): `id, session_id, agent_id, environment, status, scores, reasoning, summary, error, attempt_count, duration_ms, completed_at, created_at`।
- हर **session** आइटम (`sessions`): `session_id, agent_id, environment, status, scores, event_count, started_at, last_event_at, first_event_id, last_event_id, latest_evaluation`।

हर कमांड का `--fields` बिल्कुल अपने आइटम के फील्ड नेम्स स्वीकार करता है। सेट `sessions` और `evals` के बीच भिन्न है, तो एक के लिए वैलिड नाम दूसरे द्वारा रिजेक्ट किया जा सकता है।
