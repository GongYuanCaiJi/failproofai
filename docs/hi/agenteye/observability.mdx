observe सर्फेस वह जगहें हैं जहाँ आप अपने agents को अभी वास्तविक समय में कार्य करते हुए देखते हैं और किसी भी एक run में विस्तार से जाँच कर सकते हैं। यहाँ की हर चीज़ live है, आपके organization के लिए scoped है, और date range, environment, agent, और session के आधार पर filterable है, ताकि आप "कुछ गलत लग रहा है" से लेकर सटीक run तक कुछ ही सेकंड में पहुँच जाएँ।

![Live Event Stream, type के आधार पर रंग-कोडित और environment, agent, और session के आधार पर filterable](/agenteye/images/events-stream.png)

चार सर्फेस, प्रत्येक के अपने पृष्ठ के साथ:

- **[Event stream](/hi/agenteye/event-stream)**: हर agent में हर run का live, प्रति-step ट्रेल, सबसे नई पहले। आपके org का होम और triage के लिए पहला स्टॉप।
- **[Sessions और execution graph](/hi/agenteye/sessions)**: वे events एक row में rolled up प्रति run, साथ ही एक git-शैली की तस्वीर कि कैसे प्रत्येक run unfold हुआ।
- **[Performance metrics](/hi/agenteye/telemetry)**: latency heat-maps और आपके models, tools, और hooks के लिए p50/p95/p99 vitals, ताकि एक tail spike median से अलग दिखे।
- **[Error tracking](/hi/agenteye/error-tracking)**: हर चीज़ जो गलत हुई उसके लिए एक triage सर्फेस, एक firing alert से उस run तक जो टूटा एक क्लिक दूर।

## संबंधित

- [Evaluations](/hi/agenteye/evaluations): हर run को quality के लिए score करें।
- [Alerts](/hi/agenteye/alerts): किसी भी threshold को एक paging rule में बदलें।
- [Audits](/hi/agenteye/audits): FailproofAI Observability को sessions में failure patterns खोजने दें।
- [CLI और agents](/hi/agenteye/cli-and-agents): आपके terminal से same observability।
