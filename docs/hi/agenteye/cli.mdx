टर्मिनल या स्क्रिप्ट से FailproofAI Observability को पूरी तरह से संचालित करें: कोई डैशबोर्ड राउंड-ट्रिप नहीं। `agenteye` CLI आपके डेटा (सेशन, ईवेंट लॉग, मूल्यांकन) को क्वेरी करता है और आपके संगठन (API कुंजी, उपयोगकर्ता, सेटिंग्स, अलर्ट, घटनाएं, सहेजी गई क्वेरी) का प्रबंधन करता है, इसलिए जब आप एक जांच को स्वचालित करना चाहते हैं, CI में Observability को तार करना चाहते हैं, या एक कोडिंग एजेंट को प्रोडक्शन को निरीक्षण करने दें। हर कमांड `--json` फ़्लैग को सपोर्ट करता है, इसलिए यह आपके लिए प्रॉम्प्ट पर या कोडिंग एजेंट (Claude Code, Cursor) के लिए समान रूप से काम करता है।

एक बाइनरी से आप कर सकते हैं:

- **आपका डेटा पढ़ें**: `sessions`, `events`, `evals`, `errors` (समय, एजेंट, env, स्कोर द्वारा फ़िल्टर करें)।
- **अपने संगठन का प्रबंधन करें**: `keys`, `users`, `settings`, `alerts`, `incidents`।
- **विश्लेषण चलाएं**: सहेजी गई SQL और एक तदर्थ क्वेरी रनर (`query`)।
- **AI सहायक से पूछें**: वही केवल-पढ़ने वाले विश्लेषक जिनसे आप डैशबोर्ड में चैट करते हैं (`agent`)।

> **नोट:** यह `agenteye` CLI है, कलेक्टर डेमन (`agenteye-collector`) से एक अलग टूल। CLI आपके डैशबोर्ड से बात करता है; कलेक्टर ईवेंट को सर्वर पर भेजता है।

## त्वरित प्रारंभ

शून्य से लेकर अपने पहले परिणाम तक चार लाइनों में। CLI को अपने डैशबोर्ड पर पॉइंट करें, साइन इन करें, पुष्टि करें कि आप कौन हैं, फिर पिछले दिन के रन को पुल करें:

```bash
pipx install agenteye
agenteye --base-url https://agenteye.example.com login --email you@example.com   # emailed 6-digit code
agenteye whoami                                                                  # confirm user + active org
agenteye --json sessions --since 24h                                             # one row per agent run, last 24h
```

वह अंतिम कमांड सबसे हाल के सेशन का एक JSON ऑब्जेक्ट प्रिंट करता है (सबसे नया पहले, डिफ़ॉल्ट रूप से 50 तक सीमित)। इसे `jq` में पाइप करें इसे स्लाइस करने के लिए, या `--json` को ड्रॉप करें एक बॉक्स्ड, रंगीन तालिका के लिए। प्रत्येक पंक्ति रन की स्थिति और, यदि एक मूल्यांकनकर्ता ने इसे स्कोर किया है, तो इसके मीट्रिक स्कोर (यहाँ संक्षिप्त) ले जाती है:

```json
{
  "sessions": [
    {
      "session_id": "run-8f2a",
      "agent_id": "checkout-bot",
      "environment": "prod",
      "status": "error",
      "scores": { "helpfulness": 0.42, "tool_efficiency": 0.55 },
      "event_count": 37,
      "started_at": "2026-07-16T09:14:02Z",
      "last_event_at": "2026-07-16T09:14:48Z"
    }
  ],
  "next_cursor": null
}
```

इस पृष्ठ के बाकी हिस्से को समझाता है: [स्थापन](#installation) अलगाव में, [साइन इन करना](#authentication), [कॉन्फ़िगरेशन](#configuration), [वैश्विक सम्मेलन](#global-options--conventions) जो हर कमांड साझा करते हैं, और [पूर्ण कमांड संदर्भ](#command-reference)।

## स्थापन

CLI एक सार्वजनिक PyPI पैकेज है जिसका नाम **`agenteye`** है। इसे एक अलग वातावरण में स्थापित करें ताकि इसके पास हमेशा अपनी निर्भरताएं हों:

```bash
pipx install agenteye
# or
uv tool install agenteye
```

इसके लिए Python 3.10+ की आवश्यकता है। स्थापित कमांड **`agenteye`** है:

```bash
agenteye --version
agenteye --help
```

> **नोट:** FailproofAI Observability Python SDK भी `agenteye` वितरण नाम का उपयोग करता है। `pipx` या `uv tool` के साथ CLI को स्थापित करना (साझी virtualenv में `pip install` के बजाय) दोनों को टकराने से रोकता है। एक सादा `pip install agenteye` केवल तभी ठीक है यदि SDK एक ही वातावरण में स्थापित नहीं है।

## प्रमाणीकरण

CLI **डैशबोर्ड** के लिए एक ईमेल किए गए एकबारी कोड के साथ प्रमाणित करता है:

```bash
agenteye login --email you@example.com
# A 6-digit code is emailed to you; paste it at the prompt.
```

सेशन टोकन `~/.agenteye/cli.json` में संग्रहीत है (केवल आपके द्वारा पढ़ने योग्य, मोड `0600`) और डिफ़ॉल्ट रूप से 24 घंटे के लिए मान्य है। जब यह समाप्त हो जाता है, तो `agenteye login` फिर से चलाएं।

```bash
agenteye whoami     # show the current user, active org, and permissions
agenteye logout     # revoke the session and clear the stored token
```

`whoami` एक लापता या समाप्त सेशन पर कभी त्रुटि नहीं करता; इसके बजाय `logged_in: false` की रिपोर्ट करता है, इसलिए एक स्क्रिप्ट या एजेंट सुरक्षित रूप से auth स्थिति को जांच सकता है (यदि कोई बेस URL सेट नहीं है या डैशबोर्ड अप्राप्य है तो अभी भी गैर-शून्य से निकल सकता है)।

**आवश्यकताएं:** आपके ईमेल को डैशबोर्ड में साइन इन करने की अनुमति दी जानी चाहिए (अपने FailproofAI Observability प्रशासक से पूछें), और डैशबोर्ड अपने बेस URL पर पहुंचने योग्य होना चाहिए (देखें [कॉन्फ़िगरेशन](#configuration))। यदि आप कोड का अनुरोध करते हैं और कोई नहीं आता है, तो आपका ईमेल संभवतः अभी तक डैशबोर्ड एक्सेस के लिए सक्षम नहीं है।

## अपना org चुनना (मल्टी-टेनेंट)

यदि आपका खाता एक से अधिक org से संबंधित है, तो सक्रिय एक को **लॉगिन पर** चुनें; इसे सहेजा जाता है और हर बाद की कमांड के लिए उपयोग किया जाता है:

```bash
agenteye login --org acme           # authenticate and set the active tenant in one step
agenteye orgs list                  # the orgs you can access (the active one is marked)
agenteye orgs switch globex         # change the saved default
agenteye --org globex sessions      # override for a single command
```

यदि आप बिल्कुल एक org से संबंधित हैं तो इसे स्वचालित रूप से चुना जाता है और आप `--org` को पूरी तरह से अनदेखा कर सकते हैं। यदि आप कई से संबंधित हैं और एक को नहीं चुनते हैं, तो CLI उन्हें सूचीबद्ध करता है और आपको `--org <slug>` के साथ फिर से चलाने के लिए कहता है। सक्रिय org हर अनुरोध पर डैशबोर्ड को भेजा जाता है, और आपकी अनुमतियां **प्रति org** में हल की जाती हैं; `agenteye whoami` सक्रिय org, इसमें आपकी अनुमतियां, और सभी आपकी सदस्यताएं दिखाता है।

## कॉन्फ़िगरेशन

| सेटिंग | फ़्लैग | पर्यावरण चर | डिफ़ॉल्ट |
|---|---|---|---|
| डैशबोर्ड बेस URL | `--base-url` | `AGENTEYE_DASHBOARD_URL` | **आवश्यक** (कोई डिफ़ॉल्ट नहीं) |
| सक्रिय org/tenant | `--org` | `AGENTEYE_ORG` | लॉगिन पर चुना गया; `~/.agenteye/cli.json` में सहेजा गया |
| सेशन टोकन | `--token` | `AGENTEYE_CLI_TOKEN` | `~/.agenteye/cli.json` से |
| JSON आउटपुट | `--json` | `AGENTEYE_CLI_JSON` | बंद |
| TLS सत्यापन छोड़ें | `--insecure` / `--secure` | `AGENTEYE_INSECURE` | बंद |
| अनुरोध समय सीमा (सेकंड) | `--timeout` | _(कोई नहीं)_ | 30 |
| उपयोग टेलीमेट्री अक्षम करें | _(कोई नहीं)_ | `AGENTEYE_ANALYTICS_DISABLED` (या `DO_NOT_TRACK`) | टेलीमेट्री वर्तमान में अक्षम है; कुछ नहीं भेजा जाता है |

संकल्प क्रम **फ़्लैग → पर्यावरण चर → कॉन्फ़िग फ़ाइल** है। कोई डिफ़ॉल्ट नहीं है; आपको CLI को अपने डैशबोर्ड पर पॉइंट करना होगा, या तो प्रति-कमांड (`--base-url https://agenteye.example.com`) या एक बार पर्यावरण के माध्यम से (यह आपके पहले `login` के बाद भी सहेजा जाता है):

```bash
export AGENTEYE_DASHBOARD_URL=https://agenteye.example.com
```

कॉन्फ़िगरेशन डायरेक्टरी `AGENTEYE_HOME` को सम्मानित करती है (SDK और कलेक्टर द्वारा उपयोग किया जाने वाला समान सम्मेलन); यदि सेट किया गया है, `cli.json` `$AGENTEYE_HOME/cli.json` में रहता है।

### स्व-हस्ताक्षरित या आंतरिक TLS

यदि आपका डैशबोर्ड HTTPS पर एक स्व-हस्ताक्षरित या आंतरिक प्रमाणपत्र के साथ परोसा जाता है (उदाहरण के लिए, एक कच्चा लोड-बैलेंसर होस्टनाम), TLS सत्यापन `CERTIFICATE_VERIFY_FAILED` त्रुटि के साथ इसे अस्वीकार करता है। प्रमाणपत्र सत्यापन छोड़ने के लिए `--insecure` पास करें:

```bash
agenteye --base-url https://agenteye.internal --insecure login
```

`--insecure` **लॉगिन करते समय `cli.json` में सहेजा जाता है**, इसलिए बाद की कमांडें स्वचालित रूप से सत्यापन छोड़ती हैं; आपको फ़्लैग को दोहराना नहीं है। एक एकबारी सत्यापित कॉल के लिए `--secure` पास करें, या अपने अगले लॉगिन पर सत्यापन को वापस बंद करने के लिए। CLI किसी भी कमांड से पहले stderr में एक चेतावनी प्रिंट करता है जो डैशबोर्ड से संपर्क करता है जबकि सत्यापन अक्षम है। सत्यापन को छोड़ना आदमी-बीच-में हमलों से सुरक्षा को हटाता है; अपने डैशबोर्ड के नेटवर्क पथ पर भरोसा करने से पहले सुनिश्चित करें (VPN, निजी सबनेट, आदि)।

## टेलीमेट्री और गोपनीयता

> **नोट:** भेजा गया CLI आज **कोई उपयोग टेलीमेट्री नहीं भेजता है।** एक मास्टर किल स्विच चालू है, इसलिए आपके पर्यावरण की परवाह किए बिना कुछ भी प्रेषित नहीं होता है। निम्न अनुभाग यदि और जब टेलीमेट्री कभी सक्षम होता है तो ऑप्ट-आउट क्षमता का वर्णन करता है।

सक्षम होने पर भी, टेलीमेट्री केवल **अनाम उपयोग विश्लेषण** होगी, कभी आपके एजेंट, सेशन, या ईवेंट डेटा नहीं:

- **कोई एजेंट, सेशन, या ईवेंट डेटा कभी आपके बुनियादी ढांचे से नहीं निकलता है।** केवल CLI उपयोग की रिपोर्ट की जाएगी: कमांड और सबकमांड नाम (जैसे `keys create`), आपके द्वारा उपयोग किए गए फ़्लैग के **नाम** (कभी उनके मान नहीं), सफलता/निकास स्थिति, और अवधि, साथ ही उत्परिवर्तन के लिए एक प्रति-कार्रवाई ईवेंट (जैसे `api_key_created`, `query_run`) केवल स्थिर नाम/enums और मोटे गिनती ले जाते हैं। आपके डैशबोर्ड URL, सेशन टोकन, ईमेल, org slug, संसाधन ids, SQL, महत्वपूर्ण रहस्य, और क्वेरी फ़िल्टर **कभी भी** नहीं भेजे जाएंगे। ऑपरेटरों को केवल एक अपारदर्शी आंतरिक id द्वारा पहचाना जाता है, कभी ईमेल द्वारा नहीं।
- **पहले से ऑप्ट आउट करें** CLI के पर्यावरण में `AGENTEYE_ANALYTICS_DISABLED=1` सेट करके (CLI क्रॉस-टूल `DO_NOT_TRACK=1` सम्मेलन को भी सम्मानित करता है)। यह तुरंत प्रभावी होता है जब टेलीमेट्री कभी चालू हो जाता है, इसलिए एक गोपनीयता-सचेत पर्यावरण स्थायी रूप से ऑप्ट आउट रह सकता है।
- यदि टेलीमेट्री सक्षम होते, तो CLI सीधे PostHog (`https://us.i.posthog.com`) को भेजता; उस होस्ट को ब्लॉक किया गया एक मशीन चुप्पी से कुछ नहीं भेजता और CLI अप्रभावित होता।

## वैश्विक विकल्प और सम्मेलन

एक बार पढ़ें; यह हर कमांड पर लागू होता है।

- **वैश्विक विकल्प कमांड से पहले जाते हैं।** `agenteye --json sessions` सही है; `agenteye sessions --json` एक उपयोग त्रुटि है। वैश्विक `--json`, `--base-url`, `--org`, `--token`, `--insecure`/`--secure`, `--timeout`, `--quiet`, और `--no-color` हैं।
- **`--json` शुद्ध JSON को stdout में प्रिंट करता है, और कुछ भी नहीं।** मानव स्थिति लाइनें, चेतावनियां, और त्रुटियां **stderr** पर जाती हैं, इसलिए एक `--json` stdout कैप्चर जब भी एक स्थिति लाइन दिखाई दे तब भी `jq` में पाइप करने के लिए स्वच्छ रहता है। `--json` के बिना आप मानव आँखों के लिए एक बॉक्स्ड, रंगीन दृश्य प्राप्त करते हैं।
- **`--help` के साथ खोजें।** हर कमांड और सबकमांड के पास `--help` है (और `-h` उपनाम): `agenteye -h`, `agenteye sessions -h`, `agenteye keys create -h`। शीर्ष-स्तरीय सहायता भी निकास कोड और वैश्विक विकल्प सूचीबद्ध करती है। कोई वैश्विक मशीन-पठनीय सतह डंप नहीं है; प्रति-कमांड `--help` का उपयोग करें, साथ ही उन दोनों के लिए डोमेन-विशिष्ट `agenteye query schema` और `agenteye settings schema`।
- **पुष्टिकरण स्क्रिप्ट और एजेंटों के लिए स्वचालित-छोड़ें।** बनाएं/अपडेट/हटाएं कमांड एक इंटरैक्टिव टर्मिनल में "क्या आप निश्चित हैं?" लाइन दिखाते हैं, लेकिन **जब stdin TTY नहीं है `--json` के अंतर्गत स्वचालित-छोड़ उस लाइन को छोड़ें** (एक TTY एक इंटरैक्टिव टर्मिनल सेशन है; एक पाइप या CI रनर नहीं है), इसलिए स्क्रिप्ट और एजेंट कभी लटकते नहीं हैं। `--yes`/`-y` को स्पष्ट रूप से छोड़ने के लिए पास करें। क्योंकि लाइन एक एजेंट के लिए आग नहीं लगेगी, एक एजेंट को मानव के साथ विनाशकारी कार्यों की पुष्टि करनी चाहिए।
- **पेजिनेशन:** परिणाम सबसे नए-प्रथम हैं और कर्सर-पेजिनेट किए गए हैं (प्रत्येक पृष्ठ एक टोकन देता है जिसे आप अगले को लाने के लिए उपयोग करते हैं)। `--limit N` (उपनाम `-n`) पंक्तियों को कैप करता है और **50 के लिए डिफ़ॉल्ट**; `--all` स्वचालित-पेजिनेट करता है (**200-पंक्ति चंक्स में**) **`--limit` तक**, इसलिए एक नंगा `--all` 50 पर भी रुकता है। एक पूर्ण स्वीप के लिए एक उच्च स्पष्ट सीमा पास करें: `--all --limit 1000`। `--page-size N` प्रति-अनुरोध चंक को नियंत्रित करता है (अधिकतम 200); `--cursor <id>` एक पूर्व पृष्ठ के `next_cursor` से फिर से शुरू करता है।
- **समय फ़िल्टर:** `--since` एक सापेक्ष विंडो लेता है: `15m`, `1h`, `6h`, `24h`, `7d`, या `all` (डैशबोर्ड की प्रीसेट)। एक लंबी या कस्टम श्रेणी के लिए (कहें पिछले 30 दिन), `--from`/`--to` का उपयोग करें: स्पष्ट ISO-8601 UTC टाइमस्टैम्प **`T` और एक टाइमज़ोन के साथ** (जैसे `2026-06-01T00:00:00Z`) जो `--since` को ओवरराइड करते हैं। एक स्पेस-सेपरेटेड या टाइमज़ोन-कम मान एक उपयोग त्रुटि है।
- **`--fields a,b,c`** (on `events`, `sessions`, `evals`, `errors`) तालिका और `--json` दोनों के लिए उन कुंजियों के लिए आउटपुट को प्रतिबंधित करता है। अज्ञात नाम मान्य सूची के साथ अस्वीकार कर दिए जाते हैं, क्षेत्र नाम खोजने का एक सस्ता तरीका।
- **`--file payload.json`** (या `--file -` stdin को पढ़ने के लिए) एक पूर्ण JSON अनुरोध बॉडी आपूर्ति करता है जहां एक संसाधन के पास एक जटिल आकार है (`alerts create/update`, `settings set`, और `users create/update` पर)। सहेजी गई-क्वेरी SQL इसके बजाय `--sql @file.sql` का उपयोग करता है।
- **बहु-मान फ़िल्टर** अल्पविराम-सेपरेटेड हैं → एक सेट के रूप में मेल खाते हैं (एक फ़िल्टर के भीतर यूनियन, फ़िल्टर में AND): `--event-type tool_use,tool_result`। क्लिक विकल्प variadic नहीं हैं, इसलिए `--add a b` टूट जाता है। `--add a,b` का उपयोग करें, फ़्लैग को दोहराएं (`--add a --add b`), या उद्धृत करें (`--add "a b"`)।

## कमांड संदर्भ

### आप इन 5 कमांडों का सबसे अधिक उपयोग करेंगे

अधिकांश दिन-प्रतिदिन का काम कुछ पढ़ें कमांडों के माध्यम से चलता है। यहाँ शुरू करें, फिर नीचे दी गई पूर्ण सतह तक पहुंचें जब आपको इसकी आवश्यकता हो:

| कमांड | यह क्या करता है | इसे आजमाएं |
|---|---|---|
| `sessions` | एजेंट रन प्रति एक पंक्ति: समय, env, एजेंट, स्थिति, नवीनतम स्कोर। | `agenteye --json sessions --since 24h --status error` |
| `events` | एक रन के अंदर कच्ची प्रति-चरण पथ (पेलोड के लिए `--full` जोड़ें)। | `agenteye --json events --session-id run-001 --all` |
| `evals` | मूल्यांकन परिणाम और स्कोर; `--aggregate` उन्हें रोल अप करता है। | `agenteye --json evals --aggregate --since 7d --env prod` |
| `errors` | केवल errored ईवेंट; प्रकार के `--aggregate` गिनती के लिए। | `agenteye --json errors --since 24h --aggregate` |
| `list` | मान्य फ़िल्टर मान खोजें (एजेंट, envs, मॉडल, …)। | `agenteye list agents` |

### CLI सब कुछ कर सकता है

पूर्ण सतह अनुसरण करती है। CLI के पास **18 शीर्ष-स्तरीय कमांड** हैं। सभी पढ़ें कमांड `--json` और ऊपर वैश्विक विकल्प स्वीकार करते हैं; किसी एक के थकाऊ फ़्लैग सूची और JSON आकार के लिए `agenteye <command> -h` (या `<command> <subcommand> -h`) चलाएं।

### पहचान: `login` · `logout` · `whoami` · `orgs` · `version` · `help`

```bash
agenteye login --email you@example.com [--org acme]   # emailed one-time code; saves the session
agenteye logout                                       # clear the saved session on this machine
agenteye whoami                                       # current user, active org, permissions
agenteye version                                      # print the CLI version (same as --version)
agenteye help                                         # top-level help (same as --help)
```

`orgs` सक्रिय किरायेदार का निरीक्षण और स्विच करता है:

```bash
agenteye orgs list        # your orgs + your role in each (active one marked)
agenteye orgs switch acme # change the saved active org (omit the slug to pick from a list on a TTY)
agenteye orgs current     # identity card for the active org
agenteye orgs perms       # your permissions in the active org, grouped by resource
```

### अवलोकन (केवल-पढ़ने योग्य): `events` · `sessions` · `evals` · `errors` · `list`

इनमें से कोई भी पुष्टिकरण की जरूरत नहीं है। साझा फ़िल्टर: `--session-id`, `--agent-id`, `--env` (**नहीं** `--environment`), और समय श्रेणी (`--since` / `--from` / `--to`)।

```bash
# events (alias: the raw per-step trail), newest first
agenteye --json events --session-id run-001 --event-type tool_use,tool_result --all --limit 1000
agenteye --json events --since 1h --search timeout --all | jq '.events[].payload'

# sessions: one row per agent run (time/env/agent/session/status; no score filtering)
agenteye --json sessions --since 24h --status error
agenteye --json sessions --agent-id checkout-bot --env prod --all --limit 1000

# evals: evaluation results + scores; --score filters by metric, --aggregate rolls up
agenteye --json evals --score helpfulness:0.5..0.8 --score tool_efficiency:..0.3
agenteye --json evals --aggregate --since 7d --env prod        # status mix + per-key score stats

# errors: errored events; --aggregate for counts/sessions/agents/last-seen
agenteye --json errors --since 24h --aggregate
agenteye --json errors --since 24h --error-type timeout --all --limit 1000

# list: discover valid filter values before you filter
agenteye list envs        # also: agents event_types score_filters models hooks tools error_types
```

`--score KEY:MIN..MAX` (on **`evals`**, not `sessions`) दोहराने योग्य और AND-संयुक्त है; या तो बाउंड वैकल्पिक है (`..0.5` मतलब ≤ 0.5, `0.9..` मतलब ≥ 0.9)। प्रति अनुरोध 20 स्कोर फ़िल्टर तक। `evals --scores-full` **मानव तालिका केवल** के लिए एक प्रदर्शन ध्वज है; यह `+N` गिनती के बजाय हर स्कोर जोड़ी दिखाता है। `--json` के तहत कोई प्रभाव नहीं है, जो हमेशा पूर्ण स्कोर ऑब्जेक्ट देता है। **एक सेशन अंत-से-अंत तक** पढ़ने के लिए, ईवेंट पथ को इसके मूल्यांकन के साथ संयोजित करें:

```bash
agenteye --json events --session-id run-001 --all --limit 1000 | jq '.events[] | {ts, event_type}'
agenteye --json evals  --session-id run-001                       # its scores + status
```

### प्रबंधन (अनुमति-गेटेड): `keys` · `users` · `settings` · `alerts` · `incidents`

**`keys`**: API कुंजी। रहस्य स्थानीय रूप से उत्पादित होता है, सर्वर को भेजा जाता है (जो केवल एक हैश संग्रहीत करता है), और बनाएं/पुनर्जन्मित पर **एक बार दिखाया जाता है**; इसे तब कैप्चर करें। `--json` के साथ यह केवल `key` क्षेत्र में प्रकट होता है। **नाम** द्वारा संदर्भित।

```bash
agenteye keys list                                   # active keys first, then revoked
agenteye keys show ci-bot
agenteye keys create ci-bot --add events:read.add    # scope to what you need; prints the secret ONCE
agenteye keys create ops --permission-set standard --remove queries:run   # seed a preset, then trim
agenteye keys update ci-bot --add evaluations:read --yes
agenteye keys regenerate ci-bot --yes                # rotate the secret (the old one stops working)
agenteye keys disable ci-bot --yes                   # revoke
```

अनुमतियां `(permission-set ∪ --add) − --remove` के रूप में काम करती हैं। टोकन `slug:action` हैं (जैसे `events:read`) या कई को एक संसाधन पर विस्तारित करने के लिए `slug:action.action` (`events:read.add` → `events:read`, `events:add`)। प्रीसेट: `read-only`, `standard`, `admin`। केवल-मानव अनुमतियां (`keys:update`) एक कुंजी को दी नहीं जा सकती हैं।

**`users`**: org सदस्य, **email** द्वारा संदर्भित (एक UUID id भी स्वीकार किया जाता है)।

```bash
agenteye users list [--active-only]
agenteye users show dev@corp.com
agenteye users create dev@corp.com --permission-set standard
agenteye users update dev@corp.com --add alerts:write --remove queries:delete   # predicts + confirms
agenteye users disable dev@corp.com --yes            # has protected/self guards
agenteye users enable dev@corp.com
```

**`settings`**: एक निश्चित रजिस्ट्री (आप मौजूदा कुंजियों को पढ़ते हैं और बदलते हैं; आप नई नहीं बना सकते हैं)।

```bash
agenteye settings list                               # key · value · type · updated (secrets masked)
agenteye settings schema                             # what each key accepts (type · range · description)
agenteye settings set session_ttl_secs --value 86400 --yes
```

**`alerts`**: अलर्ट परिभाषाएं, **नाम** द्वारा संदर्भित। `create` एक स्थितीय NAME साथ ही फ़्लैग या `--file` के माध्यम से एक पूर्ण JSON बॉडी लेता है।

```bash
agenteye alerts list
agenteye alerts show high-errors
agenteye alerts create high-errors --file alert.json          # NAME is required (positional)
agenteye alerts update high-errors --severity critical --yes
agenteye alerts test high-errors --yes                        # fire a test notification
agenteye alerts delete high-errors --yes
```

**`incidents`**: अलर्ट घटनाएं, id द्वारा संदर्भित (छोटे ids स्वीकार किए जाते हैं)। `show` पूर्ण गतिविधि लॉग प्रिंट करता है; कार्य करने से पहले इसे पढ़ें।

```bash
agenteye incidents list --state firing                # also: acknowledged, resolved
agenteye incidents count
agenteye incidents show <id>
agenteye incidents ack <id>
agenteye incidents assign <id> you@corp.com           # assignee must be an operator
agenteye incidents resolve <id> --yes
agenteye incidents open --alert-id <id> --severity critical    # open one manually against an alert
agenteye incidents comment-add <id> "root cause: upstream 5xx"
agenteye incidents comment-list <id> ; agenteye incidents comment-delete <id> <comment-id>
agenteye incidents subscribe <id> ; agenteye incidents unsubscribe <id> ; agenteye incidents subscribers <id>
```

### विश्लेषण और सहायक: `query` · `agent`

**`query`**: आपके विश्लेषण स्टोर के विरुद्ध सहेजी गई SQL साथ ही एक तदर्थ रनर। सहेजी गई क्वेरी **नाम** द्वारा संदर्भित; SQL सर्वर-पक्ष में मान्य है (केवल SELECT/WITH, विवरण समय सीमा, पंक्ति कैप)।

```bash
agenteye query schema [TABLE]                         # column layout of the analytics views
agenteye query run --sql "select count(*) from analytics.events"
agenteye query run errs --arg prod --limit 100        # run a saved query + a positional $1
agenteye query list ; agenteye query show errs
agenteye query create errs --sql @errs.sql --description "errored events (24h)"
agenteye query update errs --sql @errs.sql --yes ; agenteye query delete errs --yes
```

**`agent`**: बिल्ट-इन **AI सहायक** से बात करता है (वही केवल-पढ़ने वाले विश्लेषक जिनसे आप डैशबोर्ड में चैट कर सकते हैं)। चैट एक छोटे chat-id (उपसर्ग-हल) द्वारा संदर्भित।

```bash
agenteye agent health                                 # is the AI assistant configured/reachable
agenteye agent models                                 # models you can pass to --model (default marked)
agenteye agent ask "which agents errored most in the last day?"    # starts a chat; prints its short id
agenteye agent ask --chat <short-id> "and which tools did they call?"   # continue that chat
agenteye agent chats ; agenteye agent show <short-id>
agenteye agent rename <short-id> --title "error triage" ; agenteye agent delete <short-id>
```

## निकास कोड

| कोड | अर्थ |
|---|---|
| 0 | सफलता |
| 1 | अप्रत्याशित त्रुटि (जैसे डैशबोर्ड ने 5xx लौटाया) |
| 2 | उपयोग त्रुटि (अमान्य तर्क, अज्ञात कमांड/फ़्लैग, नाम टकराव) |
| 3 | डैशबोर्ड तक पहुंचने में असमर्थ |
| 4 | लॉगिन नहीं है या सेशन समाप्त; `agenteye login` चलाएं |
| 5 | प्रमाणित, लेकिन आपके खाते में आवश्यक अनुमति की कमी है (संदेश इसे नाम करता है) |
| 6 | अनुरोधित संसाधन नहीं मिला (जैसे अज्ञात सेशन या घटना id) |

ये CLI को स्क्रिप्ट करने के लिए सुरक्षित बनाते हैं: एक कोडिंग एजेंट एक `4` पर आपको फिर से प्रमाणित करने के लिए प्रेरित करने के लिए, या एक `5` पर लापता अनुमति को सामने लाने के लिए एक शाखा बना सकता है। देखें [एजेंटों के लिए CLI व्यंजनों](/hi/agenteye/cli-recipes) exit-code-handling पैटर्न और JSON आउटपुट आकार के लिए।
