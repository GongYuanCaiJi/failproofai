Failproof AI Observability को टर्मिनल या स्क्रिप्ट से पूरी तरह चलाएं: डैशबोर्ड के चक्कर नहीं लगाने पड़ेंगे। `agenteye` CLI आपके डेटा (सेशन, इवेंट लॉग, मूल्यांकन) को क्वेरी करता है और आपके संगठन को प्रबंधित करता है (API कुंजियाँ, उपयोगकर्ता, सेटिंग्स, सतर्कताएं, घटनाएं, सहेजी गई क्वेरीज़), इसलिए जब आप कोई जांच स्वचालित करना चाहते हैं, CI में Observability को जोड़ना चाहते हैं, या किसी कोडिंग एजेंट को प्रोडक्शन का निरीक्षण करने देना चाहते हैं तो इसका उपयोग करें। प्रत्येक कमांड `--json` फ़्लैग को सपोर्ट करता है, इसलिए यह आपके लिए प्रॉम्प्ट पर समान रूप से काम करता है या किसी कोडिंग एजेंट (Claude Code, Cursor) के लिए जो आउटपुट को पार्स करता है।

एक ही बाइनरी से आप कर सकते हैं:

- **अपना डेटा पढ़ें**: `sessions`, `events`, `evals`, `errors` (समय, एजेंट, वातावरण, स्कोर के अनुसार फ़िल्टर करें)।
- **अपने संगठन को प्रबंधित करें**: `keys`, `users`, `settings`, `alerts`, `incidents`।
- **विश्लेषण चलाएं**: सहेजी गई SQL और एक ad-hoc क्वेरी रनर (`query`)।
- **AI सहायक से पूछें**: वही read-only विश्लेषक जिससे आप डैशबोर्ड में बात करते हैं (`agent`)।

> **नोट:** यह `agenteye` CLI है, कलेक्टर डेमन (`agenteye-collector`) से एक अलग टूल है। CLI आपके डैशबोर्ड से बात करता है; कलेक्टर सर्वर को इवेंट भेजता है।

## शीघ्र प्रारंभ

शून्य से आपका पहला परिणाम चार लाइनों में। CLI को अपने डैशबोर्ड पर इंगित करें, साइन इन करें, पुष्टि करें कि आप कौन हैं, फिर पिछले दिन के रन खींचें:

```bash
pipx install agenteye
agenteye --base-url https://agenteye.example.com login --email you@example.com   # emailed 6-digit code
agenteye whoami                                                                  # confirm user + active org
agenteye --json sessions --since 24h                                             # one row per agent run, last 24h
```

वह अंतिम कमांड सबसे हाल के सेशन का एक JSON ऑब्जेक्ट प्रिंट करता है (सबसे नया पहले, डिफ़ॉल्ट रूप से 50 तक सीमित)। इसे `jq` में पाइप करें इसे स्लाइस करने के लिए, या `--json` को छोड़ दें एक बॉक्सबद्ध, रंगीन तालिका के लिए। प्रत्येक पंक्ति रन की स्थिति और, यदि किसी मूल्यांकनकर्ता ने इसे स्कोर किया है, तो इसके मीट्रिक स्कोर (यहाँ संक्षिप्त) ले जाती है:

```json
{
  "sessions": [
    {
      "session_id": "run-8f2a",
      "agent_id": "checkout-bot",
      "environment": "prod",
      "status": "error",
      "scores": { "helpfulness": 0.42, "tool_efficiency": 0.55 },
      "event_count": 37,
      "started_at": "2026-07-16T09:14:02Z",
      "last_event_at": "2026-07-16T09:14:48Z"
    }
  ],
  "next_cursor": null
}
```

इस पृष्ठ का बाकी हिस्सा प्रत्येक टुकड़े की व्याख्या करता है: [अलग से स्थापित करना](#installation), [साइन इन करना](#authentication), [कॉन्फ़िगरेशन](#configuration), प्रत्येक कमांड द्वारा साझा किए जाने वाले [वैश्विक सम्मेलन](#global-options--conventions), और [संपूर्ण कमांड संदर्भ](#command-reference)।

## स्थापना

CLI एक सार्वजनिक PyPI पैकेज है जिसका नाम **`agenteye`** है। इसे एक अलग-थलग वातावरण में स्थापित करें ताकि इसके अपने निर्भरताएं हमेशा हों:

```bash
pipx install agenteye
# या
uv tool install agenteye
```

इसे Python 3.10+ की आवश्यकता है। स्थापित कमांड **`agenteye`** है:

```bash
agenteye --version
agenteye --help
```

> **नोट:** Failproof AI Observability Python SDK भी `agenteye` वितरण नाम का उपयोग करता है। CLI को `pipx` या `uv tool` के साथ स्थापित करना (साझा virtualenv में `pip install` के बजाय) दोनों को टकराने से रोकता है। एक सादा `pip install agenteye` केवल तब ठीक है यदि SDK उसी वातावरण में स्थापित नहीं है।

## प्रमाणीकरण

CLI **डैशबोर्ड** के साथ एक ईमेल किए गए एकबारी कोड से प्रमाणित करता है:

```bash
agenteye login --email you@example.com
# आपको एक 6-अंकीय कोड ईमेल किया जाता है; इसे प्रॉम्प्ट पर पेस्ट करें।
```

सेशन टोकन `~/.agenteye/cli.json` में संग्रहीत है (केवल आपके द्वारा पठनीय, मोड `0600`) और डिफ़ॉल्ट रूप से 24 घंटे के लिए वैध है। जब यह समाप्त हो जाता है, तो `agenteye login` को फिर से चलाएं।

```bash
agenteye whoami     # show the current user, active org, and permissions
agenteye logout     # revoke the session and clear the stored token
```

`whoami` कभी भी एक गायब या समाप्त सेशन पर त्रुटि नहीं देता; इसके बजाय `logged_in: false` की रिपोर्ट करता है, इसलिए एक स्क्रिप्ट या एजेंट सुरक्षित रूप से प्रमाणन स्थिति का जांच कर सकता है (यदि कोई आधार URL सेट नहीं है या डैशबोर्ड अप्राप्य है तो यह अभी भी गैर-शून्य बाहर निकल सकता है)।

**आवश्यकताएं:** आपके ईमेल को डैशबोर्ड में साइन इन करने की अनुमति दी जानी चाहिए (अपने Failproof AI Observability प्रशासक से पूछें), और डैशबोर्ड अपने आधार URL पर पहुंचने योग्य होना चाहिए ([कॉन्फ़िगरेशन](#configuration) देखें)। यदि आप कोड का अनुरोध करते हैं और कोई आता नहीं है, तो आपके ईमेल को शायद अभी तक डैशबोर्ड एक्सेस के लिए सक्षम नहीं किया गया है।

## अपना संगठन चुनना (बहु-किरायेदार)

यदि आपका खाता एक से अधिक संगठनों से संबंधित है, तो **लॉगिन पर** सक्रिय एक चुनें; इसे सहेजा जाता है और हर बाद की कमांड के लिए उपयोग किया जाता है:

```bash
agenteye login --org acme           # authenticate and set the active tenant in one step
agenteye orgs list                  # the orgs you can access (the active one is marked)
agenteye orgs switch globex         # change the saved default
agenteye --org globex sessions      # override for a single command
```

यदि आप बिल्कुल एक संगठन के हैं तो यह स्वचालित रूप से चुना जाता है और आप `--org` को पूरी तरह से नजरअंदाज कर सकते हैं। यदि आप कई से संबंधित हैं और एक नहीं चुनते हैं, तो CLI उन्हें सूचीबद्ध करता है और आपको `--org <slug>` के साथ फिर से चलाने के लिए कहता है। सक्रिय संगठन प्रत्येक अनुरोध पर डैशबोर्ड को भेजा जाता है, और आपकी अनुमतियों को **प्रति संगठन** के आधार पर हल किया जाता है; `agenteye whoami` सक्रिय संगठन, इसमें आपकी अनुमतियां, और आपकी सभी सदस्यताएं दिखाता है।

## कॉन्फ़िगरेशन

| सेटिंग | फ़्लैग | वातावरण चर | डिफ़ॉल्ट |
|---|---|---|---|
| डैशबोर्ड आधार URL | `--base-url` | `AGENTEYE_DASHBOARD_URL` | **आवश्यक** (कोई डिफ़ॉल्ट नहीं) |
| सक्रिय org/tenant | `--org` | `AGENTEYE_ORG` | लॉगिन पर चुना; `~/.agenteye/cli.json` में सहेजा गया |
| सेशन टोकन | `--token` | `AGENTEYE_CLI_TOKEN` | `~/.agenteye/cli.json` से |
| JSON आउटपुट | `--json` | `AGENTEYE_CLI_JSON` | बंद |
| TLS सत्यापन छोड़ें | `--insecure` / `--secure` | `AGENTEYE_INSECURE` | बंद (लॉगिन पर सहेजा गया) |
| अनुरोध टाइमआउट (सेकंड) | `--timeout` | _(कोई नहीं)_ | 30 |
| उपयोग टेलीमेट्री अक्षम करें | _(कोई नहीं)_ | `AGENTEYE_ANALYTICS_DISABLED` (या `DO_NOT_TRACK`) | टेलीमेट्री वर्तमान में अक्षम है; कुछ नहीं भेजा जाता है |

संकल्प क्रम **फ़्लैग → वातावरण चर → कॉन्फ़िग फ़ाइल** है। कोई डिफ़ॉल्ट नहीं है; आपको CLI को अपने डैशबोर्ड पर इंगित करना चाहिए, प्रति-कमांड (`--base-url https://agenteye.example.com`) या एक बार पर्यावरण के माध्यम से (यह आपके पहले `login` के बाद भी सहेजा जाता है):

```bash
export AGENTEYE_DASHBOARD_URL=https://agenteye.example.com
```

कॉन्फ़िगरेशन निर्देशिका `AGENTEYE_HOME` को सम्मानित करती है (SDK और कलेक्टर द्वारा उपयोग किया जाने वाला एक ही सम्मेलन); यदि सेट है, तो `cli.json` `$AGENTEYE_HOME/cli.json` में रहता है।

### स्व-हस्ताक्षरित या आंतरिक TLS

यदि आपका डैशबोर्ड HTTPS पर स्व-हस्ताक्षरित या आंतरिक प्रमाणपत्र (उदाहरण के लिए, एक कच्चा लोड-बैलेंसर होस्टनाम) के साथ परोसा जाता है, तो TLS सत्यापन इसे `CERTIFICATE_VERIFY_FAILED` त्रुटि के साथ अस्वीकार करता है। प्रमाणपत्र सत्यापन छोड़ने के लिए `--insecure` पास करें:

```bash
agenteye --base-url https://agenteye.internal --insecure login
```

`--insecure` **आपके लॉगिन पर `cli.json` में सहेजा जाता है**, इसलिए बाद की कमांड स्वचालित रूप से सत्यापन को छोड़ देती हैं; आपको फ़्लैग को दोहराना नहीं होता है। एक एकबारी सत्यापित कॉल के लिए `--secure` पास करें, या अपने अगले लॉगिन पर सत्यापन को फिर से सक्षम करने के लिए। CLI सेशन को स्टडआउट पर प्रत्येक कमांड से पहले डैशबोर्ड से संपर्क करते हुए एक चेतावनी प्रिंट करता है जबकि सत्यापन अक्षम है। सत्यापन को छोड़ना मैन-इन-द-द-मिडल हमलों से सुरक्षा हटाता है; अपने डैशबोर्ड के नेटवर्क पाथ पर निर्भर करने से पहले सुनिश्चित करें कि आप इस पर भरोसा करते हैं (VPN, निजी सबनेट, आदि)।

## टेलीमेट्री और गोपनीयता

> **नोट:** शिप किया गया CLI आज **कोई उपयोग टेलीमेट्री नहीं भेजता है।** एक मास्टर किल स्विच चालू है, इसलिए कुछ भी अपने वातावरण की परवाह किए बिना प्रेषित नहीं होता है। नीचे दिया गया अनुभाग opt-out क्षमता का वर्णन करता है यदि और जब टेलीमेट्री कभी सक्षम हो।

यहां तक कि जब सक्षम हो, टेलीमेट्री **केवल गुमनाम उपयोग विश्लेषण** होगा, कभी आपके एजेंट, सेशन, या इवेंट डेटा नहीं:

- **कोई एजेंट, सेशन, या इवेंट डेटा कभी आपके बुनियादी ढांचे को नहीं छोड़ता है।** केवल CLI उपयोग की रिपोर्ट की जाएगी: कमांड और उप-कमांड नाम (जैसे `keys create`), **नाम** आपने जो फ़्लैग का उपयोग किया (कभी उनके मान नहीं), सफलता/निकास स्थिति, और अवधि, साथ ही परिवर्तन के लिए प्रति-कार्रवाई इवेंट (जैसे `api_key_created`, `query_run`) केवल स्थिर नाम/enums और मोटे गणनाएं ले जाते हुए। आपके डैशबोर्ड URL, सेशन टोकन, ईमेल, संगठन स्लग, संसाधन ids, SQL, मुख्य रहस्य, और क्वेरी फ़िल्टर **कभी** नहीं भेजे जाएंगे। ऑपरेटरों को केवल एक अपारदर्शी आंतरिक id द्वारा पहचाना जाएगा, कभी ईमेल द्वारा नहीं।
- **पहले से ही opt out करें** `AGENTEYE_ANALYTICS_DISABLED=1` को CLI के वातावरण में सेट करके (CLI क्रॉस-टूल `DO_NOT_TRACK=1` सम्मेलन को भी सम्मानित करता है)। यह प्रभाव तब डालता है जब टेलीमेट्री कभी चालू होता है, इसलिए एक गोपनीयता-सचेत वातावरण स्थायी रूप से opted out रह सकता है।
- यदि टेलीमेट्री सक्षम थे, तो CLI सीधे PostHog (`https://us.i.posthog.com`) को भेजेगा; वह होस्ट ब्लॉक किए गए एक मशीन चुप्पी से कुछ नहीं भेजेगी और CLI प्रभावित नहीं होगा।

## वैश्विक विकल्प और सम्मेलन

यह एक बार पढ़ें; यह प्रत्येक कमांड पर लागू होता है।

- **वैश्विक विकल्प कमांड से पहले जाते हैं।** `agenteye --json sessions` सही है; `agenteye sessions --json` एक उपयोग त्रुटि है। वैश्विक `--json`, `--base-url`, `--org`, `--token`, `--insecure`/`--secure`, `--timeout`, `--quiet`, और `--no-color` हैं।
- **`--json` स्टडआउट को शुद्ध JSON प्रिंट करता है, और कुछ भी नहीं।** मानव स्थिति लाइनें, चेतावनियां, और त्रुटियां **स्टडआउट** को जाती हैं, इसलिए एक `--json` स्टडआउट कैप्चर जब एक स्थिति लाइन दिखाई जाए भी `jq` में पाइप करने के लिए स्वच्छ रहता है। `--json` के बिना आप मानव आंखों के लिए एक बॉक्सबद्ध, रंगीन दृश्य प्राप्त करते हैं।
- **`--help` के साथ खोजें।** प्रत्येक कमांड और उप-कमांड में `--help` (`-h` उपनाम) है: `agenteye -h`, `agenteye sessions -h`, `agenteye keys create -h`। शीर्ष-स्तर की सहायता भी निकास कोड और वैश्विक विकल्प सूचीबद्ध करती है। कोई वैश्विक मशीन-पठनीय सतह डंप नहीं है; प्रति-कमांड `--help` का उपयोग करें, साथ ही उन दोनों रजिस्ट्रीज़ के लिए डोमेन-विशिष्ट `agenteye query schema` और `agenteye settings schema`।
- **पुष्टिकरण स्क्रिप्ट और एजेंटों के लिए स्वचालित-छोड़।** Create/update/delete कमांड एक इंटरैक्टिव टर्मिनल में "क्या आप सुनिश्चित हैं?" प्रॉम्प्ट करते हैं, लेकिन **जब `--json` के अंतर्गत या जब स्टडइन एक TTY नहीं है तो उस प्रॉम्प्ट को स्वचालित-छोड़ दें** (एक TTY एक इंटरैक्टिव टर्मिनल सेशन है; एक पाइप या CI रनर नहीं है), इसलिए स्क्रिप्ट और एजेंट कभी हैंग नहीं करते हैं। स्पष्ट रूप से छोड़ने के लिए `--yes`/`-y` पास करें। क्योंकि प्रॉम्प्ट एक एजेंट के लिए फायर नहीं करेगा, एक एजेंट को मानव के साथ विनाशकारी कार्यों की पुष्टि करनी चाहिए।
- **पेजिनेशन:** परिणाम नए-पहले हैं और कर्सर-पेजिनेटेड हैं (प्रत्येक पेज एक टोकन लौटाता है जिसे आप अगले को फेच करने के लिए उपयोग करते हैं)। `--limit N` (उपनाम `-n`) पंक्तियों को कैप करता है और **डिफ़ॉल्ट रूप से 50**; `--all` स्वचालित-पेजिनेट करता है (200-पंक्ति चंक में) **`--limit` तक**, इसलिए एक अनावृत्त `--all` अभी भी 50 पर रुकता है। एक पूर्ण स्वीप के लिए एक उच्च स्पष्ट कैप पास करें: `--all --limit 1000`। `--page-size N` प्रति-अनुरोध चंक को नियंत्रित करता है (अधिकतम 200); `--cursor <id>` एक पूर्व पेज के `next_cursor` से फिर से शुरू करता है।
- **समय फ़िल्टर:** `--since` एक सापेक्ष विंडो लेता है: `15m`, `1h`, `6h`, `24h`, `7d`, या `all` (डैशबोर्ड की पूर्वनिर्धारितियां)। एक लंबी या कस्टम रेंज के लिए (कहें पिछले 30 दिन), `--from`/`--to` का उपयोग करें: स्पष्ट ISO-8601 UTC टाइमस्टैम्प **`T` और एक टाइमज़ोन के साथ** (जैसे `2026-06-01T00:00:00Z`) जो `--since` को override करता है। एक स्पेस-अलग या टाइमज़ोन-रहित मान एक उपयोग त्रुटि है।
- **`--fields a,b,c`** (on `events`, `sessions`, `evals`, `errors`) आउटपुट को उन कुंजियों तक सीमित करता है, टेबल और `--json` दोनों के लिए। अज्ञात नाम वैध सूची के साथ अस्वीकार किए जाते हैं, फ़ील्ड नामों को खोजने का एक सस्ता तरीका।
- **`--file payload.json`** (या `--file -` स्टडइन को पढ़ने के लिए) एक पूर्ण JSON अनुरोध बॉडी प्रदान करता है जहां एक संसाधन में एक जटिल आकार है (on `alerts create/update`, `settings set`, और `users create/update`)। सहेजी गई-क्वेरी SQL `--sql @file.sql` का उपयोग करता है।
- **बहु-मान फ़िल्टर** कोमा-अलग हैं → एक सेट के रूप में मेल खाते हैं (एक फ़िल्टर के भीतर संघ, फ़िल्टर भर में AND): `--event-type tool_use,tool_result`। क्लिक विकल्प भिन्नात्मक नहीं हैं, इसलिए `--add a b` टूट जाता है। `--add a,b`, फ़्लैग (`--add a --add b`) को दोहराएं, या उद्धरण (`--add "a b"`)।

## कमांड संदर्भ

### आप इन 5 कमांड का सबसे अधिक उपयोग करेंगे

अधिकांश दैनिक कार्य कुछ ही read कमांड के माध्यम से चलते हैं। यहाँ शुरू करें, फिर नीचे पूरी सतह तक पहुंचें जब आपको इसकी आवश्यकता हो:

| कमांड | यह क्या करता है | इसे आजमाएं |
|---|---|---|
| `sessions` | एजेंट रन प्रति एक पंक्ति: समय, env, एजेंट, स्थिति, नवीनतम स्कोर। | `agenteye --json sessions --since 24h --status error` |
| `events` | एक रन के अंदर कच्चा प्रति-चरण ट्रेल (`--full` के लिए पेलोड जोड़ें)। | `agenteye --json events --session-id run-001 --all` |
| `evals` | मूल्यांकन परिणाम और स्कोर; `--aggregate` उन्हें उठाता है। | `agenteye --json evals --aggregate --since 7d --env prod` |
| `errors` | बस की गई त्रुटि इवेंट; `--aggregate` प्रकार द्वारा गणना के लिए। | `agenteye --json errors --since 24h --aggregate` |
| `list` | वैध फ़िल्टर मान खोजें (एजेंट, envs, मॉडल, …)। | `agenteye list agents` |

### CLI सब कुछ कर सकता है

पूरी सतह निम्नानुसार है। CLI में **18 शीर्ष-स्तर कमांड** हैं। सभी read कमांड `--json` और ऊपर दिए गए वैश्विक विकल्पों को स्वीकार करते हैं; किसी भी एक के लिए संपूर्ण फ़्लैग सूची और JSON आकार के लिए `agenteye <command> -h` (या `<command> <subcommand> -h`) चलाएं।

### पहचान: `login` · `logout` · `whoami` · `orgs` · `version` · `help`

```bash
agenteye login --email you@example.com [--org acme]   # emailed one-time code; saves the session
agenteye logout                                       # clear the saved session on this machine
agenteye whoami                                       # current user, active org, permissions
agenteye version                                      # print the CLI version (same as --version)
agenteye help                                         # top-level help (same as --help)
```

`orgs` सक्रिय tenant की जांच और स्विच करता है:

```bash
agenteye orgs list        # your orgs + your role in each (active one marked)
agenteye orgs switch acme # change the saved active org (omit the slug to pick from a list on a TTY)
agenteye orgs current     # identity card for the active org
agenteye orgs perms       # your permissions in the active org, grouped by resource
```

### निरीक्षण (read-only): `events` · `sessions` · `evals` · `errors` · `list`

इनमें से कोई भी पुष्टि की आवश्यकता नहीं है। साझा फ़िल्टर: `--session-id`, `--agent-id`, `--env` (**नहीं** `--environment`), और समय रेंज (`--since` / `--from` / `--to`)।

```bash
# events (alias: the raw per-step trail), newest first
agenteye --json events --session-id run-001 --event-type tool_use,tool_result --all --limit 1000
agenteye --json events --since 1h --search timeout --all | jq '.events[].payload'

# sessions: one row per agent run (time/env/agent/session/status; no score filtering)
agenteye --json sessions --since 24h --status error
agenteye --json sessions --agent-id checkout-bot --env prod --all --limit 1000

# evals: evaluation results + scores; --score filters by metric, --aggregate rolls up
agenteye --json evals --score helpfulness:0.5..0.8 --score tool_efficiency:..0.3
agenteye --json evals --aggregate --since 7d --env prod        # status mix + per-key score stats

# errors: errored events; --aggregate for counts/sessions/agents/last-seen
agenteye --json errors --since 24h --aggregate
agenteye --json errors --since 24h --error-type timeout --all --limit 1000

# list: discover valid filter values before you filter
agenteye list envs        # also: agents event_types score_filters models hooks tools error_types
```

`--score KEY:MIN..MAX` (on **`evals`**, `sessions` पर नहीं) दोहराया जा सकता है और AND-संयोजित है; किसी भी बाध्यता को छोड़ा जा सकता है (`..0.5` मतलब ≤ 0.5, `0.9..` मतलब ≥ 0.9)। प्रति अनुरोध 20 स्कोर फ़िल्टर तक। `evals --scores-full` **मानव टेबल केवल** के लिए एक प्रदर्शन फ़्लैग है; यह पहले कुछ के बजाय प्रत्येक स्कोर जोड़ी दिखाता है साथ `+N` गणना। `--json` के अंतर्गत कोई प्रभाव नहीं है, जो हमेशा संपूर्ण स्कोर ऑब्जेक्ट लौटाता है। **एक सेशन अंत-अंत तक पढ़ने के लिए**, इवेंट ट्रेल को इसके मूल्यांकन के साथ संयोजित करें:

```bash
agenteye --json events --session-id run-001 --all --limit 1000 | jq '.events[] | {ts, event_type}'
agenteye --json evals  --session-id run-001                       # its scores + status
```

### प्रबंधित करें (अनुमति-gated): `keys` · `users` · `settings` · `alerts` · `incidents`

**`keys`**: API कुंजियां। गुप्त स्थानीय रूप से उत्पन्न होता है, सर्वर को भेजा जाता है (जो केवल एक हैश संग्रहीत करता है), और create/regenerate पर **एक बार दिखाया जाता है**; फिर इसे कैप्चर करें। `--json` के साथ यह केवल `key` फ़ील्ड में दिखाई देता है। **नाम** द्वारा संदर्भित।

```bash
agenteye keys list                                   # active keys first, then revoked
agenteye keys show ci-bot
agenteye keys create ci-bot --add events:read.add    # scope to what you need; prints the secret ONCE
agenteye keys create ops --permission-set standard --remove queries:run   # seed a preset, then trim
agenteye keys update ci-bot --add evaluations:read --yes
agenteye keys regenerate ci-bot --yes                # rotate the secret (the old one stops working)
agenteye keys disable ci-bot --yes                   # revoke
```

अनुमतियां `(permission-set ∪ --add) − --remove` के रूप में काम करती हैं। टोकन `slug:action` (जैसे `events:read`) या `slug:action.action` एक संसाधन पर कई को expand करने के लिए (`events:read.add` → `events:read`, `events:add`)। Presets: `read-only`, `standard`, `admin`। मानव-केवल अनुमतियां (`keys:update`) एक कुंजी को नहीं दी जा सकतीं।

**`users`**: org सदस्य, **ईमेल** द्वारा संदर्भित (एक UUID id भी स्वीकार किया जाता है)।

```bash
agenteye users list [--active-only]
agenteye users show dev@corp.com
agenteye users create dev@corp.com --permission-set standard
agenteye users update dev@corp.com --add alerts:write --remove queries:delete   # predicts + confirms
agenteye users disable dev@corp.com --yes            # has protected/self guards
agenteye users enable dev@corp.com
```

**`settings`**: एक निश्चित रजिस्ट्री (आप मौजूदा कुंजियों को पढ़ते और बदलते हैं; आप नई कुंजियां नहीं बना सकते)।

```bash
agenteye settings list                               # key · value · type · updated (secrets masked)
agenteye settings schema                             # what each key accepts (type · range · description)
agenteye settings set session_ttl_secs --value 86400 --yes
```

**`alerts`**: सतर्कता परिभाषाएं, **नाम** द्वारा संदर्भित। `create` एक स्थिति NAME साथ फ़्लैग या `--file` के माध्यम से एक पूर्ण JSON बॉडी लेता है।

```bash
agenteye alerts list
agenteye alerts show high-errors
agenteye alerts create high-errors --file alert.json          # NAME is required (positional)
agenteye alerts update high-errors --severity critical --yes
agenteye alerts test high-errors --yes                        # fire a test notification
agenteye alerts delete high-errors --yes
```

**`incidents`**: सतर्कता घटनाएं, id द्वारा संदर्भित (छोटे ids स्वीकार किए गए)। `show` पूर्ण गतिविधि लॉग प्रिंट करता है; कार्य करने से पहले इसे पढ़ें।

```bash
agenteye incidents list --state firing                # also: acknowledged, resolved
agenteye incidents count
agenteye incidents show <id>
agenteye incidents ack <id>
agenteye incidents assign <id> you@corp.com           # assignee must be an operator
agenteye incidents resolve <id> --yes
agenteye incidents open --alert-id <id> --severity critical    # open one manually against an alert
agenteye incidents comment-add <id> "root cause: upstream 5xx"
agenteye incidents comment-list <id> ; agenteye incidents comment-delete <id> <comment-id>
agenteye incidents subscribe <id> ; agenteye incidents unsubscribe <id> ; agenteye incidents subscribers <id>
```

### विश्लेषण और सहायक: `query` · `agent`

**`query`**: आपके विश्लेषण स्टोर के विरुद्ध सहेजी गई SQL साथ ad-hoc रनर। सहेजी गई क्वेरीज़ **नाम** द्वारा संदर्भित; SQL सर्वर-पक्ष सत्यापित है (SELECT/WITH केवल, statement टाइमआउट, पंक्ति कैप)।

```bash
agenteye query schema [TABLE]                         # column layout of the analytics views
agenteye query run --sql "select count(*) from analytics.events"
agenteye query run errs --arg prod --limit 100        # run a saved query + a positional $1
agenteye query list ; agenteye query show errs
agenteye query create errs --sql @errs.sql --description "errored events (24h)"
agenteye query update errs --sql @errs.sql --yes ; agenteye query delete errs --yes
```

**`agent`**: निर्मित **AI सहायक** से बातें करता है (वही read-only विश्लेषक जिससे आप डैशबोर्ड में बात कर सकते हैं)। चैट एक छोटे chat-id द्वारा संदर्भित हैं (prefix-resolved)।

```bash
agenteye agent health                                 # is the AI assistant configured/reachable
agenteye agent models                                 # models you can pass to --model (default marked)
agenteye agent ask "which agents errored most in the last day?"    # starts a chat; prints its short id
agenteye agent ask --chat <short-id> "and which tools did they call?"   # continue that chat
agenteye agent chats ; agenteye agent show <short-id>
agenteye agent rename <short-id> --title "error triage" ; agenteye agent delete <short-id>
```

## निकास कोड

| कोड | अर्थ |
|---|---|
| 0 | सफलता |
| 1 | अप्रत्याशित त्रुटि (जैसे डैशबोर्ड ने 5xx लौटाया) |
| 2 | उपयोग त्रुटि (अमान्य तर्क, अज्ञात कमांड/फ़्लैग, नाम टकराव) |
| 3 | डैशबोर्ड तक नहीं पहुंच सकते |
| 4 | लॉगिन नहीं है या सेशन समाप्त; `agenteye login` चलाएं |
| 5 | प्रमाणीकृत, लेकिन आपके खाते में आवश्यक अनुमति की कमी है (संदेश इसका नाम रखता है) |
| 6 | अनुरोधित संसाधन नहीं मिला (जैसे अज्ञात सेशन या घटना id) |

ये CLI को स्क्रिप्ट करने के लिए सुरक्षित बनाते हैं: एक कोडिंग एजेंट पुनः-प्रमाणीकरण करने के लिए आपको प्रॉम्प्ट करने के लिए `4` पर branch कर सकता है, या `5` पर गायब अनुमति को सतह करने के लिए। निकास-कोड-हैंडलिंग पैटर्न और JSON आउटपुट आकार के लिए [CLI एजेंट के लिए व्यंजन](/hi/agenteye/cli-recipes) देखें।
