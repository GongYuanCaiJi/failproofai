## आधिकारिक पैकेज

कैनोनिकल npm पैकेज है **`failproofai`**:

```bash
npm install -g failproofai
# या
bun add -g failproofai
```

---

## हम एलियास नामों के मालिक क्यों हैं

टाइपोस्क्वाटिंग एक सामान्य सप्लाई-चेन हमला है जहां एक दुर्भावनापूर्ण व्यक्ति एक पैकेज नाम पंजीकृत करता है जो एक लोकप्रिय पैकेज से एक कुंजीस्ट्रोक दूर होता है। जो उपयोगकर्ता इंस्टॉल कमांड में गलती करते हैं वे आक्रमणकारी-नियंत्रित कोड चलाते हैं जिसके पास पूर्ण सिस्टम एक्सेस होता है - यह ठीक वही प्रकार का खतरा है जिसके खिलाफ Failproof AI की रक्षा करने के लिए डिज़ाइन किया गया है।

इस सतह को समाप्त करने के लिए, **हम पूर्वनिर्धारित रूप से `failproofai` की सभी सामान्य गलतिलियों और स्वरूपण वेरिएंट के मालिक हैं** npm पर। इनमें से कोई भी नाम एक तीसरे पक्ष द्वारा पंजीकृत नहीं हो सकता है। प्रत्येक एक पतला प्रॉक्सी है जो वास्तविक `failproofai` पैकेज को इंस्टॉल करता है और प्रतिनिधि देता है।

## पंजीकृत एलियासेस

**स्वरूपण वेरिएंट** - "failproof ai" लिखने के विभिन्न तरीके:

| पैकेज | स्थिति |
|---------|--------|
| `failproof` | ✅ प्रकाशित |
| `failproof-ai` | ⏳ npm समर्थन की प्रतीक्षा में |
| `fail-proof-ai` | ⏳ npm समर्थन की प्रतीक्षा में |
| `failproof_ai` | ⏳ npm समर्थन की प्रतीक्षा में |
| `fail_proof_ai` | ⏳ npm समर्थन की प्रतीक्षा में |
| `fail-proofai` | ⏳ npm समर्थन की प्रतीक्षा में |

**`failprof*` टाइपो** - "proof" से एक `o` लुप्त है:

| पैकेज | स्थिति |
|---------|--------|
| `failprof` | ✅ प्रकाशित |
| `failprof-ai` | ✅ प्रकाशित |
| `failprofai` | ⏳ npm समर्थन की प्रतीक्षा में |
| `fail-prof-ai` | ⏳ npm समर्थन की प्रतीक्षा में |
| `failprof_ai` | ⏳ npm समर्थन की प्रतीक्षा में |

**`faliproof*` टाइपो** - `a` और `i` स्थानांतरित:

| पैकेज | स्थिति |
|---------|--------|
| `faliproof` | ✅ प्रकाशित |
| `faliproof-ai` | ✅ प्रकाशित |
| `faliproofai` | ⏳ npm समर्थन की प्रतीक्षा में |

> **प्रतीक्षा में क्यों?** npm की स्पैम-रोकथाम नीति ऐसे नामों को ब्लॉक करती है जो विद्यमान पैकेज के समान स्ट्रिंग में सामान्यीकृत होते हैं विराम चिन्ह हटाने और समानता जांचों को चलाने के बाद। हम इन नामों को एंटी-स्क्वाटिंग उद्देश्यों के लिए आरक्षित करने के लिए npm समर्थन से संपर्क किया है। एक बार अनुमोदित होने के बाद ये सक्रिय कर दिए जाएंगे।

आप सत्यापित कर सकते हैं कि कोई भी प्रकाशित एलियास हमारे द्वारा है:

```bash
npm info failproof
# देखें: रखरखाव कर्ता क्षेत्र में "ExosphereHost Inc."
```

## एलियासेस कैसे काम करते हैं

प्रत्येक एलियास पैकेज:

1. `failproofai` को एक निर्भरता के रूप में सूचीबद्ध करता है - इसलिए वास्तविक पैकेज (इसके `postinstall` हुक सेटअप सहित) इंस्टॉल पर चलता है
2. अपने स्वयं के नाम के साथ मेल खाने वाली एक बाइनरी को उजागर करता है (उदाहरण के लिए `failprof-ai`) जो सभी तर्कों को `failproofai` बाइनरी के लिए प्रॉक्सी करता है

प्रॉक्सी एक दो-पंक्ति Node स्क्रिप्ट है; कोई तर्क नहीं है, कोई नेटवर्क कॉल नहीं है, और कोई डेटा संग्रह नहीं है जो `failproofai` स्वयं करता है उससे परे।
