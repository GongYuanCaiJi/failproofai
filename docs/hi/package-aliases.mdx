## आधिकारिक पैकेज

कैनोनिकल npm पैकेज **`failproofai`** है:

```bash
npm install -g failproofai
# or
bun add -g failproofai
```

---

## हम एलिएस नामों का मालिक क्यों हैं

टाइपोस्क्वाटिंग एक सामान्य सप्लाई-चेन अटैक है जहां एक दुर्भावनापूर्ण व्यक्ति एक पैकेज नाम रजिस्टर करता है जो एक लोकप्रिय पैकेज से एक कीस्ट्रोक दूर होता है। जो उपयोगकर्ता इंस्टॉल कमांड में गलती करते हैं, वे अटैकर-नियंत्रित कोड को पूर्ण सिस्टम एक्सेस के साथ चलाते हैं - बिल्कुल वही प्रकार का खतरा जिसे Failproof AI से बचाव के लिए डिज़ाइन किया गया है।

इस सतह को समाप्त करने के लिए, **हम npm पर `failproofai` के सभी सामान्य गलतियों और फॉर्मेटिंग वेरिएंट्स का पूर्वसक्रिय रूप से मालिक हैं**। इनमें से कोई भी नाम किसी तीसरे पक्ष द्वारा रजिस्टर नहीं किया जा सकता है। प्रत्येक एक पतली प्रॉक्सी है जो असली `failproofai` पैकेज को इंस्टॉल करता है और डिलीगेट करता है।

## रजिस्टर की गई एलिएस

**फॉर्मेटिंग वेरिएंट्स** - "failproof ai" लिखने के विभिन्न तरीके:

| पैकेज | स्थिति |
|---------|--------|
| `failproof` | ✅ प्रकाशित |
| `failproof-ai` | ⏳ npm समर्थन की प्रतीक्षा |
| `fail-proof-ai` | ⏳ npm समर्थन की प्रतीक्षा |
| `failproof_ai` | ⏳ npm समर्थन की प्रतीक्षा |
| `fail_proof_ai` | ⏳ npm समर्थन की प्रतीक्षा |
| `fail-proofai` | ⏳ npm समर्थन की प्रतीक्षा |

**`failprof*` टाइपो** - "proof" से एक `o` गायब है:

| पैकेज | स्थिति |
|---------|--------|
| `failprof` | ✅ प्रकाशित |
| `failprof-ai` | ✅ प्रकाशित |
| `failprofai` | ⏳ npm समर्थन की प्रतीक्षा |
| `fail-prof-ai` | ⏳ npm समर्थन की प्रतीक्षा |
| `failprof_ai` | ⏳ npm समर्थन की प्रतीक्षा |

**`faliproof*` टाइपो** - `a` और `i` को स्थानांतरित किया गया:

| पैकेज | स्थिति |
|---------|--------|
| `faliproof` | ✅ प्रकाशित |
| `faliproof-ai` | ✅ प्रकाशित |
| `faliproofai` | ⏳ npm समर्थन की प्रतीक्षा |

> **प्रतीक्षा क्यों है?** npm की स्पैम-प्रिवेंशन नीति उन नामों को ब्लॉक करती है जो विद्यमान पैकेज के समान स्ट्रिंग में सामान्य होते हैं विराम चिन्ह हटाने और समानता जांच चलाने के बाद। हमने इन नामों को एंटी-स्क्वाटिंग उद्देश्यों के लिए रिजर्व करने के लिए npm समर्थन से संपर्क किया है। अनुमोदन के बाद वे सक्रिय किए जाएंगे।

आप किसी भी प्रकाशित एलिएस को सत्यापित कर सकते हैं कि यह हमारी है:

```bash
npm info failproof
# देखें: रखरखाव कर्ताओं के क्षेत्र में "ExosphereHost Inc."
```

## एलिएस कैसे काम करती हैं

प्रत्येक एलिएस पैकेज:

1. `failproofai` को एक निर्भरता के रूप में सूचीबद्ध करता है - इसलिए असली पैकेज (इसके `postinstall` हुक सेटअप सहित) इंस्टॉल पर चलता है
2. अपने नाम से मेल खाने वाली एक बाइनरी को उजागर करता है (जैसे `failprof-ai`) जो `failproofai` बाइनरी को सभी तर्कों के साथ प्रॉक्सी करता है

प्रॉक्सी एक दो-लाइन Node स्क्रिप्ट है; कोई तर्क नहीं है, कोई नेटवर्क कॉल नहीं है, और `failproofai` स्वयं जो करता है उससे परे कोई डेटा संग्रह नहीं है।
