failproofai डैशबोर्ड आपके AI एजेंट सेशन की निगरानी करने और नीतियों का प्रबंधन करने के लिए एक स्थानीय वेब एप्लिकेशन है। देखें कि आपके एजेंट आपके दूर रहते समय क्या कर रहे थे।

---

## डैशबोर्ड शुरू करना

```bash
failproofai
```

`http://localhost:8020` पर खुलता है।

डैशबोर्ड सीधे फाइल सिस्टम से पढ़ता है - आपकी Claude Code प्रोजेक्ट फोल्डर और failproofai कॉन्फ़िग फाइलें। किसी दूरस्थ सेवा को कुछ भी नहीं लिखा जाता है।

---

## पृष्ठ

### प्रोजेक्ट

आपकी मशीन पर मिले सभी Claude Code, OpenAI Codex, GitHub Copilot CLI _(बीटा)_, Cursor Agent _(बीटा)_, OpenCode _(बीटा)_, Pi _(बीटा)_, और Gemini CLI _(बीटा)_ प्रोजेक्ट को सूचीबद्ध करता है। Claude प्रोजेक्ट `~/.claude/projects/` (या `CLAUDE_PROJECTS_PATH` द्वारा निर्धारित पथ) से खोजे जाते हैं; Codex प्रोजेक्ट `~/.codex/sessions/<YYYY>/<MM>/<DD>/*.jsonl` के तहत हर ट्रांसक्रिप्ट को स्कैन करके और प्रत्येक सेशन के पहले रिकॉर्ड में दर्ज `cwd` के अनुसार समूहीकृत किए जाते हैं; Copilot CLI प्रोजेक्ट प्रत्येक `~/.copilot/session-state/<sessionId>/workspace.yaml` (`COPILOT_HOME` के माध्यम से कॉन्फ़िगर करने योग्य) को स्कैन करके और इसके `cwd` फील्ड के अनुसार समूहीकृत किए जाते हैं; Cursor Agent प्रोजेक्ट `~/.cursor/agent-sessions/<sessionId>/` के तहत प्रति-सेशन मेटाडेटा को स्कैन करके (`CURSOR_HOME` के माध्यम से कॉन्फ़िगर करने योग्य, `conversations/` और `sessions/` को फॉलबैक के रूप में खोजा जाता है) `meta.json` / `session.json` / `workspace.yaml` में एक `cwd` स्केलर के लिए खोजे जाते हैं; OpenCode प्रोजेक्ट `~/.local/share/opencode/opencode.db` पर इसके SQLite DB को `opencode db --format json` के माध्यम से क्वेरी करके खोजे जाते हैं (हम `session` और `project` टेबल पढ़ते हैं और `project_id` के अनुसार समूहीकृत करते हैं); Pi प्रोजेक्ट `~/.pi/agent/sessions/<encoded-cwd>/<timestamp>_<uuid>.jsonl` के तहत प्रति-सेशन JSONL ट्रांसक्रिप्ट को स्कैन करके (`PI_SESSIONS_DIR` के माध्यम से कॉन्फ़िगर करने योग्य) और प्रत्येक सेशन के पहले रिकॉर्ड से `cwd` निकालकर खोजे जाते हैं; Gemini CLI प्रोजेक्ट `~/.gemini/tmp/<basename>/chats/session-<timestamp>-<uuid-prefix>.jsonl` (`GEMINI_SESSIONS_DIR` के माध्यम से कॉन्फ़िगर करने योग्य) को स्कैन करके और सहायक `.project_root` टेक्स्ट मार्कर से विहित cwd को पुनः प्राप्त करके खोजे जाते हैं। एक प्रोजेक्ट जिसे कई CLI द्वारा उपयोग किया गया है, सभी मेल खाने वाले बैज के साथ एक एकल पंक्ति के रूप में प्रदर्शित होता है। एक विशिष्ट एजेंट CLI द्वारा फ़िल्टर करने के लिए तालिका के ऊपर **CLI** ड्रॉपडाउन का उपयोग करें; URL आपकी चयन को `?cli=claude|codex|copilot|cursor|opencode|pi|gemini` के रूप में संरक्षित रखता है।

प्रत्येक प्रोजेक्ट दिखाता है:
- प्रोजेक्ट का नाम (फोल्डर पथ से व्युत्पन्न)
- एक CLI बैज — `Claude Code` (नारंगी), `OpenAI Codex` (बैंगनी), `GitHub Copilot` (नीला), `Cursor Agent` (पन्ना), `OpenCode` (एम्बर), `Pi` (गुलाबी), और/या `Gemini CLI` (आसमानी)
- सबसे हाल की सेशन गतिविधि की तारीख

इसके सेशन को देखने के लिए किसी प्रोजेक्ट पर क्लिक करें।

### सेशन

किसी प्रोजेक्ट के भीतर सभी सेशन को सूचीबद्ध करता है। प्रत्येक सेशन दिखाता है:
- सेशन ID
- प्रारंभ और समाप्ति टाइमस्टैम्प
- टूल कॉल की संख्या
- हुक गतिविधि गणना (नीतियां जो सक्रिय हुईं)

सूची को कम करने के लिए दिनांक श्रेणी फ़िल्टर और सेशन ID खोज का उपयोग करें। सेशन पेजीकृत हैं।

सेशन दर्शक खोलने के लिए किसी सेशन पर क्लिक करें।

### सेशन दर्शक

सेशन दर्शक स्वायत्त एजेंट के लिए मुख्य प्रश्न का उत्तर देता है: एजेंट ने क्या किया, और क्या यह ट्रैक पर रहा? हेडर के बगल में एक CLI बैज इंगित करता है कि सेशन Claude Code, OpenAI Codex, GitHub Copilot CLI, Cursor Agent, OpenCode, Pi, या Gemini CLI ट्रांसक्रिप्ट है। यह एक सेशन में हुई हर चीज का एक समयरेखा दिखाता है:

- **संदेश** - Claude के टेक्स्ट प्रतिक्रियाएं और उपयोगकर्ता प्रॉम्प्ट
- **टूल कॉल** - हर टूल जिसे Claude ने आह्वान किया, इसके इनपुट और आउटपुट के साथ
- **नीति गतिविधि** - प्रत्येक टूल कॉल के लिए, कौन सी नीतियां सक्रिय हुईं और उन्होंने क्या निर्णय लौटाया

शीर्ष पर सांख्यिकी बार सेशन की अवधि, कुल टूल कॉल, और हुक निर्णयों का सारांश (अनुमति / अस्वीकार / निर्देश गणना) दिखाता है।

सेशन को निर्यात करने के लिए **लॉग डाउनलोड करें** बटन पर क्लिक करें। Claude Code, Codex, Copilot, Cursor, Pi, और Gemini सेशन के लिए आप मूल ऑन-डिस्क JSONL ट्रांसक्रिप्ट बाइट-फॉर-बाइट प्राप्त करते हैं; OpenCode (जिसके सेशन SQLite में रहते हैं, डिस्क पर नहीं) के लिए आप अंतर्निहित `session` / `messages` / `parts` टेबल को दर्पण करने वाली एक JSON दस्तावेज प्राप्त करते हैं।

### नीतियां

नीतियों को प्रबंधित करने और गतिविधि की समीक्षा करने के लिए एक दो-टैब पृष्ठ।

<Tabs>
  <Tab title="नीतियां टैब">
  - एकल क्लिक के साथ व्यक्तिगत नीतियों को चालू या बंद करें (`~/.failproofai/policies-config.json` में लिखता है)
  - किसी नीति के पैरामीटर को कॉन्फ़िगर करने के लिए इसे विस्तारित करें (उन नीतियों के लिए जो `policyParams` का समर्थन करती हैं)
  - किसी दिए गए दायरे के लिए हुक स्थापित या हटाएं
  - एक कस्टम नीतियां फाइल पथ सेट करें
  </Tab>
  <Tab title="गतिविधि टैब">
  - सभी सेशन में हर हुक इवेंट जो सक्रिय हुआ है की पूर्ण पेजीकृत इतिहास
  - निर्णय, इवेंट प्रकार, CLI (Claude Code / OpenAI Codex / GitHub Copilot _(बीटा)_ / Cursor Agent _(बीटा)_ / OpenCode _(बीटा)_ / Pi _(बीटा)_ / Gemini CLI _(बीटा)_), नीति का नाम, या सेशन ID द्वारा फ़िल्टर करें
  - प्रत्येक पंक्ति दिखाती है: टाइमस्टैम्प, नीति का नाम, निर्णय, CLI बैज (नारंगी = Claude Code, बैंगनी = OpenAI Codex, नीला = GitHub Copilot, पन्ना = Cursor Agent, एम्बर = OpenCode, गुलाबी = Pi, आसमानी = Gemini CLI), टूल का नाम, सेशन ID, और अस्वीकार/निर्देश निर्णयों का कारण
  - इसके ट्रांसक्रिप्ट को खोलने के लिए किसी सेशन ID पर क्लिक करें — दर्शक स्वचालित रूप से पता लगाता है कि कौन सा CLI हुक को सक्रिय किया (Claude `~/.claude/projects/…`, Codex `~/.codex/sessions/…`, Copilot CLI `~/.copilot/session-state/<id>/events.jsonl`, Cursor Agent `~/.cursor/agent-sessions/<id>/events.jsonl`, OpenCode `~/.local/share/opencode/opencode.db`, Pi `~/.pi/agent/sessions/<encoded-cwd>/<id>.jsonl`, Gemini CLI `~/.gemini/tmp/<basename>/chats/<session>.jsonl`) और हेडर में मेल खाने वाला CLI बैज प्रदान करता है
  </Tab>
</Tabs>

## स्वचालित रीफ्रेश

डैशबोर्ड के शीर्ष नेविगेशन में एक स्वचालित रीफ्रेश टॉगल है। सक्षम होने पर, वर्तमान पृष्ठ समय-समय पर नई सेशन और नीति गतिविधि को दिखाने के लिए ताज़ा होता है जैसे वे प्रकट होते हैं। दीर्घकालिक स्वायत्त एजेंट सेशन की निगरानी के लिए आवश्यक।

---

## पृष्ठों को अक्षम करना

यदि आपको डैशबोर्ड के केवल कुछ भाग की आवश्यकता है, तो `FAILPROOFAI_DISABLE_PAGES` को पृष्ठ के नामों की अल्पविराम से अलग की गई सूची पर सेट करें:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

मान्य मान: `policies`, `projects`।

---

## थीम

डैशबोर्ड हल्का और गहरा मोड का समर्थन करता है। नेविगेशन बार में बटन के माध्यम से टॉगल करें। वरीयता आपके ब्राउज़र के स्थानीय संग्रहण में संग्रहीत है।

## प्रोजेक्ट पथ को कॉन्फ़िगर करना

डिफ़ॉल्ट रूप से, डैशबोर्ड मानक Claude Code प्रोजेक्ट निर्देशिका से पढ़ता है। कस्टम सेटअप के लिए इसे ओवरराइड करें:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## गैर-localhost होस्ट से एक्सेस करना

जब डैशबोर्ड को **dev मोड** में चलाया जा रहा हो (`npm run dev`) और इसे `localhost` के अलावा किसी अन्य होस्टनाम से एक्सेस किया जा रहा हो - उदाहरण के लिए, एक कस्टम डोमेन, एक दूरस्थ IP, या एक सुरंगित URL - आप इस तरह की एक चेतावनी देख सकते हैं:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

यह Next.js अपने HMR (हॉट मॉड्यूल रीलोड) वेबसॉकेट तक क्रॉस-ऑरिजिन एक्सेस को ब्लॉक कर रहा है, जो एक dev-only विशेषता है। अपने होस्ट को अनुमति देने के लिए, `--allowed-origins` फ्लैग का उपयोग करें:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

कई होस्ट या IP के लिए, एक अल्पविराम से अलग की गई सूची पास करें:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

आप इसके बजाय `FAILPROOFAI_ALLOWED_DEV_ORIGINS` पर्यावरण चर सेट कर सकते हैं:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
यह केवल dev मोड पर लागू होता है। जब `failproofai` चलाया जा रहा हो (production मोड), कोई HMR वेबसॉकेट नहीं होता है और कोई क्रॉस-ऑरिजिन dev संसाधन समस्या नहीं होती है।
</Note>
