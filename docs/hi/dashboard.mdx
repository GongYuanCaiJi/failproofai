failproofai डैशबोर्ड आपके AI एजेंट सेशन की निगरानी करने और नीतियों को प्रबंधित करने के लिए एक स्थानीय वेब एप्लिकेशन है। देखें कि आपके एजेंट आपके अनुपस्थिति में क्या करते थे।

---

## डैशबोर्ड शुरू करना

```bash
failproofai
```

`http://localhost:8020` पर खुलता है।

डैशबोर्ड फाइलसिस्टम से सीधे पढ़ता है - आपकी Claude Code प्रोजेक्ट फ़ोल्डर और failproofai कॉन्फ़िग फ़ाइलें। किसी भी दूरस्थ सेवा में कुछ भी नहीं लिखा जाता है।

## पृष्ठ

### प्रोजेक्ट्स

आपकी मशीन पर मिली सभी Claude Code, OpenAI Codex, GitHub Copilot CLI _(beta)_, Cursor Agent _(beta)_, OpenCode _(beta)_, Pi _(beta)_, और Gemini CLI _(beta)_ प्रोजेक्ट्स की सूची। Claude प्रोजेक्ट्स `~/.claude/projects/` से खोजे जाते हैं (या `CLAUDE_PROJECTS_PATH` द्वारा सेट किए गए पथ से); Codex प्रोजेक्ट्स `~/.codex/sessions/<YYYY>/<MM>/<DD>/*.jsonl` के तहत हर ट्रांसक्रिप्ट को स्कैन करके और प्रत्येक सेशन के पहले रिकॉर्ड में दर्ज `cwd` के अनुसार समूहबद्ध करके खोजे जाते हैं; Copilot CLI प्रोजेक्ट्स प्रत्येक `~/.copilot/session-state/<sessionId>/workspace.yaml` को स्कैन करके खोजे जाते हैं (`COPILOT_HOME` के माध्यम से कॉन्फ़िगर करने योग्य) और इसके `cwd` फ़ील्ड के अनुसार समूहबद्ध करके; Cursor Agent प्रोजेक्ट्स `~/.cursor/agent-sessions/<sessionId>/` के तहत प्रति-सेशन मेटाडेटा को स्कैन करके खोजे जाते हैं (`CURSOR_HOME` के माध्यम से कॉन्फ़िगर करने योग्य, फॉलबैक के रूप में `conversations/` और `sessions/` को जांचते हुए) `meta.json` / `session.json` / `workspace.yaml` में `cwd` स्केलर के लिए; OpenCode प्रोजेक्ट्स `~/.local/share/opencode/opencode.db` पर इसके SQLite डीबी को `opencode db --format json` के माध्यम से क्वेरी करके खोजे जाते हैं (हम `session` और `project` टेबल पढ़ते हैं और `project_id` के अनुसार समूहबद्ध करते हैं); Pi प्रोजेक्ट्स `~/.pi/agent/sessions/<encoded-cwd>/<timestamp>_<uuid>.jsonl` के तहत प्रति-सेशन JSONL ट्रांसक्रिप्ट्स को स्कैन करके खोजे जाते हैं (`PI_SESSIONS_DIR` के माध्यम से कॉन्फ़िगर करने योग्य) और प्रत्येक सेशन के पहले रिकॉर्ड से `cwd` निकालते हैं; Gemini CLI प्रोजेक्ट्स `~/.gemini/tmp/<basename>/chats/session-<timestamp>-<uuid-prefix>.jsonl` को स्कैन करके खोजे जाते हैं (`GEMINI_SESSIONS_DIR` के माध्यम से कॉन्फ़िगर करने योग्य) और sibling `.project_root` टेक्स्ट मार्कर से canonical cwd को पुनः प्राप्त करते हैं। एक प्रोजेक्ट जिसका उपयोग कई CLIs द्वारा किया गया है सभी मेल खाने वाले बैज के साथ एक एकल पंक्ति के रूप में प्रदर्शित होता है। तालिका के ऊपर **CLI** ड्रॉपडाउन का उपयोग करें किसी विशिष्ट एजेंट CLI द्वारा फ़िल्टर करने के लिए; URL आपकी चयन को `?cli=claude|codex|copilot|cursor|opencode|pi|gemini` के रूप में सुरक्षित रखता है।

प्रत्येक प्रोजेक्ट दिखाता है:
- प्रोजेक्ट का नाम (फ़ोल्डर पथ से व्युत्पन्न)
- एक CLI बैज — `Claude Code` (नारंगी), `OpenAI Codex` (बैंगनी), `GitHub Copilot` (नीला), `Cursor Agent` (पन्ना), `OpenCode` (एम्बर), `Pi` (गुलाबी), और/या `Gemini CLI` (आसमानी)
- सबसे हाल की सेशन गतिविधि की तारीख

एक प्रोजेक्ट पर क्लिक करें इसके सेशन को देखने के लिए।

### सेशन

एक प्रोजेक्ट के भीतर सभी सेशन की सूची। प्रत्येक सेशन दिखाता है:
- सेशन ID
- शुरुआत और अंत की समय मुहर
- टूल कॉल की संख्या
- हुक गतिविधि गणना (नीतियाँ जो सक्रिय हुईं)

सूची को संकीर्ण करने के लिए तारीख सीमा फ़िल्टर और सेशन ID खोज का उपयोग करें। सेशन पृष्ठांकित हैं।

एक सेशन पर क्लिक करें सेशन दर्शक को खोलने के लिए।

### सेशन दर्शक

सेशन दर्शक स्वायत्त एजेंट के लिए मुख्य प्रश्न का उत्तर देता है: एजेंट ने क्या किया, और क्या यह सही मार्ग पर रहा? शीर्ष लेख के बगल में एक CLI बैज इंगित करता है कि सेशन Claude Code, OpenAI Codex, GitHub Copilot CLI, Cursor Agent, OpenCode, Pi, या Gemini CLI ट्रांसक्रिप्ट है। यह एक सेशन में हुई सभी चीजों की एक समयसारणी दिखाता है:

- **संदेश** - Claude की पाठ प्रतिक्रियाएं और उपयोगकर्ता संकेत
- **टूल कॉल** - हर टूल जिसे Claude ने आमंत्रित किया, इसके इनपुट और आउटपुट के साथ
- **नीति गतिविधि** - प्रत्येक टूल कॉल के लिए, कौन सी नीतियों ने सक्रिय हुईं और उन्होंने कौन सा निर्णय लौटाया

शीर्ष पर स्टेट्स बार सेशन अवधि, कुल टूल कॉल, और हुक निर्णयों का सारांश (allow / deny / instruct गणना) दिखाता है।

सेशन को निर्यात करने के लिए **Download Logs** बटन पर क्लिक करें। Claude Code, Codex, Copilot, Cursor, Pi, और Gemini सेशन के लिए आप मूल ऑन-डिस्क JSONL ट्रांसक्रिप्ट बाइट-दर-बाइट प्राप्त करते हैं; OpenCode के लिए (जिसके सेशन SQLite में हैं, डिस्क पर नहीं) आप एक JSON दस्तावेज़ प्राप्त करते हैं जो अंतर्निहित `session` / `messages` / `parts` टेबल को दर्पण करता है।

### नीतियाँ

नीतियों को प्रबंधित करने और गतिविधि की समीक्षा करने के लिए एक दो-टैब पृष्ठ।

<Tabs>
  <Tab title="नीतियाँ टैब">
  - एक क्लिक के साथ व्यक्तिगत नीतियों को चालू या बंद करें (`~/.failproofai/policies-config.json` में लिखता है)
  - एक नीति को विस्तृत करें इसके पैरामीटर को कॉन्फ़िगर करने के लिए (ऐसी नीतियों के लिए जो `policyParams` का समर्थन करती हैं)
  - किसी दिए गए स्कोप के लिए हुक इंस्टॉल या हटाएं
  - एक कस्टम नीति फ़ाइल पथ सेट करें
  </Tab>
  <Tab title="गतिविधि टैब">
  - सभी सेशन में सक्रिय हर हुक ईवेंट का पूर्ण पृष्ठांकित इतिहास
  - निर्णय, ईवेंट प्रकार, CLI (Claude Code / OpenAI Codex / GitHub Copilot _(beta)_ / Cursor Agent _(beta)_ / OpenCode _(beta)_ / Pi _(beta)_ / Gemini CLI _(beta)_), नीति नाम, या सेशन ID द्वारा फ़िल्टर करें
  - प्रत्येक पंक्ति दिखाता है: समय मुहर, नीति नाम, निर्णय, CLI बैज (नारंगी = Claude Code, बैंगनी = OpenAI Codex, नीला = GitHub Copilot, पन्ना = Cursor Agent, एम्बर = OpenCode, गुलाबी = Pi, आसमानी = Gemini CLI), टूल नाम, सेशन ID, और deny/instruct निर्णय का कारण
  - एक सेशन ID पर क्लिक करें इसके ट्रांसक्रिप्ट को खोलने के लिए — दर्शक स्वचालित रूप से पता लगाता है कि कौन सा CLI हुक सक्रिय किया (Claude `~/.claude/projects/…`, Codex `~/.codex/sessions/…`, Copilot CLI `~/.copilot/session-state/<id>/events.jsonl`, Cursor Agent `~/.cursor/agent-sessions/<id>/events.jsonl`, OpenCode `~/.local/share/opencode/opencode.db`, Pi `~/.pi/agent/sessions/<encoded-cwd>/<id>.jsonl`, Gemini CLI `~/.gemini/tmp/<basename>/chats/<session>.jsonl`) और शीर्ष लेख में मेल खाने वाला CLI बैज प्रदर्शित करता है
  </Tab>
</Tabs>

## स्वचालित रीफ्रेश

डैशबोर्ड में शीर्ष नेविगेशन में एक स्वचालित-रीफ्रेश टॉगल है। जब सक्षम किया जाता है, तो वर्तमान पृष्ठ समय-समय पर नए सेशन और नीति गतिविधि दिखाने के लिए रीफ्रेश करता है जैसे ही वे दिखाई देते हैं। लंबे समय तक चलने वाले स्वायत्त एजेंट सेशन की निगरानी के लिए आवश्यक।

## पृष्ठों को अक्षम करना

यदि आपको केवल डैशबोर्ड के कुछ भाग की आवश्यकता है, तो `FAILPROOFAI_DISABLE_PAGES` को पृष्ठ नामों की कॉमा-अलग सूची में सेट करें:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

वैध मान: `policies`, `projects`।

---

## प्रोजेक्ट पथ कॉन्फ़िगर करना

डिफ़ॉल्ट रूप से, डैशबोर्ड मानक Claude Code प्रोजेक्ट डायरेक्टरी से पढ़ता है। कस्टम सेटअप के लिए इसे ओवरराइड करें:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## गैर-लोकलहोस्ट होस्ट से एक्सेस करना

जब डैशबोर्ड को **dev मोड** में चलाया जाता है (`npm run dev`) और इसे `localhost` के अलावा किसी होस्टनाम से एक्सेस किया जाता है - उदाहरण के लिए, एक कस्टम डोमेन, एक दूरस्थ IP, या एक tunneled URL - आप एक चेतावनी देख सकते हैं जैसे:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

यह Next.js अपने HMR (hot module reload) वेबसॉकेट तक क्रॉस-ऑरिजिन एक्सेस को अवरुद्ध कर रहा है, जो एक dev-only विशेषता है। अपने होस्ट को अनुमति देने के लिए `--allowed-origins` फ़्लैग का उपयोग करें:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

कई होस्ट या IPs के लिए, एक कॉमा-अलग सूची पास करें:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

आप `FAILPROOFAI_ALLOWED_DEV_ORIGINS` पर्यावरण चर को इसके बजाय सेट कर सकते हैं:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
यह केवल dev मोड पर लागू होता है। जब `failproofai` (production मोड) चलाया जाता है, तो कोई HMR वेबसॉकेट नहीं होता है और कोई क्रॉस-ऑरिजिन dev संसाधन समस्या नहीं होती है।
</Note>
