failproofai डैशबोर्ड आपके AI एजेंट सेशन की निगरानी करने और नीतियों को प्रबंधित करने के लिए एक स्थानीय वेब एप्लिकेशन है। देखें कि जब आप दूर थे तो आपके एजेंट ने क्या किया।

---

## डैशबोर्ड शुरू करना

```bash
failproofai
```

`http://localhost:8020` पर खुलता है।

डैशबोर्ड सीधे फाइलसिस्टम से पढ़ता है - आपके Claude Code प्रोजेक्ट फोल्डर और failproofai कॉन्फ़िग फाइलें। किसी दूरस्थ सेवा में कुछ भी नहीं लिखा जाता है।

---

## पृष्ठ

### प्रोजेक्ट

आपकी मशीन पर मिले सभी Claude Code, OpenAI Codex, GitHub Copilot CLI _(बीटा)_, Cursor Agent _(बीटा)_, OpenCode _(बीटा)_, Pi _(बीटा)_, और Gemini CLI _(बीटा)_ प्रोजेक्ट सूचीबद्ध करता है। Claude प्रोजेक्ट `~/.claude/projects/` से खोजे जाते हैं (या `CLAUDE_PROJECTS_PATH` द्वारा सेट किया गया पथ); Codex प्रोजेक्ट `~/.codex/sessions/<YYYY>/<MM>/<DD>/*.jsonl` के अंतर्गत प्रत्येक ट्रांसक्रिप्ट को स्कैन करके और प्रत्येक सेशन के पहले रिकॉर्ड में दर्ज `cwd` के आधार पर समूहीकृत करके खोजे जाते हैं; Copilot CLI प्रोजेक्ट प्रत्येक `~/.copilot/session-state/<sessionId>/workspace.yaml` को स्कैन करके (`COPILOT_HOME` के माध्यम से कॉन्फ़िगर करने योग्य) और इसके `cwd` फील्ड के आधार पर समूहीकृत करके खोजे जाते हैं; Cursor Agent प्रोजेक्ट `~/.cursor/agent-sessions/<sessionId>/` के अंतर्गत प्रति-सेशन मेटाडेटा को स्कैन करके (कॉन्फ़िगर करने योग्य `CURSOR_HOME` के माध्यम से, `conversations/` और `sessions/` फॉलबैक के रूप में) `meta.json` / `session.json` / `workspace.yaml` में `cwd` स्केलर के लिए खोजे जाते हैं; OpenCode प्रोजेक्ट इसके SQLite DB को `~/.local/share/opencode/opencode.db` पर क्वेरी करके `opencode db --format json` के माध्यम से खोजे जाते हैं (हम `session` और `project` तालिकाएं पढ़ते हैं और `project_id` के आधार पर समूहीकृत करते हैं); Pi प्रोजेक्ट `~/.pi/agent/sessions/<encoded-cwd>/<timestamp>_<uuid>.jsonl` के अंतर्गत प्रति-सेशन JSONL ट्रांसक्रिप्ट को स्कैन करके (कॉन्फ़िगर करने योग्य `PI_SESSIONS_DIR` के माध्यम से) और प्रत्येक सेशन के पहले रिकॉर्ड से `cwd` निकालकर खोजे जाते हैं; Gemini CLI प्रोजेक्ट `~/.gemini/tmp/<basename>/chats/session-<timestamp>-<uuid-prefix>.jsonl` को स्कैन करके (कॉन्फ़िगर करने योग्य `GEMINI_SESSIONS_DIR` के माध्यम से) और भाई के `.project_root` टेक्स्ट मार्कर से canonical cwd को पुनः प्राप्त करके खोजे जाते हैं। एक प्रोजेक्ट जिसे कई CLIs द्वारा उपयोग किया गया है, सभी मिलान बैज के साथ एक एकल पंक्ति के रूप में प्रदर्शित होता है। तालिका के ऊपर **CLI** ड्रॉपडाउन का उपयोग करके किसी विशिष्ट एजेंट CLI द्वारा फ़िल्टर करें; URL आपकी चयन को `?cli=claude|codex|copilot|cursor|opencode|pi|gemini` के रूप में संरक्षित करता है।

प्रत्येक प्रोजेक्ट निम्नलिखित दिखाता है:
- प्रोजेक्ट नाम (फोल्डर पथ से व्युत्पन्न)
- एक CLI बैज — `Claude Code` (नारंगी), `OpenAI Codex` (बैंगनी), `GitHub Copilot` (नीला), `Cursor Agent` (पन्ना), `OpenCode` (एम्बर), `Pi` (गुलाबी), और/या `Gemini CLI` (आसमानी)
- सबसे हाल की सेशन गतिविधि की तारीख

इसके सेशन देखने के लिए किसी प्रोजेक्ट पर क्लिक करें।

### सेशन

एक प्रोजेक्ट के भीतर सभी सेशन सूचीबद्ध करता है। प्रत्येक सेशन निम्नलिखित दिखाता है:
- सेशन ID
- शुरुआत और अंत टाइमस्टैम्प
- टूल कॉल की संख्या
- हुक गतिविधि गणना (नीतियां जो सक्रिय हुईं)

सूची को संकीर्ण करने के लिए तारीख रेंज फ़िल्टर और सेशन ID खोज का उपयोग करें। सेशन पृष्ठांकित हैं।

सेशन दर्शक खोलने के लिए किसी सेशन पर क्लिक करें।

### सेशन दर्शक

सेशन दर्शक स्वायत्त एजेंट के लिए मुख्य प्रश्न का उत्तर देता है: एजेंट ने क्या किया, और क्या वह ट्रैक पर रहा? हेडर के बगल में एक CLI बैज इंगित करता है कि सेशन Claude Code, OpenAI Codex, GitHub Copilot CLI, Cursor Agent, OpenCode, Pi, या Gemini CLI ट्रांसक्रिप्ट है। यह एक सेशन में हुई सभी चीजों की एक समयरेखा दिखाता है:

- **संदेश** - Claude के टेक्स्ट प्रतिक्रियाएं और उपयोगकर्ता प्रॉम्प्ट
- **टूल कॉल** - हर टूल जिसे Claude ने आमंत्रित किया, इसके इनपुट और आउटपुट के साथ
- **नीति गतिविधि** - प्रत्येक टूल कॉल के लिए, कौन सी नीतियां सक्रिय हुईं और उन्होंने किस निर्णय को लौटाया

शीर्ष पर स्टेट्स बार सेशन की अवधि, कुल टूल कॉल, और हुक निर्णयों का सारांश (गणना की अनुमति दें / अस्वीकार करें / निर्देश दें) दिखाता है।

सेशन को निर्यात करने के लिए **डाउनलोड लॉग** बटन पर क्लिक करें। Claude Code, Codex, Copilot, Cursor, Pi, और Gemini सेशन के लिए आप बाइट-दर-बाइट मूल ऑन-डिस्क JSONL ट्रांसक्रिप्ट प्राप्त करते हैं; OpenCode के लिए (जिसके सेशन SQLite में रहते हैं, डिस्क पर नहीं) आप एक JSON दस्तावेज़ प्राप्त करते हैं जो अंतर्निहित `session` / `messages` / `parts` तालिकाओं को दर्पण करता है।

### ऑडिट

आपके एजेंट के बारे में कैसे वास्तव में व्यवहार कर रहे हैं इसका एक व्यक्तित्व-संचालित रिपोर्ट पिछले सेशन में। `failproofai audit` CLI के समान स्कैन चलाता है लेकिन इसे छह-खंड डैशबोर्ड के रूप में प्रदर्शित करता है:

1. **पहचान** — आपके एजेंट को 8 आर्केटाइप में से एक में वर्गीकृत करता है (`आशावादी`, `रेंगिंग`, `अन्वेषक`, `सुनहरी मछली`, `पागल आर्किटेक्ट`, `सटीकता निर्माता`, `हथौड़ा`, `भूत`) इस आधार पर कि कौन सी डिटेक्टर + नीतियां सक्रिय हुईं और कितनी भारी। एक 8×8 पिक्सल sigil, आर्केटाइप टैगलाइन, "सामान्य में" / "प्राथमिक जोखिम" फ्रेमिंग, और समापन एक-पंक्ति प्रदान करता है।
2. **अपने एजेंट को दिखाएं** - पहचान कार्ड को X / LinkedIn पर पोस्ट करने के लिए उपयुक्त 1200×630 PNG के रूप में कैप्चर करता है (`पोस्टर बनाएं` पर क्लिक करें)।
3. **ताकत** — हरी-जांचित व्यवहार जो आपका एजेंट पहले से सही तरीके से करता है, लाइव ऑडिट डेटा से व्युत्पन्न (स्वच्छ टूल-कॉल दर, औसत सेशन लंबाई, शून्य क्रेडेंशियल रिसाव, शून्य पुनः प्रयास तूफान, आदि)।
4. **स्कोर + लीडरबोर्ड** — 0–100 स्कोर अक्षर ग्रेड के साथ (S/A/B/C/D/F), एक वितरण हिस्टोग्राम दिखाता है कि आप कहां पड़ते हैं, गद्य ("एक B 71 से शुरू होता है। आप 13 अंक दूर हैं।"), और एक लीडरबोर्ड तालिका आपकी पंक्ति हाइलाइट के साथ।
5. **निष्कर्ष** — प्रभाव द्वारा रैंक किए गए प्रति-निष्कर्ष कार्ड। प्रत्येक कार्ड सतह करता है कि क्या हुआ, इसकी लागत क्या है, वास्तविक कैप्चर किए गए कमांड के साथ साक्ष्य नमूना, और failproofai नीति जो समान पैटर्न को पकड़ सकती है (`$ failproofai policy add <slug>`, क्लिक-टू-कॉपी)।
6. **निर्धारित नीतियां + रिटर्न लूप** — हर अनुपयोगी बिल्ट-इन नीति का एक ग्रिड जो एक अंतर को बंद कर देगा, अनुमानित-स्कोर कॉलआउट के साथ, साथ ही "7 दिन में पुनः-ऑडिट" CTA।

`failproofai audit` रनटाइम द्वारा संचालित — अंतर्निहित स्कैन इंजन, समर्थित फ़्लैग, और प्रति-ट्रांसक्रिप्ट कैश अपरिवर्तनीय के लिए [ऑडिट CLI](/hi/cli/audit) देखें। डैशबोर्ड सबसे हाल के परिणाम को `~/.failproofai/audit-dashboard.json` पर कैश करता है (मोड `0600`, एकल स्लॉट, नए रन अधिलेख) ताकि पुनः यात्रा तत्काल हो; **प्रति-ट्रांसक्रिप्ट और पूरे परिणाम दोनों कैश एक बार पढ़ने पर अस्वीकार कर दिए जाते हैं जब वे 7 दिन से अधिक पुराने हों** इसलिए डैशबोर्ड कभी भी चुप्पी से सप्ताह भर पुराना परिणाम नहीं देता है। पृष्ठ-के-शीर्ष `[ re-audit ]` बार हमेशा अंतिम-चलाया गया टाइमस्टैम्प दिखाता है (`3d पहले ऑडिट किया गया`), विंडो के अंतिम 24 घंटे में एक एम्बर `14h में समाप्त — ताज़ा करने के लिए पुनः-ऑडिट` चिप, और TTL के बाद एक `ऑडिट समाप्त` बैनर। शीर्ष पर `[ re-audit ]` या नीचे `[ re-audit now ]` पर क्लिक करना `/api/audit/run` पर POST करता है और डैशबोर्ड चलाया समाप्त होने तक 1Hz पर `/api/audit/status` को पोल करता है; रन के दौरान एक स्टिकी गुलाबी प्रगति पट्टी व्यूपोर्ट के शीर्ष पर पिन करता है एक अतीत टाइमर के साथ, और परिणाम सफलता पर जगह में स्वैप किया जाता है (कोई पूर्ण पृष्ठ पुनः लोड नहीं)। विफलता पर पट्टी `RerunError.kind` से कॉपी के साथ लाल हो जाता है (`timeout` / `network` / `post_failed`)। खाली स्थिति (कोई कैश नहीं) और शून्य-सेशन स्थिति (कैश मौजूद है लेकिन स्कैन को कोई ट्रांसक्रिप्ट नहीं मिला) अलग से सतह किए जाते हैं।

### नीतियां

नीतियों को प्रबंधित करने और गतिविधि की समीक्षा करने के लिए एक दो-टैब पृष्ठ।

<Tabs>
  <Tab title="नीति टैब">
  - बहु-चयन करें कि failproofai कौन से एजेंट CLIs को एक एकल पैनल से सुरक्षित रखता है — Claude Code, OpenAI Codex, GitHub Copilot, Cursor Agent, OpenCode, Pi, और Gemini CLI सभी के पास इंस्टॉल स्थिति (`सक्रिय` / `पता लगाया` / `निष्क्रिय`), उपयोगकर्ता-स्कोप सेटिंग पथ, और एक ब्रांड-रंगीन उच्चारण के साथ एक पंक्ति है। उन CLIs को जांचें या अनचेक करें जिन्हें आप चाहते हैं और एक कदम में स्थापित/अनइंस्टॉल करने के लिए `परिवर्तन लागू करें` पर क्लिक करें। CLIs जिनके बाइनरी PATH पर पता लगाया जाता है पूर्व-जांच किए जाते हैं।
  - एकल क्लिक के साथ अलग-अलग नीतियों को चालू या बंद करें (`~/.failproofai/policies-config.json` में लिखता है — हर स्थापित CLI में साझा)
  - एक नीति को अपनी मापदंड को कॉन्फ़िगर करने के लिए विस्तारित करें (उन नीतियों के लिए जो `policyParams` का समर्थन करती हैं)
  - एक कस्टम नीति फाइल पथ सेट करें
  </Tab>
  <Tab title="गतिविधि टैब">

  - निर्णय, ईवेंट प्रकार, CLI (Claude Code / OpenAI Codex / GitHub Copilot _(बीटा)_ / Cursor Agent _(बीटा)_ / OpenCode _(बीटा)_ / Pi _(बीटा)_ / Gemini CLI _(बीटा)_), नीति नाम, या सेशन ID द्वारा फ़िल्टर करें
  - प्रत्येक पंक्ति दिखाता है: टाइमस्टैम्प, नीति नाम, निर्णय, CLI बैज (नारंगी = Claude Code, बैंगनी = OpenAI Codex, नीला = GitHub Copilot, पन्ना = Cursor Agent, एम्बर = OpenCode, गुलाबी = Pi, आसमानी = Gemini CLI), टूल नाम, सेशन ID, और अस्वीकार/निर्देश निर्णयों का कारण
  - अपने ट्रांसक्रिप्ट को खोलने के लिए किसी सेशन ID पर क्लिक करें — दर्शक स्वचालित रूप से पता लगाता है कि किस CLI ने हुक को फायर किया (Claude `~/.claude/projects/…`, Codex `~/.codex/sessions/…`, Copilot CLI `~/.copilot/session-state/<id>/events.jsonl`, Cursor Agent `~/.cursor/agent-sessions/<id>/events.jsonl`, OpenCode `~/.local/share/opencode/opencode.db`, Pi `~/.pi/agent/sessions/<encoded-cwd>/<id>.jsonl`, Gemini CLI `~/.gemini/tmp/<basename>/chats/<session>.jsonl`) और हेडर में मेल खाने वाले CLI बैज को प्रदर्शित करता है

## स्वचालित ताज़ा करना

डैशबोर्ड के शीर्ष नेविगेशन में एक स्वचालित-ताज़ा टॉगल है। जब सक्षम होता है, तो वर्तमान पृष्ठ नए सेशन और नीति गतिविधि दिखाने के लिए समय-समय पर ताज़ा होता है जैसे ही वे दिखाई दें। दीर्घमेयादी स्वायत्त एजेंट सेशन की निगरानी के लिए आवश्यक।

## पृष्ठों को अक्षम करना

यदि आपको डैशबोर्ड के केवल कुछ भागों की आवश्यकता है, तो `FAILPROOFAI_DISABLE_PAGES` को पृष्ठ नामों की अल्पविराम-विभाजित सूची में सेट करें:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

मान्य मान: `policies`, `projects`, `audit`।

---

## प्रोजेक्ट पथ को कॉन्फ़िगर करना

डिफ़ॉल्ट रूप से, डैशबोर्ड मानक Claude Code प्रोजेक्ट निर्देशिका से पढ़ता है। कस्टम सेटअप के लिए इसे ओवरराइड करें:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## गैर-localhost होस्ट से पहुंचना

जब डैशबोर्ड **dev मोड** में चलाया जा रहा हो (`npm run dev`) और इसे `localhost` के अलावा किसी हોस्टनाम से एक्सेस किया जा रहा हो - उदाहरण के लिए, कस्टम डोमेन, दूरस्थ IP, या tunneled URL - आप एक चेतावनी देख सकते हैं जैसे:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

यह Next.js अपने HMR (हॉट मॉड्यूल रीलोड) वेबसॉकेट को क्रॉस-ऑरिजिन एक्सेस को ब्लॉक कर रहा है, जो एक dev-केवल सुविधा है। अपने होस्ट को अनुमति देने के लिए, `--allowed-origins` फ़्लैग का उपयोग करें:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

कई होस्ट या IP के लिए, अल्पविराम-विभाजित सूची पास करें:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

आप इसके बजाय `FAILPROOFAI_ALLOWED_DEV_ORIGINS` पर्यावरण चर भी सेट कर सकते हैं:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
यह केवल dev मोड पर लागू होता है। जब `failproofai` (उत्पादन मोड) चलाया जा रहा हो, तो कोई HMR वेबसॉकेट और कोई क्रॉस-ऑरिजिन dev संसाधन समस्या नहीं होती।
</Note>
