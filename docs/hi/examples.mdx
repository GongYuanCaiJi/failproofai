सामान्य परिस्थितियों के लिए तैयार उदाहरण। प्रत्येक एक दिखाता है कि कैसे इंस्टॉल करें और क्या उम्मीद करें।

---

## Claude Code के लिए hooks सेट अप करना

Failproof AI Claude Code के साथ इसके [hooks सिस्टम](https://docs.anthropic.com/en/docs/claude-code/hooks) के माध्यम से एकीकृत होता है। जब आप `failproofai policies --install` चलाते हैं, तो यह Claude Code के `settings.json` में hook कमांड रजिस्टर करता है जो हर टूल कॉल पर फायर होते हैं।

<Steps>
  <Step title="failproofai इंस्टॉल करें">
    ```bash
    npm install -g failproofai
    ```
  </Step>
  <Step title="सभी built-in policies सक्षम करें">
    ```bash
    failproofai policies --install
    ```
  </Step>
  <Step title="सत्यापित करें कि hooks रजिस्टर हैं">
    ```bash
    cat ~/.claude/settings.json | grep failproofai
    ```

    आपको `PreToolUse`, `PostToolUse`, `Notification`, और `Stop` इवेंट्स के लिए hook entries दिखाई देने चाहिए।
  </Step>
  <Step title="Claude Code चलाएं">
    ```bash
    claude
    ```

    Policies अब हर टूल कॉल पर स्वचालित रूप से चलते हैं। Claude को `sudo rm -rf /` चलाने के लिए कहने का प्रयास करें - यह ब्लॉक कर दिया जाएगा।
  </Step>
</Steps>

## Agents SDK के लिए hooks सेट अप करना

यदि आप [Agents SDK](https://docs.anthropic.com/en/docs/agents-sdk) के साथ बना रहे हैं, तो आप समान hook सिस्टम को प्रोग्रामेटिकली उपयोग कर सकते हैं।

<Steps>
  <Step title="अपने प्रोजेक्ट में failproofai इंस्टॉल करें">
    ```bash
    npm install failproofai
    ```
  </Step>
  <Step title="अपने agent में hooks कॉन्फ़िगर करें">
    अपने agent प्रक्रिया बनाते समय hook कमांड पास करें। hooks Claude Code में जैसे ही काम करते हैं - stdin/stdout JSON के माध्यम से:

    ```bash
    failproofai --hook PreToolUse   # प्रत्येक टूल से पहले कॉल किया जाता है
    failproofai --hook PostToolUse  # प्रत्येक टूल के बाद कॉल किया जाता है
    ```
  </Step>
  <Step title="अपने agent के लिए एक कस्टम policy लिखें">
    ```javascript
    import { customPolicies, allow, deny } from "failproofai";

    customPolicies.add({
      name: "limit-to-project-dir",
      description: "Keep the agent inside the project directory",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        const path = String(ctx.toolInput?.file_path ?? "");
        if (path.startsWith("/") && !path.startsWith(ctx.session?.cwd ?? "")) {
          return deny("Agent is restricted to the project directory");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="कस्टम policy इंस्टॉल करें">
    ```bash
    failproofai policies --install --custom ./my-agent-policies.js
    ```
  </Step>
</Steps>

## विनाशकारी कमांड को ब्लॉक करें

सबसे सामान्य सेटअप - agents को अपरिवर्तनीय नुकसान से बचाएं।

```bash
failproofai policies --install block-sudo block-rm-rf block-force-push block-curl-pipe-sh
```

यह क्या करता है:
- `block-sudo` - सभी `sudo` कमांड को ब्लॉक करता है
- `block-rm-rf` - recursive फाइल deletion को ब्लॉक करता है
- `block-force-push` - `git push --force` को ब्लॉक करता है
- `block-curl-pipe-sh` - remote scripts को shell में piping को ब्लॉक करता है

## Secret leakage से बचें

agents को tool आउटपुट में credentials देखने या लीक करने से रोकें।

```bash
failproofai policies --install sanitize-api-keys sanitize-jwt sanitize-connection-strings sanitize-bearer-tokens
```

ये `PostToolUse` पर फायर होती हैं - एक टूल चलने के बाद, वे agent को देखने से पहले आउटपुट को साफ़ करते हैं।

## agents को ध्यान की जरूरत होने पर Slack alerts प्राप्त करें

idle alerts को Slack में भेजने के लिए notification hook का उपयोग करें।

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "slack-on-idle",
  description: "Alert Slack when the agent is waiting for input",
  match: { events: ["Notification"] },
  fn: async (ctx) => {
    const webhookUrl = process.env.SLACK_WEBHOOK_URL;
    if (!webhookUrl) return allow();

    const message = String(ctx.payload?.message ?? "Agent is waiting");
    const project = ctx.session?.cwd ?? "unknown";

    try {
      await fetch(webhookUrl, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          text: `*${message}*\nProject: \`${project}\``,
        }),
        signal: AbortSignal.timeout(5000),
      });
    } catch {
      // अगर Slack अप्राप्य है तो agent को कभी ब्लॉक न करें
    }

    return allow();
  },
});
```

इसे इंस्टॉल करें:

```bash
SLACK_WEBHOOK_URL=https://hooks.slack.com/... failproofai policies --install --custom ./slack-alerts.js
```

## agents को एक branch पर रखें

agents को branch स्विच करने या protected branches में push करने से रोकें।

```javascript
import { customPolicies, allow, deny } from "failproofai";

customPolicies.add({
  name: "stay-on-branch",
  description: "Prevent the agent from checking out other branches",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+checkout\s+(?!-b)/.test(cmd)) {
      return deny("Stay on the current branch. Create a new branch with -b if needed.");
    }
    return allow();
  },
});
```

## commits से पहले tests की आवश्यकता है

agents को committing से पहले tests चलाने की याद दिलाएं।

```javascript
import { customPolicies, allow, instruct } from "failproofai";

customPolicies.add({
  name: "test-before-commit",
  description: "Remind the agent to run tests before committing",
  match: { events: ["PreToolUse"] },
  fn: async (ctx) => {
    if (ctx.toolName !== "Bash") return allow();
    const cmd = String(ctx.toolInput?.command ?? "");
    if (/git\s+commit/.test(cmd)) {
      return instruct("Run tests before committing. Use `npm test` or `bun test` first.");
    }
    return allow();
  },
});
```

## एक production repo को लॉक करें

एक project-level config को commit करें ताकि आपकी team के हर developer को समान policies मिलें।

अपने repo में `.failproofai/policies-config.json` बनाएं:

```json
{
  "enabledPolicies": [
    "block-sudo",
    "block-rm-rf",
    "block-force-push",
    "block-push-master",
    "block-env-files",
    "sanitize-api-keys",
    "sanitize-jwt"
  ],
  "policyParams": {
    "block-push-master": {
      "protectedBranches": ["main", "release", "production"]
    }
  }
}
```

फिर इसे commit करें:

```bash
git add .failproofai/policies-config.json
git commit -m "Add failproofai team policies"
```

हर team member जिसके पास failproofai इंस्टॉल है, स्वचालित रूप से इन नियमों को उठा लेगा।

## convention policies के साथ एक org-wide quality standard बनाएं

सबसे प्रभावशाली सेटअप: अपने repo में `.failproofai/policies/` को commit करें जिसमें आपके project के लिए तैयार policies हों। हर team member को वे स्वचालित रूप से मिलते हैं — कोई install कमांड नहीं, कोई config परिवर्तन नहीं।

<Steps>
  <Step title="निर्देशिका बनाएं और policies जोड़ें">
    ```bash
    mkdir -p .failproofai/policies
    ```

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    // अपनी team के पसंदीदा package manager को enforce करें
    // (या इसके बजाय built-in prefer-package-manager policy को enable करें)
    customPolicies.add({
      name: "enforce-bun",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        const cmd = String(ctx.toolInput?.command ?? "");
        if (/\bnpm\b/.test(cmd)) return deny("Use bun instead of npm.");
        return allow();
      },
    });

    // agent को committing से पहले tests चलाने की याद दिलाएं
    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Run tests before committing.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="git में commit करें">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```
  </Step>
  <Step title="सुधार करना जारी रखें">
    जब आपकी team को नई failure modes का सामना हो, policies जोड़ें और push करें। हर कोई अपने अगले `git pull` पर अपडेट प्राप्त करता है। ये policies एक living quality standard बन जाती हैं जो आपकी team के साथ बढ़ती है।
  </Step>
</Steps>

| `policies-advanced/index.js` | Transitive imports, async hooks, PostToolUse output scrubbing, Stop event handling |
