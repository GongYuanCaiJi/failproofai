---
title: नीतियों को इंस्टॉल करें
description: "नीतियों को सक्षम करें ताकि वे हर एजेंट टूल कॉल पर चलें"
---

```bash
failproofai policies --install [policy-names...] [options]
```

आपके इंस्टॉल किए गए एजेंट CLI की सेटिंग्स फ़ाइल (Claude Code, OpenAI Codex, या GitHub Copilot CLI _(beta)_) में हुक एंट्रीज लिखता है ताकि failproofai टूल कॉल को इंटरसेप्ट कर सके।

उपनाम: `failproofai p -i`

## विकल्प

| फ़्लैग | विवरण |
|------|-------------|
| `--cli claude\|codex\|copilot` | एजेंट CLI(s) जिसके लिए इंस्टॉल करना है; स्पेस-सेपरेटेड (जैसे `--cli claude codex copilot`) या दोहराया गया। छोड़ने पर इंस्टॉल किए गए CLIs का पता लगाएगा और प्रॉम्प्ट करेगा। |
| `--scope user` | यूजर-स्कोप सेटिंग्स फ़ाइल में इंस्टॉल करें (Claude: `~/.claude/settings.json`; Codex: `~/.codex/hooks.json`; Copilot: `~/.copilot/hooks/failproofai.json`)। डिफ़ॉल्ट। |
| `--scope project` | प्रोजेक्ट-स्कोप सेटिंग्स फ़ाइल में इंस्टॉल करें (Claude: `<cwd>/.claude/settings.json`; Codex: `<cwd>/.codex/hooks.json`; Copilot: `<cwd>/.github/hooks/failproofai.json`)। |
| `--scope local` | केवल Claude के लिए — `<cwd>/.claude/settings.local.json` में इंस्टॉल करता है। Codex और Copilot के पास `local` स्कोप नहीं है। |
| `--custom <path>` / `-c` | कस्टम हुक नीतियों वाली JS फ़ाइल का पथ |

## व्यवहार

- **कोई नीति नाम नहीं** - नीतियों का चयन करने के लिए एक इंटरैक्टिव प्रॉम्प्ट खोलता है
- **विशिष्ट नाम** - उन नीतियों को सक्षम करता है (पहले से सक्षम किए गए में जोड़ा गया)
- **`all`** - हर उपलब्ध नीति को सक्षम करता है

इंस्टॉलेशन एडिटिव है: `--install` को फिर से चलाने से नई नीतियां जोड़ी जाती हैं बिना पहले से मौजूद नीतियों को हटाए।

## उदाहरण

```bash
# सभी डिफ़ॉल्ट नीतियों को ग्लोबली इंस्टॉल करें (इंटरैक्टिव)
failproofai policies --install

# वर्तमान प्रोजेक्ट के लिए विशिष्ट नीतियों को इंस्टॉल करें
failproofai policies --install block-sudo sanitize-api-keys --scope project

# एक साथ सभी नीतियों को सक्षम करें
failproofai policies --install all

# कस्टम नीतियों फ़ाइल के साथ इंस्टॉल करें
failproofai policies --install --custom ./my-policies.js

# OpenAI Codex के लिए इंस्टॉल करें (प्रोजेक्ट स्कोप)
failproofai policies --install --cli codex --scope project

# वर्तमान प्रोजेक्ट के लिए GitHub Copilot CLI (beta) के लिए इंस्टॉल करें
failproofai policies --install --cli copilot --scope project

# एक साथ सभी तीन CLIs के लिए इंस्टॉल करें
failproofai policies --install --cli claude codex copilot
```

जब `--custom <path>` प्रदान किया जाता है, तो फ़ाइल को तुरंत सत्यापित किया जाता है - इसे कम से कम एक बार `customPolicies.add()` को कॉल करना चाहिए। समाधान किया गया पथ `policies-config.json` में `customPoliciesPath` के रूप में सहेजा जाता है।
