专为 **AI 故障处理**、**错误恢复**和 **LLM 可靠性**设计的 Hooks 与策略。让你的 AI 智能体在 **Claude Code**、**OpenAI Codex**、**GitHub Copilot**、**Cursor Agent**、**OpenCode**、**Pi**、**Hermes**、**OpenClaw**、**Factory Droid**、**Devin CLI**、**Antigravity CLI** 以及 **Agents SDK** 上保持稳定、自主运行。

AI 智能体的失败方式往往有规律可循：执行破坏性命令、泄露密钥、偏离任务、陷入循环，或直接推送到主分支。若无人干预，小问题会逐步演变为服务中断、凭据泄露和数据丢失。

FailproofAI 通过**策略**来解决这一问题。这些规则会介入每次智能体工具调用，**检测故障**、**进行缓解**（拦截、指引、清理），并在需要关注时**向你发出提醒**。本地仪表盘让你事后可以回顾每一次工具调用、智能体故障和恢复操作。

会话记录与策略评估均保留在本地。只有当你明确使用在线功能（如经认证的审计提醒或邀请）时，才会发送数据。

## 快速开始

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

```bash
failproofai policies --install   # enable policies (or skip — `failproofai` will offer to set them up on first run)
failproofai                      # launch the dashboard
```

完整使用流程请参阅[快速入门](/zh/getting-started)指南。
