质量问题会主动找上你，而不是等到用户投诉你才知道。只需接入一次你自己的评分服务，Failproof AI Observability 就会自动对每次已完成的运行进行评分，让帮助质量下滑或幻觉激增的问题在用户察觉之前就自动浮现。

![Sessions 网格中带有评分列：每次运行都显示评估状态标签，以及颜色编码的帮助性、事实性和工具效率徽章](/agenteye/images/sessions-list.png)

*Sessions 网格上的每次运行都带有对应的评分；红、黄、绿徽章让表现欠佳的运行一目了然，无需打开任何记录。*

## 停止手动抽查运行

过去，你只能抽查少数几次运行，然后寄希望于其余的都没问题。现在，每次完成的会话在结束的瞬间就会按照你关心的维度进行评分：帮助性、工具效率、事实性、安全性，或任何你的质量标准所要求的内容。你定义评分维度，Failproof AI Observability 存储、追踪并展示评估器返回的所有内容。没有任何一次运行会漏掉评分，你也不再需要等到收到支持工单才发现回归问题。

评分会显示在 **`/<org-slug>/sessions`**（侧边栏 → *observe* → *sessions*）的 Sessions 网格中，每行对应一组徽章。只想查看表现不达标的运行？按评分范围筛选网格，例如帮助性低于 0.5，精准定位值得仔细查看的运行。查看评分需要 `evaluations:read` 权限。

## 查看运行评分低的原因

数字告诉你某次运行表现较弱，会话页面则告诉你原因所在。打开任意运行，右侧面板最顶部显示总体摘要，随后按维度展示各项评分条，并在每项下方附上评估器自己的推理说明。从"事实性评分 0.4"到"到底哪个断言出了错"，只需几秒即可定位。

![会话的右侧面板：顶部是评估摘要，其后是带有推理说明的各维度评分条，旁边是完整的事件时间线](/agenteye/images/session-detail.png)

*会话详情视图：摘要、各维度评分条，以及每项评分背后的推理说明，与运行的事件时间线并排呈现。*

上线了更精准的评估器，或者某次运行在评分完成之前就崩溃了？**重新评估**按钮（需要 `evaluations:trigger` 权限）可以对该会话原地重新评分，并将最新结果追加到其时间线中，之前的评分作为历史记录仍然可见。该按钮位于 **`/<org-slug>/sessions/<session-id>`**。

## 跨集群监控质量趋势

一次运行评分低是噪声，整个批次持续下滑才是信号。已保存的仪表盘可将评分转化为一目了然的趋势：本周与上周的平均帮助性对比，按 Agent、按环境细分。

![质量仪表盘：各评估维度的平均分柱状图，以及随时间变化的趋势线](/agenteye/images/dashboard-quality.png)

*已保存的质量仪表盘展示你关注的评分维度趋势，让缓慢的下滑在演变为事故之前就清晰可见。*

仪表盘位于 **`/<org-slug>/dashboards`**（侧边栏 → *analyze* → *dashboards*），在整个组织内共享。每个卡片汇总匹配的会话：数量、各展示评分的平均值，以及趋势迷你图。点击"在 Sessions 中打开"可直接跳转到任意数字背后经过预筛选的运行列表。查看需要 `dashboards:read` 和 `evaluations:read` 权限。

## 只需接入一次评估器

评分功能默认关闭，需手动开启。将 Failproof AI Observability 指向一个评分服务后即可激活。你搭建一个小型 HTTP 服务（Observability 附带一个可直接复制的参考实现），在服务器上设置两个值，之后每次运行都会自动为你评分。完整操作指南、评分协议和 SDK 请参阅详细文档。

不确定哪些维度值得评分？[评估器 Agent 技能](/zh/agenteye/evaluator-skill)可以让你的编码 Agent 基于你自己的会话数据来决定评分维度，然后构建并部署服务。
