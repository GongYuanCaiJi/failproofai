Agent 一有动作，你立刻就能看到。事件流是你实时掌握生产环境中每个 Agent 状态的窗口：无需等待，无需 grep 日志，无需猜测刚才发生了什么。

![实时事件流：按颜色分类的事件行实时滚动更新，可按环境、Agent、会话、事件类型和自由文本过滤](/agenteye/images/events-stream.png)

*来自组织内每个 Agent 的所有事件，最新的排在最前，实时更新。*

## 实时掌握每个 Agent 的动态

当一个 Agent 开始运行、调用模型、触发工具、执行 hook 或遇到错误时，对应的行会在事件发生的瞬间出现在流的顶部。它会持续追踪组织内所有 Agent 的每一个事件，最新的排在最前，让你随时掌握当前状态，而不是过时的信息。

这意味着你不再需要在某台机器上 tail 日志文件，不再需要跨机器 grep，也不再需要手动拼接时间戳。打开一个页面，你就已经在观察生产环境了。

每一行都按类型进行颜色编码，让你一眼就能读懂事件流，而不必逐行解析。每行一眼就能看到：

- **事件类型**，以颜色区分：`agent_start`、`model_response`、`tool_use`、`hook_completed`、`error` 等。
- **单行摘要**，描述发生了什么，大多数情况下无需展开就能了解大意。
- **Token 计数**，对应该步骤。
- **上下文窗口占用徽章**（适用时），让 Prompt 增长和即将到来的压缩操作在造成问题之前就清晰可见。

实时观察意味着你能在坏的部署、失控循环或错误爆发时第一时间发现，而不是等到明天的日志审查才知道。

## 找到那一个关键的运行

当某些地方看起来不对时，你不需要面对全量数据洪流，你需要的是那个出问题的单次运行。事件流可以快速过滤：按环境、按 Agent、按会话、按事件类型，或按自由文本。

通过会话 ID 或 Agent ID 过滤，可以追踪一次运行从第一个事件到最后一个事件的完整过程。通过事件类型过滤，可以将某类活动单独隔离出来——例如在一个视图中查看组织内所有的 `error` 事件。叠加多个过滤条件，只需几次点击，就能从"所有地方的所有事件"精准定位到"这个 Agent，在生产环境，正在报错"，然后立即采取行动。

自由文本搜索可以直接定位到某条消息、某个工具名或某个已知的 ID，让客户反馈的问题在几秒内对应到具体的运行记录。

## 在哪里找到它

事件流就是你的组织主页。登录后，它是你看到的第一个界面，路径为 `/<org-slug>/`，让你一到就能开始排查问题。

在它背后，Agent 通过 SDK 发出事件，采集器将事件传输到你的 FailproofAI 可观测性服务器，事件流在你掌控的基础设施中实时追踪这些事件的到来。如果你想看汇总视图而不是原始记录，每次运行的所有事件会在会话页面折叠成一行，一键即达。

这是所有其他可观测性视图的原始数据来源，因此当其他地方的数据看起来有误时，事件流是你确认真实发生了什么的地方。
