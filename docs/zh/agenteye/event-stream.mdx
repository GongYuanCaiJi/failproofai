智能体一有动作，你立刻知晓。事件流是你对生产环境中每个智能体的实时脉搏监测：无需等待，无需翻查日志，无需猜测刚刚发生了什么。

![实时事件流：按颜色区分的事件行实时滚动刷新，可按环境、智能体、会话、事件类型和自由文本进行筛选](/agenteye/images/events-stream.png)

*来自你组织中每个智能体的所有事件，最新优先，实时更新。*

## 对每个智能体的实时脉搏监测

当智能体开始一次运行、调用模型、触发工具、执行 hook 或遇到错误时，该行会在事件发生的瞬间出现在流的顶部。它实时追踪你组织中每个智能体的所有事件，最新优先，让你始终掌握当前状态，而非过时信息。

这意味着无需在某台机器上追踪日志文件，无需跨机器 grep，无需手动拼接时间戳。打开一个页面，你已经在监看生产环境。

行按类型用颜色区分，让你一眼读懂事件流，而不必逐行解析。每行一览即知：

- **类型**，颜色标注：`agent_start`、`model_response`、`tool_use`、`hook_completed`、`error` 等。
- **单行摘要**，说明发生了什么，大多数时候无需点开就能明白要点。
- **Token 计数**，对应该步骤。
- **上下文窗口占用标记**（适用时），让提示词增长和即将到来的压缩操作在造成影响之前便清晰可见。

实时监看意味着你能在问题发生时就发现——糟糕的部署、失控的循环、突发的错误——而不是在明天的日志审查中才察觉。

## 定位那一次关键运行

当某些事情看起来不对劲时，你不需要铺天盖地的信息，你需要的是那一次出问题的运行。事件流的筛选很快捷：按环境、智能体、会话、事件类型或自由文本进行过滤。

通过会话 ID 或智能体 ID 筛选，可以从第一个事件到最后一个事件追踪某次运行的完整过程。通过事件类型筛选，可以隔离单一类型的活动，例如在一个视图中查看整个组织的所有 `error`。叠加筛选条件，只需几次点击即可从"所有环境，全部内容"缩小到"这个智能体，在生产环境，正在报错"，然后立即采取行动。

自由文本搜索可以直接定位到某条消息、某个工具名称或你已知的某个 ID，让一份客户反馈在数秒内对应到确切的运行记录。

## 在哪里找到它

事件流是你的组织主页。登录后，它是你首先看到的界面，位于 `/<org-slug>/`，让你一到达就能开始排查问题。

在此背后，你的智能体通过 SDK 发出事件，采集器将它们传送到你的 Failproof AI Observability 服务器，而事件流则在事件到达时实时追踪——整个基础设施由你掌控。当你想要汇总视图而非原始记录时，每次运行的事件会在"会话"中折叠为单行，一键即达。

这是所有其他观测界面的原始数据来源，因此当其他地方的数字看起来有误时，事件流就是你确认实际发生了什么的地方。
