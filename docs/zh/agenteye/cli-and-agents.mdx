一条命令，掌控您的整个 Failproof AI 可观测性部署。无需离开终端，即可检查生产环境、生成 API 密钥或确认告警事件，还可以将任意操作脚本化集成到 CI，或者用自然语言让编码智能体代您完成。

```bash
pipx install agenteye
agenteye login --email you@example.com      # 一个 6 位验证码将发送到您的邮箱
agenteye --json sessions --since 24h         # 过去 24 小时内的所有 Agent 运行记录，最新的在前
```

*`agenteye` CLI 与您的控制台通信，是一个独立工具，区别于负责将事件发送到服务器的采集器。*

## 一条命令，掌控整个部署

无需在多个标签页间来回切换来回答一个简单问题。`agenteye` CLI 通过单一二进制文件读取您的数据并管理您的组织，过去需要在控制台中点击多步才能完成的检查，现在一行命令即可搞定——可以重复执行、设置别名，或粘贴到运维手册中。它提供四个功能维度：

- **读取数据：** `sessions`、`events`、`evals` 和 `errors`，支持按时间、Agent 和环境过滤。
- **管理组织：** `keys`、`users`、`settings`、`alerts` 和 `incidents`。
- **运行分析：** 支持已保存的 SQL 查询，以及针对事件数据的即席 `query` 执行。
- **询问助手：** `agent ask` 可访问与控制台中相同的只读分析助手。

使用 `pipx` 一次性安装，通过邮件发送的 6 位验证码登录，即可开始使用。会话有效期约为一天，过期后重新运行 `agenteye login` 即可。用它来快速检查生产环境、配置密钥或分类正在触发的告警事件，全程无需打开浏览器：

```bash
agenteye errors --since 24h --aggregate      # 按错误类型分组，查看当前异常
agenteye incidents list --state firing       # 查看当前正在触发的告警
agenteye keys create ci --add events:add     # 创建一个只能推送事件的密钥，密钥仅显示一次
```

有一个使用习惯需要注意：`--json` 等全局选项需放在命令之前。`agenteye --json sessions` 是正确的写法；`agenteye sessions --json` 则不正确。

## 脚本化，集成到 CI

每个命令都支持 `--json`，这让一切大不相同。干净的 JSON 输出到 stdout，而人类可读的状态信息和警告输出到 stderr，因此 `--json` 的输出可以直接通过管道传给 `jq`，无需剔除多余内容。这也正是 CLI 既适合您在终端手动使用，也适合编码智能体解析输出的原因：

```bash
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'
```

它专为无人值守运行而设计。当没有终端附加时，确认提示会自动跳过，因此不会在流水线中卡住；每个命令都会返回有意义的退出码：`0` 表示成功，`4` 表示未登录，`5` 表示缺少权限（消息中会明确指出，例如 `alerts:write`），`3` 表示控制台不可达。脚本可以根据 `4` 进行重新认证，或根据 `5` 精确告知需要向管理员申请哪项权限，而不是盲目失败。

## 用自然语言让编码智能体来驱动

更进一步，您根本不需要记住这些参数。**CLI 技能**是一个名为 `agenteye-cli` 的小型 Agent Skill 文件夹，它可以让 Claude Code 或 Codex 等编码智能体通过自然语言请求来驱动 CLI。只需问"今天有什么异常吗？"，智能体就会自动选择合适的命令、以您的身份执行，并以自然语言回答。

对于 Claude Code，将 `agenteye-cli` 文件夹放入 `~/.claude/skills/` 即可自动发现。Failproof AI 可观测性已提供该文件夹；无需额外安装任何内容，因为它只是驱动您已安装的 CLI。请先自行登录：该技能无法替您完成邮件验证码登录流程。

由于智能体以您的身份运行 CLI，它可以执行您的登录权限允许的所有操作，包括读取和写入：创建密钥、修改设置、解决事件。CLI 的确认提示不会对智能体触发，因此该技能在执行任何变更操作前会明确说明将要运行的命令，并等待您确认。您就是那个确认步骤。

```text
you    为什么会话 run-001 失败了？

agent  正在运行：agenteye --json events --session-id run-001 --all
       checkout-agent 在第三次工具调用时遇到了 TimeoutError。
```

读取操作立即返回结果，所有写入操作均会暂停等待您确认：

```text
you    给 CI 创建一个只能推送事件的密钥。

agent  这将创建一个 API 密钥，我将运行：
         agenteye keys create ci --add events:add
       确认执行？

you    yes

agent  完成。密钥 "ci" 已创建，权限为仅 events:add。密钥仅显示一次，请立即保存。
```
