Failproof AI Observability 是一款自托管平台，用于在生产环境中观测、评估和改进您的 AI Agent。它会记录 Agent 的一切行为（每次工具调用、模型请求、hook 以及错误），对每次运行的质量进行评分，并将那些您未曾预料到的故障呈现出来——所有这些都在一个部署在您自有基础设施内的仪表板中完成。

如果您已在生产环境中部署了 AI Agent，却厌倦了猜测某次运行究竟哪里出了问题，那么这就是您的起点。本文将介绍 Failproof AI Observability 能为您提供什么，以及各个组件如何协同工作——在您安装任何东西之前先读这里。

## 团队为什么使用它

- **看清 Agent 实际做了什么。** 每次运行都会生成可读的 git 风格执行图：哪些工具并行执行、哪些子 Agent 分支启动、在哪里卡住，以及消耗了多少资源。
- **自动捕获质量回退。** 接入一个轻量评分服务后，Failproof AI Observability 会对每次完成的运行打分，帮助功能响应率下降或幻觉频率飙升等问题自动浮出水面。
- **发现您没有写规则的故障。** 定期审计会跨会话挖掘您的日志，寻找错误聚类、延迟异常值、低评分和卡死的运行，并向您提供有优先级排序、有证据支撑的调查结论。
- **在关键时刻收到告警。** 阈值规则在错误率、延迟、成本或评估分数触发时发出告警，并创建您可以确认、指派和解决的事件。
- **用自然语言提问。** 仪表板内置的 AI 助手可以基于您自己的数据回答"本周生产环境的质量趋势如何？"之类的问题，且所有变更操作均需审批。
- **数据留在您手中。** Failproof AI Observability 采用自托管方式：事件、提示词和分析数据始终保存在您可控的基础设施中。

## 您将获得什么

Failproof AI Observability 围绕三个核心概念组织（**观测**、**分析**和**管理**），对应仪表板左侧边栏的结构。

**观测**（记录真实发生的一切）：

- **[事件流](/zh/agenteye/event-stream)**：每次运行的实时逐步记录（工具调用、模型调用、hook、错误）。
- **[会话](/zh/agenteye/sessions)**：将这些事件汇总为每次运行一行，每行均可评分，并附带 git 风格执行图。
- **[性能指标](/zh/agenteye/telemetry)**：按维度细分的延迟热力图，以及模型、工具和 hook 的 p50/p95/p99 关键指标，让尾部延迟毛刺一目了然。
- **[错误追踪](/zh/agenteye/error-tracking)**：一个统一的异常分类处理界面，一键直达触发中的告警。

![工具观测页面：延迟热力图、百分位带状图以及 24 个时间桶的工具分布柱状图](/agenteye/images/tools.png)

*每个观测界面都将迷你折线图和 p50/p95/p99 关键指标与延迟热力图和百分位带状图配对展示。图示为：工具（Tools）。*

**分析**（将活动转化为洞察）：

- **[查询](/zh/agenteye/queries)**和**[仪表板](/zh/agenteye/dashboards)**：针对您的事件和评估数据执行已保存的 SQL 查询，并以图表形式呈现在团队共享、组织范围的仪表板中。
- **[评估](/zh/agenteye/evaluations)**：由您自己的评估服务生成的质量评分，附带每项评分的推理说明。
- **[审计](/zh/agenteye/audits)**：定期调查，跨会话挖掘故障模式。
- **[告警](/zh/agenteye/alerts)**和**[事件](/zh/agenteye/incidents)**：触发告警的阈值规则，以及用于分类处理的事件工作流。

**接口**（以您喜欢的方式访问数据）：

- **[CLI](/zh/agenteye/cli-and-agents)**：从终端或脚本驱动整个部署流程，也可以让编程 Agent 用自然语言代劳。
- **[AI 助手](/zh/agenteye/assistant)**：直接在仪表板中用自然语言提问，了解您的 Agent 情况。

**管理**（为您的团队运维）：

- **[API 密钥](/zh/agenteye/api-keys)**：为采集器、仪表板和助手分别颁发的范围化 Token。
- **用户**：基于邮箱的无密码登录，配合白名单管理。
- **设置**：组织级配置，包括模型上下文窗口覆盖选项。

## 各组件如何协同工作

数据沿单一方向流动，从您的 Agent 代码流向仪表板：您的 Agent（通过 Python SDK）将事件发送给 agenteye-collector，后者将事件批量传输到服务器，服务器再为仪表板提供数据。另有两个可选服务作为补充——评分服务（评估）和 AI 助手服务（仪表板内的对话功能）。

- **Python SDK**：您在 Agent 中添加若干 `agenteye.event.*` 调用；事件在本地缓冲。
- **agenteye-collector**：部署在每台 Agent 机器上的轻量级守护进程，负责批量收集事件并发送到服务器。
- **服务器**：接收您的事件，并将运营状态保存在您自己的数据库中。
- **仪表板**：您探索一切数据的地方。
- **可选服务**：评分服务（评估）和 AI 助手服务（仪表板内的对话功能）。

有关文档中使用的术语词汇（*event、session、evaluation、audit、finding、incident*），请参阅 [概念](/zh/agenteye/concepts)。
