用日常语言向你的 Agent 数据提问，并获得直接链接到证据的答案。无需编写 SQL，无需翻遍仪表板——**FailproofAI Observability** 助手是团队中任何人获取 Agent 相关答案的最快方式。

![FailproofAI Observability 助手在仪表板中回答一个自然语言问题，展示实时 Agent 活动表、每个 Agent 的模型使用情况分解以及书面总结，并内联显示其运行的查询](/agenteye/images/assistant.png)
*用日常语言提问，获得基于你自己数据构建的答案。这里它分析了哪些 Agent 最繁忙、它们使用了哪些模型，并展示了运行的查询，让你可以核实每一个数字。*

无需学习任何东西。打开对话，输入你想了解的内容，然后点击它返回的链接：

```
You:  which sessions errored today?
AI:   5 sessions errored today, newest first. Each one is linked:
      • checkout-agent   14:02   tool timeout
      • billing-agent    11:47   unhandled error
      • ...and 3 more

You:  summarize this session          (asked while viewing a run)
AI:   This run took 12 steps across 3 tools and failed near the end when a
      payment tool returned an error. It scored low on your "resolved" eval.
      Links: the session, the failing event, and that evaluation.
```

## 直接提问，直达证据

你不再需要猜测，也不再需要编写查询。问"本周生产环境的质量趋势如何？"、"今天哪些会话出错了？"或"总结一下这个会话"，几秒钟内你就能得到直接的答案，而不是自己构建查询再逐行阅读。

每个答案都附有来源依据。助手会链接到它用于得出答案的确切会话、已保存的查询和仪表板，让你可以点击确认，而不是凭空相信它的话。它还具有**页面感知**能力：当你正在查看某个会话时询问"这个会话"，它已经知道你指的是哪次运行。之后可以通过历史切换器重新打开任何早期对话，从上次离开的地方继续。

## 将好的答案保存为查询或仪表板

当一个答案值得保留时，让助手将其保存。它会为已保存的查询起草 SQL，或者从这些查询中组建仪表板，然后向你展示一张**批准 / 拒绝**卡片。在你点击批准之前不会写入任何内容，因此你可以享受"直接提问"的速度，同时始终保留最终决定权。

在**查询**页面，它更进一步，成为一个 SQL 编写者：描述你想要的查询（"显示过去 7 天内每个 Agent 的错误率"），它会将 SQL 直接流式传输到编辑器，并打开差异视图，让你在变更生效前**接受**或**拒绝**。

![可观测性查询页面及其 SQL 编辑器](/agenteye/images/query-lab.png)
*查询页面：这个编辑器是助手流式传输草稿只读查询供你接受或拒绝的地方。*

在这里通过提问创作 SQL 使用的是 `queries:run` 权限，与编辑器的**运行**按钮背后使用的权限相同。其他地方的对话需要 `agent:use` 权限。

## 可以放心开放给整个团队

你可以将助手开放给所有人，无需担心它可能触碰什么：

- **它只读取你已有权限查看的内容。** 答案的范围限定在你自己的读取权限内，因此它不会扩大你的数据访问范围。
- **每次写入操作都等待你确认。** 已保存的查询和仪表板只有在你明确点击批准后才会创建，且没有任何设置可以关闭这道门槛。
- **它永远无法删除任何内容。** 没有暴露删除工具，助手也不持有删除权限。删除操作始终由你在仪表板中亲自完成。
- **它只在你的组织内运作。** 助手只会查看你当前所在的组织。
- **你的问题属于你自己。** 提示词和答案存储在你自己的 Observability 数据库中；产品分析只记录使用元数据，从不记录你的提示词内容。

## 在哪里找到它

助手位于你的组织（`/<org-slug>/...`）下每个页面的右侧边栏。点击边栏，或按 `⌘J` / `Ctrl+J`，将其展开为完整的对话面板，并拖动其边缘以调整大小；你设置的宽度会在页面重载后保留。你需要 **`agent:use`** 权限才能使用它，否则边栏会显示为灰色。如果你的部署尚未启用它（需要 LLM 连接），你将看到一个静音的边栏而非可用的对话界面。
