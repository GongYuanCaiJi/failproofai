用自然语言向 agent 数据提问，获得直接关联证据的答案。无需编写 SQL，无需翻查仪表板——**Failproof AI Observability** 助手是团队中任何人获取 agent 相关答案的最快途径。

![Failproof AI Observability 助手在仪表板中回答一个自然语言问题，展示实时 Agent Activity 表格、每个 agent 的模型使用情况分解和书面摘要，并内联显示其执行的查询](/agenteye/images/assistant.png)
*用自然语言提问，获得基于你自己数据构建的答案。此处它分解了哪些 agent 最繁忙、它们使用了哪些模型，并展示了运行的查询，便于你核实每个数字。*

无需学习任何东西。打开对话框，输入你想知道的内容，然后点击返回的链接：

```
You:  which sessions errored today?
AI:   5 sessions errored today, newest first. Each one is linked:
      • checkout-agent   14:02   tool timeout
      • billing-agent    11:47   unhandled error
      • ...and 3 more

You:  summarize this session          (asked while viewing a run)
AI:   This run took 12 steps across 3 tools and failed near the end when a
      payment tool returned an error. It scored low on your "resolved" eval.
      Links: the session, the failing event, and that evaluation.
```

## 直接提问，直达证据

你无需再猜测，也无需再手写查询。问"本周生产环境的质量趋势如何？""今天哪些会话出现了错误？"或"总结一下这个会话"，几秒钟内即可得到直接答案，而不必自己构建查询再逐一阅读。

每个答案都附有来源依据。助手会链接它用于得出答案的确切会话、已保存的查询和仪表板，让你可以点击验证，而不必只凭信任。它还具备**页面感知**能力：当你正在查看某个会话时询问"这个会话"，它已经知道你指的是哪次运行。之后可以从历史切换器中重新打开任何之前的对话，从上次中断的地方继续。

## 将好的答案保存为查询或仪表板

当某个答案值得保留时，直接让助手保存它。它会为已保存的查询起草 SQL，或从这些查询组装仪表板，然后向你显示一张**批准 / 拒绝**卡片。在你点击批准之前，任何内容都不会被写入，因此你可以享有"直接提问"的速度，同时始终拥有最终决定权。

在**查询**页面，助手更进一步，成为 SQL 撰写者：描述你想要的查询（"显示过去 7 天各 agent 的错误率"），它会将 SQL 直接流式输出到编辑器中，并打开差异视图，让你在内容落地前**接受**或**拒绝**更改。

![Observability 查询页面及其 SQL 编辑器](/agenteye/images/query-lab.png)
*查询页面：此编辑器是助手流式生成草稿（只读查询）供你接受或拒绝的地方。*

在此处通过提问来撰写 SQL 使用的是 `queries:run` 权限，与编辑器**运行**按钮背后使用的权限相同。其他地方的对话功能需要 `agent:use` 权限。

## 可安全开放给整个团队

你可以放心地向所有人开放助手，无需担心它可能触及的内容：

- **它只读取你已有权限查看的内容。** 答案的范围受限于你自己的读取权限，因此它不会扩大你的数据访问面。
- **每次写入都等待你确认。** 已保存的查询和仪表板只有在你明确点击批准后才会创建，且没有任何设置可以关闭这道审批门槛。
- **它永远无法删除任何内容。** 没有删除工具被暴露，助手也不具备删除权限。删除操作始终由你在仪表板中执行。
- **它仅限于你的组织内部。** 助手只会查看你当前正在查看的组织。
- **你的问题属于你自己。** 提示词和答案存储在你自己的 Observability 数据库中；产品分析仅记录使用元数据，而非你的提示词文本。

## 在哪里找到它

助手位于你的组织（`/<org-slug>/...`）下每个页面的右侧边栏。点击边栏，或按 `⌘J` / `Ctrl+J`，将其展开为完整的对话面板，拖动边缘可调整大小；你设置的宽度在页面重新加载后会被记住。使用它需要 **`agent:use`** 权限，否则边栏将显示为灰色。如果你的部署尚未开启此功能（需要 LLM 连接），你会看到一个静默的边栏，而非可用的对话界面。
