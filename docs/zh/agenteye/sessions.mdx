## 一览所有运行

原始事件流记录了每一步的真实情况，但当你面对数十次运行中的数千个步骤时，你需要的是运行概览，而不是逐步明细。Sessions 页面将一次运行的所有事件汇总为一行，使一天的活动变成可快速浏览的列表，而不是信息洪流。

每行都带有状态标签，即使不点击，失败的运行也会与正常运行一眼区分。通过日期范围、环境、Agent 或会话进行筛选，只需几次点击即可从"查看全部"精准定位到"我关心的那次运行"。

接入评估器后，每次完成的运行都会自动获得评分，最新分数以徽章形式显示在对应行上。你可以按任意分数范围筛选，"显示本周所有低分生产运行"只是一个过滤条件，无需人工逐一排查。尚未配置评估器时，会话仍会完整记录运行数据，只是暂时没有分数而已。

## 以图形方式读懂整次运行

![会话的 git 风格执行图及其事件时间线，右侧为工具、模型和 Hook 的详情面板](/agenteye/images/session-detail.png)

*执行图（左侧）与事件时间线并排显示；右侧栏展示本次运行的工具、模型、Hook 及 Token 消耗详情。*

点击任意会话即可打开其执行图：这是一个类似 git 的视图，展示 Agent、工具、Hook 和模型调用随时间推进的完整过程。并行子 Agent 各自占据独立泳道，让你清晰看到哪些工作同步进行、哪个子 Agent 出现停滞、运行在哪里偏离了预期，无需在脑海中反复回放大量日志。

右侧栏提供本次运行的详细拆解：调用了哪些工具和模型、触发了哪些 Hook、消耗了多少 Token。"这次运行为何花费这么多？"或"哪个工具最慢？"——答案就在触发它们的执行图旁边，一览无余。

## 访问路径

每个仪表盘页面都以你的组织为作用域（`/<org-slug>/…`）。Sessions 位于左侧边栏 **Observe** 下方，紧邻 Events，列表顶部提供日期范围、环境、Agent 和会话等筛选器。每行只需点击一次即可查看完整执行图。

如需开启分数徽章和分数范围筛选，请接入评估器，详见 [Evaluations](/zh/agenteye/evaluations)。
