## 一览所有运行

原始事件流记录了每一个步骤的真实情况，但当你面对数十次运行、数千个步骤时，你需要的是整体视图，而非单个步骤的细节。Sessions 页面将一次运行的所有事件汇总为一行，让一天的活动变成一份可快速浏览的列表，而不是信息洪流。

每行都带有状态标签，失败的运行在你点击之前就能一眼辨认。通过日期范围、环境、Agent 或会话进行筛选，几次点击即可从"全部"定位到"我关心的那次运行"。

连接评估器后，每次已完成的运行会自动评分，最新分数以徽章形式显示在对应行上。你可以按任意分数范围筛选，"显示本周所有低分生产运行"只是一个筛选条件，无需手动逐一审查。在你设置评估器之前，会话仍会完整记录每次运行，只是暂时不带分数。

## 以图形读懂整个运行过程

![会话的类 git 执行图与事件时间线并排显示，右侧面板展示工具、模型及 Hook 的详情](/agenteye/images/session-detail.png)

*执行图（左）与事件时间线并排显示；右侧栏按运行维度列出工具、模型、Hook 及 Token 消耗。*

点击任意会话即可打开其执行图：以类 git 视图展示 Agent、工具、Hook 和模型调用随时间展开的全过程。并行的子 Agent 各自分支到独立泳道，让你清楚看到哪些工作同时进行、哪个子 Agent 出现停滞、运行在哪里偏离了轨道——无需在脑海中从一堆日志里逐行回放。

右侧栏提供每次运行的详细拆解：运行了哪些工具和模型、触发了哪些 Hook、消耗了多少 Token。"这次运行为什么这么贵？"或"哪个工具最慢？"的答案就在执行图旁边一目了然。

## 在哪里找到它

每个 Dashboard 页面都限定在你的组织范围内（`/<org-slug>/…`）。Sessions 位于左侧边栏的 **Observe** 下，与 Events 并列，列表顶部提供日期范围、环境、Agent 和会话筛选器。每行均可一键跳转至完整的执行图。

若要开启分数徽章和分数范围筛选，请连接评估器，详见[评估（Evaluations）](/zh/agenteye/evaluations)。
