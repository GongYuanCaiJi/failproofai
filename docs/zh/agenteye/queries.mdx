对 Agent 数据提出任何问题，秒级获取答案。Failproof AI Observability 为您提供一个已保存的即用查询库，涵盖您的事件和评估数据，让您从现成示例出发，而不必面对空白的 SQL 编辑器。

![已保存查询库：一个可复用查询的网格视图，包含内置预设和自定义查询](/agenteye/images/queries.png)

*您在 `/<org-slug>/queries` 的已保存查询库：内置预设与团队保存的自定义查询并排展示。*

## 从预设开始，而非空白页面

您无需记住表名，也无需从头编写 SQL。查询库打开后即展示内置预设，涵盖团队最常问的问题，与您的团队已保存并命名的查询并排显示。选择一个与您需求接近的预设，即可快速得到答案。

每个已保存查询都属于组织范围并可共享，因此团队成员编写的实用查询也会自动为您所用。为查询命名并添加描述后，组织内的任何人都可以找到它、运行它，或在之后将其结果固定到仪表盘上。

访问地址：`/<org-slug>/queries`。

## 在 SQL 编辑器中调整并运行

打开任意查询，它将在 SQL 编辑器中加载，您可以即时调整并立刻查看结果——无需导出，无需往返，无需等待他人。

![SQL 查询编辑器正在运行一个已保存查询，左侧为 schema 面板，下方为实时结果表格](/agenteye/images/query-lab.png)

*SQL 编辑器：左侧是您的查询，右侧 schema 面板让您无需猜测列名，下方是实时结果表格。*

- **Schema 面板**：列出分析表及其字段，让您无需四处寻找字段名即可构建查询。
- **实时结果表格**：运行后立即返回数据行，您可以在秒级内完成迭代，而无需反复猜测。
- **只读设计。** 查询在您的事件存储上运行，并在服务器端进行验证：仅允许 `SELECT` 和 `WITH` 语句，同时设有语句超时和行数上限。探索性查询永远不会修改您的数据，失控的查询也会被自动终止。

对结果满意？将其保存回查询库，让整个团队共享，或将输出以折线图、柱状图、面积图或饼图的形式固定到仪表盘上。

## 从终端运行，或让助手帮您编写

已保存查询可在任何工作场景下使用：

- **从终端。** `agenteye` CLI 可以列出、运行和保存完全相同的查询，让您将结果集成到脚本中、接入 CI，或传递给代码 Agent。

```bash
agenteye query list                 # 在终端中查看已保存查询
agenteye query run errs --arg prod  # 运行查询并打印数据行（添加 --json 可用于管道）
```

  完整命令集请参阅 [CLI 与 Agent](/zh/agenteye/cli-and-agents)。

- **从 AI 助手。** 不确定如何编写 SQL？用自然语言向仪表盘内的 [AI 助手](/zh/agenteye/assistant) 提问，它会为您起草查询并保存到查询库中。

运行已保存查询需要 `queries:run` 权限，该权限与创建或删除查询的权限相互独立，因此您可以在不允许所有人修改查询库的前提下授予只读访问权限。
