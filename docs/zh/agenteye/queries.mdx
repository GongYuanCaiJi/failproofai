对您的 Agent 数据提出任何问题，即可在几秒内获得答案。FailproofAI 可观测性为您提供一个针对事件和评估数据的已保存、可直接运行的查询库，让您从可用示例开始，而无需面对空白的 SQL 编辑器。

![已保存查询库：一个可复用查询的网格视图，包含内置预设和自定义查询](/agenteye/images/queries.png)

*您的已保存查询库位于 `/<org-slug>/queries`：内置预设与您团队保存的查询并排展示。*

## 从预设开始，而不是空白页

您无需记住表名或从头编写 SQL。查询库打开时即展示内置预设，涵盖团队最常提出的问题，并与您自己团队已保存和命名的查询并排显示。选择一个接近您需求的预设，您就已经完成了大半的工作。

每个已保存的查询都是按组织范围共享的，因此您的团队成员编写的实用查询也会成为您的资源。为查询命名并添加描述后，组织内任何人都可以找到它、运行它，或稍后将其结果固定到仪表板上。

访问路径：`/<org-slug>/queries`。

## 在 SQL 编辑器中调整并运行

打开任意查询，它将在 SQL 编辑器中加载，您可以随时调整并立即查看结果：无需导出，无需来回传递，无需等待他人。

![SQL 查询编辑器正在运行一个已保存的查询，左侧有 Schema 侧边栏，下方有实时结果表格](/agenteye/images/query-lab.png)

*SQL 编辑器：左侧是您的查询，Schema 侧边栏让您无需猜测列名，下方是实时结果表格。*

- **Schema 侧边栏**：列出分析表及其列名，让您无需搜寻字段名即可构建查询。
- **实时结果表格**：运行后立即返回数据行，让您在几秒内完成迭代，而无需反复猜测。
- **只读设计。** 查询在您的事件存储上运行，并在服务器端进行验证：仅允许 `SELECT` 和 `WITH` 语句，并设有语句超时和行数上限。探索性查询永远不会修改您的数据，失控的查询会自动被停止。

对结果满意？将其保存回库中，让整个团队继承，或将其输出以折线图、柱状图、面积图或饼图的形式固定到仪表板上。

## 从终端运行，或让助手来编写

已保存的查询无论您在哪里工作都可使用：

- **从终端。** `agenteye` CLI 可以列出、运行和保存完全相同的查询，因此您可以将结果嵌入脚本、接入 CI，或传递给编码 Agent。

```bash
agenteye query list                 # the same saved queries, from your terminal
agenteye query run errs --arg prod  # run one and print the rows (add --json to pipe it)
```

  完整命令集请参阅 [CLI 与 Agent](/zh/agenteye/cli-and-agents)。

- **从 AI 助手。** 不确定如何编写 SQL？用自然语言向仪表板内的 [AI 助手](/zh/agenteye/assistant)提问，它将为您草拟查询并保存到您的查询库中。

运行已保存查询需要 `queries:run` 权限，该权限与创建或删除查询的权限分开管理，因此您可以授予读取权限而不必让所有人都能修改查询库。
