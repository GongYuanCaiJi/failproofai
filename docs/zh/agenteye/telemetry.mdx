即时发现模型、工具或 hooks 何时出现性能下降或费用攀升，在用户感知到尾部延迟峰值之前将其捕获。三个专属页面将原始耗时数据转化为一目了然的 p50、p95 和 p99 指标。

![Models 页面展示延迟热力图、百分位带以及各模型的 token 用量、成本和上下文窗口占用情况](/agenteye/images/models.png)
*Models 页面：延迟热力图、百分位带，以及各模型的 token 用量、预估成本和上下文窗口占用率。*

## 别让平均值掩盖最差的那次调用

平均延迟数字看起来令人安心，实则毫无意义——它将每五十次调用中那一次导致凌晨两点报警的卡顿统统抹平了。Models、Tools 和 Hooks 页面拒绝这样做。三个页面共用相同的布局，学会一次即可举一反三：

- **24 格迷你折线图**，一眼看出趋势：情况是否在恶化？
- **关键指标条**，并排展示 p50、p95 和 p99 延迟，让典型耗时与尾部耗时一目了然。
- **延迟热力图**：横轴为 24 个时间格，纵轴为延迟区间，直观呈现慢速调用集中发生的*时段*。
- **百分位带**：以 p50 为中线，附带 p25–p75 和 p10–p90 的阴影区间以及 p99 散点，让分布情况一览无余，而非被平均值淹没。

热力图与百分位带共享悬停十字准线联动，尾部峰值在两个视图中时间对齐，不会藏在单一均值线背后。这三个页面均位于仪表盘的 **observe** 区域，可按组织维度查看，支持按日期范围、环境、agent 和会话进行筛选。

## Models：精确掌握每个模型的成本

Models 页面（如上图所示）直接回答账单上的两个问题：哪个模型，花了多少钱。在共用延迟视图的基础上，它额外展示**各模型的 token 消耗量**、**预估费用**和**上下文窗口占用率**，让 prompt 不受控制地膨胀和即将触发的上下文压缩在发生之前就暴露无遗。

Failproof AI Observability 会自动识别常见的模型 ID。如果某个窗口显示有误，或者您运行的是私有模型，可在 **Settings** 的 **model context windows** 中进行修正或添加，占用率数据将随之更新。

## Tools：区分慢速与故障

一次工具调用可能是速度慢，也可能是悄无声息地在报错——您需要在几秒内搞清楚是哪种情况，而不是翻遍日志之后才明白。

![Tools 页面展示共用的延迟热力图和百分位带，旁边是成功与失败分布图以及工具调用占比条形图](/agenteye/images/tools.png)
*Tools 页面：相同的热力图和百分位带，外加成功与失败分布图和工具调用占比条形图。*

在共用延迟视图的基础上，Tools 页面额外提供**成功与失败分布图**和**工具调用占比条形图**，让您一眼看出哪些工具被最频繁调用，哪些正在消耗您的错误预算。

## Hooks：精准定位具体的 hook 和触发事件

当某个生命周期 hook 拖慢了运行速度，"hooks 太慢了"这种结论根本无从下手。Hooks 页面帮您直接锁定问题所在。

![Hooks 页面在共用热力图和百分位带之上，按 hook 名称和触发事件展示延迟分布](/agenteye/images/hooks.png)
*Hooks 页面：按 hook 名称和触发事件细分的延迟数据。*

在相同的延迟热力图和百分位带之上，Hooks 页面将活动按 **hook 名称**和**触发事件**进行细分，让您直接定位到需要关注的那一个 hook 和那一个事件。
