Failproof AI Observability는 프로덕션 에이전트 가까이에서 동작하도록 설계되어 있으며, 프롬프트, 도구 입력 및 출력을 모두 확인합니다. 이 페이지에서는 해당 데이터를 어떻게 격리하고, 제어하며, 사용자의 손에 유지하는지 설명합니다. 보안 검토를 위해 Failproof AI Observability를 평가 중이라면 여기서 시작하세요.

---

## 데이터는 귀하의 환경에 머뭅니다

Failproof AI Observability는 셀프 호스팅 방식입니다. 이벤트, 프롬프트, 모델 응답, 분석 데이터는 모두 귀하의 환경에 있는 귀하의 데이터베이스에 저장됩니다. 서드파티 SaaS로 전송되어 저장되는 데이터는 없으며, 모든 데이터는 귀하의 클라우드 계정 안에 유지됩니다.

---

## 테넌트 격리

하나의 Failproof AI Observability 인스턴스에서 여러 조직을 호스팅할 수 있으며, 각 조직은 UI가 아닌 데이터베이스 수준에서 스토리지 레이어 격리가 적용됩니다.

- 조직의 운영 데이터(사용자, 키, 대시보드, 저장된 쿼리)는 해당 조직 범위로 한정되며, 조직 간 읽기 접근은 데이터베이스 자체에서 차단됩니다.
- 수집된 모든 이벤트에는 소유 조직이 태깅되어, 한 조직의 이벤트를 다른 조직에서 절대 읽을 수 없습니다.

모든 대시보드 라우트는 조직 슬러그(`/<org-slug>/…`) 하위로 범위가 지정됩니다.

## 로그인

Failproof AI Observability는 비밀번호 없는 이메일 기반 로그인을 사용합니다. 피싱하거나 유출될 비밀번호가 없습니다. 사용자가 일회용 코드(또는 원클릭 매직 링크)를 요청하면 이메일로 전송되며, 빠르게 만료됩니다. 로그인은 **허용 목록**으로 제한됩니다. 귀하가 허용한 이메일 주소(또는 도메인)만 인증할 수 있습니다.

## API 키를 활용한 범위 기반 접근

모든 클라이언트는 세분화된 최소 권한을 가진 API 키로 인증합니다. 수집기는 `events:add`만 필요하고, 대시보드나 어시스턴트 키는 읽기 전용으로 설정할 수 있으며, 삭제나 재생성 같은 파괴적 작업은 별도로 부여하는 권한입니다.

![API 키 페이지: 각 키의 권한이 읽기, 쓰기, 파괴적 범위에 따라 색상으로 구분되어 표시](/agenteye/images/api-keys.png)

관리자 부트스트랩 키는 설정용으로만 보관하고, 그 외의 모든 용도에는 좁은 범위의 키를 발급하세요. [API 키](/ko/agenteye/api-keys)를 참고하세요.

## 읽기 전용의 승인 기반 어시스턴트

대시보드 내 [AI 어시스턴트](/ko/agenteye/assistant)는 귀하의 데이터를 기반으로 질문에 답하지만, 설계상 다음과 같이 제한됩니다.

- **기본적으로 읽기 전용**입니다. 실행하는 SQL은 `SELECT`/`WITH` 쿼리만 허용하고, 단일 구문이며, 행 수 제한이 있는 가드를 통해 처리됩니다.
- 어시스턴트가 생성하는 모든 항목(저장된 쿼리, 대시보드)은 **승인이 필요**합니다. 쓰기가 실행되기 전에 귀하가 직접 검토하고 승인합니다.
- **절대로 삭제할 수 없습니다.**

따라서 팀원이 "이번 주에 오류가 가장 많이 발생한 에이전트는 무엇인가요?"라고 질문하고 결과에 따라 조치를 취할 수 있지만, 어시스턴트가 자체적으로 데이터를 변경하거나 삭제하는 것은 불가능합니다.

## 전송 중 보안

모든 트래픽은 HTTPS를 통해 전송됩니다. 귀하의 인증서로 TLS를 종료하므로, 수집기-서버 간 및 브라우저-서버 간 트래픽은 전송 중 암호화됩니다.
