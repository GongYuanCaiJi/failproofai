에이전트 데이터에 대해 자연어로 질문하고, 증거로 바로 연결되는 답변을 받아보세요. SQL을 작성할 필요도, 대시보드를 뒤질 필요도 없습니다. **Failproof AI Observability** 어시스턴트는 팀 누구나 에이전트에 대한 답변을 가장 빠르게 얻을 수 있는 방법입니다.

![평문 질문에 답하는 Failproof AI Observability 어시스턴트. 실시간 에이전트 활동 테이블, 에이전트별 모델 사용 현황, 텍스트 요약이 표시되며 실행된 쿼리가 인라인으로 보임](/agenteye/images/assistant.png)
*자연어로 질문하면 실제 데이터를 기반으로 한 답변을 받을 수 있습니다. 여기서는 가장 바쁜 에이전트와 각 에이전트가 사용하는 모델을 분석하고, 모든 수치를 직접 확인할 수 있도록 실행된 쿼리도 함께 보여줍니다.*

따로 배울 것이 없습니다. 채팅을 열고, 알고 싶은 것을 입력하고, 반환된 링크를 따라가기만 하면 됩니다:

```
You:  which sessions errored today?
AI:   5 sessions errored today, newest first. Each one is linked:
      • checkout-agent   14:02   tool timeout
      • billing-agent    11:47   unhandled error
      • ...and 3 more

You:  summarize this session          (asked while viewing a run)
AI:   This run took 12 steps across 3 tools and failed near the end when a
      payment tool returned an error. It scored low on your "resolved" eval.
      Links: the session, the failing event, and that evaluation.
```

## 질문하고 바로 증거로 이동하기

더 이상 추측하거나 쿼리를 직접 작성할 필요가 없습니다. "이번 주 프로덕션 품질은 어떻게 변하고 있나요?", "오늘 오류가 발생한 세션은 어떤 것들인가요?", "이 세션을 요약해줘" 같은 질문을 하면, 쿼리를 만들고 직접 해석하는 대신 몇 초 안에 명확한 답변을 받을 수 있습니다.

모든 답변에는 근거가 함께 제공됩니다. 어시스턴트는 답변 도출에 사용한 정확한 세션, 저장된 쿼리, 대시보드를 링크로 제공하므로, 그냥 믿는 대신 직접 클릭해서 확인할 수 있습니다. 또한 **페이지 인식** 기능이 있어, 특정 세션을 보는 중에 "이 세션"에 대해 질문하면 어시스턴트가 어떤 실행을 의미하는지 이미 알고 있습니다. 히스토리 전환기에서 이전 대화를 다시 열면 중단했던 곳에서 계속 이어갈 수 있습니다.

## 유용한 답변을 저장된 쿼리나 대시보드로 변환하기

보존할 가치 있는 답변이 있다면 어시스턴트에게 저장을 요청하세요. 저장된 쿼리용 SQL 초안을 작성하거나, 해당 쿼리들로 대시보드를 구성한 뒤 **승인 / 거부** 카드를 보여줍니다. 승인을 클릭하기 전까지는 아무것도 저장되지 않으므로, "그냥 물어보기"의 속도를 누리면서도 최종 결정권은 항상 사용자에게 있습니다.

**Queries** 페이지에서는 한 단계 더 나아가 SQL 작성자 역할을 합니다. 원하는 쿼리를 설명하면("최근 7일간 에이전트별 오류율 보여줘") 에디터에 SQL을 스트리밍하고, 변경 사항이 적용되기 전에 **수락** 또는 **거부**할 수 있도록 diff 뷰를 엽니다.

![Observability Queries 페이지와 SQL 에디터](/agenteye/images/query-lab.png)
*Queries 페이지: 어시스턴트가 초안 읽기 전용 쿼리를 스트리밍하는 에디터. 수락 또는 거부할 수 있습니다.*

여기서 SQL을 질문으로 작성하면 `queries:run` 권한이 사용되며, 이는 에디터의 **실행** 버튼과 동일한 권한입니다. 다른 곳에서의 채팅은 `agent:use` 권한이 필요합니다.

## 팀 전체에 안심하고 공개 가능

어시스턴트가 무엇을 건드릴지 걱정 없이 팀 모두에게 공개할 수 있습니다:

- **이미 볼 수 있는 데이터만 읽습니다.** 답변 범위가 사용자의 읽기 권한으로 제한되므로, 데이터 접근 범위가 확장되는 일이 없습니다.
- **모든 쓰기 작업은 사용자의 승인을 기다립니다.** 저장된 쿼리와 대시보드는 명시적인 승인 클릭 후에만 생성되며, 이 관문을 끄는 설정은 존재하지 않습니다.
- **어떤 것도 삭제할 수 없습니다.** 삭제 도구가 노출되어 있지 않으며 어시스턴트는 삭제 권한을 보유하지 않습니다. 삭제는 대시보드에서 사용자가 직접 수행합니다.
- **조직 내에서만 동작합니다.** 어시스턴트는 현재 보고 있는 조직의 데이터만 조회합니다.
- **질문은 사용자의 것입니다.** 프롬프트와 답변은 사용자 자신의 Observability 데이터베이스에 저장되며, 제품 분석은 사용 메타데이터만 기록하고 프롬프트 텍스트는 절대 기록하지 않습니다.

## 사용 위치

어시스턴트는 조직 하위의 모든 페이지(`/<org-slug>/...`) 오른쪽 가장자리에 항상 표시됩니다. 레일을 클릭하거나 `⌘J` / `Ctrl+J`를 눌러 전체 채팅 패널로 확장하고, 가장자리를 드래그해 크기를 조절할 수 있으며 너비는 새로고침 후에도 기억됩니다. 사용하려면 **`agent:use`** 권한이 필요하며, 없는 경우 레일이 비활성화 상태로 표시됩니다. 배포 환경에서 아직 활성화되지 않은 경우(LLM 연결이 필요함), 작동하는 채팅 대신 흐릿한 레일이 표시됩니다.
