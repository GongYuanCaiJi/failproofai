에이전트 데이터에 대해 평범한 영어로 질문하면 근거 자료로 바로 연결되는 답변을 받을 수 있습니다. SQL을 작성하거나 대시보드를 뒤질 필요가 없습니다. **FailproofAI Observability** 어시스턴트는 팀 누구든 에이전트에 관한 답변을 가장 빠르게 얻을 수 있는 방법입니다.

![대시보드 내에서 평범한 영어 질문에 답하는 FailproofAI Observability 어시스턴트 화면. 실시간 에이전트 활동 테이블, 에이전트별 모델 사용 현황, 주요 인사이트가 표시되며 실행된 쿼리도 인라인으로 확인할 수 있습니다.](/agenteye/images/assistant.png)
*평범한 영어로 질문하면 여러분의 실제 데이터를 기반으로 한 답변을 받습니다. 여기서는 가장 바쁜 에이전트와 각 에이전트가 사용하는 모델을 분석하며, 모든 수치를 직접 검증할 수 있도록 실행된 쿼리도 함께 표시합니다.*

별도로 배울 것이 없습니다. 채팅을 열고, 알고 싶은 것을 입력하고, 돌아오는 링크를 따라가기만 하면 됩니다.

```
사용자:  오늘 오류가 발생한 세션은 어디인가요?
AI:      오늘 5개의 세션에서 오류가 발생했습니다. 최신 순으로 정렬했습니다:
         • checkout-agent   14:02   tool timeout
         • billing-agent    11:47   unhandled error
         • ...외 3개

사용자:  이 세션을 요약해 줘          (특정 실행 중 화면을 보면서 질문)
AI:      이 실행은 3개의 도구에 걸쳐 12단계를 진행했으며, 결제 도구에서
         오류가 반환되면서 후반부에 실패했습니다. "resolved" 평가 항목에서
         낮은 점수를 받았습니다.
         링크: 해당 세션, 실패 이벤트, 해당 평가 결과.
```

## 그냥 물어보고, 바로 증거로 이동하세요

더 이상 추측하거나 쿼리를 직접 작성할 필요가 없습니다. "이번 주 프로덕션 품질 추이는 어떤가요?", "오늘 오류가 발생한 세션은 어디인가요?", "이 세션을 요약해 줘" 같은 질문을 하면 쿼리를 구성하고 직접 해석하는 대신 몇 초 안에 명확한 답변을 받습니다.

모든 답변에는 근거 자료가 함께 제공됩니다. 어시스턴트는 답변 도출에 사용된 정확한 세션, 저장된 쿼리, 대시보드에 대한 링크를 제공하므로, 그냥 믿는 대신 직접 클릭해서 확인할 수 있습니다. 또한 **페이지 인식** 기능도 지원합니다. 특정 세션을 보고 있는 상태에서 "이 세션"에 대해 질문하면 어떤 실행을 의미하는지 이미 파악하고 있습니다. 기록 전환기에서 이전 대화를 다시 열고 이어서 진행할 수도 있습니다.

## 좋은 답변을 저장된 쿼리나 대시보드로 만들기

보관할 가치가 있는 답변이 생기면 어시스턴트에게 저장을 요청하세요. 어시스턴트는 저장된 쿼리를 위한 SQL을 작성하거나 해당 쿼리들로 대시보드를 구성한 뒤, **승인 / 거절** 카드를 표시합니다. 승인을 클릭하기 전까지는 아무것도 저장되지 않으므로, "그냥 물어보기"의 빠른 속도와 최종 결정권이 항상 본인에게 있다는 안심감을 동시에 누릴 수 있습니다.

**Queries** 페이지에서는 한 단계 더 나아가 SQL 작성 도구로 활용할 수 있습니다. 원하는 쿼리를 설명하면 ("지난 7일간 에이전트별 오류율 표시") SQL이 에디터에 바로 스트리밍되며, 변경 사항이 적용되기 전에 **수락** 또는 **거절**할 수 있도록 diff 뷰가 열립니다.

![Observability Queries 페이지와 SQL 에디터](/agenteye/images/query-lab.png)
*Queries 페이지: 어시스턴트가 초안 읽기 전용 쿼리를 스트리밍하면 이 에디터에서 수락하거나 거절할 수 있습니다.*

여기서 질문을 통해 SQL을 작성하면 `queries:run` 권한이 사용됩니다. 이는 에디터의 **Run** 버튼과 동일한 권한입니다. 다른 모든 곳에서의 채팅에는 `agent:use` 권한이 필요합니다.

## 팀 전체에게 안전하게 공개

어시스턴트가 어떤 작업을 할지 걱정하지 않고 모든 팀원에게 공개할 수 있습니다.

- **이미 볼 수 있는 데이터만 읽습니다.** 답변은 본인의 읽기 권한 범위 내로 제한되므로 데이터 접근 범위가 절대 확장되지 않습니다.
- **모든 쓰기 작업은 본인의 승인을 기다립니다.** 저장된 쿼리와 대시보드는 명시적으로 승인 클릭을 한 후에만 생성되며, 이 검토 단계를 끄는 설정은 없습니다.
- **삭제는 절대 불가능합니다.** 삭제 도구가 노출되어 있지 않으며 어시스턴트는 삭제 권한을 보유하지 않습니다. 삭제는 대시보드에서 본인이 직접 수행해야 합니다.
- **조직 내에서만 활동합니다.** 어시스턴트는 현재 보고 있는 조직의 데이터만 볼 수 있습니다.
- **질문 내용은 본인만의 것입니다.** 프롬프트와 답변은 여러분의 Observability 데이터베이스에 저장됩니다. 제품 분석은 사용 메타데이터만 기록하며 프롬프트 텍스트는 절대 기록하지 않습니다.

## 위치 안내

어시스턴트는 조직 하위 모든 페이지(`/<org-slug>/...`)의 오른쪽 가장자리에 항상 위치합니다. 사이드 레일을 클릭하거나 `⌘J` / `Ctrl+J`를 눌러 전체 채팅 패널로 확장하고, 가장자리를 드래그해 크기를 조정할 수 있습니다. 설정한 너비는 페이지를 새로 고침해도 유지됩니다. 사용하려면 **`agent:use`** 권한이 필요하며, 권한이 없으면 레일이 비활성화(회색)됩니다. 아직 배포 환경에 활성화되지 않은 경우(LLM 연결이 필요함), 작동하는 채팅 대신 비활성화된 레일이 표시됩니다.
