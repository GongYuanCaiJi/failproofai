에이전트에서 발생한 모든 오류를 한 곳에서 확인하세요. 동일한 오류가 반복되더라도 하나의 문제로 묶어 표시하므로, "빨간 항목이 있다"는 것을 인지하는 순간부터 문제가 발생한 정확한 실행까지 클릭 한 번으로 이동할 수 있습니다. 실시간 피드를 스크롤하며 찾아 헤맬 필요가 없습니다.

*Errors 페이지: 시간 경과에 따른 오류 히스토그램과 함께, 반복 오류는 인시던트 단위로 하나의 행으로 묶입니다.*

## 모든 오류, 자동으로 수집

에이전트가 오류를 일으켰을 때, 빨간 항목이 스크롤되어 사라지기 전에 잡으려고 실시간 이벤트 스트림을 뚫어지게 들여다볼 필요가 없어야 합니다. **Errors** 페이지가 대신 수집해 드립니다. 대시보드에서 빨간색으로 표시될 모든 항목을 하나의 트리아지 화면으로 모아, 처음 눈에 들어오는 것이 "어디를 봐야 하는가"가 아닌 "무엇이 실패하고 있는가"가 되도록 합니다.

그리고 명백한 오류 이상을 잡아냅니다. 명시적인 `error` 이벤트 외에도, FailproofAI Observability는 눈에 잘 띄지 않는 실패도 감지합니다. 실패 페이로드를 포함하는 `tool_result`, `hook_completed`, `agent_end` 이벤트가 모두 여기에 표시됩니다. 오류를 반환한 도구나 비정상 종료된 훅이, 큰 예외를 던지지 않았다는 이유만으로 그냥 지나쳐지는 일이 없습니다.

페이지 상단에는 시간 경과에 따른 오류 히스토그램이 표시됩니다. 한눈에 이것이 꾸준한 배경 수준의 오류인지, 몇 분 전부터 급증한 스파이크인지 파악할 수 있어, 지금 하던 일을 멈춰야 할지 바로 판단할 수 있습니다.

다른 관측 화면과 마찬가지로, Errors 페이지는 조직 범위로 제한되며 날짜 범위, 환경, 에이전트, 세션 기준으로 필터링됩니다. 즉, 전체 플릿에 걸친 목록에서 실제로 관심 있는 특정 에이전트나 환경으로 범위를 좁힐 수 있습니다.

## 수백 개의 동일한 행이 아닌, 하나의 인시던트

의존성 하나가 깨지면 분당 수백 번 같은 오류를 발생시킬 수 있습니다. 이를 그대로 두면 거의 동일한 줄들이 벽처럼 쌓여 정작 봐야 할 것을 묻어버립니다.

FailproofAI Observability는 동일한 세션과 오류 유형을 공유하는 반복 오류를 하나의 행으로 묶습니다. 폭발적인 오류 발생이 하나의 인시던트로 읽힙니다. 로그 줄 수가 아닌 문제 수를 세게 되고, 중요한 신호가 자체 볼륨에 묻히지 않고 맨 위에 유지됩니다.

## "빨간 항목이 있다"에서 정확한 이벤트까지

어떤 행을 클릭하든 해당 실행의 세션으로 바로 이동하며, 오류가 발생한 정확한 이벤트에 위치하게 됩니다. 세션 ID를 복사하거나 문제가 발생한 순간을 찾아 스크롤할 필요 없이 바로 그 지점에 도착하며, 에이전트가 오류를 일으키기 전에 무엇을 했는지 전체 실행 그래프를 한눈에 볼 수 있습니다.

`alerts:write` 권한이 있다면, 모든 행에 **+ alert** 버튼도 표시됩니다. 클릭하면 Observability가 해당 오류를 다시 감지하도록 이미 채워진 새 알림 규칙을 열어줍니다. 방금 트리아지한 인시던트가 다음번에도 다시 여러분을 놀라게 하는 대신, 알림을 보내주는 규칙이 됩니다.

**찾는 방법:** **Errors** 페이지는 대시보드의 observe 섹션에 있으며, `/<org-slug>/errors` 경로로 접근할 수 있습니다.
