에이전트 데이터에 대해 어떤 질문이든 몇 초 안에 답변을 받아보세요. Failproof AI Observability는 이벤트와 평가 데이터를 대상으로 미리 저장된 실행 가능한 쿼리 라이브러리를 제공하므로, 빈 SQL 편집기에서 시작하는 대신 동작하는 예제부터 출발할 수 있습니다.

*`/<org-slug>/queries`의 저장된 쿼리 라이브러리: 내장 프리셋과 팀이 저장한 쿼리가 나란히 표시됩니다.*

## 빈 페이지 대신 프리셋에서 시작하세요

테이블 이름을 외우거나 SQL을 처음부터 작성할 필요가 없습니다. 라이브러리를 열면 팀이 가장 많이 묻는 질문에 대한 내장 프리셋이 팀이 저장하고 이름을 붙인 쿼리들 바로 옆에 표시됩니다. 원하는 것에 가장 가까운 프리셋을 선택하면 답변까지 절반 이상 왔다고 볼 수 있습니다.

모든 저장된 쿼리는 조직 범위로 공유되므로, 팀원이 작성한 유용한 쿼리는 여러분의 것이기도 합니다. 쿼리에 이름과 설명을 한 번만 붙여두면, 조직 내 누구든 찾아서 실행하거나 나중에 대시보드에 고정할 수 있습니다.

`/<org-slug>/queries`에서 찾아보세요.

## SQL 작성기에서 수정하고 실행하세요

쿼리를 열면 SQL 작성기로 이동하며, 여기서 쿼리를 조정하고 결과를 즉시 확인할 수 있습니다. 내보내기나 왕복 처리, 다른 사람을 기다릴 필요가 없습니다.

![저장된 쿼리를 실행 중인 SQL 작성기, 스키마 사이드바와 실시간 결과 그리드가 표시된 화면](/agenteye/images/query-lab.png)

*SQL 작성기: 왼쪽에는 쿼리, 컬럼 이름을 추측할 필요 없는 스키마 사이드바, 그리고 아래에는 실시간 결과 그리드.*

- **스키마 사이드바**는 분석 테이블과 해당 컬럼을 나열하여 필드 이름을 찾아다니지 않고도 쿼리를 구성할 수 있게 해줍니다.
- **실시간 결과 그리드**는 실행 즉시 행을 반환하므로, 반복적인 추측 없이 몇 초 만에 결과를 확인하며 수정할 수 있습니다.
- **설계상 읽기 전용입니다.** 쿼리는 이벤트 저장소에 대해 실행되며 서버에서 검증됩니다. `SELECT`와 `WITH` 구문만 허용되며, 구문 타임아웃과 행 수 제한이 적용됩니다. 탐색용 쿼리는 데이터를 수정할 수 없으며, 무한 실행되는 쿼리는 자동으로 중단됩니다.

결과가 마음에 드시나요? 팀 전체가 활용할 수 있도록 라이브러리에 다시 저장하거나, 결과를 라인, 막대, 영역, 파이 타일 형태로 대시보드에 고정하세요.

## 터미널에서 실행하거나 어시스턴트가 작성하게 하세요

저장된 쿼리는 어디서 작업하든 동일하게 사용할 수 있습니다.

- **터미널에서.** `agenteye` CLI로 동일한 쿼리를 목록 조회, 실행, 저장할 수 있어 결과를 스크립트에 넣거나, CI에 연결하거나, 코딩 에이전트에 전달할 수 있습니다.

```bash
agenteye query list                 # the same saved queries, from your terminal
agenteye query run errs --arg prod  # run one and print the rows (add --json to pipe it)
```

  전체 명령어 목록은 [CLI 및 에이전트](/ko/agenteye/cli-and-agents)를 참고하세요.

- **AI 어시스턴트에서.** SQL 표현이 막막하다면? 대시보드 내 [AI 어시스턴트](/ko/agenteye/assistant)에 일반 영어로 질문하면 쿼리를 작성해서 라이브러리에 저장해 드립니다.

저장된 쿼리 실행은 `queries:run` 권한으로 제어되며, 쿼리 생성 및 삭제 권한과 분리되어 있어 라이브러리 수정 권한 없이 읽기 접근 권한만 부여할 수 있습니다.
