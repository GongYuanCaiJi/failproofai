에이전트 데이터에 대해 어떤 질문이든 던지면 몇 초 안에 답을 얻을 수 있습니다. FailproofAI Observability는 이벤트와 평가 데이터를 대상으로 즉시 실행 가능한 저장 쿼리 라이브러리를 제공하므로, 빈 SQL 편집기 대신 바로 작동하는 예시부터 시작할 수 있습니다.

![저장 쿼리 라이브러리: 기본 제공 프리셋과 커스텀 쿼리가 격자 형태로 나열된 화면](/agenteye/images/queries.png)

*`/<org-slug>/queries`에 있는 저장 쿼리 라이브러리: 기본 제공 프리셋과 팀이 저장한 쿼리가 나란히 표시됩니다.*

## 빈 페이지가 아닌 프리셋에서 시작하세요

테이블 이름을 외우거나 SQL을 처음부터 작성할 필요가 없습니다. 라이브러리는 팀들이 가장 자주 묻는 질문에 대한 기본 제공 프리셋과 함께 열리며, 팀이 저장하고 이름 붙인 쿼리들이 바로 옆에 함께 표시됩니다. 원하는 것과 가장 비슷한 프리셋을 선택하면 답에 한결 가까워집니다.

저장된 쿼리는 모두 조직 범위로 공유되므로, 팀원이 작성한 유용한 쿼리는 곧 내 것이 되기도 합니다. 쿼리에 이름과 설명을 한 번만 붙여두면 조직 내 누구든 찾아서 실행하거나, 나중에 대시보드에 결과를 고정할 수 있습니다.

`/<org-slug>/queries`에서 확인하세요.

## SQL 컴포저에서 수정하고 바로 실행하세요

쿼리를 열면 SQL 컴포저로 이동하며, 여기서 쿼리를 수정하고 즉시 결과를 확인할 수 있습니다. 내보내기, 왕복 대기, 다른 사람을 기다릴 필요가 없습니다.

![저장된 쿼리를 실행 중인 SQL 쿼리 컴포저: 스키마 사이드바와 실시간 결과 그리드가 표시된 화면](/agenteye/images/query-lab.png)

*SQL 컴포저: 왼쪽에 쿼리, 컬럼 이름을 바로 확인할 수 있는 스키마 사이드바, 그리고 아래에 실시간 결과 그리드가 표시됩니다.*

- **스키마 사이드바**는 분석 테이블과 컬럼을 한눈에 보여주므로, 필드 이름을 찾아다닐 필요 없이 쿼리를 작성할 수 있습니다.
- **실시간 결과 그리드**는 실행하는 즉시 행을 반환하므로, 추측과 재시도 없이 몇 초 만에 반복 작업이 가능합니다.
- **읽기 전용 설계.** 쿼리는 이벤트 스토어를 대상으로 실행되며 서버에서 유효성이 검증됩니다. `SELECT`와 `WITH` 구문만 허용되며, 실행 시간 제한과 행 수 제한이 적용됩니다. 탐색용 쿼리는 절대 데이터를 수정할 수 없으며, 무한 실행되는 쿼리는 자동으로 중단됩니다.

결과가 만족스럽다면 라이브러리에 저장하여 팀 전체가 활용할 수 있도록 하거나, 결과를 대시보드에 라인, 바, 에리어, 파이 타일로 고정할 수 있습니다.

## 터미널에서 실행하거나 어시스턴트에게 작성을 맡기세요

저장된 쿼리는 어디서 작업하든 그대로 사용할 수 있습니다.

- **터미널에서.** `agenteye` CLI로 동일한 쿼리를 나열하고 실행하고 저장할 수 있으므로, 결과를 스크립트에 넣거나 CI에 연결하거나 코딩 에이전트에 전달할 수 있습니다.

```bash
agenteye query list                 # the same saved queries, from your terminal
agenteye query run errs --arg prod  # run one and print the rows (add --json to pipe it)
```

  전체 명령어는 [CLI와 에이전트](/ko/agenteye/cli-and-agents)를 참고하세요.

- **AI 어시스턴트에서.** SQL 작성이 막막하다면 대시보드 내 [AI 어시스턴트](/ko/agenteye/assistant)에게 평문 영어로 질문하세요. 어시스턴트가 쿼리를 작성하고 라이브러리에 저장해 줍니다.

저장된 쿼리 실행은 `queries:run` 권한으로 제어되며, 이는 쿼리 생성 및 삭제 권한과 분리되어 있습니다. 따라서 라이브러리 수정 권한 없이 읽기 접근만 부여할 수 있습니다.
