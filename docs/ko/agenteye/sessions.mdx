## 모든 실행을 한눈에 확인하기

원시 이벤트 기록은 모든 단계의 실제 내용을 담고 있지만, 수십 개의 실행에 걸쳐 수천 개의 단계가 있을 때는 개별 단계가 아닌 실행 전체를 파악해야 합니다. Sessions 페이지는 한 실행의 모든 이벤트를 하나의 행으로 묶어, 하루치 활동이 끝없는 로그 스트림이 아닌 한눈에 훑어볼 수 있는 목록이 됩니다.

각 행에는 상태 뱃지가 표시되어, 클릭하기 전에도 실패한 실행과 정상적인 실행을 바로 구분할 수 있습니다. 날짜 범위, 환경, 에이전트, 세션으로 필터링하면 몇 번의 클릭만으로 "전체 목록"에서 "내가 찾는 실행"으로 좁혀갈 수 있습니다.

평가자를 연결하면 완료된 모든 실행이 자동으로 채점되고, 최신 점수가 행의 뱃지로 표시됩니다. 점수 범위로 필터링할 수 있으므로 "이번 주 프로덕션에서 낮은 점수를 받은 실행 보기"가 수동 검토가 아닌 필터 하나로 해결됩니다. 평가자를 설정하기 전에도 세션은 전체 실행 내용을 캡처하며, 다만 점수 뱃지가 표시되지 않을 뿐입니다.

## 전체 실행 흐름을 그림으로 파악하기

![이벤트 타임라인 옆에 표시된 세션의 git 스타일 실행 그래프와 도구, 모델, 훅 분석 패널](/agenteye/images/session-detail.png)

*실행 그래프(왼쪽)가 이벤트 타임라인 옆에 표시되며, 오른쪽 패널에서는 해당 실행의 도구, 모델, 훅, 토큰 사용량을 분석해 줍니다.*

세션을 클릭하면 실행 그래프가 열립니다. 에이전트, 도구, 훅, 모델 호출이 시간 순으로 어떻게 전개되었는지를 git 스타일로 보여주는 뷰입니다. 병렬 서브 에이전트는 각자의 레인으로 분기되어, 어떤 작업이 동시에 실행되었는지, 어떤 서브 에이전트가 멈췄는지, 실행이 어디서 잘못되었는지를 로그 더미를 머릿속으로 재현하지 않고도 파악할 수 있습니다.

오른쪽 패널에서는 실행별 세부 내역을 확인할 수 있습니다. 어떤 도구와 모델이 실행되었는지, 어떤 훅이 발동했는지, 토큰을 얼마나 사용했는지가 모두 표시됩니다. "이 실행은 왜 이렇게 많은 비용이 들었지?" 또는 "느린 도구가 어느 것이지?"에 대한 답을 그래프 바로 옆에서 확인할 수 있습니다.

## 찾는 방법

모든 대시보드 페이지는 조직(`/<org-slug>/…`) 기준으로 범위가 지정됩니다. Sessions는 왼쪽 사이드바의 **Observe** 섹션에서 Events 옆에 위치하며, 목록 상단에 날짜 범위, 환경, 에이전트, 세션 필터가 있습니다. 모든 행은 클릭 한 번으로 전체 실행 그래프로 이동합니다.

점수 뱃지와 점수 범위 필터링을 활성화하려면 평가자를 연결하세요: [Evaluations](/ko/agenteye/evaluations)를 참고하세요.
