## 공식 패키지

npm의 공식 패키지명은 **`failproofai`**입니다:

```bash
npm install -g failproofai
# 또는
bun add -g failproofai
```

---

## 별칭을 직접 소유하는 이유

타이포스쿼팅(Typosquatting)은 흔한 공급망 공격 방식으로, 악의적인 행위자가 인기 있는 패키지명에서 키 하나만 다른 이름을 등록하는 수법입니다. 설치 명령어를 잘못 입력한 사용자는 시스템 전체에 접근할 수 있는 공격자 제어 코드를 실행하게 됩니다. 이것이 바로 Failproof AI가 방어하도록 설계된 위협 유형입니다.

이러한 공격 표면을 없애기 위해, **`failproofai`의 일반적인 오타 및 표기 변형을 npm에서 선제적으로 모두 선점**하고 있습니다. 이 이름들은 제3자가 등록할 수 없으며, 각각은 실제 `failproofai` 패키지를 설치하고 위임하는 얇은 프록시입니다.

---

## 등록된 별칭

**표기 변형** - "failproof ai"를 쓰는 다양한 방식:

| 패키지 | 상태 |
|---------|--------|
| `failproof` | ✅ 게시됨 |
| `failproof-ai` | ⏳ npm 승인 대기 중 |
| `fail-proof-ai` | ⏳ npm 승인 대기 중 |
| `failproof_ai` | ⏳ npm 승인 대기 중 |
| `fail_proof_ai` | ⏳ npm 승인 대기 중 |
| `fail-proofai` | ⏳ npm 승인 대기 중 |

**`failprof*` 오타** - "proof"에서 `o`가 하나 빠진 경우:

| 패키지 | 상태 |
|---------|--------|
| `failprof` | ✅ 게시됨 |
| `failprof-ai` | ✅ 게시됨 |
| `failprofai` | ⏳ npm 승인 대기 중 |
| `fail-prof-ai` | ⏳ npm 승인 대기 중 |
| `failprof_ai` | ⏳ npm 승인 대기 중 |

**`faliproof*` 오타** - `a`와 `i`가 뒤바뀐 경우:

| 패키지 | 상태 |
|---------|--------|
| `faliproof` | ✅ 게시됨 |
| `faliproof-ai` | ✅ 게시됨 |
| `faliproofai` | ⏳ npm 승인 대기 중 |

> **대기 중인 이유는?** npm의 스팸 방지 정책은 구두점을 제거하고 유사도 검사를 수행한 후 기존 패키지와 동일한 문자열로 정규화되는 이름의 등록을 차단합니다. 저희는 스쿼팅 방지 목적으로 이 이름들을 예약하기 위해 npm 지원팀에 연락한 상태입니다. 승인되는 즉시 활성화될 예정입니다.

게시된 별칭이 저희 소유인지 다음 명령어로 확인할 수 있습니다:

```bash
npm info failproof
# 확인 사항: maintainers 필드에 "ExosphereHost Inc." 포함 여부
```

## 별칭의 작동 방식

각 별칭 패키지는:

1. `failproofai`를 의존성으로 등록하여 설치 시 실제 패키지(및 `postinstall` 훅 설정 포함)가 실행되도록 합니다.
2. 자체 이름과 일치하는 바이너리(예: `failprof-ai`)를 노출하여 모든 인수를 `failproofai` 바이너리로 프록시합니다.

프록시는 두 줄짜리 Node 스크립트로 구성되어 있으며, 별도의 로직이나 네트워크 호출이 없고, `failproofai` 자체가 수행하는 것 이상의 데이터 수집도 없습니다.
