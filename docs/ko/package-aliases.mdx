## 공식 패키지

npm의 표준 패키지명은 **`failproofai`**입니다:

```bash
npm install -g failproofai
# or
bun add -g failproofai
```

---

## 별칭 이름을 소유하는 이유

타이포스쿼팅(typosquatting)은 공급망 공격의 일반적인 수법으로, 악의적인 공격자가 인기 패키지명에서 키 하나만 다른 이름을 등록합니다. 설치 명령어를 잘못 입력한 사용자는 시스템 전체 접근 권한을 가진 공격자 제어 코드를 실행하게 됩니다. 이것이 바로 Failproof AI가 방어하고자 설계된 위협의 전형입니다.

이러한 취약점을 원천 차단하기 위해, **npm에서 `failproofai`의 흔한 오타 및 표기 변형 이름을 모두 선제적으로 확보**했습니다. 이 이름들은 제3자가 등록할 수 없으며, 각각 실제 `failproofai` 패키지에 설치 및 실행을 위임하는 얇은 프록시입니다.

---

## 등록된 별칭

**표기 변형** - "failproof ai"를 작성하는 다양한 방식:

| 패키지 | 상태 |
|--------|------|
| `failproof` | ✅ 게시됨 |
| `failproof-ai` | ⏳ npm 승인 대기 중 |
| `fail-proof-ai` | ⏳ npm 승인 대기 중 |
| `failproof_ai` | ⏳ npm 승인 대기 중 |
| `fail_proof_ai` | ⏳ npm 승인 대기 중 |
| `fail-proofai` | ⏳ npm 승인 대기 중 |

**`failprof*` 오타** - "proof"에서 `o` 하나가 빠진 경우:

| 패키지 | 상태 |
|--------|------|
| `failprof` | ✅ 게시됨 |
| `failprof-ai` | ✅ 게시됨 |
| `failprofai` | ⏳ npm 승인 대기 중 |
| `fail-prof-ai` | ⏳ npm 승인 대기 중 |
| `failprof_ai` | ⏳ npm 승인 대기 중 |

**`faliproof*` 오타** - `a`와 `i`가 뒤바뀐 경우:

| 패키지 | 상태 |
|--------|------|
| `faliproof` | ✅ 게시됨 |
| `faliproof-ai` | ✅ 게시됨 |
| `faliproofai` | ⏳ npm 승인 대기 중 |

> **왜 대기 중인가요?** npm의 스팸 방지 정책은 구두점을 제거하고 유사도 검사를 수행했을 때 기존 패키지와 동일한 문자열로 정규화되는 이름의 등록을 차단합니다. 스쿼팅 방지 목적으로 이 이름들을 확보하기 위해 npm 지원팀에 문의한 상태이며, 승인되는 즉시 활성화될 예정입니다.

게시된 별칭이 저희 소유인지 확인하려면 다음 명령어를 사용하세요:

```bash
npm info failproof
# Look for: "ExosphereHost Inc." in the maintainers field
```

## 별칭의 작동 방식

각 별칭 패키지는 다음과 같이 동작합니다:

1. `failproofai`를 의존성으로 선언하여, 설치 시 실제 패키지(및 해당 `postinstall` 훅 설정 포함)가 실행됩니다.
2. 자체 이름(예: `failprof-ai`)에 해당하는 바이너리를 노출하며, 모든 인수를 `failproofai` 바이너리로 프록시합니다.

프록시는 두 줄짜리 Node 스크립트로, 어떠한 로직도, 네트워크 호출도, `failproofai` 자체가 수행하는 것 이상의 데이터 수집도 없습니다.
