감사는 이제 CLI 하위 명령어가 아닌 **/audit 대시보드 페이지**로 제공됩니다. 대시보드 내비게이션 바(Policies와 Projects 사이)에서 열거나, `failproofai`를 로컬에서 실행 중일 때 `http://localhost:8020/audit`에 직접 접속하세요.

```bash
failproofai          # 대시보드를 열고 "Audit"을 클릭하세요
```

대시보드는 이 머신에 저장된 과거 에이전트 CLI 트랜스크립트(Claude Code, Codex, Copilot, Cursor, OpenCode, Pi, Gemini)를 스캔하여, failproofai가 차단하도록 설계된 행동들—환경 변수 확인, 강제 푸시, 불필요한 `cd <cwd>` 접두사, sleep 폴링 루프, 방금 편집한 파일 재읽기 등—이 얼마나 자주 발생했는지 보고합니다.

각 트랜스크립트에 대해, 모든 도구 사용 이벤트는 39개의 내장 정책 **및** 런타임 정책에서 아직 다루지 않는 패턴을 잡아내는 8개의 감사 전용 디텍터를 통해 재실행됩니다. 집계 결과는 모든 세션에 걸쳐 정책/디텍터별로 산출됩니다.

## 제공되는 정보

`/audit` 페이지는 여섯 개의 섹션으로 구성됩니다:

1. **Identity** — 감사된 모든 트랜스크립트의 가중치 신호를 기반으로 에이전트를 8가지 원형 중 하나(`optimist`, `cowboy`, `explorer`, `goldfish`, `paranoid architect`, `precision builder`, `hammer`, `ghost`)로 분류합니다.
2. **Strengths** — 스캔에서 도출된 실제 수치(클린 호출 비율, "0건의 자격 증명 유출" 등)로, 관련 sanitize 정책이 실제로 작동했을 때만 표시됩니다.
3. **Score** — 0-100 점수로 S/A/B/C/D/F 등급을 제공하며, 권장 정책을 모두 활성화했을 때의 예상 향상치도 보여줍니다.
4. **Findings** — 정책별 카드로, 발생한 내용, 비용, 수집된 증거, 그리고 해당 상황을 실시간으로 잡아낼 수 있는 내장 정책을 활성화하는 정확한 `failproofai policy add <slug>` 명령어를 제공합니다.
5. **Prescribed policies** — 단일 `failproofai policies --install` 명령으로 한 번에 설치할 수 있는 집계된 설치 목록입니다.
6. **Re-audit reminder** — "더 나아진 상태로 돌아오세요." API 서버를 통해 7일 이메일 알림을 설정할 수 있습니다(로그인 필요; [`failproofai auth`](/ko/cli/auth) 참조).

## 감사 전용 디텍터

이 디텍터들은 실시간으로 아직 적용되지 않는 "비효율적인 행동" 패턴을 감지합니다. 감사 중에만 실행되며 라이브 도구 호출을 절대 차단하지 않습니다.

| 디텍터 | 집계 내용 |
|---|---|
| `redundant-cd-cwd` | 명령어가 이미 `cwd`에서 실행되고 있음에도 `cd <cwd> && …`로 시작하는 Bash 명령어. |
| `prefer-edit-over-read-cat` | 단일 소스 파일에 대한 `cat`/`head`/`tail`/`less`/`more` 사용 — `Read` 도구를 사용해야 합니다. |
| `prefer-edit-over-sed-awk` | `sed -i` / `awk … > file` 인플레이스 편집 — `Edit` 도구를 사용해야 합니다. |
| `prefer-write-over-heredoc` | Heredoc / 멀티라인 `echo > file`로 파일 작성 — `Write` 도구를 사용해야 합니다. |
| `sleep-polling-loop` | 긴 `sleep N` (≥ 30초) 또는 `while …; sleep …; done` 폴링 루프. |
| `find-from-root` | `find /`, `find /home`, `find /usr` 등 — `cwd`로 범위를 제한해야 합니다. |
| `git-commit-no-verify` | `git commit … --no-verify` / `-n`으로 훅을 건너뛰는 경우. |
| `reread-after-edit` | 같은 세션에서 `Edit`/`Write`한 직후 해당 파일을 `Read`하는 경우. |

## 캐시

- **트랜스크립트별 캐시**는 `~/.failproofai/cache/audit/<sha1>.json`에 `(mtime, size, engineVersion, detectorVersion)`을 키로 저장됩니다. 트랜스크립트나 정책/디텍터 코드가 변경되면 자동으로 무효화됩니다. 각 항목에는 **TTL 메타데이터**로서 `cachedAt` 타임스탬프도 저장됩니다(캐시 키의 일부가 아님). **7일**이 지난 항목은 읽을 때 거부되므로, 오래된 결과가 진화하는 디텍터 의도를 넘어 지속되지 않습니다.
- **전체 결과 캐시**는 `~/.failproofai/audit-dashboard.json` (모드 0600)에 저장됩니다. 내비게이션 시 대시보드를 재실행 없이 즉시 렌더링할 수 있게 합니다. 마찬가지로 **7일 TTL**이 지나면 읽기가 거부되며, `/audit`은 `audit expired — re-audit to refresh` 배너와 함께 빈 상태로 표시됩니다. 페이지 상단의 `[ re-audit ]` 또는 하단의 `[ re-audit now ]`를 클릭하여 새로고침하세요. 새 실행은 상단 고정 스트립을 통해 진행 상황을 스트리밍하며, 성공 시 페이지 새로고침 없이 결과가 즉시 교체됩니다.

## 참고 사항

- **변경 없음.** 감사는 읽기 전용 모드로 재실행됩니다. `warn-repeated-tool-calls`는 건너뜁니다. 그렇지 않으면 세션별 사이드카가 수정될 수 있기 때문입니다.
- **워크플로 정책 건너뜀.** `require-*-before-stop` 정책은 `Stop` 이벤트와 라이브 git 상태에 대한 `execSync`에서만 작동합니다. "2025년에 어떤 일이 있었을까"에 대한 의미 있는 해석이 없으므로 감사 집계에 나타나지 않습니다.
- **커스텀 정책 건너뜀.** 사용자가 제공한 커스텀 훅은 재실행되지 않습니다(원래 세션 이후 변경되었을 수 있기 때문입니다).
