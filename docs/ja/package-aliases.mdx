## 公式パッケージ

npmの正規パッケージは **`failproofai`** です：

```bash
npm install -g failproofai
# or
bun add -g failproofai
```

---

## エイリアス名を所有している理由

タイポスクワッティングは、悪意のある攻撃者が人気パッケージに1文字違いの名前を登録するサプライチェーン攻撃の手口です。インストールコマンドをタイプミスした無防備なユーザーが、攻撃者の制御するコードをシステム全権限で実行してしまいます。まさにこのような脅威に対して、Failproof AI は設計されています。

この攻撃面を排除するため、**npmにおける `failproofai` のよくあるスペルミスや表記バリアントをすべて先回りして取得**しています。これらの名前はサードパーティに登録されることはありません。各エイリアスは、実際の `failproofai` パッケージにインストールを委譲する薄いプロキシです。

## 登録済みエイリアス

**表記バリアント** - "failproof ai" のさまざまな書き方：

| パッケージ | ステータス |
|---------|--------|
| `failproof` | ✅ 公開済み |
| `failproof-ai` | ⏳ npm サポート対応待ち |
| `fail-proof-ai` | ⏳ npm サポート対応待ち |
| `failproof_ai` | ⏳ npm サポート対応待ち |
| `fail_proof_ai` | ⏳ npm サポート対応待ち |
| `fail-proofai` | ⏳ npm サポート対応待ち |

**`failprof*` タイポ** - "proof" から `o` が1つ抜けたもの：

| パッケージ | ステータス |
|---------|--------|
| `failprof` | ✅ 公開済み |
| `failprof-ai` | ✅ 公開済み |
| `failprofai` | ⏳ npm サポート対応待ち |
| `fail-prof-ai` | ⏳ npm サポート対応待ち |
| `failprof_ai` | ⏳ npm サポート対応待ち |

**`faliproof*` タイポ** - `a` と `i` が入れ替わったもの：

| パッケージ | ステータス |
|---------|--------|
| `faliproof` | ✅ 公開済み |
| `faliproof-ai` | ✅ 公開済み |
| `faliproofai` | ⏳ npm サポート対応待ち |

> **なぜ対応待ちなのか？** npmのスパム防止ポリシーにより、句読点を除去して類似度チェックを行った結果、既存パッケージと同一文字列になる名前はブロックされます。これらの名前をアンチスクワッティング目的で予約するよう npm サポートに連絡済みです。承認され次第、有効化されます。

公開済みのエイリアスが当社所有であることは次のコマンドで確認できます：

```bash
npm info failproof
# Look for: "ExosphereHost Inc." in the maintainers field
```

## エイリアスの仕組み

各エイリアスパッケージは以下の動作をします：

1. `failproofai` を依存関係として登録しているため、インストール時に本体パッケージ（`postinstall` フックのセットアップを含む）が実行されます
2. 自身の名前（例：`failprof-ai`）と一致するバイナリを公開し、すべての引数を `failproofai` バイナリにプロキシします

プロキシは2行のNodeスクリプトであり、ロジックもネットワーク通信も、`failproofai` 本体が行う以外のデータ収集も一切ありません。
