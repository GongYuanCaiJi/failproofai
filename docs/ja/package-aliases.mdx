## 公式パッケージ

npm の正式パッケージは **`failproofai`** です：

```bash
npm install -g failproofai
# または
bun add -g failproofai
```

---

## エイリアス名を所有している理由

タイポスクワッティングは、悪意のある攻撃者が人気パッケージ名から1文字違いのパッケージ名を登録するサプライチェーン攻撃の一般的な手口です。インストールコマンドを誤って入力したユーザーが、攻撃者の管理するコードをシステムへのフルアクセス権限付きで実行してしまいます。これはまさに Failproof AI が防ぐことを目的とした脅威です。

この攻撃経路をなくすため、**npm 上の `failproofai` に関するよくあるスペルミスや表記のバリエーションをすべて先回りして取得しています**。これらの名前はいずれも第三者に登録されることはありません。各エイリアスは、実際の `failproofai` パッケージをインストールして委譲するシンプルなプロキシです。

## 登録済みエイリアス

**表記バリエーション** - "failproof ai" のさまざまな書き方：

| パッケージ | ステータス |
|---------|--------|
| `failproof` | ✅ 公開済み |
| `failproof-ai` | ⏳ npm サポート待ち |
| `fail-proof-ai` | ⏳ npm サポート待ち |
| `failproof_ai` | ⏳ npm サポート待ち |
| `fail_proof_ai` | ⏳ npm サポート待ち |
| `fail-proofai` | ⏳ npm サポート待ち |

**`failprof*` タイポ** - "proof" の `o` が1つ欠けているもの：

| パッケージ | ステータス |
|---------|--------|
| `failprof` | ✅ 公開済み |
| `failprof-ai` | ✅ 公開済み |
| `failprofai` | ⏳ npm サポート待ち |
| `fail-prof-ai` | ⏳ npm サポート待ち |
| `failprof_ai` | ⏳ npm サポート待ち |

**`faliproof*` タイポ** - `a` と `i` が入れ替わっているもの：

| パッケージ | ステータス |
|---------|--------|
| `faliproof` | ✅ 公開済み |
| `faliproof-ai` | ✅ 公開済み |
| `faliproofai` | ⏳ npm サポート待ち |

> **なぜ「サポート待ち」なのか？** npm のスパム防止ポリシーにより、句読点を除去して類似性チェックを行った際に既存パッケージと同一文字列になる名前はブロックされます。アンチスクワッティングを目的としてこれらの名前を確保するよう npm サポートに連絡済みです。承認され次第、有効化される予定です。

公開済みエイリアスが当社所有であることは以下で確認できます：

```bash
npm info failproof
# maintainers フィールドに "ExosphereHost Inc." が表示されることを確認
```

## エイリアスの仕組み

各エイリアスパッケージは：

1. `failproofai` を依存関係として指定しているため、インストール時に本物のパッケージ（`postinstall` フックのセットアップを含む）が実行されます
2. 独自のパッケージ名に対応するバイナリ（例：`failprof-ai`）を公開し、すべての引数を `failproofai` バイナリにプロキシします

プロキシは2行の Node スクリプトで構成されており、ロジックもネットワーク呼び出しもなく、`failproofai` 自体が行う範囲を超えたデータ収集は一切ありません。
