## 公式パッケージ

npm の正規パッケージは **`failproofai`** です：

```bash
npm install -g failproofai
# または
bun add -g failproofai
```

---

## エイリアス名を取得している理由

タイポスクワッティングとは、攻撃者が人気パッケージの名前から1文字だけ違うパッケージ名を登録するサプライチェーン攻撃の手法です。インストールコマンドを打ち間違えた無防備なユーザーが、攻撃者の管理するコードをフルシステムアクセス権限で実行してしまう——これは、まさに Failproof AI が防ぐために設計された脅威です。

この攻撃面を排除するため、**`failproofai` のよくあるスペルミスやフォーマットバリエントをすべて先手を打って取得しています**。これらの名前はいずれもサードパーティが登録できません。それぞれが薄いプロキシとして機能し、本物の `failproofai` パッケージをインストール・委譲します。

## 登録済みエイリアス

**フォーマットバリアント** — "failproof ai" のさまざまな表記：

| パッケージ | ステータス |
|-----------|----------|
| `failproof` | ✅ 公開済み |
| `failproof-ai` | ⏳ npm サポート対応待ち |
| `fail-proof-ai` | ⏳ npm サポート対応待ち |
| `failproof_ai` | ⏳ npm サポート対応待ち |
| `fail_proof_ai` | ⏳ npm サポート対応待ち |
| `fail-proofai` | ⏳ npm サポート対応待ち |

**`failprof*` タイポ** — "proof" から `o` が1つ欠けたもの：

| パッケージ | ステータス |
|-----------|----------|
| `failprof` | ✅ 公開済み |
| `failprof-ai` | ✅ 公開済み |
| `failprofai` | ⏳ npm サポート対応待ち |
| `fail-prof-ai` | ⏳ npm サポート対応待ち |
| `failprof_ai` | ⏳ npm サポート対応待ち |

**`faliproof*` タイポ** — `a` と `i` が入れ替わったもの：

| パッケージ | ステータス |
|-----------|----------|
| `faliproof` | ✅ 公開済み |
| `faliproof-ai` | ✅ 公開済み |
| `faliproofai` | ⏳ npm サポート対応待ち |

> **対応待ちの理由について** npm のスパム防止ポリシーにより、句読点を除去して類似チェックを実行した際に既存パッケージと同一の文字列に正規化される名前はブロックされます。アンチスクワッティングを目的としてこれらの名前を予約するよう npm サポートに連絡済みです。承認され次第、順次有効化されます。

公開済みのエイリアスが当社所有であることは次のコマンドで確認できます：

```bash
npm info failproof
# maintainers フィールドに "ExosphereHost Inc." が表示されれば OK
```

## エイリアスの仕組み

各エイリアスパッケージは：

1. `failproofai` を依存関係として宣言しており、本物のパッケージがインストールされてそのバイナリが使用可能になります
2. 自身の名前に対応するバイナリ（例：`failprof-ai`）を公開しており、すべての引数を `failproofai` バイナリに転送します

プロキシは2行の Node スクリプトで構成されており、ロジックもネットワーク呼び出しも、`failproofai` 自体が行う以外のデータ収集も一切ありません。
