Failproof AI Observability のデプロイ全体を、コマンド一つで操作できます。ターミナルを離れることなく本番環境の確認、API キーの発行、インシデントの確認が可能です。CI にスクリプトとして組み込んだり、コーディングエージェントに自然言語で実行させることもできます。

```bash
pipx install agenteye
agenteye login --email you@example.com      # 6桁のコードがメールで届きます
agenteye --json sessions --since 24h         # 過去1日のエージェント実行一覧（新しい順）
```

*`agenteye` CLI はダッシュボードと通信します。イベントをサーバーに送信するコレクターとは別のツールです。*

## デプロイ全体をコマンド一つで

ちょっとした確認のためにタブを切り替えるのはもう終わりにしましょう。`agenteye` CLI は単一のバイナリからデータの参照と組織の管理を行えるため、ダッシュボードをクリックして回っていた確認作業が、一行のコマンドで完結します。そのコマンドは再実行・エイリアス化・ランブックへの貼り付けも自在です。次の4つの操作が可能です：

- **データの参照：** `sessions`、`events`、`evals`、`errors` を時間・エージェント・環境でフィルタリング。
- **組織の管理：** `keys`、`users`、`settings`、`alerts`、`incidents`。
- **アナリティクスの実行：** 保存済み SQL および イベントデータに対するアドホックな `query` の実行。
- **アシスタントへの問い合わせ：** `agent ask` でダッシュボードの読み取り専用アナリストと会話できます。

`pipx` で一度インストールし、メールで届いた6桁のコードでサインインするだけで準備完了です。セッションは約1日間有効で、期限切れになったら `agenteye login` を再実行してください。ブラウザを開かずに、本番環境のスポットチェック、キーのプロビジョニング、発火中のインシデントのトリアージをすべて行えます：

```bash
agenteye errors --since 24h --aggregate      # エラータイプ別にグループ化した障害状況
agenteye incidents list --state firing       # 現在発火中のインシデント
agenteye keys create ci --add events:add     # イベントのプッシュのみ可能なキー（シークレットは一度だけ表示）
```

一つ覚えておきたい点があります：`--json` などのグローバルオプションはコマンドの前に置きます。`agenteye --json sessions` が正しく、`agenteye sessions --json` は誤りです。

## スクリプト化・CI への組み込み

すべてのコマンドは `--json` オプションに対応しており、これが大きな違いをもたらします。クリーンな JSON は stdout に出力され、人間向けのステータスや警告は stderr に出力されます。そのため `--json` でキャプチャした出力は、余計な行を取り除くことなくそのまま `jq` にパイプできます。これにより CLI は、ターミナルで使う場合でも、出力をパースするコーディングエージェントにとっても同様に使いやすくなっています：

```bash
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'
```

無人実行向けに設計されています。端末が接続されていない場合は確認プロンプトが自動的にスキップされるため、パイプラインで処理が止まることはありません。また、各コマンドは意味のある終了コードを返します：`0` は成功、`4` は未ログイン、`5` は権限不足（メッセージに例えば `alerts:write` のように具体的な権限名が表示されます）、`3` はダッシュボードへの接続不可です。スクリプトでは `4` で再認証の分岐処理を行ったり、`5` で管理者に何を申請すべきかを正確に把握したりでき、理由不明のまま失敗するという事態を防げます。

## コーディングエージェントに自然言語で操作させる

さらに理想的な使い方として、これらのフラグをすべて覚える必要さえありません。**CLI スキル** は `agenteye-cli` という名前の小さな Agent Skill フォルダで、Claude Code や Codex などのコーディングエージェントが自然言語のリクエストから CLI を操作できるようにします。「今日、何か壊れているものはある？」と聞くだけで、エージェントが適切なコマンドを選択し、あなたの権限で実行して、結果を文章で回答してくれます。

Claude Code の場合、`agenteye-cli` フォルダを `~/.claude/skills/` に配置するだけで自動的に認識されます。Failproof AI Observability がそのフォルダを提供しており、すでにインストール済みの CLI を操作するだけなので、追加のインストールは不要です。ただし、事前にご自身でログインしておく必要があります。メールで届くコードを使ったログインはスキルでは完了できません。

エージェントはあなたのログイン権限で CLI を実行するため、キーの作成、設定の変更、インシデントの解決など、あなたのログインで許可されているすべての操作（読み取り・書き込みを含む）を行えます。エージェントに対しては CLI の確認プロンプトが表示されないため、スキルは変更を加える前に実行するコマンドを明示した上で承認を待つよう設計されています。確認ステップはあなた自身が担います。

```text
you    session run-001 はなぜ失敗したの？

agent  実行中: agenteye --json events --session-id run-001 --all
       checkout-agent が3回目のツール呼び出しで TimeoutError が発生しました。
```

読み取りは即座に完了し、書き込みはすべて一時停止してあなたの確認を求めます：

```text
you    CI にイベントのプッシュだけできるキーを発行して。

agent  API キーを作成します。次のコマンドを実行します：
         agenteye keys create ci --add events:add
       実行してよいですか？

you    yes

agent  完了しました。「ci」キーが events:add のみの権限で作成されました。シークレットは一度しか表示されないので、今すぐ保存してください。
```
