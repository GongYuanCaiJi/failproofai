エージェントのデータに自然な日本語で質問するだけで、根拠へのリンク付きの回答が得られます。SQLを書く必要も、ダッシュボードを探し回る必要もありません。**FailproofAI Observability**アシスタントは、チームの誰もがエージェントに関する答えを最速で得るための手段です。

![ダッシュボード内でFailproofAI Observabilityアシスタントが自然言語の質問に答えている画面。ライブのエージェントアクティビティテーブル、エージェントごとのモデル使用状況の内訳、テキストによるまとめが表示され、実行されたクエリがインラインで確認できる](/agenteye/images/assistant.png)
*自然言語で質問するだけで、自分のデータから構築された回答が得られます。ここではどのエージェントが最も多くのリクエストを処理しているか、そしてそのエージェントが使用しているモデルを分解し、すべての数値を検証できるよう実行したクエリも表示しています。*

学習は不要です。チャットを開き、知りたいことを入力して、返ってきたリンクをたどるだけです：

```
You:  which sessions errored today?
AI:   5 sessions errored today, newest first. Each one is linked:
      • checkout-agent   14:02   tool timeout
      • billing-agent    11:47   unhandled error
      • ...and 3 more

You:  summarize this session          (asked while viewing a run)
AI:   This run took 12 steps across 3 tools and failed near the end when a
      payment tool returned an error. It scored low on your "resolved" eval.
      Links: the session, the failing event, and that evaluation.
```

## 質問するだけで、すぐに根拠へジャンプ

推測をやめ、クエリを書くのをやめましょう。「今週のプロダクション環境で品質はどう推移している？」「今日エラーになったセッションはどれ？」「このセッションを要約して」と聞けば、クエリを自分で組み立てて読む代わりに、数秒で的確な回答が得られます。

すべての回答には証拠が付いてきます。アシスタントは回答の根拠として使用した正確なセッション、保存済みクエリ、ダッシュボードをリンクするので、言葉を鵜呑みにするのではなく、クリックして確認できます。また**ページ認識機能**も備えています。セッションを閲覧中に「このセッション」と尋ねれば、どの実行を意味するかすでに把握しています。履歴スイッチャーから以前の会話を後で再度開き、続きから再開することもできます。

## 良い回答を保存済みクエリやダッシュボードに変換する

回答を残しておく価値がある場合は、アシスタントに保存するよう依頼してください。保存済みクエリ用のSQLを下書きしたり、そのクエリからダッシュボードを組み立てたりして、**承認 / 拒否**カードを表示します。承認をクリックするまで何も書き込まれないので、「質問するだけ」のスピードを保ちながら、最終決定は常に自分の手に残ります。

**クエリ**ページではさらに一歩進んで、SQLの作成者となります。欲しいクエリを説明し（「過去7日間のエージェント別エラーレートを表示して」）、SQLをエディタに直接ストリーミングして、差分ビューを開くので変更が反映される前に**承認**か**拒否**を選べます。

![ObservabilityのクエリページとそのSQLエディタ](/agenteye/images/query-lab.png)
*クエリページ：このエディタはアシスタントが読み取り専用のドラフトクエリをストリーミングする場所で、承認または拒否を選択できます。*

ここで質問してSQLを作成する場合は`queries:run`パーミッションを使用します。これはエディタの**実行**ボタンと同じパーミッションです。他の場所でのチャットには`agent:use`が必要です。

## チーム全体に安心して開放できる

アシスタントへのアクセスを全員に開放しても、何に触れるかを心配する必要はありません：

- **あなたがすでに見られるものしか読みません。** 回答はあなた自身の読み取り権限のスコープ内に限定されるため、データの範囲が広がることはありません。
- **すべての書き込みはあなたを待ちます。** 保存済みクエリとダッシュボードは、明示的な承認クリック後にのみ作成され、そのゲートをオフにする設定はありません。
- **削除は一切できません。** 削除ツールは公開されておらず、アシスタントは削除権限を持っていません。削除操作はダッシュボード上のあなたの手に留まります。
- **組織の外には出ません。** アシスタントは現在閲覧中の組織のみを参照します。
- **質問はあなたのものです。** プロンプトと回答はあなた自身のObservabilityデータベースに保存されます。プロダクトアナリティクスは使用メタデータのみを記録し、プロンプトのテキストは決して記録しません。

## アクセス方法

アシスタントはあなたの組織（`/<org-slug>/...`）のすべてのページの右端に表示されます。レールをクリックするか、`⌘J` / `Ctrl+J`を押してフルチャットパネルに展開し、端をドラッグしてサイズを変更できます。幅はリロードをまたいで記憶されます。使用するには**`agent:use`**パーミッションが必要で、ない場合はレールがグレーアウトされます。デプロイメントでまだ有効化されていない場合（LLM接続が必要です）、動作するチャットの代わりに淡色のレールが表示されます。
