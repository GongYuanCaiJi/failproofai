エージェントデータにどんな質問でも投げかけ、数秒で答えを得られます。Failproof AI Observabilityは、イベントや評価に対してすぐに実行できる保存済みクエリのライブラリを提供しているため、空白のSQLエディタからではなく、動作するサンプルから作業を始められます。

![保存済みクエリライブラリ：再利用可能なクエリのグリッド表示（組み込みプリセットとカスタムクエリの両方）](/agenteye/images/queries.png)

*`/<org-slug>/queries` にある保存済みクエリライブラリ：組み込みプリセットとチームが保存したクエリが並んで表示されます。*

## 白紙からではなく、プリセットから始める

テーブル名を覚えたり、SQLをゼロから書いたりする必要はありません。ライブラリを開くと、チームがよく尋ねる質問向けの組み込みプリセットが、自チームが保存・命名したクエリのすぐ隣に並んでいます。目的に近いものを選べば、答えにたどり着くまでの道のりは大半が済んだも同然です。

保存済みクエリはすべてorg単位でスコープが設定され、共有されます。そのため、チームメンバーが書いた有用なクエリはあなたのものにもなります。クエリに名前と説明を一度付けるだけで、org内の誰でもそれを見つけて実行したり、後でダッシュボードに結果をピン留めしたりできます。

`/<org-slug>/queries` からアクセスできます。

## SQLコンポーザーで調整して実行する

クエリを開くとSQLコンポーザーに読み込まれ、その場で調整してすぐに答えを確認できます。エクスポートも、往復の待ち時間も、他の人を待つ必要もありません。

![保存済みクエリを実行中のSQLクエリコンポーザー（スキーマサイドバーとライブ結果グリッド付き）](/agenteye/images/query-lab.png)

*SQLコンポーザー：左にクエリ、列名を調べる手間を省くスキーマサイドバー、下部にライブ結果グリッド。*

- **スキーマサイドバー**は分析テーブルとその列を一覧表示するため、フィールド名を探し回ることなくクエリを組み立てられます。
- **ライブ結果グリッド**は実行と同時に行を返すため、何度も推測し直すことなく数秒でイテレーションできます。
- **読み取り専用設計。** クエリはイベントストアに対して実行され、サーバー側で検証されます。`SELECT` と `WITH` 文のみが許可されており、ステートメントのタイムアウトと行数上限が設定されています。探索的なクエリでデータが変更されることはなく、暴走したクエリは自動的に停止されます。

結果に満足したら、チーム全体が利用できるようライブラリに保存して戻すか、折れ線・棒・面・円グラフのタイルとしてダッシュボードに出力をピン留めしましょう。

## ターミナルから実行する、またはアシスタントに書いてもらう

同じ保存済みクエリは、どこで作業していても利用できます。

- **ターミナルから。** `agenteye` CLIを使えば、まったく同じクエリの一覧表示・実行・保存が可能です。スクリプトに結果を渡したり、CIに組み込んだり、コーディングエージェントに渡したりできます。

```bash
agenteye query list                 # the same saved queries, from your terminal
agenteye query run errs --arg prod  # run one and print the rows (add --json to pipe it)
```

  コマンドの完全なセットは [CLI and agents](/ja/agenteye/cli-and-agents) を参照してください。

- **AIアシスタントから。** SQLの書き方がわからない場合は、ダッシュボード内の [AIアシスタント](/ja/agenteye/assistant) に普通の言葉で質問してください。クエリを作成してライブラリに保存してくれます。

保存済みクエリの実行は `queries:run` 権限によって制御されており、クエリの作成・削除の権限とは分離されています。そのため、ライブラリの書き換えを許可せずに読み取りアクセスのみを付与できます。
