エージェントのデータに対してどんな質問でも投げかけ、数秒で答えを得られます。FailproofAI Observability には、イベントと評価に対してすぐに実行できる保存済みクエリのライブラリが用意されており、空の SQL エディタからではなく、動作するサンプルから始められます。

![保存済みクエリのライブラリ：再利用可能なクエリのグリッド。組み込みプリセットとカスタムクエリの両方が表示されている](/agenteye/images/queries.png)

*`/<org-slug>/queries` にある保存済みクエリのライブラリ：組み込みプリセットとチームが保存したクエリが並んで表示されます。*

## 白紙のページではなく、プリセットから始める

テーブル名を覚えたり、SQL をゼロから書いたりする必要はありません。ライブラリは、チームがよく尋ねる質問に対応した組み込みプリセットと、自分のチームが保存・命名したクエリが隣り合わせで表示された状態で開きます。求めているものに近いものを選べば、答えのほとんどは得られています。

保存済みクエリはすべて組織スコープで共有されるため、チームメンバーが書いた便利なクエリはあなたのものにもなります。クエリに名前と説明を一度つけておけば、組織内の誰でも見つけて実行したり、後でダッシュボードに結果をピン留めしたりできます。

`/<org-slug>/queries` からアクセスできます。

## SQL コンポーザーで調整して実行する

クエリを開くと SQL コンポーザーが表示され、そこで調整してすぐに答えを確認できます。エクスポートも往復も、他の誰かを待つ必要もありません。

![保存済みクエリを実行している SQL クエリコンポーザー。スキーマサイドバーとライブ結果グリッドが表示されている](/agenteye/images/query-lab.png)

*SQL コンポーザー：左側にクエリ、列名を推測しなくて済むスキーマサイドバー、そして下部にライブ結果グリッドが表示されます。*

- **スキーマサイドバー**により、分析テーブルとそのカラムが一覧表示されるため、フィールド名を探し回らずにクエリを組み立てられます。
- **ライブ結果グリッド**は実行した瞬間に行を返すため、何度も推測を繰り返すのではなく、数秒で反復作業を進められます。
- **設計上、読み取り専用です。** クエリはイベントストアに対して実行され、サーバー側で検証されます。`SELECT` および `WITH` ステートメントのみが許可され、ステートメントのタイムアウトと行数の上限が設けられています。探索的なクエリがデータを変更することはなく、暴走したクエリは自動的に停止されます。

結果に満足したら、チーム全体が利用できるようにライブラリに保存したり、折れ線・棒・面・円のタイルとしてダッシュボードに出力をピン留めしたりできます。

## ターミナルから実行する、またはアシスタントに作成させる

保存済みクエリは、どこで作業していても同じように使えます。

- **ターミナルから。** `agenteye` CLI は同じ保存済みクエリを一覧表示、実行、保存できるため、スクリプトに結果を組み込んだり、CI に組み入れたり、コーディングエージェントに渡したりできます。

```bash
agenteye query list                 # the same saved queries, from your terminal
agenteye query run errs --arg prod  # run one and print the rows (add --json to pipe it)
```

  完全なコマンド一覧については [CLI とエージェント](/ja/agenteye/cli-and-agents) を参照してください。

- **AI アシスタントから。** SQL をどう書けばよいかわからない場合は、ダッシュボード内の [AI アシスタント](/ja/agenteye/assistant) に平易な言葉で尋ねると、クエリを作成してライブラリに保存してくれます。

保存済みクエリの実行は `queries:run` 権限によって管理され、クエリの作成や削除の権限とは別になっています。そのため、ライブラリを書き換える権限を与えることなく、読み取りアクセスだけを付与できます。
