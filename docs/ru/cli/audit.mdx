<Note>
  **Бета-функция.** Аудит поставляется в статусе бета, пока мы собираем ранние отзывы.
  Каталог детекторов и формат отчёта могут измениться перед следующим стабильным релизом.
  Пожалуйста, откройте issue, если что-то выглядит неправильно.
</Note>

Аудит теперь доступен как страница **/audit в дашборде**, а не как подкоманда CLI. Откройте его из навигационной панели дашборда (между Policies и Projects) или посетите `http://localhost:8020/audit` напрямую при запуске `failproofai` локально.

```bash
failproofai          # откройте дашборд, затем нажмите "Audit"
```

Дашборд сканирует прошлые транскрипты агентских вызовов CLI на этой машине (Claude Code, Codex, Copilot, Cursor, OpenCode, Pi, Gemini) и сообщает, как часто агент совершал действия, которые failproofai создан для остановки — проверки env-var, force push'и, избыточные префиксы `cd <cwd>`, циклы sleep-polling, перечитывание только что отредактированных файлов и многое другое.

Для каждого транскрипта каждое событие tool-use проигрывается через 39 встроенных политик **и** через 8 аудит-только детекторов, которые ловят паттерны, ещё не охватываемые runtime политиками. Счётчики агрегируются по политикам / детекторам во всех сессиях.

## Что вы получите

Страница `/audit` состоит из шести секций:

1. **Identity** — ваш агент классифицируется в один из 8 архетипов (`optimist`, `cowboy`, `explorer`, `goldfish`, `paranoid architect`, `precision builder`, `hammer`, `ghost`) на основе взвешенного сигнала по всем проаудированным транскриптам.
2. **Strengths** — реальные числа из сканирования (чистых вызовов %, "0 утечек учётных данных" и т.д.), отфильтрованные по срабатыванию соответствующих sanitize политик.
3. **Score** — 0-100 с диапазонами S/A/B/C/D/F и прогнозируемым улучшением, если каждая рекомендуемая политика будет включена.
4. **Findings** — карточки по политикам с описанием произошедшего, стоимостью, собранными доказательствами и точной командой `failproofai policy add <slug>` для включения встроенной runtime политики, которая это поймала бы.
5. **Prescribed policies** — агрегированный список установки с одной командой `failproofai policies --install`.
6. **Re-audit reminder** — "вернитесь лучше". Установите 7-дневное напоминание по email через api-server (требует входа; см. [`failproofai auth`](/ru/cli/auth)).

## Аудит-только детекторы

Они обнаруживают паттерны "глупого поведения", которые (пока) не выполняются в реальном времени. Они работают только во время аудита и никогда не блокируют live tool-вызов.

| Детектор | Что он считает |
|---|---|
| `redundant-cd-cwd` | Bash-команды, начинающиеся с `cd <cwd> && …`, хотя команды уже выполняются в `cwd`. |
| `prefer-edit-over-read-cat` | `cat`/`head`/`tail`/`less`/`more` на одном исходном файле — используйте инструмент `Read`. |
| `prefer-edit-over-sed-awk` | `sed -i` / `awk … > file` встроенные редактирования — используйте инструмент `Edit`. |
| `prefer-write-over-heredoc` | Heredoc / многострочный `echo > file` для записи файлов — используйте инструмент `Write`. |
| `sleep-polling-loop` | Длинные `sleep N` (≥ 30s) или `while …; sleep …; done` циклы polling. |
| `find-from-root` | `find /`, `find /home`, `find /usr` и т.д. — используйте `cwd` вместо этого. |
| `git-commit-no-verify` | `git commit … --no-verify` / `-n`, пропуск хуков. |
| `reread-after-edit` | `Read` файла, который был только что `Edit`/`Write` в той же сессии. |

## Кэши

- **Кэш по транскрипту** в `~/.failproofai/cache/audit/<sha1>.json` с ключом `(mtime, size, engineVersion, detectorVersion)` — автоматически инвалидируется при изменении транскрипта или кода политик/детекторов. Каждая запись также хранит `cachedAt` timestamp как **TTL метаданные** (не часть ключа кэша); записи старше **7 дней** отклоняются при чтении, чтобы долгоживущие результаты не переживали эволюцию намерения детектора.
- **Кэш полного результата** в `~/.failproofai/audit-dashboard.json` (режим 0600). Позволяет дашборду отрендериться мгновенно при навигации без перезапуска. Также отклоняется при чтении после **7-дневного TTL** — `/audit` затем переходит в пустое состояние с баннером `audit expired — re-audit to refresh`. Нажмите `[ re-audit ]` в верхней части страницы (или `[ re-audit now ]` ближе к низу) для обновления; новый запуск потоком выводит прогресс через неподвижную верхнюю полосу и меняет результат на месте при успехе (без перезагрузки страницы).

## Примечания

- **Без мутаций.** Аудит воспроизводится в режиме только для чтения. `warn-repeated-tool-calls` пропускается, потому что её per-session сайдкар был бы иначе изменён.
- **Политики рабочего процесса пропущены.** `require-*-before-stop` политики срабатывают только на событиях `Stop` и `execSync` против live git состояния — они не имеют значимой интерпретации "что было бы произошло в 2025", поэтому они не появляются в аудит счётчиках.
- **Пользовательские политики пропущены.** Пользовательские хуки не воспроизводятся (они могли измениться со времени исходной сессии).
