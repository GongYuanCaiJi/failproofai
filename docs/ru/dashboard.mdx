Панель управления failproofai — это локальное веб-приложение для мониторинга сеансов вашего AI-агента и управления политиками. Смотрите, что делали ваши агенты, пока вас не было.

---

## Запуск панели управления

```bash
failproofai
```

Открывается по адресу `http://localhost:8020`.

Панель управления читает данные непосредственно из файловой системы — ваши папки проектов Claude Code и конфигурационные файлы failproofai. Ничего не отправляется на удалённый сервис.

## Страницы

### Проекты

Показывает все проекты Claude Code, OpenAI Codex, GitHub Copilot CLI _(beta)_, Cursor Agent _(beta)_, OpenCode _(beta)_, Pi _(beta)_ и Gemini CLI _(beta)_, найденные на вашей машине. Проекты Claude обнаруживаются из `~/.claude/projects/` (или пути, установленного переменной `CLAUDE_PROJECTS_PATH`); проекты Codex обнаруживаются путём сканирования каждой записи под `~/.codex/sessions/<YYYY>/<MM>/<DD>/*.jsonl` и группировки по `cwd`, записанному в первой записи каждого сеанса; проекты Copilot CLI обнаруживаются путём сканирования каждого файла `~/.copilot/session-state/<sessionId>/workspace.yaml` (настраивается через `COPILOT_HOME`) и группировки по полю `cwd`; проекты Cursor Agent обнаруживаются путём сканирования метаданных каждого сеанса в `~/.cursor/agent-sessions/<sessionId>/` (настраивается через `CURSOR_HOME`, с резервной проверкой `conversations/` и `sessions/`) в поисках скалярного значения `cwd` в `meta.json` / `session.json` / `workspace.yaml`; проекты OpenCode обнаруживаются путём запроса к его БД SQLite в `~/.local/share/opencode/opencode.db` через `opencode db --format json` (мы читаем таблицы `session` и `project` и группируем по `project_id`); проекты Pi обнаруживаются путём сканирования JSONL-записей каждого сеанса в `~/.pi/agent/sessions/<encoded-cwd>/<timestamp>_<uuid>.jsonl` (настраивается через `PI_SESSIONS_DIR`) и извлечения `cwd` из первой записи каждого сеанса; проекты Gemini CLI обнаруживаются путём сканирования `~/.gemini/tmp/<basename>/chats/session-<timestamp>-<uuid-prefix>.jsonl` (настраивается через `GEMINI_SESSIONS_DIR`) и восстановления канонического пути `cwd` из соседнего текстового маркера `.project_root`. Проект, используемый несколькими интерфейсами CLI, отображается в виде одной строки со всеми соответствующими значками. Используйте раскрывающееся меню **CLI** над таблицей, чтобы отфильтровать по определённому агенту CLI; URL сохраняет ваш выбор в виде `?cli=claude|codex|copilot|cursor|opencode|pi|gemini`.

Каждый проект показывает:
- Имя проекта (получено из пути папки)
- Значок CLI — `Claude Code` (оранжевый), `OpenAI Codex` (фиолетовый), `GitHub Copilot` (голубой), `Cursor Agent` (изумрудный), `OpenCode` (янтарный), `Pi` (розовый) и/или `Gemini CLI` (небесный)
- Дату последней активности сеанса

Щёлкните на проект, чтобы просмотреть его сеансы.

### Сеансы

Показывает все сеансы в проекте. Каждый сеанс отображает:
- ID сеанса
- Время начала и окончания
- Количество вызовов инструментов
- Количество активностей hook (срабатываний политик)

Используйте фильтр диапазона дат и поиск по ID сеанса, чтобы сузить список. Сеансы разбиты по страницам.

Щёлкните на сеанс, чтобы открыть просмотр сеанса.

### Просмотр сеанса

Просмотр сеанса отвечает на ключевой вопрос для автономных агентов: что делал агент и оставался ли он в курсе? Значок CLI рядом с заголовком указывает, является ли сеанс записью Claude Code, OpenAI Codex, GitHub Copilot CLI, Cursor Agent, OpenCode, Pi или Gemini CLI. Он показывает временную шкалу всего, что произошло в сеансе:

- **Сообщения** — текстовые ответы Claude и подсказки пользователя
- **Вызовы инструментов** — каждый инструмент, который вызвал Claude, с его входными и выходными данными
- **Активность политик** — для каждого вызова инструмента показано, какие политики сработали и какое решение они вернули

Панель статистики в верхней части показывает длительность сеанса, общее количество вызовов инструментов и сводку решений hook (количество разрешений / отказов / инструкций).

Нажмите кнопку **Download Logs**, чтобы экспортировать сеанс. Для сеансов Claude Code, Codex, Copilot, Cursor, Pi и Gemini вы получите исходную JSONL-запись на диске без изменений; для OpenCode (чьи сеансы хранятся в SQLite, а не на диске) вы получите документ JSON, отражающий базовые таблицы `session` / `messages` / `parts`.

## Автообновление

Панель управления имеет переключатель автообновления в верхней навигации. Когда он включён, текущая страница периодически обновляется, чтобы отображать новые сеансы и активность политик по мере их появления. Это необходимо для мониторинга долгоживущих сеансов автономных агентов.

---

## Отключение страниц

Если вам нужна только часть панели управления, установите переменную `FAILPROOFAI_DISABLE_PAGES` со списком имён страниц, разделённым запятыми:

```bash
FAILPROOFAI_DISABLE_PAGES=policies failproofai
```

Допустимые значения: `policies`, `projects`.

## Тема

Панель управления поддерживает светлый и тёмный режим. Переключайте через кнопку в панели навигации. Предпочтение сохраняется в локальном хранилище вашего браузера.

---

## Настройка пути проектов

По умолчанию панель управления читает из стандартного каталога проектов Claude Code. Переопределите его для пользовательских установок:

```bash
CLAUDE_PROJECTS_PATH=/custom/path/to/projects failproofai
```

## Доступ с удалённого хоста

При запуске панели управления в **режиме разработки** (`npm run dev`) и доступе к ней с имени хоста, отличного от `localhost` — например, пользовательского домена, удалённого IP или туннелированного URL — вы можете увидеть предупреждение:

```text
⚠ Blocked cross-origin request to Next.js dev resource /_next/webpack-hmr from "dashboard.example.com".
```

Это Next.js блокирует кроссисходный доступ к своему websocket HMR (горячая перезагрузка модулей), который доступен только в режиме разработки. Чтобы разрешить ваш хост, используйте флаг `--allowed-origins`:

```bash
npm run dev -- --allowed-origins dashboard.example.com
```

Для нескольких хостов или IP передайте список, разделённый запятыми:

```bash
npm run dev -- --allowed-origins dashboard.example.com,192.168.1.5
```

Вместо этого можно установить переменную окружения `FAILPROOFAI_ALLOWED_DEV_ORIGINS`:

```bash
FAILPROOFAI_ALLOWED_DEV_ORIGINS=dashboard.example.com npm run dev
```

<Note>
Это касается только режима разработки. При запуске `failproofai` (режим производства) нет websocket HMR и нет проблем с кроссисходным доступом к ресурсам разработки.
</Note>
