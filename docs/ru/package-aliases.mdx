## Официальный пакет

Канонический npm-пакет — **`failproofai`**:

```bash
npm install -g failproofai
# или
bun add -g failproofai
```

---

## Почему мы владеем этими алиасами

Typosquatting — распространённая атака на цепочку поставок, при которой злоумышленник регистрирует название пакета, отличающееся от популярного на один символ. Невнимательные пользователи, допустившие опечатку в команде установки, запускают контролируемый атакующим код с полным доступом к системе — именно от такой угрозы Failproof AI предназначен защищать.

Чтобы устранить эту уязвимость, **мы проактивно владеем всеми распространёнными опечатками и вариантами форматирования** `failproofai` на npm. Никто из третьих лиц не сможет зарегистрировать эти названия. Каждый из них — это тонкий прокси, который устанавливает и делегирует управление настоящему пакету `failproofai`.

## Зарегистрированные алиасы

**Варианты форматирования** — разные способы написания "failproof ai":

| Пакет | Статус |
|---------|--------|
| `failproof` | ✅ Опубликован |
| `failproof-ai` | ⏳ Ожидание поддержки npm |
| `fail-proof-ai` | ⏳ Ожидание поддержки npm |
| `failproof_ai` | ⏳ Ожидание поддержки npm |
| `fail_proof_ai` | ⏳ Ожидание поддержки npm |
| `fail-proofai` | ⏳ Ожидание поддержки npm |

**Опечатки `failprof*`** — пропущена одна буква `o` в слове "proof":

| Пакет | Статус |
|---------|--------|
| `failprof` | ✅ Опубликован |
| `failprof-ai` | ✅ Опубликован |
| `failprofai` | ⏳ Ожидание поддержки npm |
| `fail-prof-ai` | ⏳ Ожидание поддержки npm |
| `failprof_ai` | ⏳ Ожидание поддержки npm |

**Опечатки `faliproof*`** — переставлены буквы `a` и `i`:

| Пакет | Статус |
|---------|--------|
| `faliproof` | ✅ Опубликован |
| `faliproof-ai` | ✅ Опубликован |
| `faliproofai` | ⏳ Ожидание поддержки npm |

> **Почему ожидание?** Политика защиты от спама npm блокирует названия, которые нормализуются в ту же строку, что и существующий пакет, после удаления пунктуации и проверки схожести. Мы связались с поддержкой npm для резервирования этих названий в целях защиты от typosquatting. Они будут активированы после одобрения.

Вы можете проверить, что любой опубликованный алиас принадлежит нам:

```bash
npm info failproof
# Ищите: "ExosphereHost Inc." в поле maintainers
```

## Как работают алиасы

Каждый пакет-алиас:

1. Указывает `failproofai` в качестве зависимости — таким образом при установке запускается настоящий пакет (включая его hook `postinstall`)
2. Предоставляет бинарный файл, соответствующий его собственному названию (например `failprof-ai`), который проксирует все аргументы в бинарный файл `failproofai`

Прокси — это двухстрочный Node.js скрипт; здесь нет логики, сетевых запросов и сбора данных, выходящих за рамки того, что делает сам `failproofai`.
