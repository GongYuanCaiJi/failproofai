Хуки и политики для **обработки отказов AI**, **восстановления после ошибок** и **надёжности LLM**. Делайте ваших AI-агентов надёжными и способными работать автономно с **Claude Code**, **OpenAI Codex**, **GitHub Copilot**, **Cursor Agent**, **OpenCode**, **Pi**, **Hermes**, **OpenClaw**, **Factory Droid**, **Devin CLI**, **Antigravity CLI** и **Agents SDK**.

AI-агенты отказывают предсказуемо. Они выполняют деструктивные команды, утекают секреты, отклоняются от задачи, застревают в циклах или пушат прямо в main. Если оставить без присмотра, небольшие сбои каскадируют в отказы в обслуживании, утечки учётных данных и потерю работы.

FailproofAI решает эту проблему с помощью **политик**. Эти правила встраиваются в каждый вызов инструмента агента, чтобы **обнаруживать отказы**, **смягчать их** (блокировать, давать инструкции, санитизировать) и **оповещать вас** когда требуется внимание. Локальная панель управления позволяет вам просмотреть каждый вызов инструмента, отказ агента и действие восстановления впоследствии.

Расшифровки и оценка политик остаются на вашем компьютере. Данные отправляются только когда вы явно используете онлайн-функцию, такую как аутентифицированные напоминания об аудите или приглашения.

## Быстрый старт

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

```bash
failproofai policies --install   # включить политики (или пропустить — `failproofai` предложит их установить при первом запуске)
failproofai                      # запустить панель управления
```

Смотрите руководство [Начало работы](/ru/getting-started) для полного пошагового руководства.
