## Модель данных

**Event (событие)**
Наименьшая единица данных. Одно событие записывает один шаг, который сделал ваш агент: `tool_use`, `model_request`, `hook_completed`, `error` и так далее. Ваш агент генерирует события через [Python SDK](/ru/agenteye/python-sdk); они появляются в реальном времени на странице **Events**.

**Session (сессия)**
Один запуск агента, идентифицируемый по `session_id`. Сессия — это все события, которые имеют одинаковый id, сворачиваемые в одну строку на странице **Sessions** и отображаемые как граф выполнения на детальной странице. Сессия обычно начинается с `agent_start` и заканчивается `agent_end`.

**Agent (агент)**
Именованный исполнитель внутри запуска, идентифицируемый по `agent_id`. Один запуск может включать нескольких агентов: например, планировщик, который запускает вспомогательного агента-суммаризатора. Вспомогательные агенты имеют `parent_id`, что позволяет Failproof AI Observability отображать их на отдельных линиях в графе выполнения.

**Environment (окружение)**
Метка места, где был запущен агент: `production`, `staging`, `dev`. Вы устанавливаете её один раз при конфигурации SDK. Почти все страницы дашборда могут фильтровать по окружению.

**Context-window fill (заполнение контекстного окна)**
Процент контекстного окна модели, потреблённый ответом. Failproof AI Observability указывает его на событиях `model_response` для моделей, которые она распознаёт, так что рост подсказки и предстоящее сжатие видны прямо в потоке событий.

## Качество

**Evaluation (оценка)**
Оценка качества завершённой сессии, полученная от вашего сервиса оценки. Оценки опциональны: до подключения evaluator'а сессии записываются, но не оцениваются. Каждая оценка может содержать несколько именованных баллов (например `helpfulness`, `factuality`, `tool_efficiency`), каждый с кратким комментарием. Смотрите [Evaluation suite](/ru/agenteye/evaluation-suite).

**Score key (ключ оценки)**
Имя одного измерения, о котором сообщает evaluator, например `helpfulness`. Оповещения и проверки могут отслеживать конкретный ключ оценки со временем.

**Evaluator (оценщик)**
Ваш сервис оценки. Failproof AI Observability отправляет POST с расшифровкой завершённого запуска и сохраняет возвращаемые баллы. Она не поставляет evaluator'а по умолчанию; логика оценки — это ваша задача.

## Поиск и исправление ошибок

**Hook (хук)**
Защита или побочный эффект, который ваш агент-фреймворк запускает вокруг шага: проверка безопасности контента, редактирование PII, охрана бюджета. Хуки генерируют события `hook_triggered` / `hook_completed` с `outcome` (allow, deny, modify) и имеют собственную страницу наблюдения.

**Alert rule (правило оповещения)**
Правило, которое срабатывает, когда метрика пересекает установленный вами порог: частота ошибок, p95 латентность, стоимость токенов или оценка evaluator'а. Когда правило срабатывает, оно открывает инцидент и уведомляет выбранные каналы (электронная почта, Slack, webhook, в дашборде). Смотрите [Alerts](/ru/agenteye/alerts).

**Incident (инцидент)**
Открытая проблема, созданная при срабатывании правила оповещения. Инциденты имеют жизненный цикл (acknowledge, assign, resolve) и временную шкалу активности, которая записывает каждое действие. Вы также можете открыть его вручную.

**Audit (проверка)**
Повторяющееся исследование (ежечасное до еженедельного), которое анализирует ваши логи *между* сессиями в поиске паттернов ошибок, которые вы ещё не описали: кластеры ошибок, низкие оценки, выбросы латентности, циклы вызовов инструментов и запуски, которые никогда не завершились. Если оповещение отслеживает уже известную вам метрику, проверка подскажет вам, на что обратить внимание дальше. Смотрите [Audits](/ru/agenteye/audits).

**Finding (находка)**
Один ранжированный результат проверки, подкреплённый доказательствами. Находка называет паттерн, ссылается на точные сессии, стоящие за ним, и имеет жизненный цикл сортировки (acknowledge, resolve, mute, dismiss). Failproof AI Observability дедублицирует находки между запусками проверок, так что известный паттерн обновляется вместо накопления.

**The AI assistant (AI помощник)**
Чат в дашборде, который отвечает на вопросы о ваших агентах на простом английском языке, на основе ваших собственных данных. По умолчанию он доступен только для чтения; всё, что он создаёт (сохранённый запрос, дашборд), требует одобрения, и он никогда не может удалять. Смотрите [AI assistant](/ru/agenteye/assistant).

## Запуск

**Organization (tenant) (организация)**
Изолированное рабочее пространство. Один экземпляр Failproof AI Observability может содержать много организаций, каждая со своими пользователями, ключами и данными. Каждый URL дашборда находится в области действия вашей организации (`/<org-slug>/…`).

**Collector (сборщик)**
`agenteye-collector`, лёгкий демон, который работает на каждой машине агента, объединяет события, которые SDK записывает на диск, и отправляет их на сервер.

**API key (ключ API)**
Ограниченный токен, который аутентифицирует клиент на сервере. Ключи имеют детальные разрешения (например `events:add` для сборщика, области только для чтения для ключа дашборда). Смотрите [API keys](/ru/agenteye/api-keys).

**Server (сервер)**
Сервис приёма и API. Он принимает события, сохраняет операционное состояние в ваших базах данных и обслуживает дашборд и CLI.

**Dashboard (дашборд)**
Веб-интерфейс. Каждая страница находится в области действия организации и читает данные через API сервера.
