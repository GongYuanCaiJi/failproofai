Вся ваша сквозная инфраструктура Failproof AI Observability, одна команда — и готово. Проверьте production, создайте API ключ или подтвердите инцидент, не выходя из терминала, а затем автоматизируйте всё это в CI или позвольте агенту кодирования сделать это на простом английском.

```bash
pipx install agenteye
agenteye login --email you@example.com      # a 6-digit code lands in your inbox
agenteye --json sessions --since 24h         # every agent run from the last day, newest first
```

*CLI `agenteye` взаимодействует с вашим dashboard. Это отдельный инструмент от collector'а, который отправляет события на сервер.*

## Вся ваша инфраструктура, одна команда — и готово

Хватит переключаться между вкладками на быстрый вопрос. CLI `agenteye` читает ваши данные и управляет организацией из одного бинарного файла, поэтому проверка, которая раньше означала клики по dashboard, становится одной строкой, которую вы можете переиспользовать, создать alias или вставить в runbook. У вас есть четыре поверхности:

- **Чтение ваших данных:** `sessions`, `events`, `evals` и `errors`, отфильтрованные по времени, агенту и окружению.
- **Управление организацией:** `keys`, `users`, `settings`, `alerts` и `incidents`.
- **Запуск аналитики:** сохранённые SQL запросы и ad-hoc `query` runner над вашими данными событий.
- **Обращение к ассистенту:** `agent ask` обращается к тому же аналитику, только для чтения, что и в dashboard.

Установите один раз с `pipx`, войдите с 6-значным кодом по email, и готово. Сессия длится около дня; перезапустите `agenteye login`, когда она истечёт. Используйте для spot-check production, создания ключа или триажа инцидента, всё без открытия браузера:

```bash
agenteye errors --since 24h --aggregate      # what is breaking, grouped by error type
agenteye incidents list --state firing       # what is on fire right now
agenteye keys create ci --add events:add     # a key that can only push events, secret shown once
```

Один обычай, который нужно знать: глобальные опции вроде `--json` идут перед командой. `agenteye --json sessions` правильно; `agenteye sessions --json` неправильно.

## Автоматизируйте, интегрируйте в CI

Каждая команда принимает `--json`, и это меняет всё. Чистый JSON выводится на stdout, а статус и предупреждения для человека — на stderr, поэтому захват `--json` пройдёт прямо в `jq` без каких-либо лишних строк для удаления. Это делает CLI одинаково хорошим для вас в командной строке и для агента кодирования, разбирающего вывод:

```bash
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'
```

Это построено для работы без присмотра. Подтверждения автоматически пропускаются, когда терминала нет, поэтому ничто не зависает в pipeline, и каждая команда возвращает значимый код выхода: `0` успех, `4` не вошли, `5` отсутствует разрешение (сообщение назовёт его, например `alerts:write`), `3` dashboard недоступен. Скрипт может разветвиться на `4` для переаутентификации или `5` чтобы сказать вам ровно то, о чём попросить администратора, вместо слепого отказа.

## Позвольте агенту кодирования управлять им на простом английском

Ещё лучше — вам не должно быть нужно помнить все эти флаги. **CLI skill** — это маленькая папка Agent Skill с именем `agenteye-cli`, которая учит агента кодирования, такого как Claude Code или Codex, управлять CLI из обычных английских запросов. Спросите «что-нибудь сломалось сегодня?» и агент выбирает команду, запускает её от вас и отвечает прозой.

Для Claude Code бросьте папку `agenteye-cli` в `~/.claude/skills/` и она будет автоматически найдена. Failproof AI Observability предоставляет папку; нечего лишнего устанавливать, потому что она только управляет CLI, который вы уже установили. Сначала войдите сами: skill не может завершить для вас вход по коду по email.

Поскольку агент запускает CLI от вас, он может делать всё, что позволяет ваш login, читает и записи: создавать ключи, менять настройки, разрешать инциденты. Подтверждение «вы уверены?» не срабатывает для агента, поэтому skill написан так, чтобы указывать точную команду и ждать вашего ОК перед любым изменением. Вы — шаг подтверждения.

```text
you    Why did session run-001 fail?

agent  Running: agenteye --json events --session-id run-001 --all
       The checkout-agent hit a TimeoutError on its third tool call.
```

Чтения остаются мгновенными, и каждая запись ждёт вас:

```text
you    Give CI a key that can only push events.

agent  This creates an API key. I'll run:
         agenteye keys create ci --add events:add
       OK to proceed?

you    yes

agent  Done. Key "ci" created with events:add only. The secret is shown once, so store it now.
```
