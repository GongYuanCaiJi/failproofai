Вся инфраструктура FailproofAI Observability — одна команда. Проверьте production, создайте API ключ или обработайте инцидент прямо из терминала, а затем автоматизируйте это в CI или позвольте кодирующему агенту сделать это на простом английском.

```bash
pipx install agenteye
agenteye login --email you@example.com      # a 6-digit code lands in your inbox
agenteye --json sessions --since 24h         # every agent run from the last day, newest first
```

*CLI `agenteye` взаимодействует с вашей панелью управления. Это отдельный инструмент от collector, который отправляет события на сервер.*

## Вся инфраструктура, одна команда

Перестаньте прыгать по вкладкам, чтобы ответить на быстрый вопрос. CLI `agenteye` читает ваши данные и администрирует вашу организацию из одного бинарного файла, поэтому проверка, которая раньше требовала кликов в панель управления, теперь одна строка, которую вы можете повторить, создать для неё alias или вставить в runbook. Вы получаете четыре интерфейса:

- **Чтение данных:** `sessions`, `events`, `evals` и `errors`, отфильтрованные по времени, агенту и окружению.
- **Управление организацией:** `keys`, `users`, `settings`, `alerts` и `incidents`.
- **Запуск аналитики:** сохранённые SQL запросы плюс ad-hoc `query` runner для ваших данных событий.
- **Общение с ассистентом:** `agent ask` обращается к тому же read-only аналитику, с которым вы чатитесь в панели управления.

Установите один раз с `pipx`, авторизуйтесь с 6-значным кодом, отправленным на почту, и всё готово. Сессия длится около дня; запустите `agenteye login` снова, когда срок истечёт. Используйте для быстрой проверки production, выдачи ключа или триажа срабатывающего инцидента, всё без открытия браузера:

```bash
agenteye errors --since 24h --aggregate      # what is breaking, grouped by error type
agenteye incidents list --state firing       # what is on fire right now
agenteye keys create ci --add events:add     # a key that can only push events, secret shown once
```

Один важный момент: глобальные опции типа `--json` идут перед командой. `agenteye --json sessions` правильно; `agenteye sessions --json` неправильно.

## Автоматизируйте и интегрируйте в CI

Каждая команда поддерживает `--json`, и это меняет всё. Чистый JSON идёт в stdout, а статус и предупреждения идут в stderr, поэтому вывод с `--json` напрямую можно piped в `jq` без лишних строк. Именно это делает CLI одинаково удобным для вас в терминале и для кодирующего агента, который парсит вывод:

```bash
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'
```

Он построен для unattended запуска. Prompts подтверждения автоматически пропускаются, когда нет подключённого терминала, поэтому ничто не зависает в pipeline, и каждая команда возвращает meaningful exit code: `0` успех, `4` не авторизован, `5` недостаточно прав (сообщение назовёт их, например `alerts:write`), `3` панель управления недоступна. Скрипт может разветвляться на `4` для переавторизации или на `5`, чтобы сказать вам, что именно запросить у администратора, вместо того чтобы молча упасть.

## Позвольте кодирующему агенту управлять этим на простом английском

Ещё лучше: вам вообще не нужно помнить эти флаги. **CLI skill** — это маленькая папка Agent Skill с именем `agenteye-cli`, которая учит кодирующего агента типа Claude Code или Codex управлять CLI по простым английским командам. Спросите «сломалось ли что-нибудь сегодня?» и агент подберёт команду, запустит её от вашего имени и ответит прозой.

Для Claude Code просто поместите папку `agenteye-cli` в `~/.claude/skills/` и она будет автоматически обнаружена. FailproofAI Observability предоставляет эту папку; ничего дополнительного устанавливать не нужно, потому что она только управляет уже установленным CLI. Авторизуйтесь самостоятельно первым: skill не может завершить login с кодом из письма за вас.

Поскольку агент запускает CLI от вашего имени, он может делать всё, что позволяют ваши права: создавать ключи, менять настройки, разрешать инциденты. Prompt подтверждения CLI не срабатывает для агента, поэтому skill написан так, чтобы показать точную команду и подождать вашего подтверждения перед любым изменением. Вы — шаг подтверждения.

```text
вы     Почему session run-001 упал?

агент  Running: agenteye --json events --session-id run-001 --all
       The checkout-agent hit a TimeoutError on its third tool call.
```

Чтения остаются мгновенными, а каждая запись pauses для вас:

```text
вы     Дайте CI ключ, который может только отправлять события.

агент  This creates an API key. I'll run:
         agenteye keys create ci --add events:add
       OK to proceed?

вы     yes

агент  Done. Key "ci" created with events:add only. The secret is shown once, so store it now.
```
