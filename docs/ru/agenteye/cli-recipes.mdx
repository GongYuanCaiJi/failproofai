Извлекайте данные сеансов, событий и оценок (и запускайте переоценки) прямо из скрипта или кодирующего агента с чистым JSON на stdout, который передается в `jq`. Эти рецепты превращают данные Failproof AI Observability в то, что пользователь терминала или AI-агент кодирования (Claude Code, Cursor) может запрашивать и автоматизировать без нажатия на кнопки в панели управления.

Приведённые ниже примеры готовы для копирования в CLI Failproof AI Observability (`agenteye`). Для установки, аутентификации и полного списка опций см. [CLI](/ru/agenteye/cli); выполните `agenteye -h` или `agenteye <command> -h` для встроенной справки.

## Золотые правила

1. **Глобальные опции идут *перед* командой.** `agenteye --json sessions` верно; `agenteye sessions --json` неверно. Глобальные опции: `--json`, `--base-url`, `--org`, `--token`, `--insecure`/`--secure`, `--timeout`, `--quiet`, `--no-color`.
2. **Передавайте `--json` каждый раз, когда вы парсите выход.** Данные идут в **stdout** как JSON; статус и ошибки идут в **stderr**, так что stdout остаётся чистым для передачи в `jq`.
3. **Ветвитесь по коду выхода, а не по тексту stderr**: `0` ОК · `1` непредвиденная ошибка · `2` неправильные аргументы · `3` невозможно доступиться до панели управления · `4` не авторизован или сеанс истёк · `5` отсутствует разрешение · `6` ресурс не найден.
4. **Исследуйте с помощью `-h`.** Каждая команда документирует свои фильтры, форматы значений и форму JSON.

## Одноразовая настройка

```bash
export AGENTEYE_DASHBOARD_URL=https://agenteye.example.com   # чтобы не повторять --base-url
agenteye login --email you@example.com                       # вставьте отправленный код; действителен ~24ч
```

## Проверьте аутентификацию перед работой

`whoami` никогда не выдаёт ошибку при отсутствующем или истёкшем сеансе; вместо этого он сообщает `logged_in:false`, так что агент может безопасно проверить состояние аутентификации. (Он всё ещё может выдать ненулевой код, если не установлен базовый URL или панель управления недоступна.)

```bash
if [ "$(agenteye --json whoami | jq -r .logged_in)" != "true" ]; then
  echo "Not authenticated. Run: agenteye login" >&2; exit 1
fi
```

## Найдите сеансы с ошибками или низкими оценками

```bash
# сеансы за последние 24ч, в которых оценка завершилась с ошибкой
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'

# оценки с баллом <= 0.5 по полезности для одного агента
agenteye --json evals --agent-id checkout-bot --score helpfulness:..0.5 \
  | jq '.evaluations[] | {session_id, scores}'
```

Фильтрация оценок работает на **`evals`**, а не на `sessions`. `--score KEY:MIN..MAX` повторяется и комбинируется оператором AND; любая граница опциональна (`..0.5` означает ≤ 0.5, `0.9..` означает ≥ 0.9). Вы можете передать до 20 фильтров оценок за запрос; больше возвращает HTTP 400. `sessions` распределяет фильтры `--env`, `--status`, `--agent-id`, `--session-id` и диапазон времени с `evals`, но не имеет `--score`.

## Прочитайте один сеанс целиком

Единственной команды `session show` нет. Объедините цепь событий с оценкой сеанса:

```bash
# последняя оценка сеанса (статус + оценки)
agenteye --json evals --session-id run-001 | jq '.evaluations[0] | {status, scores}'

# каждое событие в запуске (увеличьте --limit для полной проверки)
agenteye --json events --session-id run-001 --all --limit 1000 | jq '.events[] | {ts, event_type}'

# только вызовы инструментов в сеансе (--full требуется для получения исходной полезной нагрузки)
agenteye --json events --full --session-id run-001 --event-type tool_use,tool_result --all \
  | jq '.events[].payload'
```

> **Примечание:** По умолчанию `events` читает быструю ленту без полезной нагрузки. Каждое событие содержит рассчитанную на сервере однострочную `summary` плюс флаги типа `is_error` и подсчёты токенов, но `payload` возвращается как `{}`. Чтобы извлечь исходную полезную нагрузку, добавьте `--full` (или `--fields payload`). Полная лента работает медленнее в масштабе, поэтому держите её ограниченной: комбинируйте `--full` с одним `--session-id`.

## Получите всё (постраничность)

Результаты новейшие-первыми и разбиты на страницы по курсору.

```bash
# в один приём: получите до 500 строк по 200 строк на странице
agenteye --json events --session-id run-001 --limit 500 --all > events.json

# ручная постраничность: передайте next_cursor обратно
page=$(agenteye --json events --limit 100)
cursor=$(echo "$page" | jq -r '.next_cursor // empty')
[ -n "$cursor" ] && agenteye --json events --limit 100 --cursor "$cursor"
```

## Уменьшите выход с помощью --fields

Ограничьте ключи (и в таблице, и в `--json`) чтобы сократить то, что должен прочитать агент.

```bash
agenteye --json sessions --since 7d --fields session_id,status,scores | jq -c '.sessions[]'
agenteye --json events --session-id run-001 --fields ts,event_type --all
```

Неизвестные имена полей отклоняются (выход `2`) с допустимым списком, дешёвый способ обнаружить имена полей.

## Обнаружьте допустимые значения фильтров

```bash
agenteye --json list envs | jq -r '.values[]'           # значения для --env
agenteye --json list tools | jq -r '.values[]'          # имена инструментов; также agents, models, event_types, …
agenteye --json list score_filters | jq -r '.values[]'  # допустимый KEY для --score KEY:MIN..MAX
```

## Выберите вашу организацию (мультитенантность)

Если вы принадлежите более чем одной организации, выберите активного клиента при входе (он сохраняется):

```bash
agenteye login --org acme --email you@corp.com   # установите клиента на том же этапе входа
agenteye --json orgs list | jq -r '.orgs[].org_slug'
agenteye --org globex --json sessions --since 24h   # переопределите для одной команды
```

Вход в мультиорганизацию без `--org` выдаёт ненулевой код и выводит организации на выбор.

## Подготовьте API-ключ для SDK/collector

```bash
# секрет выводится ОДИН РАЗ, при --json это поле .key
key=$(agenteye --json keys create ci-bot --add events:read.add | jq -r '.key')
agenteye keys regenerate ci-bot --yes    # ротация; agenteye keys disable ci-bot --yes чтобы отозвать
```

## Запустите сохранённый или ad-hoc запрос

```bash
agenteye --json query run --sql "select count(*) from analytics.events" | jq '.rows'
agenteye --json query run errs --arg prod | jq '.rows'   # сохранённый запрос + позиционный $1
```

## Классифицируйте инцидент неинтерактивно

```bash
id=$(agenteye --json incidents list --state firing | jq -r '.incidents[0].id')
agenteye incidents ack "$id"
agenteye incidents assign "$id" --assignee you@corp.com
agenteye incidents resolve "$id" --yes
```

> **Примечание:** Мутации автоматически пропускают запрос подтверждения при `--json` или когда stdin не является TTY, так что агенты никогда не зависают; передайте `--yes`/`-y` чтобы пропустить его явно в другом месте.

## Обработка кодов выхода в скрипте

```bash
out=$(agenteye --json sessions --since 1h) || code=$?
case "${code:-0}" in
  0) echo "$out" | jq '.sessions | length' ;;
  4) echo "Session expired - run 'agenteye login'." >&2 ;;
  5) echo "Missing permission (ask an admin for evaluations:read)." >&2 ;;
  3) echo "Dashboard unreachable - check the URL." >&2 ;;
  *) echo "Unexpected error (exit ${code})." >&2 ;;
esac
```

## Формы JSON выхода

| Команда | stdout JSON (с `--json`) |
|---|---|
| `whoami` | `{"logged_in": true, "id", "email", "is_instance_admin", "active_org", "permissions": [...], "memberships": [...]}` или `{"logged_in": false}` |
| `orgs list` | `{"active_org", "orgs": [{"org_slug","org_name","permission_set","permissions"}]}` |
| `events` | `{"events": [...], "next_cursor": <cursor or null>}` |
| `evals` | `{"evaluations": [...], "next_cursor": <cursor or null>}` |
| `sessions` | `{"sessions": [...], "next_cursor": <cursor or null>}` |
| `errors` | `{"errors": [...], "next_cursor": <cursor or null>}` |
| `list <kind>` | `{"kind", "values": [...]}` |
| `keys list` / `keys create` | `{"keys": [...]}` / `{id, name, permissions, created_at, key}` (`key` показан один раз) |
| `query run` | `{columns: [{name,type}], rows: [[...]], truncated, elapsed_ms}` |
| `users list` / `settings list` | `{"users": [...]}` / `{"settings": [...]}` |
| `alerts list` / `incidents list` | `{"alerts": [...]}` / `{"incidents": [...]}` |
| create/update/delete (любой) | объект ресурса или `{"deleted": true, "id"}` для удалений |
| failure (любой, с `--json`) | `{"error": "...", "exit_code": <n>, "status"?: <http>, "hint"?: "..."}` на stdout |

- Каждый элемент **event** (`events`): `id, session_id, agent_id, event_type, ts, payload, environment, summary, is_error, error_type, output_tokens, context_window, context_fill`. Обратите внимание, что `payload` — это `{}` если вы не запросили полную ленту с помощью `--full` (или `--fields payload`).
- Каждый элемент **evaluation** (`evals`): `id, session_id, agent_id, environment, status, scores, reasoning, summary, error, attempt_count, duration_ms, completed_at, created_at`.
- Каждый элемент **session** (`sessions`): `session_id, agent_id, environment, status, scores, event_count, started_at, last_event_at, first_event_id, last_event_id, latest_evaluation`.

Каждая команда `--fields` принимает ровно имена полей своего собственного элемента. Набор различается между `sessions` и `evals`, поэтому имя допустимое для одного может быть отклонено другим.
