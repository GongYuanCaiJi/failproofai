## Видите каждый запуск с первого взгляда

Сырой поток событий — это истина каждого шага, но когда у вас есть тысячи шагов в дюжине запусков, вам нужен запуск, а не шаг. Страница сеансов свёртывает все события запуска в одну строку, так что день активности становится сканируемым списком вместо потока информации.

Каждая строка содержит индикатор статуса, поэтому неудачный запуск выделяется на фоне здорового ещё до того, как вы что-то нажмёте. Фильтруйте по диапазону дат, окружению, агенту или сеансу — переходите от "всё" к "запуск, который меня интересует" в пару кликов.

Когда вы подключите оценщик, каждый завершённый запуск автоматически получит оценку, и её актуальное значение будет отображаться на строке как значок. Вы можете фильтровать по любому диапазону оценок — так что "покажи мне все низкооценённые prod-запуски за эту неделю" становится фильтром, а не ручной проверкой. До тех пор пока вы не установите фильтр, сеансы по-прежнему фиксируют полный запуск; они просто ещё не имеют оценки.

## Читайте весь запуск как картину

![Граф выполнения в стиле git рядом с временной шкалой событий, с панелью разбора инструментов, моделей и хуков](/agenteye/images/session-detail.png)

*Граф выполнения (слева) расположен рядом с временной шкалой событий; правая колонка показывает разбор инструментов, моделей, хуков и расходов токенов для запуска.*

Нажмите на любой сеанс, чтобы открыть его граф выполнения: вид в стиле git того, как агенты, инструменты, хуки и вызовы моделей разворачивались во времени. Каждый параллельный суб-агент получает свою ветку, поэтому вы можете увидеть, какая работа выполнялась одновременно, какой суб-агент зависал и где запуск пошёл не по плану — всё без необходимости мысленно воспроизводить это по стене логов.

Правая колонка даёт вам разбор по запуску: какие инструменты и модели запускались, какие хуки срабатывали и сколько токенов потратил запуск. Это ответ на вопрос "почему этот запуск стоил так дорого?" или "какой инструмент медленный?" — всё это расположено прямо рядом с графом, который это вызвал.

## Где это найти

Каждая страница панели инструментов привязана к вашей организации (`/<org-slug>/…`). Сеансы находятся в разделе **Observe** в левой боковой панели, рядом с событиями, с фильтрами диапазона дат, окружения, агента и сеанса в верхней части списка. Каждая строка — один клик до её полного графа выполнения.

Чтобы включить значки оценок и фильтрацию по диапазону оценок, подключите оценщик: см. [Evaluations](/ru/agenteye/evaluations).

## Связанное

- [Event stream](/ru/agenteye/event-stream): сырой поток, состоящий из каждого шага, из которого свёртывается каждый сеанс.
- [Evaluations](/ru/agenteye/evaluations): подключите оценщик, чтобы каждый запуск получал значок оценки, по которому вы можете фильтровать.
- [Telemetry](/ru/agenteye/telemetry): как запуски попадают из вашего агента в эти сеансы.
