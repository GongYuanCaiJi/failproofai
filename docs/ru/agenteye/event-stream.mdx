В момент, когда ваш агент что-то делает, вы это видите. Event Stream — это ваш живой пульс каждого агента в продакшене: без ожидания, без поиска в логах, без гадания о том, что произошло.

![Live Event Stream: цветные строки событий, идущие в реальном времени, фильтруемые по окружению, агенту, сессии, типу события и свободному тексту](/agenteye/images/events-stream.png)

*Каждое событие от каждого агента в вашей организации, самые свежие первыми, обновляется по мере поступления.*

## Ваш живой пульс каждого агента

Когда агент запускает run, обращается к модели, вызывает tool, запускает hook или попадает в ошибку, строка появляется в верхней части потока в тот момент, когда это происходит. Он отслеживает каждое событие от каждого агента в вашей организации, самые свежие первыми, так что у вас всегда есть актуальная картина вместо устаревшей.

Это означает, что вам не нужно следить за файлами логов на каком-то сервере, не нужно искать данные на разных машинах, не нужно вручную совмещать временные метки. Вы открываете одну страницу и уже наблюдаете за продакшеном.

Строки имеют цветовую кодировку по типам, так что вы можете читать поток с одного взгляда, а не разбирать каждую строку. На первый взгляд каждая строка показывает вам:

- **Её тип**, цветом: `agent_start`, `model_response`, `tool_use`, `hook_completed`, `error` и другие.
- **Однострочное резюме** того, что произошло, так что вам редко нужно открывать что-либо, чтобы просто понять суть.
- **Количество токенов** для этого шага.
- **Значок заполнения контекстного окна**, где это применимо, чтобы рост промпта и приближающаяся компактизация были видны до того, как причинят вред.

Наблюдение в реальном времени означает, что вы поймаете плохой deploy, бесконечный цикл или всплеск ошибок в момент их возникновения, а не при проверке логов на следующий день.

## Найдите нужный вам run

Когда что-то выглядит не так, вам не нужна вся информация сразу. Вам нужен единственный run, который сломался. Поток фильтруется быстро: по окружению, по агенту, по сессии, по типу события или по свободному тексту.

Фильтруйте по id сессии или id агента, чтобы следить за одним run от его первого события до последнего. Фильтруйте по типу события, чтобы выделить один вид активности, например все `error` по всей организации в одном представлении. Комбинируйте фильтры, чтобы сужать от "всё, везде" до "этот агент, в prod, с ошибками" за пару кликов, затем действуйте на основе найденного.

Поиск по свободному тексту быстро находит сообщение, имя tool или id, который у вас уже есть под рукой, так что отчёт от клиента превращается в точный run за считанные секунды.

## Где его найти

Event Stream — это дом вашей организации. После входа это первая поверхность, на которую вы попадаете, по адресу `/<org-slug>/`, так что сортировка начинается с момента вашего прибытия.

В фоне ваши агенты отправляют события через SDK, коллектор доставляет их на ваш FailproofAI Observability сервер, и поток отслеживает их по мере поступления в управляемую вами инфраструктуру. Когда вам нужен сводный вид вместо raw логов, события каждого run сворачиваются в одну строку на Sessions, одним кликом.

Это raw источник истины, на котором построены все остальные поверхности наблюдения, так что когда число выглядит неправильно в другом месте, поток — это место, где вы подтверждаете, что действительно произошло.
