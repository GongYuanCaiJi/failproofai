Проблемы с качеством обнаруживаются сейчас, а не в жалобе пользователя. Подключите свой собственный сервис оценки один раз, и Failproof AI Observability автоматически оценит каждый завершённый сеанс, так что снижение качества помощи или всплеск галлюцинаций проявятся сами по себе, прежде чем это заметит пользователь.

![Сетка сеансов с колонкой оценки: каждый запуск имеет статус оценки и цветные значки полезности, факт-проверки и эффективности инструментов](/agenteye/images/sessions-list.png)

*Каждый запуск в сетке сеансов содержит свои оценки; красные, жёлтые и зелёные значки выделяют слабые запуски без необходимости открывать транскрипции.*

## Перестаньте вручную проверять выборки запусков

Раньше вы выборочно проверяли несколько запусков и надеялись, что остальные в порядке. Теперь каждый завершённый сеанс оценивается сразу после завершения по интересующим вас параметрам: полезность, эффективность использования инструментов, фактическая точность, безопасность — всё, что входит в ваш стандарт качества. Вы определяете ключи оценки; Failproof AI Observability хранит, отслеживает и выводит всё, что возвращает ваша система оценки. Ни один запуск не остаётся без оценки, и вы перестаёте узнавать о регрессии из письма в поддержку.

Оценки отображаются в сетке сеансов по адресу **`/<org-slug>/sessions`** (боковая панель → *observe* → *sessions*), один набор значков на строку. Хотите увидеть только запуски, которые не прошли проверку? Отфильтруйте сетку по диапазону оценок, например полезность менее 0,5, и получите ровно те запуски, которые стоит прочитать. Для просмотра оценок требуется разрешение `evaluations:read`.

## Посмотрите, почему запуск получил низкую оценку

Число говорит о том, что запуск был слабым; страница сеанса объясняет почему. Откройте любой запуск, и правая панель начинается с кратого резюме, затем показывает полосу для каждого параметра с рассуждениями вашей системы оценки под каждой, так что вы переходите от «это получило 0,4 по факт-проверке» к точному утверждению, которое она неправильно определила, за секунды.

![Правая панель сеанса: сводка оценки сверху, затем полосы оценки для каждого параметра с рассуждениями, рядом с полной временной шкалой событий](/agenteye/images/session-detail.png)

*Вид деталей сеанса: сводка, полосы оценки для каждого параметра и рассуждения, стоящие за каждой оценкой, прямо рядом с временной шкалой событий запуска.*

Развернули улучшенную систему оценки или смотрите на запуск, который упал до оценки? Кнопка **re-evaluate** (контролируется `evaluations:trigger`) переоценивает сеанс на месте и добавляет свежий результат в его временную шкалу, так что более ранние оценки остаются видимыми как история. Вы найдете её по адресу **`/<org-slug>/sessions/<session-id>`**.

## Наблюдайте за тенденциями качества по всему парку

Один низкий запуск — это шум; целая группа скользящих вниз — это сигнал. Сохранённые панели мониторинга превращают ваши оценки в тенденцию, которую вы можете отслеживать с первого взгляда: среднюю полезность на этой неделе против прошлой, по агентам, по окружениям.

![Панель мониторинга качества: полосы средней оценки для каждого параметра оценки рядом с тенденцией во времени](/agenteye/images/dashboard-quality.png)

*Сохранённая панель мониторинга качества отслеживает ключи оценки, которые вы выбрали, так что медленный дрейф становится очевидным задолго до того, как это станет инцидентом.*

Панели мониторинга находятся по адресу **`/<org-slug>/dashboards`** (боковая панель → *analyze* → *dashboards*), общие для всей организации, и каждая карточка суммирует соответствующие сеансы: сколько их, среднее значение каждой выбранной оценки и спарклайн тенденции. «Open in sessions» доставляет вас прямо в предварительно отфильтрованные запуски позади любого числа. Для просмотра требуются `dashboards:read` и `evaluations:read`.

## Подключите оценщик один раз

Оценка опциональна и полностью отключена, пока вы не нацелите Failproof AI Observability на оценщик. Вы разворачиваете один небольшой HTTP-сервис (Observability поставляется с работающей ссылкой, которую можно скопировать), устанавливаете два значения на вашем сервере, и каждый запуск с этого момента оценивается для вас. Полное пошаговое руководство, контракт оценки и SDK находятся в подробном руководстве.

Не уверены, какие параметры стоит оценивать в первую очередь? [Навык агента-оценщика](/ru/agenteye/evaluator-skill) поможет вашему кодирующему агенту разобраться в этом на основе ваших собственных сеансов, а затем построить и развернуть сервис.
