Управляйте всеми возможностями Failproof AI Observability из терминала или скрипта: без необходимости обращаться к панели инструментов. CLI `agenteye` запрашивает ваши данные (сессии, журналы событий, оценки) и администрирует вашу организацию (API ключи, пользователи, параметры, оповещения, инциденты, сохранённые запросы), поэтому используйте его, когда нужно автоматизировать проверку, интегрировать Observability в CI или дать агенту доступ к анализу production. Каждая команда поддерживает флаг `--json`, поэтому она работает одинаково хорошо как для вас в командной строке, так и для кодирующего агента (Claude Code, Cursor), который выполняет команду и парсит результат.

С одним бинарником вы можете:

- **Читать ваши данные**: `sessions`, `events`, `evals`, `errors` (фильтровать по времени, агенту, окружению, оценке).
- **Управлять организацией**: `keys`, `users`, `settings`, `alerts`, `incidents`.
- **Запускать аналитику**: сохранённые SQL-запросы и инструмент для ad-hoc запросов (`query`).
- **Общаться с AI-помощником**: тот же аналитик, доступный только для чтения, с которым вы общаетесь в панели инструментов (`agent`).

> **Примечание:** Это CLI `agenteye`, отличный инструмент от демона-сборщика (`agenteye-collector`). CLI взаимодействует с вашей панелью инструментов; сборщик отправляет события на сервер.

## Быстрый старт

От нуля до первого результата за четыре строки. Укажите CLI на вашу панель инструментов, войдите, подтвердите вашу личность, затем извлеките последний день запусков:

```bash
pipx install agenteye
agenteye --base-url https://agenteye.example.com login --email you@example.com   # код из 6 цифр отправляется по почте
agenteye whoami                                                                  # подтвердить пользователя + активную организацию
agenteye --json sessions --since 24h                                             # одна строка для каждого запуска агента, последние 24 часа
```

Последняя команда выводит JSON-объект с самыми последними сессиями (новые первыми, ограничено 50 по умолчанию). Передайте в `jq` для среза или удалите `--json` для табличного представления в рамках с цветами. Каждая строка содержит статус запуска и, если оценщик его оценил, его метрические оценки (сокращено здесь):

```json
{
  "sessions": [
    {
      "session_id": "run-8f2a",
      "agent_id": "checkout-bot",
      "environment": "prod",
      "status": "error",
      "scores": { "helpfulness": 0.42, "tool_efficiency": 0.55 },
      "event_count": 37,
      "started_at": "2026-07-16T09:14:02Z",
      "last_event_at": "2026-07-16T09:14:48Z"
    }
  ],
  "next_cursor": null
}
```

Остаток этой страницы объясняет каждую часть: [установка](#installation) отдельно, [вход](#authentication), [конфигурация](#configuration), [глобальные соглашения](#global-options--conventions), которые разделяют все команды, и [полный справочник команд](#command-reference).

## Установка

CLI — это публичный пакет PyPI с названием **`agenteye`**. Установите его в изолированную среду, чтобы он всегда имел свои собственные зависимости:

```bash
pipx install agenteye
# или
uv tool install agenteye
```

Требуется Python 3.10+. Установленная команда — **`agenteye`**:

```bash
agenteye --version
agenteye --help
```

> **Примечание:** Пакет Failproof AI Observability Python SDK также использует имя дистрибьютива `agenteye`. Установка CLI с помощью `pipx` или `uv tool` (вместо `pip install` в общий virtualenv) предотвращает их конфликт. Простой `pip install agenteye` подходит только если SDK не установлен в той же среде.

## Аутентификация

CLI аутентифицируется на **панели инструментов** с помощью отправленного по почте одноразового кода:

```bash
agenteye login --email you@example.com
# Код из 6 цифр отправляется вам по почте; вставьте его в приглашение.
```

Токен сессии хранится в `~/.agenteye/cli.json` (читаемый только вами, режим `0600`) и по умолчанию действителен 24 часа. Когда он истечёт, выполните `agenteye login` снова.

```bash
agenteye whoami     # показать текущего пользователя, активную организацию и разрешения
agenteye logout     # отозвать сессию и удалить сохранённый токен
```

`whoami` никогда не выдаёт ошибку при отсутствующей или истёкшей сессии; вместо этого сообщает `logged_in: false`, поэтому скрипт или агент могут безопасно проверить состояние аутентификации (но может выйти с ненулевым кодом, если не установлен базовый URL или панель инструментов недоступна).

**Требования:** ваша электронная почта должна быть разрешена для входа на панель инструментов (попросите администратора Failproof AI Observability), и панель инструментов должна быть доступна по своему базовому URL (см. [Конфигурация](#configuration)). Если вы запросили код и он не пришёл, ваша электронная почта, вероятно, ещё не включена для доступа к панели инструментов.

## Выбор вашей организации (мультитенантность)

Если ваш аккаунт принадлежит нескольким организациям, выберите активную **при входе**; она сохраняется и используется для каждой последующей команды:

```bash
agenteye login --org acme           # аутентифицировать и установить активного тенанта в один шаг
agenteye orgs list                  # организации, к которым вы можете получить доступ (активная отмечена)
agenteye orgs switch globex         # изменить сохранённое значение по умолчанию
agenteye --org globex sessions      # переопределить для одной команды
```

Если вы принадлежите ровно одной организации, она выбирается автоматически и вы можете полностью игнорировать `--org`. Если вы принадлежите нескольким и не выбрали одну, CLI перечислит их и попросит вас повторить команду с `--org <slug>`. Активная организация отправляется на панель инструментов в каждом запросе, и ваши разрешения разрешаются **для каждой организации**; `agenteye whoami` показывает активную организацию, ваши разрешения в ней и все ваши членства.

## Конфигурация

| Параметр | Флаг | Переменная окружения | По умолчанию |
|---|---|---|---|
| Базовый URL панели инструментов | `--base-url` | `AGENTEYE_DASHBOARD_URL` | **обязателен** (нет значения по умолчанию) |
| Активная организация/тенант | `--org` | `AGENTEYE_ORG` | выбирается при входе; сохраняется в `~/.agenteye/cli.json` |
| Токен сессии | `--token` | `AGENTEYE_CLI_TOKEN` | из `~/.agenteye/cli.json` |
| JSON вывод | `--json` | `AGENTEYE_CLI_JSON` | отключено |
| Пропустить проверку TLS | `--insecure` / `--secure` | `AGENTEYE_INSECURE` | отключено (сохраняется при входе) |
| Время ожидания запроса (секунды) | `--timeout` | _(нет)_ | 30 |
| Отключить телеметрию использования | _(нет)_ | `AGENTEYE_ANALYTICS_DISABLED` (или `DO_NOT_TRACK`) | телеметрия в настоящий момент отключена; ничего не отправляется |

Порядок разрешения: **флаг → переменная окружения → файл конфигурации**. Значения по умолчанию нет; вы должны указать CLI на вашу панель инструментов либо для каждой команды (`--base-url https://agenteye.example.com`), либо один раз через переменную окружения (она также сохраняется после вашего первого `login`):

```bash
export AGENTEYE_DASHBOARD_URL=https://agenteye.example.com
```

Директория конфигурации учитывает `AGENTEYE_HOME` (то же соглашение, используемое SDK и сборщиком); если установлено, `cli.json` находится в `$AGENTEYE_HOME/cli.json`.

### Самоподписанный или внутренний TLS

Если ваша панель инструментов подаётся по HTTPS с самоподписанным или внутренним сертификатом (например, имя хоста raw load-balancer'а), проверка TLS отклонит его ошибкой `CERTIFICATE_VERIFY_FAILED`. Передайте `--insecure` для пропуска проверки сертификата:

```bash
agenteye --base-url https://agenteye.internal --insecure login
```

`--insecure` **сохраняется в `cli.json` при входе**, поэтому последующие команды автоматически пропускают проверку; вам не нужно повторять флаг. Передайте `--secure` для единовременного проверенного вызова или для сохранения проверки при следующем входе. CLI выводит предупреждение в stderr перед любой командой, которая обращается к панели инструментов с отключённой проверкой. Пропуск проверки удаляет защиту от атак man-in-the-middle; убедитесь, что вы доверяете сетевому пути к вашей панели инструментов (VPN, приватная подсеть и т.д.) перед тем, как полагаться на это.

## Телеметрия и конфиденциальность

> **Примечание:** Отправленный CLI **в настоящий момент не отправляет никакую телеметрию использования.** Главный переключатель включён, поэтому ничего не передаётся независимо от вашего окружения. Раздел ниже описывает возможность отказа на случай, если телеметрия когда-либо будет включена.

Даже при включении телеметрия будет только **анонимной аналитикой использования**, никогда не данными вашего агента, сессии или событий:

- **Данные агента, сессии или события никогда не покидают вашу инфраструктуру.** Только использование CLI будет сообщено: имя команды и подкоманды (например, `keys create`), **имена** используемых флагов (никогда их значения), статус успеха/выхода и продолжительность, плюс событие за действие для мутаций (например, `api_key_created`, `query_run`) содержащее только статические имена/enum и приблизительные подсчёты. Ваш URL панели инструментов, токен сессии, электронная почта, slug организации, ID ресурсов, SQL, секреты ключей и фильтры запросов **никогда** не будут отправлены. Операторы будут идентифицированы только по опaque internal id, никогда по электронной почте.
- **Откажитесь заранее**, установив `AGENTEYE_ANALYTICS_DISABLED=1` в окружении CLI (CLI также соблюдает кроссплатформенное соглашение `DO_NOT_TRACK=1`). Это вступит в силу в момент включения телеметрии, поэтому среда, ориентированная на конфиденциальность, может остаться отключённой навсегда.
- Если бы телеметрия была включена, CLI отправлял бы непосредственно на PostHog (`https://us.i.posthog.com`); машина с заблокированным этим хостом молча ничего не отправляла бы и CLI не был бы затронут.

## Глобальные параметры и соглашения

Прочитайте это один раз; это применяется ко всем командам.

- **Глобальные параметры идут ДО команды.** `agenteye --json sessions` правильно; `agenteye sessions --json` — это ошибка использования. Глобальные параметры: `--json`, `--base-url`, `--org`, `--token`, `--insecure`/`--secure`, `--timeout`, `--quiet` и `--no-color`.
- **`--json` выводит чистый JSON на stdout и ничего больше.** Строки статуса, предупреждения и ошибки идут на **stderr**, поэтому захват stdout с `--json` остаётся чистым для передачи в `jq` даже если показана строка статуса. Без `--json` вы получаете табличный вид с цветами для человеческих глаз.
- **Открывайте с помощью `--help`.** Каждая команда и подкоманда имеют `--help` (и псевдоним `-h`): `agenteye -h`, `agenteye sessions -h`, `agenteye keys create -h`. Справка верхнего уровня также перечисляет коды выхода и глобальные параметры. Нет глобальной машиночитаемой surface dump'а; используйте по-командной `--help`, плюс специфичные для домена `agenteye query schema` и `agenteye settings schema` для этих двух реестров.
- **Подтверждения автоматически пропускаются для скриптов и агентов.** Команды создания/обновления/удаления выводят приглашение «вы уверены?» в интерактивном терминале, но **автоматически пропускают это приглашение под `--json` или когда stdin не является TTY** (TTY — это интерактивная сессия терминала; pipe или CI runner не являются), поэтому скрипты и агенты никогда не зависают. Передайте `--yes`/`-y` для явного пропуска. Поскольку приглашение не будет выводиться для агента, агент должен подтвердить деструктивные действия с человеком в первую очередь.
- **Пагинация:** результаты — новые первыми и курсорная пагинация (каждая страница возвращает токен, который вы используете для получения следующей). `--limit N` (псевдоним `-n`) ограничивает строки и **по умолчанию 50**; `--all` автоматически пагинирует (в порциях по 200 строк) **до `--limit`**, поэтому голый `--all` всё ещё останавливается на 50. Для полного прохода передайте явный высокий лимит: `--all --limit 1000`. `--page-size N` контролирует порцию за запрос (макс 200); `--cursor <id>` возобновляет с `next_cursor` предыдущей страницы.
- **Фильтры времени:** `--since` принимает относительное окно: `15m`, `1h`, `6h`, `24h`, `7d` или `all` (предустановки панели инструментов). Для более длинного или пользовательского диапазона (скажем, последние 30 дней), используйте `--from`/`--to`: явные временные метки ISO-8601 UTC **с `T` и часовым поясом** (например, `2026-06-01T00:00:00Z`), которые переопределяют `--since`. Значение, разделённое пробелом или без часового пояса, — это ошибка использования.
- **`--fields a,b,c`** (в `events`, `sessions`, `evals`, `errors`) ограничивает вывод этими ключами, как для таблицы, так и для `--json`. Неизвестные имена отклоняются со списком правильных, дешёвый способ открыть имена полей.
- **`--file payload.json`** (или `--file -` для чтения из stdin) предоставляет полное тело JSON-запроса, где ресурс имеет сложную форму (на `alerts create/update`, `settings set` и `users create/update`). Сохранённые запросы SQL используют `--sql @file.sql` вместо этого.
- **Многозначные фильтры** разделены запятыми → сопоставлены как набор (объединение в одном фильтре, И между фильтрами): `--event-type tool_use,tool_result`. Параметры клика не вариативны, поэтому `--add a b` нарушается. Используйте `--add a,b`, повторите флаг (`--add a --add b`) или кавычки (`--add "a b"`).

## Справочник команд

### Эти 5 команд вы будете использовать чаще всего

Большинство ежедневной работы выполняется через несколько команд чтения. Начните здесь, затем обращайтесь к полной поверхности ниже, когда вам это понадобится:

| Команда | Что она делает | Попробуйте |
|---|---|---|
| `sessions` | Одна строка на запуск агента: время, окружение, агент, статус, последняя оценка. | `agenteye --json sessions --since 24h --status error` |
| `events` | Сырой след за шаг внутри запуска (добавьте `--full` для payload'ов). | `agenteye --json events --session-id run-001 --all` |
| `evals` | Результаты и оценки оценки; `--aggregate` свёртывает их. | `agenteye --json evals --aggregate --since 7d --env prod` |
| `errors` | Только errored события; `--aggregate` для подсчётов по типу. | `agenteye --json errors --since 24h --aggregate` |
| `list` | Откройте правильные значения фильтров (агенты, окружения, модели, …). | `agenteye list agents` |

### Всё, что может делать CLI

Полная поверхность следует. CLI имеет **18 команд верхнего уровня**. Все команды чтения принимают `--json` и глобальные параметры выше; выполните `agenteye <command> -h` (или `<command> <subcommand> -h`) для исчерпывающего списка флагов и JSON-формы любой из них.

### Личность: `login` · `logout` · `whoami` · `orgs` · `version` · `help`

```bash
agenteye login --email you@example.com [--org acme]   # отправляется одноразовый код; сохраняет сессию
agenteye logout                                       # очистить сохранённую сессию на этой машине
agenteye whoami                                       # текущий пользователь, активная организация, разрешения
agenteye version                                      # вывести версию CLI (то же, что --version)
agenteye help                                         # справка верхнего уровня (то же, что --help)
```

`orgs` проверяет и переключает активного тенанта:

```bash
agenteye orgs list        # ваши организации + ваша роль в каждой (активная отмечена)
agenteye orgs switch acme # изменить сохранённую активную организацию (опустите slug для выбора из списка на TTY)
agenteye orgs current     # карточка личности для активной организации
agenteye orgs perms       # ваши разрешения в активной организации, сгруппированные по ресурсу
```

### Наблюдение (только чтение): `events` · `sessions` · `evals` · `errors` · `list`

Ни один из них не требует подтверждения. Общие фильтры: `--session-id`, `--agent-id`, `--env` (**не** `--environment`) и диапазон времени (`--since` / `--from` / `--to`).

```bash
# events (псевдоним: сырой след за шаг), новые первыми
agenteye --json events --session-id run-001 --event-type tool_use,tool_result --all --limit 1000
agenteye --json events --since 1h --search timeout --all | jq '.events[].payload'

# sessions: одна строка на запуск агента (время/окружение/агент/сессия/статус; без фильтрации оценки)
agenteye --json sessions --since 24h --status error
agenteye --json sessions --agent-id checkout-bot --env prod --all --limit 1000

# evals: результаты оценок + оценки; --score фильтрует по метрике, --aggregate свёртывает
agenteye --json evals --score helpfulness:0.5..0.8 --score tool_efficiency:..0.3
agenteye --json evals --aggregate --since 7d --env prod        # микс статусов + статистика оценок за ключ

# errors: errored события; --aggregate для подсчётов/сессий/агентов/последнего прохода
agenteye --json errors --since 24h --aggregate
agenteye --json errors --since 24h --error-type timeout --all --limit 1000

# list: откройте правильные значения фильтров перед тем, как фильтровать
agenteye list envs        # также: agents event_types score_filters models hooks tools error_types
```

`--score KEY:MIN..MAX` (на **`evals`**, не `sessions`) повторяемо и AND-комбинировано; любая граница опциональна (`..0.5` означает ≤ 0.5, `0.9..` означает ≥ 0.9). До 20 фильтров оценки за запрос. `evals --scores-full` — это флаг отображения **только для человеческой таблицы**; показывает каждую пару оценок вместо первых нескольких плюс счёт `+N`. Не имеет эффекта под `--json`, который всегда возвращает полный объект оценки. Для чтения **одной сессии от начала до конца**, объедините след события с его оценкой:

```bash
agenteye --json events --session-id run-001 --all --limit 1000 | jq '.events[] | {ts, event_type}'
agenteye --json evals  --session-id run-001                       # её оценки + статус
```

### Управление (привратниковые разрешения): `keys` · `users` · `settings` · `alerts` · `incidents`

**`keys`**: API ключи. Секрет генерируется локально, отправляется на сервер (который хранит только хеш) и **показан один раз** при создании/регенерации; захватите его тогда. С `--json` он появляется только в поле `key`. Ссылается по **названию**.

```bash
agenteye keys list                                   # активные ключи первыми, затем отозванные
agenteye keys show ci-bot
agenteye keys create ci-bot --add events:read.add    # область до того, что вам нужно; выводит секрет ОДИН РАЗ
agenteye keys create ops --permission-set standard --remove queries:run   # затравить предустановку, затем обрезать
agenteye keys update ci-bot --add evaluations:read --yes
agenteye keys regenerate ci-bot --yes                # ротировать секрет (старый прекращает работу)
agenteye keys disable ci-bot --yes                   # отозвать
```

Разрешения работают как `(permission-set ∪ --add) − --remove`. Токены — `slug:action` (например, `events:read`) или `slug:action.action` для развёртывания нескольких на один ресурс (`events:read.add` → `events:read`, `events:add`). Предустановки: `read-only`, `standard`, `admin`. Разрешения, только для человека (`keys:update`), не могут быть предоставлены ключу.

**`users`**: члены организации, на которые ссылаются по **электронной почте** (также принимается ID UUID).

```bash
agenteye users list [--active-only]
agenteye users show dev@corp.com
agenteye users create dev@corp.com --permission-set standard
agenteye users update dev@corp.com --add alerts:write --remove queries:delete   # предсказывает + подтверждает
agenteye users disable dev@corp.com --yes            # имеет защищённые/саморегулируемые охраны
agenteye users enable dev@corp.com
```

**`settings`**: фиксированный реестр (вы читаете и изменяете существующие ключи; вы не можете создавать новые).

```bash
agenteye settings list                               # ключ · значение · тип · обновлено (секреты замаскированы)
agenteye settings schema                             # что принимает каждый ключ (тип · диапазон · описание)
agenteye settings set session_ttl_secs --value 86400 --yes
```

**`alerts`**: определения оповещений, на которые ссылаются по **названию**. `create` принимает позиционный NAME плюс флаги или полное тело JSON через `--file`.

```bash
agenteye alerts list
agenteye alerts show high-errors
agenteye alerts create high-errors --file alert.json          # NAME обязателен (позиционный)
agenteye alerts update high-errors --severity critical --yes
agenteye alerts test high-errors --yes                        # отправить тестовое уведомление
agenteye alerts delete high-errors --yes
```

**`incidents`**: оповещения инцидентов, на которые ссылаются по id (принимаются короткие id). `show` выводит полный журнал активности; прочитайте его перед действием.

```bash
agenteye incidents list --state firing                # также: acknowledged, resolved
agenteye incidents count
agenteye incidents show <id>
agenteye incidents ack <id>
agenteye incidents assign <id> you@corp.com           # назначенный должен быть оператором
agenteye incidents resolve <id> --yes
agenteye incidents open --alert-id <id> --severity critical    # открыть вручную против оповещения
agenteye incidents comment-add <id> "root cause: upstream 5xx"
agenteye incidents comment-list <id> ; agenteye incidents comment-delete <id> <comment-id>
agenteye incidents subscribe <id> ; agenteye incidents unsubscribe <id> ; agenteye incidents subscribers <id>
```

### Аналитика и помощник: `query` · `agent`

**`query`**: сохранённый SQL для вашего хранилища аналитики плюс инструмент ad-hoc runner. Сохранённые запросы на которые ссылаются по **названию**; SQL проверяется на стороне сервера (только SELECT/WITH, время ожидания выполнения, cap строк).

```bash
agenteye query schema [TABLE]                         # структура столбцов аналитических представлений
agenteye query run --sql "select count(*) from analytics.events"
agenteye query run errs --arg prod --limit 100        # запустить сохранённый запрос + позиционный $1
agenteye query list ; agenteye query show errs
agenteye query create errs --sql @errs.sql --description "errored events (24h)"
agenteye query update errs --sql @errs.sql --yes ; agenteye query delete errs --yes
```

**`agent`**: общается с встроенным **AI-помощником** (тот же аналитик, доступный только для чтения, с которым вы можете общаться в панели инструментов). Чаты на которые ссылаются по short chat-id (разрешение префикса).

```bash
agenteye agent health                                 # AI-помощник настроен/достижим
agenteye agent models                                 # модели, которые вы можете передать в --model (по умолчанию отмечена)
agenteye agent ask "which agents errored most in the last day?"    # запускает чат; выводит его short id
agenteye agent ask --chat <short-id> "and which tools did they call?"   # продолжить тот чат
agenteye agent chats ; agenteye agent show <short-id>
agenteye agent rename <short-id> --title "error triage" ; agenteye agent delete <short-id>
```

---

## Коды выхода

| Код | Значение |
|---|---|
| 0 | Успех |
| 1 | Неожиданная ошибка (например, панель инструментов вернула 5xx) |
| 2 | Ошибка использования (недействительные аргументы, неизвестная команда/флаг, конфликт имён) |
| 3 | Не удаётся достичь панель инструментов |
| 4 | Не вошли или сессия истекла; выполните `agenteye login` |
| 5 | Аутентифицировано, но вашему аккаунту не хватает требуемого разрешения (сообщение называет его) |
| 6 | Запрашиваемый ресурс не найден (например, неизвестный session или incident id) |

Это делает CLI безопасным для скриптов: кодирующий агент может разветвиться на `4` для приглашения вас к повторной аутентификации или на `5` для поверхности отсутствующего разрешения. См. [Рецепты CLI для агентов](/ru/agenteye/cli-recipes) для паттернов обработки кодов выхода и форм JSON-вывода.
