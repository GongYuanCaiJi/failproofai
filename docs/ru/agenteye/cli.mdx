Управляйте FailproofAI Observability прямо из терминала или скрипта — без необходимости открывать панель управления. CLI `agenteye` запрашивает ваши данные (сессии, логи событий, оценки) и администрирует организацию (API-ключи, пользователи, настройки, оповещения, инциденты, сохранённые запросы). Используйте его для автоматизации проверок, интеграции Observability в CI или предоставления агенту доступа к production. Каждая команда поддерживает флаг `--json`, поэтому работает одинаково хорошо как для вас в терминале, так и для кодирующего агента (Claude Code, Cursor), который вызывает её и разбирает результат.

С одного бинарного файла вы можете:

- **Читать данные**: `sessions`, `events`, `evals`, `errors` (фильтровать по времени, агенту, окружению, оценке).
- **Управлять организацией**: `keys`, `users`, `settings`, `alerts`, `incidents`.
- **Запускать аналитику**: сохранённый SQL и интерактивный редактор запросов (`query`).
- **Использовать AI-помощник**: того же встроенного аналитика, с которым вы общаетесь в панели (`agent`).

> **Примечание:** Это CLI `agenteye` — отдельный инструмент от демона-сборщика (`agenteye-collector`). CLI общается с вашей панелью; сборщик отправляет события на сервер.

## Быстрый старт

От нуля до первого результата в четыре строки. Укажите CLI адрес вашей панели, войдите, подтвердите личность, а затем получите данные за последний день:

```bash
pipx install agenteye
agenteye --base-url https://agenteye.example.com login --email you@example.com   # 6-значный код придёт по почте
agenteye whoami                                                                  # подтверждение пользователя + активной организации
agenteye --json sessions --since 24h                                             # по одной строке на запуск агента, за последние 24 часа
```

Последняя команда выводит JSON-объект с самыми свежими сессиями (новейшие первыми, по умолчанию максимум 50). Направьте вывод в `jq` для фильтрации или удалите `--json` для красивой таблицы в рамке с цветовым выделением. Каждая строка содержит статус запуска и, если оценщик его обработал, значения метрик (здесь сокращено):

```json
{
  "sessions": [
    {
      "session_id": "run-8f2a",
      "agent_id": "checkout-bot",
      "environment": "prod",
      "status": "error",
      "scores": { "helpfulness": 0.42, "tool_efficiency": 0.55 },
      "event_count": 37,
      "started_at": "2026-07-16T09:14:02Z",
      "last_event_at": "2026-07-16T09:14:48Z"
    }
  ],
  "next_cursor": null
}
```

На остальной части страницы рассказывается о каждом компоненте: [установка](#installation) в изолированной среде, [аутентификация](#authentication), [конфигурация](#configuration), [глобальные параметры](#global-options--conventions), которые общие для всех команд, и [полный справочник команд](#command-reference).

## Установка

CLI — это публичный пакет PyPI с именем **`agenteye`**. Установите его в изолированной среде, чтобы он всегда имел собственные зависимости:

```bash
pipx install agenteye
# или
uv tool install agenteye
```

Требуется Python 3.10+. Установленная команда — **`agenteye`**:

```bash
agenteye --version
agenteye --help
```

> **Примечание:** Python SDK FailproofAI Observability также использует имя дистрибутива `agenteye`. Установка CLI через `pipx` или `uv tool` (вместо `pip install` в общую virtualenv) предотвращает конфликты. Обычный `pip install agenteye` допустим только если SDK не установлен в той же среде.

## Аутентификация

CLI аутентифицируется на **панели** с помощью одноразового кода, отправленного по почте:

```bash
agenteye login --email you@example.com
# На ваш адрес придёт 6-значный код; введите его в приглашение.
```

Токен сессии сохраняется в `~/.agenteye/cli.json` (доступен только вам, режим `0600`) и по умолчанию действителен 24 часа. После истечения снова запустите `agenteye login`.

```bash
agenteye whoami     # показать текущего пользователя, активную организацию и права
agenteye logout     # отозвать сессию и удалить сохранённый токен
```

`whoami` никогда не выдаёт ошибку при отсутствии или истекшей сессии; вместо этого она сообщает `logged_in: false`, поэтому скрипт или агент может безопасно проверить состояние аутентификации (может выйти с ненулевым кодом, если не установлен базовый URL или панель недоступна).

**Требования:** ваш адрес электронной почты должен быть разрешён для входа на панель (попросите администратора FailproofAI Observability), а панель должна быть доступна по её базовому URL (см. [Конфигурация](#configuration)). Если код не приходит, ваш адрес вероятно ещё не включён для доступа к панели.

## Выбор организации (мультитенантность)

Если ваш аккаунт принадлежит нескольким организациям, выберите активную **при входе**; она сохраняется и используется для всех следующих команд:

```bash
agenteye login --org acme           # аутентифицироваться и установить активный тенант за раз
agenteye orgs list                  # организации, к которым у вас есть доступ (активная отмечена)
agenteye orgs switch globex         # изменить сохранённое значение по умолчанию
agenteye --org globex sessions      # переопределить для одной команды
```

Если вы принадлежите только одной организации, она выбирается автоматически и вы можете полностью игнорировать `--org`. Если вы принадлежите нескольким и не выбрали организацию, CLI выведет список и попросит повторить команду с `--org <slug>`. Активная организация отправляется на панель в каждом запросе, и ваши права определяются **для каждой организации отдельно**; `agenteye whoami` показывает активную организацию, ваши права в ней и все ваши членства.

## Конфигурация

| Параметр | Флаг | Переменная окружения | Значение по умолчанию |
|---|---|---|---|
| Базовый URL панели | `--base-url` | `AGENTEYE_DASHBOARD_URL` | **обязательно** (нет значения по умолчанию) |
| Активная организация/тенант | `--org` | `AGENTEYE_ORG` | выбирается при входе; сохраняется в `~/.agenteye/cli.json` |
| Токен сессии | `--token` | `AGENTEYE_CLI_TOKEN` | из `~/.agenteye/cli.json` |
| JSON-вывод | `--json` | `AGENTEYE_CLI_JSON` | отключено |
| Пропустить проверку TLS | `--insecure` / `--secure` | `AGENTEYE_INSECURE` | отключено (сохраняется при входе) |
| Таймаут запроса (секунды) | `--timeout` | _(нет)_ | 30 |
| Отключить телеметрию использования | _(нет)_ | `AGENTEYE_ANALYTICS_DISABLED` (или `DO_NOT_TRACK`) | телеметрия отключена; ничего не отправляется |

Порядок разрешения: **флаг → переменная окружения → файл конфигурации**. Нет значения по умолчанию; вы должны указать CLI адрес своей панели либо в каждой команде (`--base-url https://agenteye.example.com`), либо один раз через окружение (она также сохраняется после первого `login`):

```bash
export AGENTEYE_DASHBOARD_URL=https://agenteye.example.com
```

Директория конфигурации учитывает `AGENTEYE_HOME` (то же соглашение используется SDK и сборщиком); если установлена, `cli.json` находится в `$AGENTEYE_HOME/cli.json`.

### Самоподписанные или внутренние TLS-сертификаты

Если ваша панель обслуживается по HTTPS с самоподписанным или внутренним сертификатом (например, имя хоста обычного балансировщика нагрузки), проверка TLS отклонит его с ошибкой `CERTIFICATE_VERIFY_FAILED`. Передайте `--insecure` чтобы пропустить проверку сертификата:

```bash
agenteye --base-url https://agenteye.internal --insecure login
```

`--insecure` **сохраняется в `cli.json` при входе**, поэтому последующие команды пропускают проверку автоматически; вам не нужно повторять флаг. Передайте `--secure` для разовой проверенной команды или чтобы сохранить проверку обратно при следующем входе. CLI выводит предупреждение в stderr перед любой командой, которая обращается к панели с отключённой проверкой. Пропуск проверки удаляет защиту от атак «человек посередине»; убедитесь, что вы доверяете пути сети к вашей панели (VPN, приватная подсеть и т. д.) перед использованием.

## Телеметрия и конфиденциальность

> **Примечание:** Поставляемый CLI **не отправляет сегодня никакую телеметрию использования.** Главный выключатель включён, поэтому ничего не передаётся независимо от вашего окружения. Раздел ниже описывает возможность отключения на случай, если телеметрия когда-то будет включена.

Даже если включена, телеметрия была бы **только анонимной аналитикой использования**, никогда не данными вашего агента, сессии или событий:

- **Данные агента, сессии или события никогда не покидают вашу инфраструктуру.** Сообщалось бы только об использовании CLI: имя команды и подкоманды (например `keys create`), **имена** флагов, которые вы использовали (никогда их значения), статус успеха/выхода и длительность, плюс событие на действие для мутаций (например `api_key_created`, `query_run`) с только статическими именами/перечислениями и грубыми подсчётами. Ваш URL панели, токен сессии, адрес электронной почты, slug организации, id ресурсов, SQL, секреты ключей и фильтры запросов **никогда** не будут отправлены. Операторы идентифицировались бы только по непрозрачному внутреннему id, никогда по адресу электронной почты.
- **Отключитесь заранее**, установив `AGENTEYE_ANALYTICS_DISABLED=1` в окружении CLI (CLI также поддерживает кроссплатформенное соглашение `DO_NOT_TRACK=1`). Это вступит в силу в момент включения телеметрии, поэтому конфиденциальное окружение может остаться отключённым навсегда.
- Если бы телеметрия была включена, CLI отправлял бы напрямую в PostHog (`https://us.i.posthog.com`); машина с заблокированным этим хостом молча не отправляла бы ничего и CLI был бы не затронут.

## Глобальные параметры и соглашения

Прочитайте это один раз; это применимо ко всем командам.

- **Глобальные параметры идут ДО команды.** `agenteye --json sessions` правильно; `agenteye sessions --json` — ошибка использования. Глобальные — это `--json`, `--base-url`, `--org`, `--token`, `--insecure`/`--secure`, `--timeout`, `--quiet` и `--no-color`.
- **`--json` выводит чистый JSON в stdout и ничего больше.** Строки статуса для людей, предупреждения и ошибки идут в **stderr**, поэтому захват `--json` stdout остаётся чистым для направления в `jq` даже если показана строка статуса. Без `--json` вы получите красивую окрашенную таблицу для человеческих глаз.
- **Открывайте через `--help`.** Каждая команда и подкоманда имеет `--help` (и псевдоним `-h`): `agenteye -h`, `agenteye sessions -h`, `agenteye keys create -h`. Справка верхнего уровня также выводит коды выхода и глобальные параметры. Нет глобального машиночитаемого поверхностного дампа; используйте `--help` для каждой команды плюс域-специфичные `agenteye query schema` и `agenteye settings schema` для этих двух реестров.
- **Подтверждения автоматически пропускаются для скриптов и агентов.** Команды создания/обновления/удаления выводят подсказку «вы уверены?» в интерактивном терминале, но **автоматически пропускают эту подсказку при `--json` или когда stdin не является TTY** (TTY — это интерактивный сеанс терминала; pipe или CI-runner — нет), поэтому скрипты и агенты никогда не зависают. Передайте `--yes`/`-y` чтобы явно пропустить. Поскольку подсказка не будет выводиться для агента, агент должен подтвердить деструктивные действия с человеком сначала.
- **Пагинация:** результаты отсортированы новейшие-первыми и используют курсор-пагинацию (каждая страница возвращает токен для получения следующей). `--limit N` (псевдоним `-n`) ограничивает строки и **по умолчанию 50**; `--all` автоматически пагинирует (по 200 строк на шаг) **вплоть до `--limit`**, поэтому простой `--all` всё ещё остановится на 50. Для полного сканирования передайте высокий явный лимит: `--all --limit 1000`. `--page-size N` управляет размером шага за запрос (максимум 200); `--cursor <id>` возобновляет со слова `next_cursor` предыдущей страницы.
- **Фильтры времени:** `--since` принимает относительное окно: `15m`, `1h`, `6h`, `24h`, `7d` или `all` (предустановки панели). Для более длинного или пользовательского диапазона (скажем последние 30 дней) используйте `--from`/`--to`: явные ISO-8601 UTC-временные метки **с `T` и часовым поясом** (например `2026-06-01T00:00:00Z`) которые переопределяют `--since`. Значение с пробелом или без часового пояса — ошибка использования.
- **`--fields a,b,c`** (на `events`, `sessions`, `evals`, `errors`) ограничивает вывод этими ключами как для таблицы, так и `--json`. Неизвестные имена отклоняются с корректным списком, дешёвый способ обнаружить имена полей.
- **`--file payload.json`** (или `--file -` чтобы читать из stdin) предоставляет полное тело JSON-запроса где ресурс имеет сложную форму (на `alerts create/update`, `settings set` и `users create/update`). Сохранённый SQL запрос использует `--sql @file.sql` вместо этого.
- **Многозначные фильтры** разделены запятыми → совпадают как множество (объединение в одном фильтре, AND через фильтры): `--event-type tool_use,tool_result`. Опции клика не вариадичны, поэтому `--add a b` нарушается. Используйте `--add a,b`, повторите флаг (`--add a --add b`) или используйте кавычки (`--add "a b"`).

## Справочник команд

### Вы будете использовать эти 5 команд чаще всего

Большая часть ежедневной работы проходит через несколько команд чтения. Начните отсюда, затем используйте полную поверхность ниже когда понадобится:

| Команда | Что она делает | Попробуйте |
|---|---|---|
| `sessions` | По одной строке на запуск агента: время, окружение, агент, статус, последняя оценка. | `agenteye --json sessions --since 24h --status error` |
| `events` | Сырой per-step след внутри запуска (добавьте `--full` для полезных нагрузок). | `agenteye --json events --session-id run-001 --all` |
| `evals` | Результаты оценки и оценки; `--aggregate` их суммирует. | `agenteye --json evals --aggregate --since 7d --env prod` |
| `errors` | Только ошибочные события; `--aggregate` для подсчётов по типам. | `agenteye --json errors --since 24h --aggregate` |
| `list` | Обнаружьте допустимые значения фильтров (агенты, окружения, модели, …). | `agenteye list agents` |

### Всё что может делать CLI

Полная поверхность ниже. CLI имеет **18 команд верхнего уровня**. Все команды чтения принимают `--json` и глобальные параметры выше; запустите `agenteye <command> -h` (или `<command> <subcommand> -h`) для исчерпывающего списка флагов и JSON-формы любой из них.

### Идентификация: `login` · `logout` · `whoami` · `orgs` · `version` · `help`

```bash
agenteye login --email you@example.com [--org acme]   # одноразовый код отправлен по почте; сохраняет сессию
agenteye logout                                       # очистить сохранённую сессию на этой машине
agenteye whoami                                       # текущий пользователь, активная организация, права
agenteye version                                      # вывести версию CLI (то же что и --version)
agenteye help                                         # справка верхнего уровня (то же что и --help)
```

`orgs` проверяет и переключает активного тенанта:

```bash
agenteye orgs list        # ваши организации + ваша роль в каждой (активная отмечена)
agenteye orgs switch acme # изменить сохранённую активную организацию (пропустите slug чтобы выбрать из списка на TTY)
agenteye orgs current     # карточка активной организации
agenteye orgs perms       # ваши права в активной организации, сгруппированные по ресурсам
```

### Наблюдение (только чтение): `events` · `sessions` · `evals` · `errors` · `list`

Ни одна из них не требует подтверждения. Общие фильтры: `--session-id`, `--agent-id`, `--env` (**не** `--environment`) и диапазон времени (`--since` / `--from` / `--to`).

```bash
# events (псевдоним: сырой per-step след), новейшие первыми
agenteye --json events --session-id run-001 --event-type tool_use,tool_result --all --limit 1000
agenteye --json events --since 1h --search timeout --all | jq '.events[].payload'

# sessions: одна строка на запуск агента (время/окружение/агент/сессия/статус; без фильтрации оценок)
agenteye --json sessions --since 24h --status error
agenteye --json sessions --agent-id checkout-bot --env prod --all --limit 1000

# evals: результаты оценки + оценки; --score фильтрует по метрике, --aggregate суммирует
agenteye --json evals --score helpfulness:0.5..0.8 --score tool_efficiency:..0.3
agenteye --json evals --aggregate --since 7d --env prod        # микс статусов + статистика оценок на ключ

# errors: ошибочные события; --aggregate для подсчётов/сессий/агентов/последний-раз-замечено
agenteye --json errors --since 24h --aggregate
agenteye --json errors --since 24h --error-type timeout --all --limit 1000

# list: обнаружьте допустимые значения фильтров перед фильтрацией
agenteye list envs        # также: agents event_types score_filters models hooks tools error_types
```

`--score KEY:MIN..MAX` (на **`evals`**, не `sessions`) повторяемо и AND-комбинируется; любое граничное значение опционально (`..0.5` означает ≤ 0.5, `0.9..` означает ≥ 0.9). До 20 фильтров оценок на запрос. `evals --scores-full` — флаг отображения только для **человеческой таблицы**; он показывает каждую пару оценок вместо первых нескольких плюс счётчик `+N`. Он не влияет на `--json`, который всегда возвращает полный объект оценок. Чтобы прочитать **одну сессию конец-в-конец**, комбинируйте trail событий с её оценкой:

```bash
agenteye --json events --session-id run-001 --all --limit 1000 | jq '.events[] | {ts, event_type}'
agenteye --json evals  --session-id run-001                       # её оценки + статус
```

### Управление (с доступом по правам): `keys` · `users` · `settings` · `alerts` · `incidents`

**`keys`**: API-ключи. Секрет генерируется локально, отправляется на сервер (который хранит только хеш) и **показывается один раз** при создании/переинициализации; запишите его тогда. С `--json` он появляется только в поле `key`. На них ссылаются по **имени**.

```bash
agenteye keys list                                   # активные ключи первыми, затем отозванные
agenteye keys show ci-bot
agenteye keys create ci-bot --add events:read.add    # область видимости ограничьте тем что нужно; выводит секрет ОДИН РАЗ
agenteye keys create ops --permission-set standard --remove queries:run   # инициализируйте предустановку, затем сократьте
agenteye keys update ci-bot --add evaluations:read --yes
agenteye keys regenerate ci-bot --yes                # вращение секрета (старый перестаёт работать)
agenteye keys disable ci-bot --yes                   # отозвать
```

Права работают как `(permission-set ∪ --add) − --remove`. Токены — это `slug:action` (например `events:read`) или `slug:action.action` чтобы развернуть несколько на одном ресурсе (`events:read.add` → `events:read`, `events:add`). Предустановки: `read-only`, `standard`, `admin`. Права только для людей (`keys:update`) не могут быть выданы ключу.

**`users`**: члены организации, на них ссылаются по **адресу электронной почты** (также принимается UUID id).

```bash
agenteye users list [--active-only]
agenteye users show dev@corp.com
agenteye users create dev@corp.com --permission-set standard
agenteye users update dev@corp.com --add alerts:write --remove queries:delete   # предсказывает + подтверждает
agenteye users disable dev@corp.com --yes            # имеет защиту/самозащиту
agenteye users enable dev@corp.com
```

**`settings`**: фиксированный реестр (вы читаете и изменяете существующие ключи; вы не можете создавать новые).

```bash
agenteye settings list                               # ключ · значение · тип · обновлено (секреты скрыты)
agenteye settings schema                             # что каждый ключ принимает (тип · диапазон · описание)
agenteye settings set session_ttl_secs --value 86400 --yes
```

**`alerts`**: определения оповещений, на них ссылаются по **имени**. `create` принимает позиционный NAME плюс флаги или полное тело JSON через `--file`.

```bash
agenteye alerts list
agenteye alerts show high-errors
agenteye alerts create high-errors --file alert.json          # NAME обязателен (позиционный)
agenteye alerts update high-errors --severity critical --yes
agenteye alerts test high-errors --yes                        # запустить тестовое уведомление
agenteye alerts delete high-errors --yes
```

**`incidents`**: инциденты оповещений, на них ссылаются по id (принимаются короткие id). `show` выводит полный лог активности; прочитайте его перед действием.

```bash
agenteye incidents list --state firing                # также: acknowledged, resolved
agenteye incidents count
agenteye incidents show <id>
agenteye incidents ack <id>
agenteye incidents assign <id> you@corp.com           # присвоенный должен быть оператором
agenteye incidents resolve <id> --yes
agenteye incidents open --alert-id <id> --severity critical    # открыть один вручную против оповещения
agenteye incidents comment-add <id> "root cause: upstream 5xx"
agenteye incidents comment-list <id> ; agenteye incidents comment-delete <id> <comment-id>
agenteye incidents subscribe <id> ; agenteye incidents unsubscribe <id> ; agenteye incidents subscribers <id>
```

### Аналитика и помощник: `query` · `agent`

**`query`**: сохранённый SQL против вашего хранилища аналитики плюс интерактивный редактор. Сохранённые запросы на них ссылаются по **имени**; SQL валидируется серверной стороной (только SELECT/WITH, таймаут выполнения, лимит строк).

```bash
agenteye query schema [TABLE]                         # раскладка столбцов views аналитики
agenteye query run --sql "select count(*) from analytics.events"
agenteye query run errs --arg prod --limit 100        # запустить сохранённый запрос + позиционный $1
agenteye query list ; agenteye query show errs
agenteye query create errs --sql @errs.sql --description "errored events (24h)"
agenteye query update errs --sql @errs.sql --yes ; agenteye query delete errs --yes
```

**`agent`**: общается со встроенным **AI-помощником** (тем же аналитиком, который доступен только для чтения, с которым вы можете общаться на панели). Чаты на них ссылаются по короткому chat-id (разрешение префиксом).

```bash
agenteye agent health                                 # настроен ли AI-помощник/доступен
agenteye agent models                                 # модели, которые вы можете передать `--model` (по умолчанию отмечена)
agenteye agent ask "which agents errored most in the last day?"    # начинает чат; выводит его короткий id
agenteye agent ask --chat <short-id> "and which tools did they call?"   # продолжить этот чат
agenteye agent chats ; agenteye agent show <short-id>
agenteye agent rename <short-id> --title "error triage" ; agenteye agent delete <short-id>
```

---

## Коды выхода

| Код | Значение |
|---|---|
| 0 | Успех |
| 1 | Неожиданная ошибка (например панель вернула 5xx) |
| 2 | Ошибка использования (неверные аргументы, неизвестная команда/флаг, конфликт имён) |
| 3 | Невозможно достичь панель |
| 4 | Не вошли в систему или сессия истекла; запустите `agenteye login` |
| 5 | Аутентифицирован, но ваш аккаунт не имеет требуемого права (сообщение его называет) |
| 6 | Требуемый ресурс не найден (например неизвестный session или incident id) |

Это делает CLI безопасным для скриптов: кодирующий агент может ветвиться на `4` чтобы попросить вас повторно аутентифицироваться, или на `5` чтобы вывести отсутствующее право.
