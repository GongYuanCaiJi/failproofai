## Требования

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (опционально - требуется только для сборки из исходного кода)

## Установка

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

## Быстрый старт

<Steps>
  <Step title="Включите политики">
    Политики — это правила, которые выполняются до и после каждого вызова инструмента агентом. Они перехватывают деструктивные команды, утечки секретов и другие режимы отказа до того, как они причинят ущерб.

    ```bash
    failproofai policies --install
    ```

    Это записывает записи хука в установленные CLI агентов (Claude Code в `~/.claude/settings.json`, OpenAI Codex в `~/.codex/hooks.json`, GitHub Copilot CLI в `~/.copilot/hooks/failproofai.json`, Cursor Agent в `~/.cursor/hooks.json`, OpenCode в созданный плагин `~/.config/opencode/plugins/failproofai.mjs` плюс запись регистрации в массиве `plugin` файла `~/.config/opencode/opencode.json`, Pi в `~/.pi/agent/settings.json` или Gemini CLI в `~/.gemini/settings.json`). Когда присутствует более одного, вам будет предложено выбрать; передайте `--cli claude codex copilot cursor opencode pi gemini` (любое подмножество), чтобы пропустить запрос.

    GitHub Copilot CLI, Cursor Agent, OpenCode, Pi и Gemini CLI находятся в статусе **бета** — установите с помощью `--cli copilot`, `--cli cursor`, `--cli opencode`, `--cli pi` или `--cli gemini`.

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install --cli copilot --scope project
    failproofai policies --install --cli cursor --scope project
    failproofai policies --install --cli opencode --scope project
    failproofai policies --install --cli pi --scope project
    failproofai policies --install --cli gemini --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="Проверьте">
    ```bash
    failproofai policies
    ```

    Показывает каждую политику, включена ли она, и любые настроенные параметры.
  </Step>
  <Step title="Запустите панель управления">
    ```bash
    failproofai
    ```

    Открывает локальную панель управления по адресу `http://localhost:8020`, где вы можете просматривать сессии, проверять вызовы инструментов и управлять политиками.
  </Step>
  <Step title="Запустите вашего агента">
    Запустите Claude Code как обычно. Если агент попытается сделать что-то рискованное, failproofai перехватит это автоматически. Оставьте его работать без присмотра и просмотрите результаты в панели управления.
  </Step>
</Steps>

## Как работают политики

Каждый раз, когда агент запускает инструмент, Claude Code вызывает failproofai как подпроцесс:

```text
Claude Code  →  failproofai --hook PreToolUse  →  читает JSON из stdin
                                                 оценивает политики
                                                 записывает решение в stdout
```

Каждая политика возвращает одно из трёх решений:

- **allow** - агент продолжает работу нормально
- **deny** - действие блокируется, агенту сообщается причина
- **instruct** - дополнительный контекст добавляется в приглашение агента

<Note>
Политики выполняются в вашем локальном процессе. Ничего не отправляется на удалённый сервис.
</Note>

## Установка командных политик с использованием политик на основе соглашений

Самый быстрый способ установить стандарты качества в вашей команде — использовать соглашение `.failproofai/policies/`. Поместите файлы политик в этот каталог, и они загружаются автоматически — без флагов, без изменений конфигурации, без команд установки.

<Steps>
  <Step title="Создайте каталог политик">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="Добавьте файлы политик">
    Скопируйте примеры для начала работы или напишите свои:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    Или создайте новый:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Запустите тесты перед коммитом.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="Закоммитьте в git">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```

    Каждый член команды, у которого установлен failproofai, автоматически получит эти политики. Никакой отдельной настройки для каждого разработчика не требуется.
  </Step>
</Steps>

<Tip>
Закоммитьте `.failproofai/policies/` в ваш репозиторий, чтобы вся команда придерживалась одних и тех же стандартов. По мере того как ваша команда обнаруживает новые режимы отказа, добавляйте политики и отправляйте их — все получат обновление при следующем `git pull`. Со временем эти политики становятся живым стандартом качества, который постоянно улучшается.
</Tip>

## Хранение данных

Вся конфигурация и логи остаются на вашем компьютере:

| Путь | Что хранится |
|------|----------------|
| `~/.failproofai/policies-config.json` | Глобальная конфигурация политик |
| `~/.failproofai/hook-activity.jsonl` | История выполнения хуков |
| `~/.failproofai/hook.log` | Журнал отладки для ошибок пользовательских хуков |
| `.failproofai/policies-config.json` | Конфигурация для каждого проекта (коммитится) |
| `.failproofai/policies-config.local.json` | Личные переопределения (игнорируется в git) |

---

## Удаление

```bash
failproofai policies --uninstall
```

Удаляет записи хука из `~/.claude/settings.json`. Файлы конфигурации в `~/.failproofai/` сохраняются.
