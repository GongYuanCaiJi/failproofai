Hook e politiche per la **gestione dei fallimenti dell'AI**, il **recupero dagli errori** e l'**affidabilità degli LLM**. Mantieni i tuoi agenti AI affidabili e in esecuzione autonoma su **Claude Code**, **OpenAI Codex**, **GitHub Copilot**, **Cursor Agent**, **OpenCode**, **Pi**, **Hermes**, **OpenClaw**, **Factory Droid**, **Devin CLI**, **Antigravity CLI** e l'**Agents SDK**.

Gli agenti AI falliscono in modi prevedibili. Eseguono comandi distruttivi, fanno trapelare segreti, si allontanano dal compito assegnato, rimangono bloccati in loop o eseguono push direttamente su main. Se lasciati incustoditi, i piccoli fallimenti si trasformano in interruzioni, fughe di credenziali e lavoro perso.

FailproofAI risolve questo con le **politiche**. Queste regole si integrano in ogni chiamata di tool dell'agente per **rilevare i fallimenti**, **mitigarli** (bloccare, istruire, sanificare) e **avvisarti** quando qualcosa necessita attenzione. Un dashboard locale ti permette di revisionare ogni chiamata di tool, fallimento dell'agente e azione di recupero in seguito.

Le trascrizioni e la valutazione delle politiche rimangono sul tuo computer. I dati vengono inviati solo quando usi esplicitamente una funzione online, come i promemoria di audit autenticati o gli inviti.

## Avvio rapido

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

```bash
failproofai policies --install   # abilita le politiche (o salta — `failproofai` ti offrirà di configurarle al primo avvio)
failproofai                      # avvia il dashboard
```

Consulta la guida [Inizia](/it/getting-started) per la procedura dettagliata completa.
