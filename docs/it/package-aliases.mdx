## Package ufficiale

Il package npm canonico è **`failproofai`**:

```bash
npm install -g failproofai
# oppure
bun add -g failproofai
```

---

## Perché possediamo i nomi degli alias

Il typosquatting è un comune attacco alla catena di approvvigionamento in cui un attore malevolo registra un nome di package a una tastiera di distanza da un package popolare. Gli utenti ignari che digitano male il comando di installazione finiscono per eseguire codice controllato dall'attaccante con accesso completo al sistema - esattamente il tipo di minaccia contro cui Failproof AI è progettato per difendersi.

Per eliminare questa superficie di attacco, **possediamo preventivamente tutte le comuni varianti di ortografia e formattazione** di `failproofai` su npm. Nessuno di questi nomi può essere registrato da una terza parte. Ognuno è un thin proxy che installa e delega al vero package `failproofai`.

## Alias registrati

**Varianti di formattazione** - diversi modi di scrivere "failproof ai":

| Package | Stato |
|---------|-------|
| `failproof` | ✅ Pubblicato |
| `failproof-ai` | ⏳ In attesa di supporto npm |
| `fail-proof-ai` | ⏳ In attesa di supporto npm |
| `failproof_ai` | ⏳ In attesa di supporto npm |
| `fail_proof_ai` | ⏳ In attesa di supporto npm |
| `fail-proofai` | ⏳ In attesa di supporto npm |

**Typo `failprof*`** - manca una `o` da "proof":

| Package | Stato |
|---------|-------|
| `failprof` | ✅ Pubblicato |
| `failprof-ai` | ✅ Pubblicato |
| `failprofai` | ⏳ In attesa di supporto npm |
| `fail-prof-ai` | ⏳ In attesa di supporto npm |
| `failprof_ai` | ⏳ In attesa di supporto npm |

**Typo `faliproof*`** - `a` e `i` invertiti:

| Package | Stato |
|---------|-------|
| `faliproof` | ✅ Pubblicato |
| `faliproof-ai` | ✅ Pubblicato |
| `faliproofai` | ⏳ In attesa di supporto npm |

> **Perché in sospeso?** La politica di prevenzione dello spam di npm blocca i nomi che si normalizzano nella stessa stringa di un package esistente dopo aver rimosso la punteggiatura ed eseguito controlli di somiglianza. Abbiamo contattato il supporto npm per riservare questi nomi per scopi anti-squatting. Saranno attivati una volta approvati.

Puoi verificare che qualsiasi alias pubblicato sia di nostra proprietà:

```bash
npm info failproof
# Cerca: "ExosphereHost Inc." nel campo maintainers
```

## Come funzionano gli alias

Ogni package alias:

1. Elenca `failproofai` come dipendenza - così il vero package (inclusa la configurazione del suo hook `postinstall`) viene eseguito all'installazione
2. Espone un binary che corrisponde al suo stesso nome (ad es. `failprof-ai`) che proxya tutti gli argomenti al binary `failproofai`

Il proxy è uno script Node di due righe; non c'è logica, nessuna chiamata di rete e nessuna raccolta di dati oltre a quella che fa lo stesso `failproofai`.
