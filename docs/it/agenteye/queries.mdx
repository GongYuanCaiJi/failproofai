Fai qualsiasi domanda sui dati del tuo agent e ottieni una risposta in pochi secondi. FailproofAI Observability ti mette a disposizione una libreria di query salvate e pronte all'uso sui tuoi eventi e valutazioni, così puoi partire da un esempio funzionante invece che da un editor SQL vuoto.

![La libreria delle query salvate: una griglia di query riutilizzabili, sia preset integrati che personalizzati](/agenteye/images/queries.png)

*La tua libreria di query salvate in `/<org-slug>/queries`: i preset integrati affiancati dalle query salvate dal tuo team.*

## Inizia da un preset, non da una pagina vuota

Non devi ricordare i nomi delle tabelle o scrivere SQL da zero. La libreria si apre con i preset integrati per le domande che i team pongono più frequentemente, proprio accanto alle query che il tuo team ha salvato e nominato. Scegline una che si avvicina a quello che cerchi e avrai già risolto la maggior parte del problema.

Ogni query salvata è a livello organizzativo e condivisa, quindi quelle utili scritte dai tuoi colleghi diventano tue. Nomina una query, assegnale una descrizione una volta sola, e chiunque nella tua organizzazione potrà trovarla, eseguirla o fissarne i risultati su un dashboard in seguito.

La trovi in `/<org-slug>/queries`.

## Modificala ed eseguila nel composer SQL

Apri qualsiasi query e apparirà nel composer SQL, dove potrai regolarla e vedere la risposta istantaneamente: niente esportazioni, niente andirivieni, niente attese.

![Il composer di query SQL che esegue una query salvata, con una barra laterale dello schema e una griglia di risultati live](/agenteye/images/query-lab.png)

*Il composer SQL: la tua query a sinistra, una barra laterale dello schema così non dovrai mai indovinare un nome di colonna, e una griglia di risultati live sottostante.*

- **Una barra laterale dello schema** illustra le tabelle di analitici e le loro colonne, così puoi costruire una query senza cercare i nomi dei campi.
- **Una griglia di risultati live** restituisce le righe nel momento in cui le esegui, così iteri in pochi secondi invece di indovinare e ricominciare.
- **Progettato in sola lettura.** Le query vengono eseguite sul tuo archivio eventi e validate dal server: solo i comandi `SELECT` e `WITH` sono consentiti, con un timeout per l'istruzione e un limite di righe. Una query esplorativa non potrà mai modificare i tuoi dati, e una query anomala sarà fermata automaticamente.

Soddisfatto del risultato? Salvalo di nuovo nella libreria così tutto il team ne beneficia, oppure fissa il suo output su un dashboard come tile lineare, a barre, ad area o a torta.

## Eseguile dal terminale, o lascia che l'assistente le scriva

Le stesse query salvate ti seguono ovunque lavori:

- **Dal terminale.** La CLI `agenteye` elenca, esegue e salva le stesse identiche query, così puoi inserire un risultato in uno script, integrarlo in CI, o passarlo a un coding agent.

```bash
agenteye query list                 # le stesse query salvate, dal tuo terminale
agenteye query run errs --arg prod  # esegui una e stampa le righe (aggiungi --json per usarlo in pipe)
```

  Consulta [CLI e agents](/it/agenteye/cli-and-agents) per l'elenco completo dei comandi.

- **Dall'assistente AI.** Non sei sicuro di come formulare l'SQL? Chiedi all'[assistente AI](/it/agenteye/assistant) nel dashboard in inglese semplice e ti redige la query e la salva nella tua libreria.

L'esecuzione di una query salvata è gestita dal permesso `queries:run`, mantenuto separato dai permessi per creare o eliminare query, così puoi concedere accesso in lettura senza permettere a tutti di riscrivere la libreria.
