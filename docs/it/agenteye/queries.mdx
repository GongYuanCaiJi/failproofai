Poni qualsiasi domanda sui dati del tuo agente e ricevi una risposta in secondi. Failproof AI Observability ti fornisce una libreria di query salvate e pronte all'uso sui tuoi eventi e valutazioni, così puoi iniziare da un esempio funzionante invece da un editor SQL vuoto.

![La libreria di query salvate: una griglia di query riutilizzabili, sia preset integrati che personalizzati](/agenteye/images/queries.png)

*La libreria di query salvate su `/<org-slug>/queries`: preset integrati affiancati alle query salvate dal tuo team.*

## Inizia da un preset, non da una pagina vuota

Non devi ricordare i nomi delle tabelle o scrivere SQL da zero. La libreria si apre con preset integrati per le domande che i team fanno più frequentemente, insieme alle query salvate e nominate dal tuo team. Scegline una simile a quello che cerchi e avrai già la maggior parte della risposta.

Ogni query salvata è nell'ambito dell'organizzazione e condivisa, quindi le query utili scritte dai tuoi colleghi diventano anche tue. Una volta nominata una query e aggiunta una descrizione, chiunque nella tua organizzazione può trovarla, eseguirla o fissare i suoi risultati su un dashboard in seguito.

Trovala su `/<org-slug>/queries`.

## Adattala ed eseguila nel compositore SQL

Apri qualsiasi query e si carica nel compositore SQL, dove puoi modificarla e vedere la risposta immediatamente: nessuna esportazione, nessun andirivieni, nessuna attesa di qualcun altro.

![Il compositore di query SQL che esegue una query salvata, con una barra laterale dello schema e una griglia di risultati in tempo reale](/agenteye/images/query-lab.png)

*Il compositore SQL: la tua query a sinistra, una barra laterale dello schema in modo che tu non debba indovinare un nome di colonna, e una griglia di risultati in tempo reale sottostante.*

- **Una barra laterale dello schema** espone le tabelle di analisi e le loro colonne, così puoi strutturare una query senza cercare i nomi dei campi.
- **Una griglia di risultati in tempo reale** restituisce le righe non appena le esegui, così iteri in secondi anziché indovinare e reindivinare.
- **Progettata come sola lettura.** Le query vengono eseguite sul tuo archivio eventi e validate dal server: sono consentiti solo gli statement `SELECT` e `WITH`, con un timeout e un limite di righe. Una query esplorativa non può mai modificare i tuoi dati, e una che va fuori controllo viene fermata per te.

Soddisfatto del risultato? Salvalo di nuovo nella libreria in modo che il team intero lo erediti, oppure fissa il suo output su un dashboard come una tile lineare, a barre, ad area o a torta.

## Eseguile dal terminale, o lascia che l'assistente le scriva

Le stesse query salvate ti seguono dovunque tu lavori:

- **Dal terminale.** La CLI `agenteye` elenca, esegue e salva le stesse identiche query, così puoi inserire un risultato in uno script, collegarlo a una CI o passarlo a un agente di codifica.

```bash
agenteye query list                 # le stesse query salvate, dal tuo terminale
agenteye query run errs --arg prod  # esegui una e stampa le righe (aggiungi --json per piparla)
```

  Vedi [CLI e agenti](/it/agenteye/cli-and-agents) per l'insieme completo di comandi.

- **Dall'assistente AI.** Non sei sicuro di come formulare l'SQL? Chiedi all'[assistente AI](/it/agenteye/assistant) nel dashboard in inglese semplice e lui elaborerà la query e la salverà nella tua libreria per te.

L'esecuzione di una query salvata è controllata dal permesso `queries:run`, mantenuto separato dai permessi per creare o eliminare query, così puoi concedere l'accesso in lettura senza consentire a tutti di riscrivere la libreria.
