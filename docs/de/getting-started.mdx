## Voraussetzungen

- **Node.js** >= 20.9.0
- **Bun** >= 1.3.0 (optional – nur für das Erstellen aus dem Quellcode erforderlich)

---

## Installation

<CodeGroup>

```bash npm
npm install -g failproofai
```

```bash bun
bun add -g failproofai
```

</CodeGroup>

## Schnellstart

<Steps>
  <Step title="Richtlinien aktivieren">
    Richtlinien sind Regeln, die vor und nach jedem Werkzeugaufruf eines Agenten ausgeführt werden. Sie erkennen destruktive Befehle, das Durchsickern von Geheimnissen und andere Fehlermuster, bevor diese Schaden anrichten können.

    ```bash
    failproofai policies --install
    ```

    Damit werden Hook-Einträge in Ihren installierten Agenten-CLIs hinterlegt (Claude Codes `~/.claude/settings.json`, OpenAI Codex' `~/.codex/hooks.json`, GitHub Copilot CLIs `~/.copilot/hooks/failproofai.json`, Cursor Agents `~/.cursor/hooks.json`, OpenCodes generiertem Plugin-Shim unter `~/.config/opencode/plugins/failproofai.mjs` sowie einem Registrierungseintrag im `plugin`-Array von `~/.config/opencode/opencode.json`, Pis `~/.pi/agent/settings.json` oder Gemini CLIs `~/.gemini/settings.json`). Sind mehrere vorhanden, werden Sie zur Auswahl aufgefordert; übergeben Sie `--cli claude codex copilot cursor opencode pi gemini` (beliebige Teilmenge), um die Abfrage zu überspringen.

    Die Unterstützung für GitHub Copilot CLI, Cursor Agent, OpenCode, Pi und Gemini CLI befindet sich im **Beta**-Stadium – Installation mit `--cli copilot`, `--cli cursor`, `--cli opencode`, `--cli pi` oder `--cli gemini`.

    ```bash
    failproofai policies --install --scope project
    failproofai policies --install --cli codex --scope project
    failproofai policies --install --cli copilot --scope project
    failproofai policies --install --cli cursor --scope project
    failproofai policies --install --cli opencode --scope project
    failproofai policies --install --cli pi --scope project
    failproofai policies --install --cli gemini --scope project
    failproofai policies --install block-sudo block-rm-rf sanitize-api-keys
    ```
  </Step>
  <Step title="Überprüfen">
    ```bash
    failproofai policies
    ```

    Zeigt alle Richtlinien, deren Aktivierungsstatus und konfigurierte Parameter an.
  </Step>
  <Step title="Dashboard starten">
    ```bash
    failproofai
    ```

    Öffnet ein lokales Dashboard unter `http://localhost:8020`, in dem Sie Sitzungen durchsuchen, Werkzeugaufrufe inspizieren und Richtlinien verwalten können.
  </Step>
  <Step title="Agenten starten">
    Starten Sie Claude Code wie gewohnt. Wenn der Agent etwas Riskantes versucht, greift failproofai automatisch ein. Lassen Sie ihn unbeaufsichtigt laufen und überprüfen Sie anschließend im Dashboard, was passiert ist.
  </Step>
</Steps>

## Funktionsweise von Richtlinien

Jedes Mal, wenn ein Agent ein Werkzeug ausführt, ruft Claude Code failproofai als Teilprozess auf:

```text
Claude Code  →  failproofai --hook PreToolUse  →  reads stdin JSON
                                                 evaluates policies
                                                 writes decision to stdout
```

Jede Richtlinie gibt eine von drei Entscheidungen zurück:

- **allow** – der Agent fährt normal fort
- **deny** – die Aktion wird blockiert, dem Agenten wird der Grund mitgeteilt
- **instruct** – dem Prompt des Agenten wird zusätzlicher Kontext hinzugefügt

<Note>
Richtlinien laufen in Ihrem lokalen Prozess. Es werden keine Daten an einen externen Dienst übertragen.
</Note>

## Team-Richtlinien mit konventionsbasierten Richtlinien einrichten

Der schnellste Weg, Qualitätsstandards im gesamten Team zu etablieren, ist die `.failproofai/policies/`-Konvention. Legen Sie Richtliniendateien in dieses Verzeichnis – sie werden automatisch geladen, ohne Flags, Konfigurationsänderungen oder Installationsbefehle.

<Steps>
  <Step title="Richtlinienverzeichnis erstellen">
    ```bash
    mkdir -p .failproofai/policies
    ```
  </Step>
  <Step title="Richtliniendateien hinzufügen">
    Kopieren Sie die Starter-Beispiele oder schreiben Sie eigene:

    ```bash
    cp node_modules/failproofai/examples/convention-policies/*.mjs .failproofai/policies/
    ```

    Oder erstellen Sie eine neue:

    ```js
    // .failproofai/policies/team-policies.mjs
    import { customPolicies, allow, deny, instruct } from "failproofai";

    customPolicies.add({
      name: "test-before-commit",
      match: { events: ["PreToolUse"] },
      fn: async (ctx) => {
        if (ctx.toolName !== "Bash") return allow();
        if (/git\s+commit/.test(ctx.toolInput?.command ?? "")) {
          return instruct("Run tests before committing.");
        }
        return allow();
      },
    });
    ```
  </Step>
  <Step title="In Git committen">
    ```bash
    git add .failproofai/policies/
    git commit -m "Add team quality policies"
    ```

    Jedes Teammitglied, das failproofai installiert hat, übernimmt diese Richtlinien automatisch. Es ist keine Einrichtung pro Entwickler erforderlich.
  </Step>
</Steps>

<Tip>
Committen Sie `.failproofai/policies/` in Ihr Repository, damit das gesamte Team dieselben Standards teilt. Wenn Ihr Team neue Fehlermuster entdeckt, fügen Sie Richtlinien hinzu und pushen Sie – alle erhalten das Update beim nächsten `git pull`. Mit der Zeit werden diese Richtlinien zu einem lebendigen Qualitätsstandard, der sich kontinuierlich verbessert.
</Tip>

## Datenspeicherung

Alle Konfigurationen und Protokolle verbleiben auf Ihrem Rechner:

| Pfad | Inhalt |
|------|--------|
| `~/.failproofai/policies-config.json` | Globale Richtlinienkonfiguration |
| `~/.failproofai/hook-activity.jsonl` | Hook-Ausführungsverlauf |
| `~/.failproofai/hook.log` | Debug-Protokoll für benutzerdefinierte Hook-Fehler |
| `.failproofai/policies-config.json` | Projektspezifische Konfiguration (committet) |
| `.failproofai/policies-config.local.json` | Persönliche Überschreibungen (per gitignore ausgeschlossen) |

---

## Deinstallation

```bash
failproofai policies --uninstall
```

Entfernt Hook-Einträge aus `~/.claude/settings.json`. Konfigurationsdateien in `~/.failproofai/` bleiben erhalten.
