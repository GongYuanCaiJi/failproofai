### Einen echten Aufruf instrumentieren

    model="claude-sonnet-4-6",
    max_tokens=512,
    messages=messages,

    session_id="run-001",
    agent_id="planner",
    model=reply.model,
    stop_reason=reply.stop_reason,

    output_tokens=reply.usage.output_tokens,
    content=[block.model_dump() for block in reply.content],

## configure()

## Event-Referenz

## Human-in-the-Loop-Events

Human-in-the-Loop-Events geben Ihnen Kontrolle ĂĽber die Momente, in denen eine Person in die AusfĂĽhrung des Agenten eingreift (auf Genehmigung warten, Eingaben bereitstellen, pausieren oder den Agenten stoppen).     prompt="Do you approve this action?",        # str | None - the question shown to the human
    options=["approve", "reject", "defer"],      # list[str] | None - choices presented to the human

```

    agent_id="planner",
    input_id="inp-abc",    # str, required - must match the prior human_wait

### `event.human_pause()`

Wird ausgelĂ¶st, wenn ein Mensch den Agenten aktiv pausiert (z. B. ĂĽber eine Dashboard-Steuerung). Der Agent wird angehalten, aber nicht beendet.

### `event.human_interrupt()`

Wird ausgelĂ¶st, wenn ein Mensch den Agenten mitten in der AusfĂĽhrung aktiv stoppt. Im Gegensatz zu `human_pause` wird die Arbeit des Agenten beendet und nicht nur ausgesetzt.

    agent_id="planner",

```

## Wie Events geschrieben werden

Events werden prozessintern gepuffert und alle `flush_interval` Sekunden auf Disk geschrieben (Standard: 500 ms). Jeder Flush schreibt eine JSONL-Datei:

```text

```

Der Collector ĂĽberwacht dieses Verzeichnis und lĂ¤dt Dateien automatisch hoch. Sie mĂĽssen diese Dateien nicht direkt verwalten.

Jede Datei wird atomar geschrieben: Das SDK schreibt in eine temporĂ¤re Datei und benennt sie dann an die endgĂĽltige Position um, sodass der Collector nie eine unvollstĂ¤ndig geschriebene Datei sieht. Ein abschlieĂźender Flush wird auch beim Beenden Ihres Prozesses ausgefĂĽhrt, damit Events, die im letzten Intervall gepuffert wurden, nicht verloren gehen. Wenn der Collector offline ist, akkumulieren sich Events einfach als Dateien auf Disk und werden ĂĽbertragen, sobald er wieder online ist.
