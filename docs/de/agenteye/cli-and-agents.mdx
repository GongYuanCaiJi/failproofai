Ihre gesamte Failproof AI Observability-Umgebung, ein Befehl entfernt. Prüfen Sie die Produktion, erstellen Sie einen API-Schlüssel oder bestätigen Sie einen Vorfall, ohne Ihr Terminal zu verlassen – und skripten Sie das alles in CI oder lassen Sie einen Coding-Agent es in einfachem Englisch für Sie erledigen.

```bash
pipx install agenteye
agenteye login --email you@example.com      # a 6-digit code lands in your inbox
agenteye --json sessions --since 24h         # every agent run from the last day, newest first
```

*Die `agenteye` CLI kommuniziert mit Ihrem Dashboard. Sie ist ein anderes Werkzeug als der Collector, der Events an den Server übermittelt.*

## Ihre gesamte Umgebung, ein Befehl entfernt

Hören Sie auf, zwischen Tabs zu wechseln, nur um eine schnelle Frage zu beantworten. Die `agenteye` CLI liest Ihre Daten und verwaltet Ihre Organisation über eine einzige Binary – eine Prüfung, die früher mehrere Klicks im Dashboard erforderte, ist jetzt eine Zeile, die Sie wiederholen, als Alias anlegen oder in ein Runbook einfügen können. Sie haben vier Bereiche:

- **Daten lesen:** `sessions`, `events`, `evals` und `errors`, gefiltert nach Zeit, Agent und Umgebung.
- **Organisation verwalten:** `keys`, `users`, `settings`, `alerts` und `incidents`.
- **Analysen ausführen:** gespeichertes SQL sowie ein Ad-hoc-`query`-Runner über Ihre Event-Daten.
- **Den Assistenten befragen:** `agent ask` erreicht dieselbe schreibgeschützte Analysefunktion, die Sie auch im Dashboard nutzen.

Installieren Sie es einmalig mit `pipx`, melden Sie sich mit einem per E-Mail zugesandten 6-stelligen Code an, und Sie sind startklar. Die Sitzung läuft etwa einen Tag; führen Sie `agenteye login` erneut aus, wenn sie abläuft. Nutzen Sie die CLI, um die Produktion zu überprüfen, einen Schlüssel zu erstellen oder einen aktiven Vorfall zu triagieren – ganz ohne Browser:

```bash
agenteye errors --since 24h --aggregate      # what is breaking, grouped by error type
agenteye incidents list --state firing       # what is on fire right now
agenteye keys create ci --add events:add     # a key that can only push events, secret shown once
```

Eine wichtige Konvention: Globale Optionen wie `--json` kommen vor dem Befehl. `agenteye --json sessions` ist korrekt; `agenteye sessions --json` ist es nicht.

## Skripten Sie es, binden Sie es in CI ein

Jeder Befehl unterstützt `--json`, und das verändert alles. Sauberes JSON geht an stdout, während Statusmeldungen und Warnungen für Menschen an stderr gehen – eine `--json`-Ausgabe kann also direkt in `jq` gepipt werden, ohne störende Zeilen herausfiltern zu müssen. Das macht die CLI gleichermaßen nützlich für Sie an der Eingabeaufforderung und für einen Coding-Agent, der die Ausgabe verarbeitet:

```bash
agenteye --json sessions --since 24h --status error | jq -r '.sessions[].session_id'
```

Sie ist für den unbeaufsichtigten Betrieb ausgelegt. Bestätigungsabfragen werden automatisch übersprungen, wenn kein Terminal angeschlossen ist – nichts hängt sich in einer Pipeline auf. Jeder Befehl gibt einen aussagekräftigen Exit-Code zurück: `0` für Erfolg, `4` für nicht eingeloggt, `5` für fehlende Berechtigung (die Meldung nennt sie, z. B. `alerts:write`), `3` für Dashboard nicht erreichbar. Ein Skript kann bei `4` eine Neuauthentifizierung einleiten oder bei `5` genau angeben, was Sie bei einem Admin anfragen müssen – statt blind zu scheitern.

## Lassen Sie einen Coding-Agent es auf Englisch steuern

Noch besser: Sie sollten sich gar keine dieser Flags merken müssen. Der **CLI-Skill** ist ein kleiner Agent-Skill-Ordner namens `agenteye-cli`, der einen Coding-Agent wie Claude Code oder Codex beibringt, die CLI auf Basis von Anfragen in einfachem Englisch zu bedienen. Fragen Sie „Ist heute etwas kaputt?" und der Agent wählt den passenden Befehl, führt ihn als Sie aus und antwortet in Prosa.

Für Claude Code legen Sie den Ordner `agenteye-cli` in `~/.claude/skills/` ab, und er wird automatisch erkannt. Failproof AI Observability stellt den Ordner bereit; es gibt nichts zusätzlich zu installieren, da er nur die bereits installierte CLI steuert. Melden Sie sich zuerst selbst an: Der Skill kann den E-Mail-Code-Login nicht für Sie abschließen.

Da der Agent die CLI als Sie ausführt, kann er alles tun, was Ihr Login erlaubt – Lesen und Schreiben: Schlüssel erstellen, Einstellungen ändern, Vorfälle auflösen. Die „Sind Sie sicher?"-Abfrage der CLI wird für einen Agent nicht ausgelöst, daher ist der Skill so geschrieben, dass er den genauen Befehl angibt und auf Ihre Bestätigung wartet, bevor eine Änderung vorgenommen wird. Sie sind der Bestätigungsschritt.

```text
you    Why did session run-001 fail?

agent  Running: agenteye --json events --session-id run-001 --all
       The checkout-agent hit a TimeoutError on its third tool call.
```

Lesezugriffe bleiben sofort, und jeder Schreibzugriff wartet auf Sie:

```text
you    Give CI a key that can only push events.

agent  This creates an API key. I'll run:
         agenteye keys create ci --add events:add
       OK to proceed?

you    yes

agent  Done. Key "ci" created with events:add only. The secret is shown once, so store it now.
```
