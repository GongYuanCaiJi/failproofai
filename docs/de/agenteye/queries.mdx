Stellen Sie beliebige Fragen zu Ihren Agent-Daten und erhalten Sie in Sekunden eine Antwort. Failproof AI Observability bietet Ihnen eine Bibliothek gespeicherter, sofort ausführbarer Abfragen über Ihre Events und Auswertungen – so starten Sie mit einem funktionierenden Beispiel statt mit einem leeren SQL-Editor.

![Die Bibliothek gespeicherter Abfragen: ein Raster wiederverwendbarer Abfragen, sowohl eingebaute Vorlagen als auch eigene](/agenteye/images/queries.png)

*Ihre Bibliothek gespeicherter Abfragen unter `/<org-slug>/queries`: eingebaute Vorlagen neben den Abfragen, die Ihr Team gespeichert hat.*

## Mit einer Vorlage beginnen, nicht mit einer leeren Seite

Sie müssen sich keine Tabellennamen merken oder SQL von Grund auf schreiben. Die Bibliothek öffnet sich mit eingebauten Vorlagen für die Fragen, die Teams am häufigsten stellen – direkt neben den Abfragen, die Ihr eigenes Team gespeichert und benannt hat. Wählen Sie eine aus, die Ihrem Bedarf nahekommt, und Sie sind dem Ergebnis schon sehr nah.

Jede gespeicherte Abfrage ist organisationsweit verfügbar und geteilt, sodass nützliche Abfragen Ihrer Kollegen auch Ihnen zugutekommen. Benennen Sie eine Abfrage einmal und versehen Sie sie mit einer Beschreibung – dann kann jeder in Ihrer Organisation sie finden, ausführen oder ihre Ergebnisse später in ein Dashboard einbinden.

Sie finden die Bibliothek unter `/<org-slug>/queries`.

## Im SQL-Composer anpassen und ausführen

Öffnen Sie eine Abfrage, und sie landet direkt im SQL-Composer, wo Sie sie anpassen und die Antwort sofort sehen können: kein Export, kein Umweg, kein Warten auf jemand anderen.

![Der SQL-Abfrage-Composer mit einer gespeicherten Abfrage, einer Schema-Seitenleiste und einem Live-Ergebnisraster](/agenteye/images/query-lab.png)

*Der SQL-Composer: Ihre Abfrage links, eine Schema-Seitenleiste damit Sie nie nach Spaltennamen suchen müssen, und ein Live-Ergebnisraster darunter.*

- **Eine Schema-Seitenleiste** zeigt die Analysetabellen und ihre Spalten übersichtlich an, sodass Sie eine Abfrage gestalten können, ohne mühsam nach Feldnamen zu suchen.
- **Ein Live-Ergebnisraster** liefert Zeilen sofort nach dem Ausführen, sodass Sie in Sekunden iterieren können, anstatt immer wieder zu raten.
- **Standardmäßig nur lesend.** Abfragen werden gegen Ihren Event-Store ausgeführt und serverseitig validiert: Nur `SELECT`- und `WITH`-Anweisungen sind erlaubt, mit einem Anweisungs-Timeout und einer Zeilenbegrenzung. Eine Erkundungsabfrage kann Ihre Daten niemals verändern, und eine außer Kontrolle geratene wird automatisch gestoppt.

Mit dem Ergebnis zufrieden? Speichern Sie es zurück in die Bibliothek, damit das gesamte Team davon profitiert, oder pinnen Sie die Ausgabe als Linien-, Balken-, Flächen- oder Kreisdiagramm-Kachel in ein Dashboard.

## Vom Terminal aus ausführen oder vom Assistenten schreiben lassen

Die gespeicherten Abfragen stehen Ihnen überall zur Verfügung:

- **Vom Terminal aus.** Die `agenteye`-CLI listet, führt aus und speichert dieselben Abfragen, sodass Sie ein Ergebnis in ein Skript einfügen, in CI einbinden oder an einen Coding-Agenten übergeben können.

```bash
agenteye query list                 # die gleichen gespeicherten Abfragen, vom Terminal aus
agenteye query run errs --arg prod  # eine ausführen und die Zeilen ausgeben (--json zum Weiterleiten hinzufügen)
```

  Den vollständigen Befehlssatz finden Sie unter [CLI and agents](/de/agenteye/cli-and-agents).

- **Über den KI-Assistenten.** Sie sind sich nicht sicher, wie Sie das SQL formulieren sollen? Fragen Sie den [KI-Assistenten](/de/agenteye/assistant) im Dashboard auf normalem Deutsch – er entwirft die Abfrage und speichert sie für Sie in Ihrer Bibliothek.

Das Ausführen einer gespeicherten Abfrage erfordert die Berechtigung `queries:run`, die getrennt von den Berechtigungen zum Erstellen oder Löschen von Abfragen verwaltet wird. So können Sie Lesezugriff gewähren, ohne dass jeder die Bibliothek umschreiben kann.
