Sehen Sie jeden Fehler Ihrer Agenten an einem Ort, gruppiert, sodass ein gehäufter Ausbruch als ein einziges Problem erscheint. Sie erhalten einen Ein-Klick-Pfad von „etwas ist rot" zum genauen Lauf, der den Fehler verursacht hat – ohne in einem Live-Feed scrollen zu müssen.

## Jeder Fehler, bereits für Sie gesammelt

Wenn ein Agent abbricht, sollten Sie nicht in einem Live-Ereignisstrom scrollen müssen, um die roten Zeilen zu erwischen, bevor sie verschwinden. Die **Fehler**-Seite übernimmt das Sammeln für Sie. Sie bündelt alles, was das Dashboard rot einfärben würde, auf einer einzigen Triage-Oberfläche – damit Sie auf Anhieb sehen, was fehlschlägt, anstatt erst suchen zu müssen.

Und sie erfasst mehr als die offensichtlichen Fälle. Neben expliziten `error`-Ereignissen bringt Failproof AI Observability auch die stillen Fehler ans Licht: Jedes `tool_result`, `hook_completed` oder `agent_end`, dessen Payload einen Fehler enthält, erscheint hier. Ein Tool, das einen Fehler zurückgegeben hat, oder ein Hook, der schlecht beendet wurde, entgeht Ihrer Aufmerksamkeit nicht länger, nur weil keine laute Exception geworfen wurde.

Am oberen Rand zeigt ein Histogramm Fehler über die Zeit. Ein Blick genügt, um zu erkennen, ob es sich um ein stetiges Hintergrundrauschen oder um eine Spitze handelt, die vor wenigen Minuten begann – so wissen Sie sofort, ob Sie alles stehen und liegen lassen müssen.

Wie jede Beobachtungsoberfläche ist die Fehlerseite auf Ihre Organisation zugeschnitten und lässt sich nach Zeitraum, Umgebung, Agent und Sitzung filtern. Das bedeutet, Sie können eine flottenweit Liste nehmen und sie auf den einen Agenten oder die eine Umgebung eingrenzen, die Sie wirklich interessiert.

## Ein Vorfall, nicht hundert identische Zeilen

Eine einzige defekte Abhängigkeit kann denselben Fehler hunderte Male pro Minute auslösen. Unverarbeitet ergibt das eine Wand nahezu identischer Zeilen, die genau das begräbt, was Sie eigentlich sehen müssen.

Failproof AI Observability fasst wiederholte Fehler, die dieselbe Sitzung und denselben Fehlertyp teilen, in einer einzigen Zeile zusammen. Ein Ausbruch erscheint als ein Vorfall. Sie zählen Probleme, keine Logzeilen – und das Signal, das wichtig ist, bleibt oben, anstatt von seinem eigenen Volumen übertönt zu werden.

## Von „etwas ist rot" zum genauen Ereignis

Klicken Sie auf eine Zeile, um direkt in die Sitzung dieses Laufs zu gelangen, positioniert auf dem genauen Ereignis, das fehlgeschlagen ist. Kein Kopieren von Sitzungs-IDs, kein Scrollen auf der Suche nach dem Moment, an dem etwas schiefging: Sie landen direkt dort, mit dem vollständigen Ausführungsgraphen auf einen Blick, sodass Sie sehen können, was der Agent in den Momenten vor dem Fehler getan hat.

Wenn Sie `alerts:write` besitzen, enthält jede Zeile außerdem einen **+ alert**-Knopf. Klicken Sie darauf, und Observability öffnet eine neue Alert-Regel, die bereits vorkonfiguriert ist, um genau denselben Fehler erneut zu erfassen. Der Vorfall, den Sie gerade triagiert haben, wird derjenige, der Sie beim nächsten Mal benachrichtigt – anstatt Sie zweimal zu überraschen.

**Wo Sie es finden:** Die **Fehler**-Seite befindet sich im Beobachtungsbereich des Dashboards unter `/<org-slug>/errors`.

- [Sessions](/de/agenteye/sessions): Den vollständigen Lauf hinter einem Fehler öffnen.
