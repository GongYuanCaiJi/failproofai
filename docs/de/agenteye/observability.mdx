Die Observe-Oberflächen zeigen Ihnen, was Ihre Agents gerade tun, und ermöglichen es Ihnen, jeden einzelnen Run im Detail zu untersuchen. Alles hier ist live, auf Ihre Organisation beschränkt und nach Zeitraum, Umgebung, Agent und Session filterbar – so gelangen Sie in Sekunden von „irgendetwas stimmt nicht" zum genauen Run.

![Der Live-Event-Stream, farblich nach Typ kodiert und nach Umgebung, Agent und Session filterbar](/agenteye/images/events-stream.png)

Vier Oberflächen, jede mit einer eigenen Seite:

- **[Event Stream](/de/agenteye/event-stream)**: Die live, schrittweise Verlaufsspur jedes Runs über alle Agents hinweg, neueste zuerst. Ihr Organisations-Einstiegspunkt und erste Anlaufstelle für die Fehlersuche.
- **[Sessions und Ausführungsgraph](/de/agenteye/sessions)**: Diese Events zu einer Zeile pro Run zusammengefasst, plus eine Git-ähnliche Darstellung, wie sich jeder Run entfaltet hat.
- **[Performance-Metriken](/de/agenteye/telemetry)**: Latenz-Heatmaps sowie p50/p95/p99-Kennwerte für Ihre Modelle, Tools und Hooks – damit sticht ein Ausreißer sofort aus dem Median hervor.
- **[Fehlerverfolgung](/de/agenteye/error-tracking)**: Eine zentrale Triage-Oberfläche für alles, was schiefgelaufen ist – ein Klick von einem ausgelösten Alert zum betroffenen Run.

## Verwandte Themen

- [Evaluierungen](/de/agenteye/evaluations): Jeden Run auf Qualität bewerten.
- [Alerts](/de/agenteye/alerts): Jeden Schwellenwert in eine Benachrichtigungsregel umwandeln.
- [Audits](/de/agenteye/audits): FailproofAI Observability lässt sitzungsübergreifende Fehlermuster automatisch erkennen.
- [CLI und Agents](/de/agenteye/cli-and-agents): Dieselbe Observability direkt aus Ihrem Terminal.
