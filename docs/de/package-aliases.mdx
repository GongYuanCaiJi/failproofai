## Offizielles Paket

Das kanonische npm-Paket ist **`failproofai`**:

```bash
npm install -g failproofai
# or
bun add -g failproofai
```

---

## Warum wir die Alias-Namen besitzen

Typosquatting ist ein verbreiteter Supply-Chain-Angriff, bei dem ein böswilliger Akteur einen Paketnamen registriert, der sich nur um einen Tastendruck vom Namen eines populären Pakets unterscheidet. Ahnungslose Nutzer, die beim Installationsbefehl einen Tippfehler machen, führen am Ende Code unter Angreiferkontrolle mit vollem Systemzugriff aus – genau die Art von Bedrohung, gegen die Failproof AI schützen soll.

Um diese Angriffsfläche zu eliminieren, **besitzen wir präventiv alle gängigen Falschschreibungen und Formatierungsvarianten** von `failproofai` auf npm. Keiner dieser Namen kann von Dritten registriert werden. Jeder davon ist ein schlanker Proxy, der das echte `failproofai`-Paket installiert und an dieses delegiert.

## Registrierte Aliase

**Formatierungsvarianten** – verschiedene Schreibweisen von „failproof ai":

| Paket | Status |
|---------|--------|
| `failproof` | ✅ Veröffentlicht |
| `failproof-ai` | ⏳ Ausstehend (npm-Freigabe) |
| `fail-proof-ai` | ⏳ Ausstehend (npm-Freigabe) |
| `failproof_ai` | ⏳ Ausstehend (npm-Freigabe) |
| `fail_proof_ai` | ⏳ Ausstehend (npm-Freigabe) |
| `fail-proofai` | ⏳ Ausstehend (npm-Freigabe) |

**`failprof*`-Tippfehler** – ein `o` fehlt in „proof":

| Paket | Status |
|---------|--------|
| `failprof` | ✅ Veröffentlicht |
| `failprof-ai` | ✅ Veröffentlicht |
| `failprofai` | ⏳ Ausstehend (npm-Freigabe) |
| `fail-prof-ai` | ⏳ Ausstehend (npm-Freigabe) |
| `failprof_ai` | ⏳ Ausstehend (npm-Freigabe) |

**`faliproof*`-Tippfehler** – vertauschtes `a` und `i`:

| Paket | Status |
|---------|--------|
| `faliproof` | ✅ Veröffentlicht |
| `faliproof-ai` | ✅ Veröffentlicht |
| `faliproofai` | ⏳ Ausstehend (npm-Freigabe) |

> **Warum ausstehend?** Die Spam-Schutzrichtlinie von npm blockiert Namen, die nach dem Entfernen von Satzzeichen und dem Ausführen von Ähnlichkeitsprüfungen auf denselben String wie ein bestehendes Paket normalisiert werden. Wir haben den npm-Support kontaktiert, um diese Namen zu Anti-Squatting-Zwecken zu reservieren. Sie werden nach der Genehmigung freigeschaltet.

Sie können überprüfen, ob ein veröffentlichter Alias uns gehört:

```bash
npm info failproof
# Look for: "ExosphereHost Inc." in the maintainers field
```

## Wie die Aliase funktionieren

Jedes Alias-Paket:

1. Listet `failproofai` als Abhängigkeit auf – sodass das echte Paket (einschließlich seines `postinstall`-Hook-Setups) bei der Installation ausgeführt wird
2. Stellt ein Binary bereit, das seinem eigenen Namen entspricht (z. B. `failprof-ai`) und alle Argumente an das `failproofai`-Binary weiterleitet

Der Proxy ist ein zweizeiliges Node-Skript; es enthält keine Logik, keine Netzwerkanfragen und keine Datenerhebung über das hinaus, was `failproofai` selbst tut.
